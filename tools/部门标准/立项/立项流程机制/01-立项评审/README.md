# 立项评审

这里是立项阶段的决策节点。

这一阶段不再等同于旧的“五张卡串联”。

阶段目标：

- 判断这个方向是否值得进入下一阶段验证
- 判断当前证据是否足够进入小测
- 判断是否需要进入 `02-纸面原型` 做补证
- 判断长线经营假设是否已经写清并具备后续验证路径

评审输入：

- 项目目录中的 `00-立项文档`
- [原型立项闸门表.md](/Users/mt/Documents/Codex/tools/部门标准/立项/立项流程机制/原型立项闸门表.md) 对应证据
- [长线经营假设卡.md](/Users/mt/Documents/Codex/tools/部门标准/立项/立项流程机制/长线经营假设卡.md)
- 若当前证据不足，需要明确缺口属于：
  `需要纸面原型补证`
  `需要竞品/题材/发行信息补证`
  `需要终止`

如果结论是 `补其他证据再评`，默认按下面方式承接：

- 题材、竞品、发行信息补证：
  回到项目目录的 `00-立项文档/` 或本目录的 `03-题材选择/`
- 成本、资源、团队承接能力补证：
  回到项目目录的 `00-立项文档/`
- 补证完成后，再回到 `01-立项评审`

评审面只保留两块：

1. `原型立项闸门评审`
2. `长线经营假设评审`

旧的 `方向 / 母型 / 参与方式 / 核心机制 / 玩法循环` 仍可作为立项定义和内部分析材料，但不再作为独立决策卡逐张给最终结论。

使用入口：

- [标准评审模板文档.md](/Users/mt/Documents/Codex/tools/部门标准/立项/立项流程机制/01-立项评审/标准评审模板文档.md)
- [原型立项闸门表.md](/Users/mt/Documents/Codex/tools/部门标准/立项/立项流程机制/原型立项闸门表.md)
- [长线经营假设卡.md](/Users/mt/Documents/Codex/tools/部门标准/立项/立项流程机制/长线经营假设卡.md)

本阶段结论只能选一个：

- `进入纸面原型补证`
- `补其他证据再评`
- `进入小测`
- `终止`

建议责任分工：

- `项目负责人 / 制作人`：主持结论，决定是否进入小测
- `主策`：负责原型闸门定义和 `02-纸面原型` 补证材料
- `发行 / 买量负责人`：负责 `三秒买点`、`一分钟承接` 和创意持续性判断
- `产品 / 商业化负责人`：负责 `首付理由` 和长线经营假设

如果结论是 `进入纸面原型补证`，则进入：

- [02-纸面原型/README.md](/Users/mt/Documents/Codex/tools/部门标准/立项/立项流程机制/02-纸面原型/README.md)
