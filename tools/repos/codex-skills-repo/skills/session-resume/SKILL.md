# session-resume

从记忆库中读取任务状态，输出结构化续接摘要，让新会话直接从"下一步"继续。

## 执行步骤

### Step 1：确定续接目标

**如果用户提供了任务名**：
- 直接定位 `memory/task_{任务名}.md`

**如果用户未提供任务名**：
- 读取 `/Users/mt/.claude/projects/-Users-mt-Documents-Codex/memory/MEMORY.md`
- 列出所有 `task_` 开头的条目（任务名 + 一句话状态）
- 请用户选择要续接的任务

### Step 2：读取任务状态

从对应 `task_*.md` 文件中提取：

| 字段 | 来源 |
|---|---|
| 任务目标 | `## 任务概述` 或 `## 目标` |
| 当前进度 | `## 当前进度` 或 `当前进行` |
| 下一步 | `## 下一步` |
| 已确认约束 | `## 已确认约束` |
| 阻塞项 | `阻塞项` 字段 |
| 参考文件 | `参考文件` 字段 |

### Step 3：判断是否需要补充读取

如果状态文件中的"参考文件"列出了关键文件路径，且续接任务需要理解这些文件的当前内容，询问用户：

> 任务涉及 {N} 个参考文件，是否现在读取以完整恢复上下文？（Y/N）

用户确认后读取；不确认则跳过，记录为"待读取"。

### Step 4：输出续接摘要

格式固定，≤200 token：

```
✓ 已恢复任务上下文：{任务名}

目标：{一句话}
当前进度：{到哪一步了}
下一步：{具体的下一个动作，必须是可执行的，不是方向}
关键约束：{N 条，逐行列出}
阻塞项：{有 → 列出；无 → 无}
参考文件：{有 → 列出路径；无 → 无}

续接后请确认：目标未变 / 进度描述准确 / 下一步可执行
```

### Step 5：等待用户确认

输出摘要后，等待用户说"继续"或纠正任何不准确的地方，再开始执行下一步动作。

不要在输出摘要后立即开始执行任务——给用户一个校正的机会。

## 质量检查

- [ ] "下一步"是具体动作，不是方向（如"修改 line 145"而不是"继续优化"）
- [ ] 阻塞项已列出（包括"上次 compact 时尚未解决的问题"）
- [ ] 没有把调试过程或失败尝试带进摘要
- [ ] 如果任务状态文件显示已 Archived，告知用户任务已完成，询问是否要做后续工作
