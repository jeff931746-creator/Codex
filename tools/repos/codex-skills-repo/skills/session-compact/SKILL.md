# session-compact

把当前会话的有效信息提炼成结构化摘要，写入记忆库，丢弃过程噪音。

## 执行步骤

### Step 1：提炼任务状态

按以下结构从当前会话中提取信息：

**必须提取**：
- 任务名称和核心目标
- 已确认的约束条件（用户明确说过的、测试验证过的）
- 已得出的结论（有证据支撑的，不是猜测）
- 当前进行到哪一步
- 下一步是什么

**必须丢弃**：
- 调试过程和中间输出
- 失败的尝试（只记录"失败原因"，不记录过程）
- 重复的解释和说明
- 临时的探索性内容

### Step 2：判断写入目标

根据内容性质选择写入位置：

| 内容类型 | 写入位置 |
|---|---|
| 当前任务的进行状态 | 复制任务状态模板，填写后存为 `memory/task_{任务名}.md` |
| 跨任务有效的结论或约束 | 写入对应 project / feedback / user 类型记忆文件 |
| 外部资源的指针 | 写入 reference 类型记忆文件 |

### Step 3：写入记忆文件

**任务状态文件**格式（从模板复制）：

```markdown
---
name: Task State — {任务名}
description: {任务名} 的当前进行状态，compact 于 {日期}
type: project
---

# 任务状态 — {任务名}

## 目标
{一句话描述}

## 成功标准
{可验证的完成条件}

## 已确认约束
- {约束1}
- {约束2}

## 已得出结论
- {结论1}（依据：{简短说明}）

## 当前进度
{当前到了哪一步}

## 下一步
{具体的下一个动作}

## 失败记录
- {失败原因简记，不展开过程}

**Why:** compact 于 {日期}，原始会话因{原因}需要压缩。
**How to apply:** 新会话开始时从"当前进度"和"下一步"继续。
```

### Step 4：更新 MEMORY.md 索引

在 `/Users/mt/.claude/projects/-Users-mt-Documents-Codex/memory/MEMORY.md` 末尾追加：

```
- [Task State — {任务名}](task_{任务名}.md) — {任务一句话描述}，compact 于 {日期}
```

### Step 5：输出摘要

向主会话返回一个 ≤ 200 token 的摘要，格式：

```
✓ 已写入记忆：{文件名}

任务：{任务名}
目标：{一句话}
当前进度：{当前到哪步}
下一步：{具体动作}
已确认结论：{N 条}
```

## 质量检查

写完后确认：
- [ ] 摘要能让新会话在不看原始对话的情况下继续任务
- [ ] 没有把调试过程写进去
- [ ] 结论都有依据，不是猜测
- [ ] MEMORY.md 索引已更新
