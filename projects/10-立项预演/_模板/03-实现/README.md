# 实现

这里放 demo 的代码、配置、资源、模块拆分、接口说明和构建说明。

建议保持下面的边界：

- 代码只写实现，不写长篇复盘
- 配置只写可调参数，不散落魔法数字
- 资源只放当前 demo 需要的内容
- 模块拆分和架构说明单独记录

如果 demo 还没开始实现，这里也可以先放一份空目录说明，避免后面临时补结构。

## 如果使用 Godot

Godot demo 建议再按下面的实现层骨架继续拆：

```text
03-实现/
  Godot/
    project.godot
    scenes/
    scripts/
    autoload/
    assets/
    configs/
    ui/
    docs/
```

### Godot 目录边界

- `project.godot`：项目根配置，只保留 Godot 必需项和少量全局设置。
- `scenes/`：所有 `.tscn` 场景文件。
- `scripts/`：所有 `.gd` 脚本和脚本辅助说明。
- `autoload/`：单例、全局状态、全局事件总线。
- `assets/`：图片、音频、图标、字体、VFX 资源。
- `configs/`：节奏参数、关卡配置、卡池配置、数值表。
- `ui/`：界面场景、面板、HUD、菜单。
- `docs/`：Godot 实现说明、节点树约定、输入映射、导出说明。

### Godot 实现约定

- 场景负责组合，脚本负责行为，配置负责数值。
- 单例只放跨场景共享内容，不要把业务逻辑全塞进 autoload。
- 每个核心系统尽量拆成独立场景或独立脚本，不要把战斗、UI、商店、存档揉进同一个脚本。
- 节点命名和路径最好固定，避免后续重构时引用失效。
- 输入映射、窗口尺寸、分辨率、伸缩模式、发布导出配置都应单独记录。

### Godot 新 demo 启动清单

如果新 demo 选 Godot，建议先准备：

1. `project.godot`
2. 至少一个主场景
3. 输入映射表
4. `autoload` 单例占位
5. 基础配置文件
6. 导出预设说明
