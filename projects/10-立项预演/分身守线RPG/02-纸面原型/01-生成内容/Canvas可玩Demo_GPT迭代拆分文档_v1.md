## 一、总拆分策略

1. `第1轮`：先把基础 Canvas 场景和主循环跑起来

5. `第5轮`：加入精英、Boss、胜负结算和重开
6. `第6轮`：补 UI 可读性、数值调优和轻度打磨

每一轮都要求 GPT：

- 在已有文件基础上改
- 不重写整个项目
- 不引入框架
- 保持 `index.html / styles.css / script.js` 三文件结构

## 三、分轮拆解

## 第1轮：搭基础运行框架

### 本轮目标

只做一个可运行的 Canvas 战斗场景骨架。

### 本轮要做

1. 创建基础页面结构
2. 页面中放一个 Canvas
3. 建立游戏主循环：
   - `update()`
   - `render()`
   - `requestAnimationFrame`
4. 画出基础战场：
   - 背景
   - 左侧水晶占位
   - 右侧敌人刷新区域
   - 中部 6 个分身阵位占位
5. 做简单的状态栏占位：
   - 水晶血量
   - 主英雄血量
   - 波次
   - 当前能量

### 本轮不要做

- 不做敌人
- 不做主英雄
- 不做分身
- 不做碰撞
- 不做升级

### 本轮验收

满足以下条件即可：

1. 页面能打开
2. Canvas 正常渲染
3. 战场布局能看懂
4. 有持续刷新的主循环

### 给 GPT 的 prompt

```text
请在 /Users/mt/Documents/Codex/playground/web-demo/ 下实现一个原生 Canvas 2D 的网页 demo 基础框架。

要求：
1. 只使用 index.html、styles.css、script.js
2. 不使用任何框架或外部依赖
3. 直接打开 index.html 就能运行
4. 本轮只做基础战场骨架，不做战斗系统

请实现：
- 一个 1280x720 左右的 Canvas 战场
- 游戏主循环 update/render
- 左侧核心水晶占位
- 中部 6 个分身阵位占位
- 右侧敌人生成区占位
- 顶部或侧边简易状态栏占位

要求代码结构清晰，不要把所有逻辑堆在一个函数里。
不要实现敌人、主英雄、分身、升级和 Boss。
```

## 第2轮：主英雄、敌人、水晶、基础战斗

### 本轮目标

让场景“活起来”，形成最基础的守线循环。

### 本轮要做

1. 加入主英雄：
   - `WASD` 移动
   - 自动攻击最近敌人
2. 加入敌人：
   - 从右侧生成
   - 向左侧水晶推进
3. 加入水晶：
   - 敌人接触后造成伤害
4. 加入简单碰撞与伤害
5. 加入主英雄技能：
   - `Space` 释放范围雷暴脉冲
   - 有冷却

### 本轮不要做

- 不做分身系统
- 不做升级系统
- 不做精英和 Boss
- 不做复杂特效

### 本轮验收

1. 可以移动主英雄
2. 敌人会刷出并推进
3. 主英雄会自动打敌人
4. 敌人接触水晶后水晶掉血
5. 可以用 `Space` 开技能清一波

### 给 GPT 的 prompt

```text
请基于现有 /Users/mt/Documents/Codex/playground/web-demo/ 文件继续迭代，不要重写整个项目。

本轮目标：加入主英雄、敌人、水晶和最基础战斗。

请实现：
1. 主英雄：
- WASD 移动
- 自动攻击最近敌人
- 使用简单几何图形绘制

2. 敌人：
- 从右侧定时生成
- 按路径或直线向左推进
- 接近水晶后攻击水晶

3. 水晶：
- 有生命值
- 生命值归零时游戏结束

4. 主英雄技能：
- 按 Space 释放范围冲击波
- 有冷却

注意：
- 本轮不要做分身、升级、Boss、精英
- 保持代码结构清晰
- 运行后必须可以直接玩到“主英雄守水晶”的基础循环
```

## 第3轮：分身阵位与 4 类分身

验证“分身能不能替代塔”。

### 本轮要做

1. 允许玩家用数字键切换分身类型：
   - `1` 盾卫
   - `2` 寒霜
   - `3` 雷灵
   - `4` 战旗
2. 鼠标点击空阵位部署分身
3. 加入 4 类分身行为：
   - 盾卫：近战拦截/吸怪
   - 寒霜：远程减速
   - 雷灵：远程主输出
   - 战旗：范围增益
4. 加入部署消耗：
   - 使用能量点
   - 能量点随时间恢复

### 本轮不要做

- 不做升级三选一
- 不做精英和 Boss
- 不做复杂 build 联动

### 本轮验收

1. 玩家能切换和部署分身
2. 不同分身职责清晰
3. 场面已经能看出“不是摆塔，而是布阵位分身”
4. 主英雄和分身都能参与守线

### 给 GPT 的 prompt

```text

请实现：

2. 鼠标点击空阵位部署分身
3. 分身部署要消耗能量点
4. 能量点随时间缓慢恢复

四类分身：
- 盾卫分身：近战、拦截、抗线
- 寒霜分身：远程、减速、控制
- 雷灵分身：远程、主力清杂输出
- 战旗分身：不攻击，给附近主英雄和分身提供增益
```

## 第4轮：经验、升级、三选一 build

### 本轮目标

让 demo 开始具备“这一局有 build”的感觉。

### 本轮要做

1. 击杀敌人获得经验
2. 经验满后升级
3. 升级时暂停游戏，弹出 `3选1`
4. 本轮先做 `12个左右` 升级项，分三类：
   - 主英雄强化
   - 分身强化
   - 混合强化
5. 选择后立即生效并继续战斗

### 推荐升级项

- 主英雄普攻伤害提升
- 主英雄普攻弹射 +1
- 主英雄技能范围增加
- 盾卫生命增加
- 寒霜减速增强
- 雷灵伤害提升
- 战旗增益范围扩大
- 解锁额外阵位
- 主英雄靠近分身时增伤
- 冰冻敌人被雷击时额外伤害
- 分身数量达到 3 时全体攻速提升
- 能量恢复速度提升

### 本轮不要做

- 不做复杂稀有度
- 不做装备系统
- 不做局外成长

### 本轮验收

1. 玩家能升级
2. 升级时会暂停游戏
3. 选择后能明显感受到战斗风格变化
4. 至少能玩出 2-3 种不同倾向

### 给 GPT 的 prompt

```text

请实现：
1. 敌人死亡给经验
2. 经验满后升级
3. 升级时暂停战斗，弹出三选一面板
4. 本轮实现约 12 个升级项，覆盖：
- 主英雄强化
- 分身强化
- 混合强化

要求：
- 选项说明要简洁可读
- 选择后立即生效
- 本轮不要做 Boss、精英和复杂稀有系统
```

## 五、推荐喂给 GPT 的节奏

推荐不要一口气连跑 6 轮。

建议节奏：

1. 先跑第1轮，确认基础框架可运行
2. 再跑第2轮，亲自试玩 1 分钟
3. 再跑第3轮，确认“分身替代塔”的体感是否成立
4. 只有第3轮成立，才继续第4轮和第5轮
5. 第6轮只在核心体验成立后再做

原因：

- 第3轮才是这个方向的生死点
- 如果第3轮就感觉“不像守线”或“太乱”，后面再堆升级和 Boss 意义不大

## 六、最重要的判断点

在试玩过程中，重点只看这 4 个问题：

1. 玩家能不能快速理解“分身 = 阵地单位”
2. 主英雄会不会太忙
3. build 是否足够清楚
4. 它更像“分身守线”，还是更像“幸存者刷怪”

如果前两条不成立，就先别继续加内容。  
先回去改主循环和分身逻辑。
