## 1. 项目摘要

### 1.1 一句话定义

- 一款以“国王带队守线”为核心体验、结合 War3 守家味道、局内 Build、短程打野和神器跃迁的中度 RPG 塔防产品。

### 1.2 立项结论

- 立项建议：`可试做`
- 核心理由：
  1. 当前塔防赛道在移动端和小游戏端仍有稳定需求，但主流产品明显分化为 `轻量模板化塔防` 与 `单英雄割草/僵尸like` 两大方向，中间存在“更有角色代入感、又不至于太重”的中度守线空档。
  2. 项目核心体验“守线 + 带队 + Build + 短程打野”具备明确识别度，不是纯复刻 War3，也不是《向僵尸开炮》式单英雄站桩。
  3. 商业主核已相对聚焦在 `国王 / 随从 / 神器` 三条线上，避免了主角、防线、建筑多核并立导致的付费锚点发散。
- 最大机会点：把 War3 守家与 RPG Build 的熟悉感，翻译成更适合短局、买量和手游节奏的“现代中度守线产品”。
- 最大风险点：题材和玩法表达如果不够直给，容易在买量端被用户误识别为“普通割草”或“普通塔防”，导致新意无法被一眼感知。

### 1.3 项目概览

- 目标赛道：中度塔防 / RPG塔防 / 守线 Build 产品
- 目标平台：优先手游与小游戏可转化框架，后续可评估海外移动
- 目标用户：喜欢 War3 守家、Build、守线、轻 RPG 成长和男性向数值反馈的中度策略用户
- 核心玩法：国王带领随从守住基地，通过三选一成长、神器构筑和前场打野完成单局 Build
- 核心题材：中世纪奇幻 / 王国最后防线 / 国王统御
- 核心破局点：`带队守线`、`前场打野`、`神器 Build 跃迁`

## 2. 找准切入点

### 2.1 市场时机判断

- 目标品类当前阶段：`红海中的结构性机会期`
- 结论：
  1. 塔防不是市场空白，但“中度守线 + 队伍感 + Build”的组合仍有切口。
  2. 小游戏与移动端中，轻量塔防和肉鸽塔防仍有活跃样本，说明用户需求未消失；问题不在“有没有需求”，而在“有没有足够新且足够好懂的表达”。
- 依据：
  1. 本地 2025 资料库中，移动端样本同时存在 `正中靶心`、`永远的蔚蓝星球`、`魔法之路`、`塔防精灵` 等不同方向，说明塔防需求仍在，但头部并不统一。
  2. 微信小游戏侧 `躺平发育` 等样本证明塔防依然能吃到大众传播，但大多靠极低理解成本和强题材钩子。
  3. PC/Steam 侧 2025 样本如 `Orcs Must Die! Deathtrap`、`Dungeon Warfare 3`、`Gnomes`、`ShapeHero Factory` 的存在，说明塔防的创新仍有空间，只是创新方向高度分化。
- 当前平台是否存在阶段性流量红利：
  - 有，但不是“塔防大盘红利”，而是 `题材包装 + 低理解成本 + 强短视频表现` 的结构性红利。
- 是否存在市场空白：
  - 存在“单英雄割草”和“纯摆塔模板”之间的中度空白，尤其是“有队伍、有守线、有 Build，但不做过重大地图和多线养成”的细分切口。

### 2.2 用户判断

- 目标用户是谁：
  1. 主核心：War3 守家、RPG守图、暗黑 Build、僵尸like、塔防用户重叠群体
  2. 次核心：喜欢轻操作、强反馈、装备或词条成长的中度男性向用户
  3. 泛用户：喜欢“最后防线”“王国防守”“带队作战”这类本能题材的休闲策略用户
- 年龄段：25-40 为核心，18-24 可作为新增量
- 性别倾向：男性为主，若美术与角色设计柔化可拓展中性用户
- 游戏偏好：Build、守线、刷成长、短局、策略感、扭转战局、轻社交
- 主要需求：
  - 杀时间：6-8 分钟单局可闭环
  - 成就感：从守不住到顶住 Boss 的明显跃迁
  - 代入感：我不是塔防旁观者，而是“国王/统帅”
  - 构筑乐趣：这把 Build 成型了
  - 风险收益：要不要前出抢奖励
- 用户迁移动机：
  - 现有轻塔防过于模板化
  - 现有割草塔防队伍感弱、守线感弱
  - 现有重度塔防或 War3 向产品太老、太重、太难转化

### 2.3 获客判断

- 预期获客成本区间：中等偏上，需要靠题材与玩法共同降低获客成本
- 是否适合走买量：适合，但前提是素材能明确讲清“国王带队守线”和“守线前出打野”的独特性
- 题材是否具备低成本吸量潜力：
  - 中世纪奇幻 + 王国最后防线有天然理解优势，但不属于纯题材红利型，需要靠玩法共同成立
- 画风是否适合当前投放环境：
  - 应优先追求 `清晰、轻快、可识别的中世纪奇幻卡通化风格`，而不是纯硬核暗黑写实

### 2.4 阶段结论

- 当前切入成立，但不是靠“塔防赛道天然好做”，而是靠明确的细分组合切口成立。
- 成立前提：
  1. “带队守线”的视觉感知必须非常强
  2. 打野奖励必须真能改变 Build
  3. 单局节奏必须在 6-8 分钟内完成明显起承转合
- 如果这三点有任一点做不出来，该项目会退化为普通守线产品，竞争力会显著下降。

## 3. 竞品与缺口分析

### 3.1 竞品池范围

- 竞品池定义：
  1. 直接竞品：中度塔防 / RPG塔防 / 守线 Build 产品
  2. 邻近竞品：单英雄割草塔防、多人轻塔防、War3 守家系产品
- 样本筛选规则：
  - 能代表当前市场主流或潜在用户认知的产品优先

### 3.2 TOP3 竞品拆解

#### 竞品A：正中靶心

- 核心循环：守怪潮 -> 选技能/词条 -> 继续守更高压波次
- 长期留存依赖：局内随机成长、短局重复游玩、轻量 Build
- 商业化模式：中度手游常见成长与活动驱动
- 核心卖点：理解成本低、战斗爽点直给、抖音传播强
- 明显短板：单英雄体验重，队伍感弱，守线战略层较浅
- 可借鉴点：短局节奏、素材表达、肉鸽成长反馈

#### 竞品B：魔法之路

- 核心循环：守线/刷敌 -> 选 Build -> 强化装备与技能 -> 推更高强度
- 长期留存依赖：暗黑词条、Build 深度、成长兑现
- 商业化模式：手游成长、活动、礼包
- 核心卖点：Build 感强，男性向数值反馈强
- 明显短板：塔防表达容易被割草表达覆盖，策略层可能被战斗爽感吃掉
- 可借鉴点：神器/词条包装、成长跃迁感、男性向表达

#### 竞品C：Orcs Must Die! Deathtrap

- 核心循环：布置陷阱 -> 手操战斗 -> 获取成长 -> 抗更高压关卡
- 长期留存依赖：动作反馈、合作、roguelite 元进程
- 商业化模式：买断制，商业模型不直接可复用
- 核心卖点：动作与塔防融合、强反馈、合作乐趣
- 明显短板：平台与商业模型不适合直接迁移到手游/小游戏
- 可借鉴点：角色主动参与守线、陷阱/系统联动、Boss 压迫感

### 3.3 竞品共性与缺口

- 同类产品共性优势：
  1. 用户都能快速理解“敌潮 -> 防守 -> 成长”
  2. 守线类天然适合做爽点反馈和素材
  3. Build 与词条是长期留存的重要支点
- 同类产品共性问题：
  1. 单英雄产品队伍感弱
  2. 轻量塔防产品深度不足
  3. 重度塔防产品获客难、表达慢
  4. 纯模板合成塔防缺少长期记忆点
- 是否存在明显短板：
  - 存在，主要集中在 `题材套路化`、`体验模板化`、`队伍感不足`、`Build 记忆点弱`
- 这个缺口是否足以让用户迁移：
  - 有机会，但前提是我们能把“国王带队守线”的差异化在前 10 秒内讲明白

### 3.4 阶段结论

- 我们的切口是：
  - 用 `国王带队守线` 替代 `单英雄守门`
  - 用 `短程前场打野` 替代纯站桩守线
  - 用 `神器驱动 Build 跃迁` 替代纯数值升级
- 为什么这个切口值得做：
  - 它兼顾了用户熟悉感和陌生感：看起来像懂的守线产品，玩起来又有明确新意

## 4. 破局点分析

### 4.1 胜率来源判断

- 主要靠什么赢：`靠产品 + 靠发行表达`
- 核心破局点：
  1. 带队守线的识别度
  2. 前场打野的 War3 味
  3. 神器 Build 的单局跃迁感

### 4.2 如果靠发行

- 是否存在明显地缘套利空间：暂不作为主结论，先以通用中世纪奇幻表达覆盖国内与潜在海外
- 是否存在平台套利空间：
  - 有，小游戏和手游端都能承接，但前提是轻操作和短局感要做出来
- 素材套利空间是否明显：
  - 有潜力，素材可以同时做 `最后防线`、`国王带队`、`前出抢神器`、`Boss压门` 四种方向
- 研发速度是否足够快：
  - 若做 MVP，可控；若一开始追求大而全系统，会错失窗口
- 产品综合体验是否过关：
  - 当前设计上可成立，但必须通过白模验证

### 4.3 如果靠产品

- 差异化类型：`组合创新`
- 超长板是什么：
  - 把守线、队伍、Build、短程打野压缩进单局 6-8 分钟里
- 用户是否能一眼感知到新意：
  - 能，但必须明确看到“国王 + 队伍 + 基地 + 怪潮 + 前场奖励点”同屏
- 竞品是否难以短期复制：
  - 中等。概念可抄，但节奏组织、队伍感、风险收益和 Build 密度不容易短时间一起抄明白

### 4.4 破局点逐条论证

#### 破局点1：国王带队守线

- 破局点描述：玩家主控国王，随从自动协同作战，基地与怪潮同屏，形成明确“我在带队守住最后防线”的感受。
- 为什么能吸引用户：
  - 比纯摆塔更有角色代入感
  - 比单英雄割草更有队伍感和战场感
- 为什么能促成迁移：
  - 对现有塔防用户来说更有新鲜度
  - 对 Build 用户来说比传统塔防更有角色感
- 为什么能支撑留存：
  - 队伍变化、随从组合和神器联动能带来更多复玩空间
- 风险：
  - 如果同屏信息过乱，用户会只看到“乱战”，看不到“统御感”

#### 破局点2：前场短程打野

- 破局点描述：在守线节奏中安排近场高价值事件，玩家需要冒风险前出处理，以换取高质量成长或神器。
- 为什么能吸引用户：
  - 有明确的风险收益抉择，不是纯站桩
- 为什么能促成迁移：
  - War3 守家用户会感到熟悉
  - 一般塔防用户会感到“这把多了能抢的内容”
- 为什么能支撑留存：
  - 每把事件位置、奖励和时机不同，能提升复玩
- 风险：
  - 如果事件打断主线节奏，会让玩家觉得累赘

#### 破局点3：神器驱动的 Build 跃迁

- 破局点描述：局内通过普通三选一、高质量三选一和神器，显著改变国王、随从或二者联动方式。
- 为什么能吸引用户：
  - Build 成型的爽点直给
- 为什么能促成迁移：
  - 对现在大量肉鸽和 Build 用户是熟悉语言
- 为什么能支撑留存：
  - “这把是什么流派”是复玩关键
- 风险：
  - 如果神器只加数值、不改体验，会很快失去记忆点

## 5. 吸引力验证

### 5.1 题材向素材验证

- 验证方式：静态概念图 / 动态模拟镜头 / 题材海报
- 素材方向：
  1. 国王站在最后防线前指挥军队
  2. 中世纪奇幻怪潮压门
  3. Boss 压境与城墙危机
- 预期结论：
  - “最后防线 + 国王统御”比纯奇幻塔防更容易讲清角色身份

### 5.2 玩法向素材验证

- 验证方式：AI 白模录屏 / 原型战斗短视频
- 素材方向：
  1. 国王前出抢高价值目标
  2. 三选一后队伍明显变强
  3. 神器让技能或随从发生质变
- 预期结论：
  - 单纯玩法向素材必须把“带队守线”拍明白，否则容易被误判成普通割草

### 5.3 题材+玩法综合验证

- 验证方式：素材 A/B 测试
- 素材方向：
  1. 王国最后防线 + 国王带队 + 怪潮压门
  2. 前出打野抢神器 + 回防顶 Boss
- 预期结论：
  - 综合向素材应是最优解，因为它同时解释了题材和差异化玩法

### 5.4 综合判断

- 哪类素材表现最好：预判为 `题材+玩法综合向`
- 素材表现是否稳定：待验证
- 素材拓展空间是否足够大：足够大，可扩为 `国王技能`、`随从流派`、`神器构筑`、`Boss 压门`、`打野抢件`
- 是否值得继续投放验证：值得

## 6. 可玩性验证

### 6.1 当前验证方式

- 验证方式：`AI白模 + 轻玩法原型`
- 验证目标：
  1. 玩家能否理解带队守线
  2. 短程打野是否爽
  3. 单局 6-8 分钟是否足够成型
- 参与测试人数：待定
- 测试对象：内部策划、美术、制作、外部目标用户小样本

### 6.2 核心体验判断

- 实操体验是否好玩：
  - 推断：如果国王操作感、随从协同感和怪潮压迫感同时成立，会有较高可玩性
- 最强爽点：
  - 国王技能 + 随从体系 + 神器联动一起顶住 Boss 的时刻
- 最明显问题：
  - 若队伍行为不够清晰，玩家会失去统御感
- 用户是否愿意继续玩第二局：
  - 决定因素不是关卡内容，而是“第二局能不能做出完全不同的 Build”

### 6.3 玩法拓展空间

- 核心玩法拓展空间：中高
- 是否支持中期成长：支持，国王、随从、神器可组成稳定中期骨架
- 是否支持长线内容：支持，但要谨慎控制不要做成多核大系统
- 是否存在快速内容枯竭风险：存在，如果神器和随从联动不足，流派数量会很快见顶

### 6.4 结论

- 当前玩法值得继续做 MVP
- 下一轮需要验证：
  1. 队伍自动战斗是否足够聪明
  2. 玩家是否会主动追打野事件
  3. Build 成型是否足够被感知

## 7. 落地能力评估

### 7.1 产品设计难度

- 核心玩法设计难度：中高
- 关键系统设计难度：中
- 核心风险点：
  1. 队伍 AI 与战场清晰度
  2. 神器/三选一的可读性和差异度
  3. 打野事件与主线节奏的平衡
- 团队是否有对应经验：
  - 若团队具备塔防、RPG成长、轻 roguelike 或多单位战斗经验，则可以承接

### 7.2 技术落地能力

- 主要技术难点：
  1. 多单位同屏与性能控制
  2. 随从自动行为逻辑
  3. 怪潮、Boss、事件点与路径管理
- 现有技术栈是否覆盖：理论上常规手游引擎可覆盖
- 是否需要专项预研：需要，小规模验证多单位、技能特效与路径表现
- 技术风险等级：中

### 7.3 美术与内容难度

- 美术风格难度：中
- 资源生产压力：中
- 内容生产压力：中高
- 是否依赖高成本表现：
  - 不应依赖高保真，核心应是清晰、可读、同屏层次明确

### 7.4 资源投入评估

- 预计人数：8-15 人可完成 MVP 到验证版
- 预计周期：8-12 周完成首轮白模到可测原型
- 核心岗位：主策、战斗策划、数值策划、客户端、TA/技术、美术、UI、发行素材支持
- 预估成本：中
- 是否在公司资源带宽内：若以 MVP 思路推进，在带宽内；若一开始做大世界、多线局外和复杂防线，将超带宽

### 7.5 综合落地结论

- 能否落地：能，但必须先做小而清晰的 MVP
- 最大落地风险：试图一次性做成“大而全 War3 守家手游”
- 建议资源方案：优先验证核心战斗环，不先做重局外

## 8. 风险与止损条件

### 8.1 市场风险

- 风险描述：素材无法讲清与现有塔防/割草产品的差异
- 风险等级：高
- 应对方式：前置做多方向素材测试

### 8.2 产品风险

- 风险描述：带队守线最终只剩表面概念，没有真实统御感
- 风险等级：高
- 应对方式：优先验证国王、随从、防线三者关系

### 8.3 研发风险

- 风险描述：多单位同屏 + 自动行为复杂，导致开发周期拖长
- 风险等级：中
- 应对方式：先做 1 国王 + 6-8 随从 + 1 Boss 的最小闭环

### 8.4 投放风险

- 风险描述：题材不弱，但玩法表达过重，导致点击和转化不稳定
- 风险等级：中高
- 应对方式：题材向、玩法向、综合向三套素材并行

### 8.5 止损条件

- 当 `题材+玩法综合素材无法形成明显优于普通塔防素材的点击优势` 时，停止扩大验证
- 当 `白模中玩家无法在第一局理解带队守线与打野风险收益` 时，调整核心玩法
- 当 `Build 成型感不明显，第二局复玩意愿弱` 时，暂停正式立项推进

## 10. 最终结论

### 10.1 立项判断

- 是否建议立项：建议进入 `MVP验证`
- 建议立项级别：`可试做`

### 10.2 必须回答的三个问题

- 为什么现在做：
  - 因为塔防赛道虽然不再是纯红利期，但“中度守线 + 带队 + Build”的细分空档仍存在，且当前平台依然奖励易传播、短局、有角色感的产品。
- 为什么我们能赢：
  - 因为项目不是单点创新，而是围绕“国王带队守线”形成了完整识别链：题材、玩法、事件、Build、Boss 收束。
- 为什么我们做得出来：
  - 因为核心环可以先收敛到 1 国王、6-8 随从、1 防线、1 Boss、几次三选一和 1-2 个打野事件，不必一开始做成大而全项目。

### 10.3 最终建议

- 建议结论：
  - 继续推进，但必须严格按 MVP 路线验证，不要提前膨胀到重度大系统
- 建议下一步：
  1. 做战斗白模
  2. 做素材方向测试
  3. 做首轮 10-20 人目标用户定性反馈
