## 一、合并结论

四个方向可以合并，但不能并列合并。

正确做法不是：

- 同时做 `单王守线`
- 同时做 `国王布防`
- 同时做 `国王打宝`
- 同时做 `国王技能流`

因为这会导致：

- 玩家身份混乱
- 高频动作超标
- 中段爽点来源冲突
- 主循环失焦

当前最优合并方式是：

- `单王守线` 作为操作身份层
- `国王打宝` 作为主循环差异层
- `国王技能流` 作为成长与爽点层
- `国王布防` 作为轻托底层

一句话定义：

`玩家只控制国王本人，在中央防线守城；波次里抓短窗口前压打宝，拿回技能神器后回防顶波次，并依靠少量固定防线位托底，最终击败Boss。`

## 二、合并后的正式纸面原型

### 1. 一句话定义

玩家扮演独自守住中央城门的国王，在怪潮压力中短程前压争夺高价值掉落，拿回技能神器后完成个人 Build 跃迁，并借助轻量布防托底，最终顶住 Boss。

### 2. 20字核心循环

`守线 -> 前压打宝 -> 拿技能神器 -> 回防 -> 打Boss`

### 3. 核心体验

- 我是独自守门的国王，目标清楚
- 我不是一直钉在门口，而是会抓窗口出去抢关键件
- 我拿到的奖励会明显改变技能形态和处理方式
- 我有轻防线托底，但真正决定胜负的还是国王自己
- Boss 来时，我会检验整局的出击、成长和回防是否合理

### 4. 本次只验证的假设

- 假设 1：`单王守线 + 轻布防` 能提供足够清楚的守城目标
- 假设 2：`短程打宝` 能进入主循环，而不是变成离岗行为
- 假设 3：`技能神器` 能成为单局中段最强爽点

### 5. 本次不验证的内容

- 小队系统
- 多职业系统
- 自由建塔
- 多地图与章节
- 局外养成

### 6. Kill 条件

- 轻布防戏份过重，国王存在感下降
- 打宝行为与守线行为像两套玩法
- 技能神器只加数值，不改处理方式
- 删除打宝系统后，玩法仍几乎不变

## 三、最小系统模型

### 战场

只做 `1张横屏单战区地图`：

`前场事件区 -> 交战区 -> 中央防线 -> 城门`

### 玩家侧要素

只保留 4 个核心战斗件：

1. `国王`
2. `箭塔`
3. `路障`
4. `陷阱位`

约束：

- 国王是绝对主角
- 防线只负责托底，不负责主输出
- 所有前场事件都必须在离开主防线 `3-5 秒` 内完成

### 资源

只保留：

- `城门耐久`
- `升级经验`

### 成长

只保留：

1. `普通三选一`
2. `技能神器`

## 四、玩家动作集

### 高频动作

- 控制国王移动
- 释放国王技能
- 选择强化 / 拾取神器

### 中频动作

- 判断这次要不要前压打宝
- 决定本局技能 Build 往哪边偏

### 低频动作

- 临时补防线
- 打断 Boss 关键动作

### 判断

- 高频动作为 3 个，没有超约束
- 关键决策为 2 个，仍然稳定
- 玩家在 30 秒内能理解自己在“守中央城门”

## 五、单局时间脚本（7-8分钟）

### 0:00 - 1:30 建立守线目标

#### 玩家看到什么

- 中央城门和轻防线槽清晰可见
- 第一波普通怪推进
- 国王需要主动清怪，轻防线只能补下限

#### 玩家做什么

- 调整站位
- 释放初始技能
- 完成第一次普通三选一

#### 系统给什么

- 城门受损反馈
- 第一轮小成长

#### 这一段的目的

- 建立“单王守城”的目标感
- 明确防线不是主角

#### 风险点

- 如果箭塔和路障太强，会误判成塔防游戏

### 1:30 - 3:30 第一次前压打宝

#### 玩家看到什么

- 前场刷新 `神器祭坛`
- 主线怪潮继续推进

#### 玩家做什么

- 判断当前城门和轻防线能否撑住
- 前压 3-5 秒拿第一件技能神器

#### 系统给什么

- 第一件技能神器
- 拿完后不久出现第一只精英怪

#### 这一段的目的

- 验证打宝行为是否自然进入主循环
- 验证第一件神器能否立刻改玩法

#### 风险点

- 如果神器只加伤害，打宝价值会不足

### 3:30 - 5:30 Build 分化与第二次争夺

#### 玩家看到什么

- 技能形态已有变化
- 中线压力提升
- 第二次前场事件 `宝箱怪` 刷新

#### 玩家做什么

- 基于当前技能流派决定是否二次前压
- 平衡贪收益和守线压力

#### 系统给什么

- 第二件技能神器或强三选一
- Boss 前最后一次 Build 收束

#### 这一段的目的

- 验证第二次争夺是否仍然有吸引力
- 验证技能 Build 是否已明显分化

#### 风险点

- 如果第二次争夺吸引力不够，主循环会在中段变平

### 5:30 - 8:00 Boss 收束

#### 玩家看到什么

- Boss 压线
- Boss 会冲撞路障、召分流怪、读条重击城门

#### 玩家做什么

- 回防
- 用技能链和站位处理 Boss
- 在必要时补防线或用陷阱拖时间

#### 系统给什么

- 最终收束
- Boss 结算占位

#### 这一段的目的

- 检验单王守线、打宝收益、技能 Build 和轻布防是否形成闭环

#### 风险点

- 如果 Boss 主要由塔位解决，主角会失焦

## 六、首批神器方向

只保留技能形态神器，不做纯面板神器。

### 1. 王令连斩

- 冲锋后触发回身双斩

### 2. 陨火敕令

- 直线火球改为穿透陨石

### 3. 王怒追击

- 击杀快跑怪后技能冷却缩短

### 4. 守城余烬

- 路障被破坏后留下燃烧带

### 5. 断罪号角

- 打断 Boss 重击后触发一次范围爆发

### 6. 前线旗印

- 国王离开主防线时，箭塔短时间攻速提升

原则：

- 神器必须改变技能使用时机、攻击范围或回防方式
- 不接受 `攻击+20%` 作为主成长件

## 七、对四个分支的分析

### 1. 单王守线的价值与问题

价值：

- 身份最清楚
- 操作最轻
- 最容易做最小 Demo

问题：

- 差异点不足
- 非常容易退化成常见守线成长

结论：

- 适合作为合并版的 `身份层`
- 不适合单独作为最终差异化方向

### 2. 国王布防的价值与问题

价值：

- 守城感最强
- 用户最容易第一眼看懂
- 有利于提供前压时的托底容错

问题：

- 如果防线系统一重，国王就会被边缘化
- 很容易看起来像轻塔防

结论：

- 适合作为合并版的 `托底层`
- 不适合单独做主核

### 3. 国王打宝的价值与问题

价值：

- 差异点最强
- 最接近 War3 守家里的关键记忆点
- 最有机会成为主循环中的“为什么这产品不一样”

问题：

- 如果距离稍远，就会变成跑图
- 如果收益不够香，就会变成伪选择

结论：

- 适合作为合并版的 `主差异层`
- 必须保留，不能削成装饰

### 4. 国王技能流的价值与问题

价值：

- 单局爽点最直观
- 最容易在 8 分钟内做出成长弧线
- 与打宝奖励结合最自然

问题：

- 如果只剩技能变强，产品辨识度仍会变弱

结论：

- 适合作为合并版的 `成长与爽点层`
- 必须和打宝绑定，不能单独悬空

## 八、为什么这样合并

这套合并不是平均主义，而是按职责收编：

| 分支 | 在合并版里的职责 |
|---|---|
| 单王守线 | 提供玩家身份和目标感 |
| 国王布防 | 提供守城托底和容错 |
| 国王打宝 | 提供主循环差异点 |
| 国王技能流 | 提供中段爽点和 Build 分化 |

这样合并后的好处是：

- 玩家身份清楚
- 主循环仍只有一条
- 爽点来源明确
- 各分支优点被保留，缺点被压低

## 九、继续与回滚条件

### 继续当前合并版的条件

- 玩家能清楚感知自己是“单王守城”
- 两次打宝都成立，且不是伪选择
- 第一件技能神器拿到后，处理方式明显变化
- Boss 战里国王仍是绝对主角

### 需要局部回调的条件

- 轻布防存在感过强
- 第一件神器强度不足
- 第二次打宝吸引力弱于第一次

### 需要拆回分支验证的条件

- 打宝系统怎么调都像额外事件，不像主循环
- 技能神器怎么调都无法形成明显中段爽点
- 单王守线本身支撑不起目标感和压力感

一句话总结：

`四方向合并后，最合理的新原型不是“四核并立”，而是“单王守线打宝技能流”，再用极轻的布防系统托底。`
