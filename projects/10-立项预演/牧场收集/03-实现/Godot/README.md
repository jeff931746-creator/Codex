# Godot 闭环演示

这里是“牧场收集”立项方向的 Godot 二维闭环演示。

当前版本验证：

`能量消耗 -> 防守战斗 -> 碎片掉落 -> 孵化/升级伙伴 -> 战力提升 -> 区域解锁/能量回流`

## 运行方式

1. 用 Godot 4.x 打开本目录。
2. 运行主场景 `res://scenes/Main.tscn`。
3. 在主界面按阶段提示进入战斗。
4. 战斗中用方向键移动，按住空格或鼠标左键攻击。
5. 胜利回到主界面后，用碎片孵化或升级伙伴。

更详细的实现说明见：

- `docs/implementation_notes.md`

## 当前结构

```text
Godot/
  project.godot
  scenes/
  scripts/
  autoload/
  assets/
  configs/
  ui/
  docs/
```

## 目录边界

- `project.godot`：项目配置入口。
- `scenes/`：所有场景文件。
- `scripts/`：所有脚本。
- `autoload/`：全局单例与事件总线。
- `assets/`：图片、音频、字体、特效等资源。
- `configs/`：数值、节奏、卡包、关卡等配置。
- `ui/`：界面场景和战斗信息层。
- `docs/`：实现说明、输入映射、导出说明、节点约定。

## 实现约定

- 一个核心玩法至少对应一个独立场景或脚本边界。
- 配置优先外置，避免把节奏、波次、卡牌和商店价格写死在脚本里。
- 用 `autoload` 管全局状态，但不要把所有逻辑都堆进去。
- 场景树和节点命名尽量固定，避免后续引用路径不稳定。
