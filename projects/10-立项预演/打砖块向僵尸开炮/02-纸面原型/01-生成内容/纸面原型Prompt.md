# 打砖块向僵尸开炮 纸面原型 Prompt

用途：直接给 GPT 生成一个 16:9 的可玩 demo 描述稿，或作为原型 UI/交互设计说明。

---

## 一、原型目标

做一个 16:9 横版战场的纸面原型，核心验证两件事：

1. 打砖块不是刷分，而是会改变炮弹弹道的战场结构。
2. 僵尸不是装饰敌人，而是会持续压线、逼迫玩家切换目标的时间压力。

这一版 demo 要让人一眼看懂：

- 玩家是底线炮手
- 炮弹会反弹、穿透、分裂
- 砖块被打碎后会改写后续弹道
- 僵尸会从上方或侧边压下来
- 玩家在“破砖”和“清僵尸”之间做实时取舍

---

## 二、画面规格

- 画面比例：`16:9`
- 建议分辨率：`1920x1080`
- 视角：`固定横版战场`
- 镜头：`全程不切镜头`
- 画风：`轻度休闲 + 清晰 UI + 强反馈`
- UI 要求：`战场信息一眼可读，不能花`

---

## 三、画面布局

### 1. 整体构图

画面分成三个层次：

1. 上半区：砖墙区和僵尸出生区
2. 中间区：主战场弹道区
3. 下半区：玩家炮台、底线、防守区

### 2. 具体布局

- 画面底部中央放玩家炮台
- 炮台左右各有一个简洁的角度辅助线
- 画面中上部铺满砖块阵列
- 砖块不是整齐纯平面，要有少量缺口，方便看出弹道变化
- 僵尸从画面上方或右上角持续出现，向下压线
- 底线附近有一条非常明显的“防守线”
- 右侧或左上角放简洁 HUD
- 中央上方放当前波次 / 剩余时间 / Boss 提示

### 3. 画面元素分层

#### 背景层

- 深色战场背景
- 轻微网格或科技纹理
- 不抢主体

#### 战场层

- 砖块阵列
- 僵尸单位
- 炮弹轨迹
- 反弹火花

#### 前景层

- 炮台
- 底线
- 数值飘字
- 掉落道具

#### UI层

- 生命值
- 当前波次
- 炮弹类型
- 强化选择
- 倒计时

## 四、核心视觉元素

### 1. 炮台

- 位置：底部中央
- 形态：简洁重炮或发射器
- 状态：
  - 待发射
  - 装填中
  - 强化后形态变化
- 发射时要有明显后坐力和光效

### 2. 炮弹

- 需要让轨迹非常清楚
- 飞行中要有尾焰或光轨
- 碰到砖块时要有清晰反弹火花
- 打碎砖块时要有碎裂粒子

### 3. 砖块

- 砖块是规则矩阵，但不要太整齐死板
- 砖块可分颜色或材质等级：
  - 普通砖
  - 强化砖
  - 反弹砖
  - 特殊砖
- 被打碎后应留下缺口，缺口会改变后续弹道

### 4. 僵尸

- 形象要明显像“压线敌人”
- 动作要朝底线推进
- 可以有普通僵尸、护甲僵尸、冲锋僵尸
- Boss 要足够大，压迫感强

### 5. 掉落强化

- 砖块打碎后掉落金币、弹药、强化卡
- 掉落物不需要复杂，但要明显可拾取

## 五、玩法说明

### 玩家高频动作

1. 观察砖墙和僵尸位置
2. 调整发射角度
3. 发射炮弹
4. 在三选一里选择强化

### 玩家关键决策

1. 当前这一发优先破砖还是优先清僵尸
2. 当前强化选择更偏弹射还是更偏守线

### 基础规则

- 炮弹会在砖块之间反弹
- 炮弹可以打碎砖块
- 砖块被打碎后会改变弹道空间
- 僵尸会持续下压
- 僵尸到达底线即危险
- 玩家必须一边改写弹道，一边阻止僵尸突破

## 六、单局脚本

### 开局 0:00 - 1:00

- 玩家看到完整砖墙和第一波僵尸
- 玩家先试探角度发射
- 系统展示第一次破砖和反弹爽感

### 中段 1:00 - 3:00

- 僵尸数量增加
- 砖墙出现缺口
- 玩家开始明显权衡：
  - 打砖开路
  - 还是优先清怪保线
- 弹出第一次三选一强化

### 中后段 3:00 - 5:00

- 战场被逐步打开
- 僵尸压线变强
- 玩家通过强化让炮弹出现变化
- 形成一次明显 Build 方向

### 收束段 5:00 - 8:00

- Boss 或高压波次出现
- 玩家用成型炮弹路线清掉关键敌人
- 结束时给出胜利/失败结算
