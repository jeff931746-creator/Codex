# 项目区

`projects/` 只放项目和项目模板，不包含任何标准或工具。

## 目录结构

- `10-立项预演/`：立项评估阶段的项目，阶段制管理（00-05）
- `_正式项目模板/`：立项通过后的正式开发项目模板，部门制管理（策划/开发/美术 + 输入/输出）

部门能力标准在 `tools/部门标准/`，工作流链路规则在 `.claude/rules/workflow-chain.md`。

## 立项预演标准结构

每个立项预演项目统一采用：

```
项目名/
  00-立项文档/
  01-评审/
  02-纸面原型/
```

- `00-立项文档/`：立项文档正文、玩法模型、GDD 骨架等正式论证材料
- `01-评审/`：初审稿、评审意见、评审规则和决策记录
- `02-纸面原型/`：纸面原型、概念图 prompt、辅助脚本、探索性输出

## 使用流程

1. 新点子先进入 `10-立项预演/`，按立项预演标准结构推进
2. 执行专项工作前先查 `tools/部门标准/{部门}/`，有标准按标准做
3. 立项评审通过后，复制 `_正式项目模板/` 创建正式项目目录
4. 可复用的方法、规则和模板上收到 `tools/部门标准/`，不在项目内重复维护
