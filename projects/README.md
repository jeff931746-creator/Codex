# 项目区

`projects/` 只分三类内容：

- `00-共享能力/`：跨项目复用的方法、规则、角色提示词、流程资产
- `10-立项预演/`：还在做方向探索、评审准备和立项论证的项目

当前默认只保留前两类目录。
如果后续有项目明确进入正式制作，再按需要单独创建开发目录。

## 共享能力分类原则

`00-共享能力/` 下只保留两类：

- `立项/`：立项阶段复用的评审模板、角色包、方法论
- `开发/`：开发阶段复用的策划、规则、执行流程、协作资产

注意：

- `共享能力` 下不再使用 `立项预演/` 目录
- `策划` 相关内容统一归到 `00-共享能力/开发/`

## 立项预演标准结构

每个立项预演项目统一采用：

```text
项目名/
  00-立项文档/
  01-评审/
  02-纸面原型/
```

- `00-立项文档/`：立项文档正文、玩法模型、GDD 骨架等正式论证材料
- `01-评审/`：初审稿、评审意见、评审规则和决策记录
- `02-纸面原型/`：纸面原型、概念图 prompt、辅助脚本、探索性输出

## 使用流程

1. 新点子先进入 `10-立项预演/`
2. 项目内部按 `立项文档 -> 评审 -> 纸面原型` 归档
3. 如果项目进入正式制作，再按需要为该项目单独拆出开发目录
4. 能复用的方法、规则和模板再上收进 `00-共享能力/`
5. 做项目拆解时，先套用 `research/资料/项目拆解模板.md`，再往项目文档里写结论
