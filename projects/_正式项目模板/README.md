# 正式项目模板

立项通过后，正式开发阶段的项目目录结构模板。

工作流全链路规则详见 [`.claude/rules/workflow-chain.md`](/Users/mt/Documents/Codex/.claude/rules/workflow-chain.md)。

## 使用时机

当 `10-立项预演/{项目名}/01-评审/` 给出"通过"结论后：

1. 在 `projects/` 下新建 `{项目名}/` 目录
2. 复制本模板的目录结构（含 `.gitkeep` 占位）
3. 将立项阶段文档从 `10-立项预演/{项目名}/` 迁入 `{项目名}/立项/`
4. 开始按部门推进开发

## 目录结构

```
{项目名}/
  立项/                  归档：立项阶段全部文档（从 10-立项预演 迁入）
  策划/
    输入/
      立项文档/          来自立项阶段的核心定义
      竞品参考/          参考来源：research/资料库/竞品库/
      机制库引用/        参考来源：research/资料库/机制库/
      用户调研/
    输出/
      系统设计/
      玩法规则/
      数值框架/
      GDD/               Game Design Document
  开发/
    输入/
      策划文档/          来源：策划/输出/
      技术规范/
    输出/
      核心模块/
      配置表/
  美术/
    输入/
      策划需求/          来源：策划/输出/
      风格参考/
    输出/
      概念图/
      UI设计/
      图标资产/
```

## 部门间依赖

```
策划/输出/ → 开发/输入/策划文档/
策划/输出/ → 美术/输入/策划需求/
research/资料库/ → 策划/输入/（只读引用，不复制文件）
```

## 能力标准

执行各部门工作前，先查 `projects/00-共享能力/{部门}/`。无标准时停止，告知用户。
