# 纸面原型

## 阶段定位

这一阶段是 `01-立项评审` 通过后的下一步。

它的任务不是继续发散方向，而是把已经通过评审的玩法骨架压成：

- 可内部讨论
- 可纸面推演
- 可版本迭代
- 可被推翻
- 可决定是否进入白模、灰盒或可玩原型

## 进入条件

- `方向卡` 已通过
- `母型卡` 已通过
- `参与方式卡` 已通过
- `核心机制卡` 已通过
- `01-立项评审` 已完成，且玩法循环结论为通过

## 本阶段输入

- 前五步的结论
- 当前版本约束
- 想验证的 2-3 个关键假设
- 若是迭代版，还需要上一版纸面原型结论和本轮改动点

## 本阶段产出

- 一版 `5-8 分钟` 的单局纸面原型
- 最小系统表
- 三轮纸面推演
- 问题清单
- 版本迭代评估
- 下一步动作和退回路径

## 在立项预演项目中的落盘规则

如果纸面原型内容需要落到 `立项预演` 项目目录中，默认按两类输出分开：

1. `02-纸面原型/01-生成内容/`
2. `02-纸面原型/02-评审结果/`

说明：

- `01-生成内容/`：放纸面原型正文、概念图 prompt、辅助脚本、规则参考、参考输出和探索材料
- `02-评审结果/`：放纸面原型评审、打分、结论、回滚判断和退回路径

## 本阶段结论

只能从以下结论中选择：

- `纸面原型可继续`
- `暂不进入灰盒，需要补一轮纸面推演`
- `继续，但必须按新结构重开 v2`
- `回滚到上一版`
- `退回上一阶段`
