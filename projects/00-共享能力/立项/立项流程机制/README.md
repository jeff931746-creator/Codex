## 常见使用场景

### 场景 1：新项目启动

1. 完成立项定义（项目基线、竞品、承诺）
2. 阅读 [00-流程总览.md](00-流程总览.md) 理解流程
3. 进入 [01-立项评审](01-立项评审/)
4. 根据评审结论选择下一步

### 场景 2：评审要求补证

1. 明确本轮要回答的闸门和阈值
2. 进入 [02-纸面原型](02-纸面原型/)
3. 完成补证后回到 [01-立项评审](01-立项评审/) 复评

### 场景 3：需要补充题材信息

1. 进入 [补证支线-题材选择](补证支线-题材选择/)
2. 完成题材分析
3. 回到 [01-立项评审](01-立项评审/) 复评相关闸门

## 结构说明

- `01-立项评审/`：立项阶段的决策节点，用来决定项目是 `进入纸面原型补证 / 补其他证据再评 / 进入小测 / 终止`
- `02-纸面原型/`：立项补证节点，用纸面原型或轻灰盒回答 `01-立项评审` 尚未通过的原型闸门
- `原型立项闸门表.md`：面向买量产品的早期闸门，判断的是”是否具备进入小测的资格”，不是直接判断 `ROI` 结果
- `长线经营假设卡.md`：立项时用于写清长线 `ROI / 留存` 的经营模型、内容引擎、验证节点和止损条件
- `补证支线-题材选择/`：题材和风格相关的辅助材料，是可选前置输入，不是主流程的独立通过节点

## 执行原则

- 立项阶段不再直接把 `方向卡 -> 母型卡 -> 参与方式卡 -> 核心机制卡 -> 玩法循环卡` 当作最终评审结论；这些内容现在属于立项定义和补充分析材料。
- `01-立项评审` 只看两类东西：
  `原型立项闸门`
  `长线经营假设`
- 如果原型闸门证据不足，进入 `02-纸面原型` 做补证，再回到 `01-立项评审` 复评。
- 如果结论是 `补其他证据再评`，默认由项目目录的 `00-立项文档/` 或本目录的 `补证支线-题材选择/` 承接补证，不额外拆独立共享阶段。
- `02-纸面原型` 不直接宣告 `ROI 成立`、`长线留存成立` 或”可以大做”；它只负责补证。
- 真实短线数据由后续小测验证；真实长线结果由小测、扩量和运营阶段继续验证。
