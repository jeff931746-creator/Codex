# 原型立项闸门表

## 使用边界

- 本表适用于以买量为核心的内购产品。
- 本表用于判断项目是否具备 `进入小测` 的资格。
- 本表不直接验证 `长线 ROI`、`长线留存` 或真实 `LTV` 结果。
- 本表验证的是两类前提：
  `买量回收前提`
  `付费留存前提`

## 使用方式

- 在项目目录的 `00-立项文档/` 里先写清项目基线和本轮目标。
- 在 `01-立项评审/` 里按本表逐项核验。
- 如果某些闸门证据不足，进入 `02-纸面原型/` 做补证，再回到评审会复评。

## 六个闸门

### 1. 三秒买点

- 要回答的问题：
  用户看到首屏或短素材时，能不能立刻看懂“这游戏玩什么、爽点是什么”。
- 必交产物：
  `15 秒素材脚本`
  `首屏录屏 / 序列图 / 关键帧`
- 最小通过阈值：
  找 `5` 位非项目成员，至少 `3` 位能在 `3` 秒内复述目标和爽点。
- 通过意味着：
  当前题材表达和买量承诺具备继续验证价值。
- 不通过时回退：
  回到题材表达、买量承诺或核心冲突设计。

### 2. 一分钟承接

- 要回答的问题：
  用户点进来后，首局能不能在 `1` 分钟内兑现素材承诺。
- 必交产物：
  `60 秒首局承接脚本`
  `可操作轻灰盒` 或 `明确交互流程图`
- 最小通过阈值：
  找 `5` 位测试者，至少 `4` 位无需口头说明就能完成关键操作，并在 `60` 秒内打出第一次爽点。
- 通过意味着：
  当前产品承接能力足以继续往下验证。
- 不通过时回退：
  回到核心交互、目标表达或第一次爽点设计。

### 3. 十分钟成型

- 要回答的问题：
  玩家在 `10` 分钟内能不能形成明确打法，而不是一直在试探。
- 必交产物：
  `10 分钟单局流程`
  `至少 3 套 build / 解法`
- 最小通过阈值：
  测试者结束后能说清“这局我是怎么赢的”以及“下一局我准备补什么”。
- 通过意味着：
  当前玩法已经具备中前期成型感和继续深化的空间。
- 不通过时回退：
  回到成长结构、分支差异或局内节奏设计。

### 4. 二刷目标

- 要回答的问题：
  首局结束后，玩家是否会主动想再开一局，而且知道下一局想验证什么。
- 必交产物：
  `首局测试记录`
  `下一局目标记录`
- 最小通过阈值：
  至少 `60%` 的测试者主动要求再来一局，或明确说出下一局想追求的目标。
- 通过意味着：
  当前方向具备基础复玩意愿。
- 不通过时回退：
  回到局内目标、结算反馈或差异化收益设计。

### 5. 首付理由

- 要回答的问题：
  玩家为什么会在当前体验里自然产生第一笔付费动机。
- 必交产物：
  `首个付费点草案`
  `付费场景说明`
  `局外承接草案`
- 最小通过阈值：
  评审会上能用一句话说清“玩家此刻为什么愿意付这笔钱”，且这笔钱是在放大当前欲望，不是在解除纯卡点。
- 通过意味着：
  当前项目存在自然生长的首付理由。
- 不通过时回退：
  回到核心欲望、局外承接或付费挂点设计。

### 6. 爽点成本

- 要回答的问题：
  当前核心爽点是否能用低成本、可重复的生产方式持续提供。
- 必交产物：
  `核心爽点拆解`
  `最小生产清单`
  `最高成本项说明`
- 最小通过阈值：
  去掉高规格包装后，核心爽点仍然成立；不存在必须依赖重内容、重数值堆料或高定制资源才能成立的关键节点。
- 通过意味着：
  当前方向具备继续验证 `ROI 前提` 的成本基础。
- 不通过时回退：
  回到爽点来源、内容密度或生产方案。

## 一票否决项

以下任一项成立，立即终止项目，不进入补证：

- **承接失败**：首局 `1` 分钟内，≥60% 测试者无法完成关键操作
- **成型失败**：`10` 分钟内，≥60% 玩家说不清自己在玩什么打法，也说不出下一局目标
- **付费结构性缺陷**：首个付费点只能依赖纯卡点、纯体力或纯数值焦虑，无法找到放大欲望的自然付费场景
- **爽点成本不可控**：去掉高规格包装后，核心爽点直接失效，或核心爽点必须依赖重内容、重数值堆料或高定制资源

## 决策规则

### 基础决策矩阵

| 通过数 | 补证数 | 不通过数 | 一票否决 | 决策 |
|--------|--------|----------|----------|------|
| 6 | 0 | 0 | 否 | **进入小测** |
| 5 | 1 | 0 | 否 | **进入纸面原型补证** |
| 4 | 2 | 0 | 否 | **进入纸面原型补证** |
| 4 | 1 | 1 | 否 | **补其他证据再评** 或 **退回立项定义** |
| 3 | 2 | 1 | 否 | **补其他证据再评** 或 **退回立项定义** |
| 3 | 3 | 0 | 否 | **补其他证据再评** 或 **退回立项定义** |
| ≤2 | - | ≥2 | 否 | **退回立项定义** 或 **终止** |
| - | - | - | 是 | **终止** |

### 决策说明

**进入小测**：
- 条件：所有核心闸门通过，且没有触发一票否决项
- 下一步：进入小测阶段，验证真实数据

**进入纸面原型补证**：
- 条件：1-2 个闸门证据不足，但方向值得验证，未触发一票否决项
- 下一步：进入 `02-纸面原型/` 做针对性补证，完成后回到 `01-立项评审` 复评
- 限制：连续 2 轮补证仍未通过 → 强制终止或退回立项定义

**补其他证据再评**：
- 条件：≥3 个闸门需要补证，或有 1 个闸门明确不通过但未触发一票否决
- 下一步：补充题材/竞品/发行信息等外部证据，回到 `01-立项评审` 复评相关闸门
- 判断标准：如果不通过的闸门属于方向性问题（如核心承诺不成立），应选择”退回立项定义”

**退回立项定义**：
- 条件：核心假设被证伪，需要重新设计方向
- 标志：冻结基线需要改动（目标用户、核心承诺、核心玩法等）
- 下一步：回到立项定义阶段，重新定义项目基线

**终止**：
- 条件：触发任意一票否决项，或 ≥2 个核心闸门明确不成立
- 下一步：项目终止，总结失败原因，归档材料

### 核心闸门定义

- 对买量型产品，以下为核心闸门：
  - 三秒买点
  - 一分钟承接
  - 十分钟成型
  - 二刷目标
  - 爽点成本
- 如果项目以内购为主，`首付理由` 也应视为核心闸门

## 说明

- 本表得到的是”是否值得进入小测”的结论，不是”已经证明会赚钱”的结论。
- 真实短线数据由后续小测验证；真实长线结果由小测、扩量和运营阶段继续验证。
