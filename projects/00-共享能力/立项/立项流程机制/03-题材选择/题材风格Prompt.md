# 游戏题材选择框架

## 使用说明

- 本文件仅保留题材判断框架和落地模板。
- 所有可直接复制使用的提示词已移除。
- 将方括号中的内容替换成你的项目情况后再使用模板。

## 建议用法

如果你愿意，可以把你的团队信息、平台、预算和候选题材发我，我可以直接基于这份框架帮你填一版“定制版”。

## 落地模板

下面这套模板适合真实立项场景，建议按 `项目约束 -> 候选题材 -> 打分比较 -> 结论决策` 的顺序使用。

### 模板一：项目约束卡

先把下面内容补全，再进入题材筛选。这个步骤的目标是防止“题材讨论脱离项目现实”。

```text
# 项目约束卡

## 1. 项目目标
- 本项目最重要的目标：[利润 / 流水 / 下载量 / DAU / 品牌验证 / 找发行 / 团队练兵]
- 成功标准：[例如 首月回本 / 3个月内验证留存 / 拿到发行合作]

## 2. 基础条件
- 目标平台：[手游 / Steam / 小程序 / 主机 / 跨平台]
- 目标市场：[中国 / 日本 / 美国 / 全球 / 其他]
- 开发周期：[ ]
- 总预算：[ ]
- 团队规模：[ ]

## 3. 团队能力
- 擅长玩法：[ ]
- 擅长美术风格：[ ]
- 技术强项：[ ]
- 现有可复用资源：[ ]
- 明显短板：[ ]

## 4. 项目限制
- 不能做的方向：[例如 高成本3D / 重剧情 / MMO / 强PVP]
- 必须满足的条件：[例如 可买量 / 可低成本验证 / 可半年上线]
- 风险容忍度：[低 / 中 / 高]

## 5. 商业要求
- 主要变现方式预期：[内购 / 抽卡 / 皮肤 / 广告 / 买断 / DLC]
- 预期用户规模：[ ]
- 是否要求长线运营：[是 / 否]
- 是否需要强传播性：[是 / 否]
```

### 模板三：题材打分表

建议每个维度 1-10 分。  
1-3 分代表明显不成立，4-6 分代表勉强可做，7-8 分代表成立，9-10 分代表非常强。

```text
# 题材打分表

评分规则：
- 1-3 分：明显弱项，可能构成立项障碍
- 4-6 分：可做，但有明显问题
- 7-8 分：基本成立
- 9-10 分：优势明显

维度说明：
- 欲望强度：玩家是否会天然被这个题材吸引
- 市场需求：目标市场中是否有稳定需求
- 竞争友好度：后来者是否还有切入口
- 玩法适配度：题材能否自然放大核心玩法
- 差异化空间：是否能讲出明确新意
- 开发可控性：是否适合当前预算、周期、团队
- 内容延展性：是否适合持续做内容
- 传播潜力：是否容易讲清卖点、做素材、做展示
- 商业化适配：是否和变现方式自然匹配
- 团队适配度：团队是否真的擅长做

| 题材 | 欲望强度 | 市场需求 | 竞争友好度 | 玩法适配度 | 差异化空间 | 开发可控性 | 内容延展性 | 传播潜力 | 商业化适配 | 团队适配度 | 综合结论 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |

```

- 最大风险点1：[ ]

## 推荐使用顺序

1. 先填“项目约束卡”。
2. 再列 3-5 个候选题材，不要一开始只讨论 1 个。
3. 用“题材打分表”先做一轮粗筛。
4. 让前 2-3 名进入“单题材深挖卡”。
5. 最后用“立项结论页”出正式结论。

## 快速判断口诀

可以用这句话快速判断一个题材值不值得继续看：

这个题材是否能承载强欲望，放大核心玩法，支持持续内容，并且在团队能力和商业目标范围内被低风险做出来？
