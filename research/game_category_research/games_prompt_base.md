# 通用版：游戏品类信息收集、简单拆解和分类

```text
你现在是一名资深游戏分析师，请帮我收集并整理不同品类游戏的信息，并对每款游戏进行简单拆解和分类。

任务目标：
1. 收集各个主要游戏品类的代表产品信息
2. 对每款游戏做简要拆解
3. 按统一维度进行分类，方便后续横向对比和分析
4. 输出结果尽量结构化、清晰、可继续扩展

请按以下要求执行：

一、品类范围
请尽量覆盖常见游戏品类，包括但不限于：
- MOBA
- FPS / TPS
- 战术射击
- 开放世界
- RPG
- ARPG
- 卡牌
- 二次元养成
- SLG
- 模拟经营
- 生存建造
- Roguelike / Roguelite
- 休闲益智
- 派对游戏
- 放置 / Idle
- MMORPG
- 音乐节奏
- 体育竞速
- 沙盒创造

二、每个品类的输出内容
对于每个品类，请输出：
1. 品类名称
2. 品类核心特点
3. 主要用户需求
4. 常见核心循环
5. 常见商业化方式
6. 常见长线留存方式
7. 代表游戏（3-5款）

三、每款游戏的拆解维度
对于每款代表游戏，请按以下结构做简单拆解：
1. 游戏名称
2. 所属品类
3. 平台
4. 核心玩法
5. 核心循环
6. 主要成长线
7. 主要付费点
8. 社交/对抗/合作属性
9. 局内体验特征
10. 局外系统特征
11. 目标用户画像
12. 该游戏最突出的设计卖点
13. 可以归入的细分类标签

四、分类要求
请对游戏进行多维分类，至少包括：
1. 按核心玩法分类
2. 按对抗形式分类（PVP / PVE / PVPVE / 单机）
3. 按成长驱动分类
4. 按商业化模式分类
5. 按节奏分类（长局 / 短局 / 碎片化）
6. 按社交强度分类
7. 按操作门槛分类
8. 按内容消耗方式分类（关卡型 / 沙盒型 / 赛季型 / 刷子型 / 剧情型）

五、输出格式要求
请按照以下顺序输出：
1. 先给出整体分类框架
2. 再按品类逐个整理
3. 每个品类下列出代表游戏
4. 每款游戏用统一格式简要拆解
5. 最后给出一个总表，方便横向对比

六、风格要求
1. 以分析和归纳为主，不要写成宣传文案
2. 内容尽量简洁，但要有信息密度
3. 如果某款游戏可能跨多个品类，请说明主品类和次品类
4. 如果信息不完整，可基于行业常识做合理推断，但要明确说明是推断
5. 尽量使用表格或结构化列表输出

七、额外要求
在最后补充：
1. 哪些品类最适合做中轻度项目
2. 哪些品类最适合做长线运营
3. 哪些品类最适合小团队切入
4. 哪些品类竞争最激烈
5. 哪些品类更适合创新混合玩法

如果内容过多，请优先保证分类框架完整，再保证每个品类至少有 2-3 个代表游戏的简要拆解。
```
