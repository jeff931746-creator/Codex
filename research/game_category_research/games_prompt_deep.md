# 深度版

```text
请你作为资深游戏行业分析师、竞品研究员和系统策划，帮我系统性收集各个品类游戏的信息，并完成基础拆解、分类和立项参考分析。

研究目标：
1. 建立完整的游戏品类认知框架
2. 收集各品类代表产品信息
3. 对代表游戏进行统一维度拆解
4. 输出适合立项、竞品研究、玩法参考和商业化参考的结构化内容

研究范围：
请尽量覆盖主流及常见细分品类，包括但不限于：
- MOBA
- FPS / TPS
- 战术射击
- RPG
- ARPG
- MMORPG
- 开放世界
- 卡牌
- 二次元养成
- SLG
- 模拟经营
- 生存建造
- Roguelike / Roguelite
- 休闲益智
- 派对游戏
- 放置 / Idle
- 音乐节奏
- 体育竞速
- 沙盒创造
- 生存竞技
- 剧情互动
- 塔防
- 自走棋

请按以下结构输出：

一、品类全景框架
请先输出一张总览表，字段包括：
- 品类名称
- 一句话定义
- 核心玩法特征
- 核心用户需求
- 常见局内循环
- 常见局外循环
- 常见商业化模式
- 长线运营方式
- 开发复杂度
- 市场竞争强度
- 代表游戏

二、按品类展开分析
对于每个品类，请分别输出：
1. 品类定义
2. 核心体验
3. 用户动机
4. 常见玩法结构
5. 常见成长结构
6. 常见商业化方式
7. 常见运营手段
8. 该品类的开发难点
9. 该品类的竞争风险
10. 该品类的机会点
11. 适合什么样的团队切入
12. 代表游戏（3-5款）

三、代表游戏拆解
对每款代表游戏，请统一按以下字段输出表格：
- 游戏名称
- 主品类
- 次品类/标签
- 平台
- 目标市场
- 核心玩法
- 核心循环
- 局内体验特征
- 局外系统特征
- 主要成长线
- 商业化模式
- 主要付费点
- 社交强度
- PVP/PVE属性
- 节奏类型
- 操作门槛
- 内容更新方式
- 核心卖点
- 用户画像
- 参考价值
- 风险提示

四、每款游戏的简要拆解说明
每款游戏再额外补充以下内容：
1. 成功关键点
2. 最值得参考的设计点
3. 最值得参考的商业化点
4. 最值得参考的运营点
5. 不适合直接照搬的地方

五、多维分类汇总
请把所有游戏再按以下维度分类汇总：
- 按玩法类型
- 按对抗形式
- 按成长驱动
- 按商业模式
- 按社交强度
- 按操作门槛
- 按内容消耗方式
- 按项目体量
- 按适合参考阶段分类（立项/玩法/商业化/运营/UI）

六、立项结论
请在最后输出：
1. 最适合小团队切入的品类
2. 最适合中型团队切入的品类
3. 最适合长线运营的品类
4. 最适合创新混合玩法的品类
5. 当前竞争最激烈的品类
6. 当前值得重点研究的 10 个品类
7. 当前最值得重点研究的 20 款游戏
8. 如果要做新项目，优先建议研究哪些样本，并说明原因

输出要求：
- 先给摘要，再给详细内容
- 优先用 Markdown 表格
- 表达要结构化
- 以分析和归纳为主，不要写宣传文案
- 信息不确定时标注“推测”或“待确认”
- 如果内容过长，可以分批输出，但要保证结构完整
```
