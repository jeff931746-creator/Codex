# 字段说明

## 核心字段

| 字段 | 说明 | 填写规则 |
|---|---|---|
| 产品名 | 产品或地图名称 | 使用公开最常见名称 |
| 产品类型 | 游戏、地图、模组等 | 先统一写 `游戏` 或 `War3地图` |
| 平台 | 运行平台 | 如 `Warcraft III`、`iOS/Android`、`Steam` |
| 发售状态 | 已上线、未上线、停运等 | 无法确认时写 `待确认` |
| 主品类 | 一级归类 | 如 `塔防`、`守图`、`RPG` |
| 次级标签 | 二级玩法标签 | 可写多个，用 ` / ` 分隔 |
| 上线时间 | 公开首发日或最早公开出现时间 | 拿不准就写 `待确认` |
| 时间口径 | 时间来源口径 | `官方首发日 / 首次公开出现日 / 待确认` |
| 地区/市场 | 主要面向区域 | 如 `全球`、`中国大陆` |
| 核心玩法 | 一句话说明玩法 | 强调核心决策，不写宣传语 |
| 核心玩法分析 | 对玩法重心的进一步拆解 | 说明它主要考验什么决策、和同类相比差在哪 |
| 核心循环 | 玩家重复动作链路 | 推荐按 `A -> B -> C` 写 |
| 核心循环分析 | 对循环驱动力的进一步拆解 | 说明循环的奖励、压力、成长和复玩来源 |
| 局内体验 | 单局核心体验 | 如摆位、守波次、发兵、刷装 |
| 局外系统 | 单局外成长或系统 | 如存档、职业成长、任务 |
| 主要成长线 | 主要数值/构筑成长方式 | 英雄、装备、塔系、科技等 |
| 内容更新方式 | 内容扩展方式 | 新关卡、新塔系、新角色、新模式等 |
| 付费模式 | 买断、广告、IAP 等 | War3 地图可写 `无直接商业化` |
| 主要付费点 | 变现重点 | 无则写 `无` |
| 社交属性 | 单人、多人协作、强对抗等 | 用于判断传播和留存 |
| PVP/PVE属性 | PVP、PVE、混合 | 不强行二选一 |
| 节奏类型 | 短局、长局、碎片化等 | 按真实游玩节奏填写 |
| 操作门槛 | 低、中、高 | 看实时操作与理解成本 |
| 目标用户画像 | 主要受众 | 如硬核策略玩家、War3 地图玩家 |
| 核心卖点 | 最值得记住的点 | 一句话突出差异化 |
| 参考价值 | 为什么值得研究 | 从玩法、商业化、运营、留存角度写 |
| 风险提示 | 该样本的局限或风险 | 如平台老化、商业化不可复用 |
| 研发复杂度 | 低、中、高 | 看系统量、内容量、联机需求 |
| 运营复杂度 | 低、中、高 | 看更新节奏和活动运营需求 |
| 榜单排名 | 可公开验证的榜单信息 | 没有就写 `待确认` |
| 收入规模 | 头部、腰部、长尾、待确认 | 只做分级 |
| 收入口径 | 收入判断依据 | 如评测量、畅销榜表现、产品形态 |
| 是否适合小团队 | 是、部分适合、否 | 结合系统复杂度判断 |
| 题材差异化 | 低、中、高 | 看题材稀缺度和包装力度 |
| 买量素材强度 | 低、中、高 | 看冲突表达与爽点展示能力 |
| 抖音来源 | 抖音账号、话题、视频链接 | 优先记录官方号、达人试玩、代表素材 |
| 抖音传播观察 | 抖音侧传播判断 | 记录首屏钩子、用户第一理解点、常见吐槽点 |
| 微信来源 | 公众号、视频号、小游戏、小程序入口 | 优先记录可复核入口或文章链接 |
| 微信生态观察 | 微信侧生态判断 | 记录是否适合小游戏化、社交扩散、轻量传播 |
| 数据来源 | 公开证据链接 | 多个来源用 ` | ` 分隔 |
| 备注 | 额外补充信息 | 用于记录边界与判断说明 |

## 当前口径边界

- 本地首版样本以 `War3 塔防 / 守图 / ORPG 邻近样本` 为主
- 不是“全球塔防全量目录”，而是一个可持续扩充的研究底库
- 对于老产品，优先记录“公开可检索到的最早出现信息”，不要伪造精确日期
