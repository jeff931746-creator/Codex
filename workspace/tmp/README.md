# tmp

Workspace scratch area. Files here are temporary working artifacts, not long-term project records.

## Directory Layout

- `browser-captures/` — temporary browser exports such as HTML, TXT, or DOCX captures.
- `level_progress_analysis/` — scratch workspace for level progress and churn analysis.
- `scripts/` — one-off helper scripts that are not yet promoted into `tools/`.
- `skill-drafts/` — temporary Skill drafts or update staging folders.

## Rules

- Keep only active scratch files here.
- Move reusable scripts to `tools/` once they are used beyond a single task.
- Move final research, reports, and project decisions to `projects/` or `research/` when they become durable.
- Do not keep installers, downloaded runtimes, package caches, `.DS_Store`, `__pycache__`, or `node_modules` here.
- Name task folders by purpose, not by vague dates. Prefer `level_progress_analysis/` over `2026-04-28-temp/`.
- Add a local `README.md` when a task folder contains more than scripts and one output file.
