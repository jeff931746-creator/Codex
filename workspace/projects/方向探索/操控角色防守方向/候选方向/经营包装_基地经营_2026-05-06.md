# 基地经营包装

## 核心定位

**底层经营**：投入资源建造设施 → 设施产出功能 → 功能支持战斗 → 战斗获得资源

**表层包装**：避难所、太空站、地下基地、移动堡垒

**认知负荷**：中（建造有空间规划）

## 四要素验证

| 要素 | 验证 |
|---|---|
| 时间分离 | ✓ 局外建造，局内战斗 |
| 低操作频次 | ⚠️ 建造需要一定操作频次，但低于商店经营 |
| 本能目标 | ✓ "我建造的基地"有成就感 |
| 成长/策略 | ✓ 建造规划、资源分配 |

**双约束验证**：
- **约束一（结构合法性）**：✓ 基地建造本身是独立玩法（Fallout Shelter、RimWorld可以不战斗只建造）
- **约束二（体验闭环）**：✓ 建造设施→设施产出→战斗使用→获得材料→升级设施，循环完整且紧密

## 核心循环

```
局外建造阶段
├─ 规划基地布局
├─ 建造功能设施
└─ 升级现有设施
    ↓
设施产出
├─ 生产设施产出资源
├─ 训练设施提升能力
└─ 防御设施提供保护
    ↓
进入战斗
    ↓
局内战斗阶段
├─ 操控角色防守基地
├─ 依托基地设施战斗
└─ 设施提供支援
    ↓
战斗结束
├─ 获得建造材料
├─ 解锁新设施
└─ 基地可能受损
    ↓
回到局外建造
```

## 具体设计方案

### 方案A：避难所模式

**核心机制**：
- 建造避难所设施
- 设施提供生存资源
- 资源用于战斗准备
- 战斗保护避难所

**设施类型**：
- **生产设施**：食物、水、弹药
- **训练设施**：提升角色能力
- **医疗设施**：战斗后恢复
- **防御设施**：战斗中提供支援

**建造系统**：
- 地下空间有限，需要规划
- 设施有等级，可以升级
- 设施之间有协同效果
- 设施需要电力/人力维护

**战斗关联**：
- 生产设施产出战斗资源
- 训练设施提升战斗能力
- 防御设施在战斗中提供火力支援
- 战斗失败可能导致设施受损

**长线驱动**：
- 解锁所有设施类型
- 优化基地布局
- 建造完美避难所

**参考对标**：
- Fallout Shelter（避难所经营）
- This War of Mine（避难所生存）
- 60 Seconds!（避难所管理）

### 方案B：太空站模式

**核心机制**：
- 建造太空站模块
- 模块提供不同功能
- 功能支持太空战斗
- 战斗保护太空站

**模块类型**：
- **生产模块**：资源采集、加工
- **研究模块**：解锁新科技
- **武器模块**：提供火力支援
- **生活模块**：提升船员能力

**建造系统**：
- 模块化建造，可以自由组合
- 模块之间需要连接通道
- 模块有能源需求
- 模块可以旋转、移动

**战斗关联**：
- 武器模块在战斗中自动开火
- 研究模块解锁新武器/技能
- 生产模块提供弹药补给
- 战斗中保护关键模块

**长线驱动**：
- 解锁所有模块类型
- 建造巨型太空站
- 优化模块布局

**参考对标**：
- Oxygen Not Included（太空殖民）
- RimWorld（殖民地管理）
- Space Haven（太空站建造）

### 方案C：移动堡垒模式

**核心机制**：
- 建造移动堡垒
- 堡垒可以移动到不同战场
- 堡垒设施在战斗中提供支援
- 战斗获得材料升级堡垒

**堡垒系统**：
- **车体**：决定移动速度和载重
- **武器**：提供火力支援
- **装甲**：提供防护
- **功能舱**：生产、维修、储存

**建造系统**：
- 堡垒有载重限制
- 需要平衡武器、装甲、功能
- 可以更换不同车体
- 可以安装不同模块

**战斗关联**：
- 堡垒武器在战斗中自动开火
- 堡垒装甲保护核心区域
- 功能舱提供补给
- 战斗中操控角色+堡垒协同

**长线驱动**：
- 解锁所有车体和模块
- 建造终极堡垒
- 优化堡垒配置

**参考对标**：
- Metal Slug（移动战车）
- Frostpunk（移动城市）
- Mortal Engines（移动城市）

## 与操控防守的结合

**方案A（避难所）**：
- 我建造的避难所产出资源
- 我用这些资源战斗
- 战斗保护避难所不被摧毁

**方案B（太空站）**：
- 我建造的太空站提供支援
- 战斗中太空站武器自动开火
- 我操控角色配合太空站防守

**方案C（移动堡垒）**：
- 我建造的堡垒和我一起战斗
- 堡垒提供火力和防护
- 我操控角色在堡垒周围战斗

**资源循环**：
- 建造设施 → 设施产出 → 战斗使用 → 战斗奖励 → 升级设施
- 循环清晰，互相喂养

**成长可见**：
- 基地规模越来越大
- 设施越来越多
- 战斗中能看到设施效果

## 参考对标

**成功案例**：
- **Fallout Shelter（避难所经营）**：建造设施+资源管理+自动战斗，建造本身有深度（空间规划、设施协同），但战斗是自动的
- **RimWorld（殖民地管理）**：建造+生存+自动战斗，建造规划极深，但玩家不直接操控角色战斗
- **Oxygen Not Included（基地建造）**：建造+资源链+自动化，建造系统复杂，但无战斗部分
- **Frostpunk（城市建造）**：建造+资源管理+叙事选择，建造有策略深度，但无操控战斗

**与Thronefall的对比**：
- **相同点**：都是"局外建造+局内战斗"的结构
- **核心区别**：
  - Thronefall：建造的是**防御布局**（塔、墙、兵营），建筑本身就是防御力量
  - 基地经营：建造的是**功能设施**（生产、训练、研究），设施产出资源/能力，不是直接防御
  - Thronefall：战斗中建筑是主要防御，玩家操控角色是辅助
  - 基地经营：战斗中玩家操控角色是主要防御，设施提供支援是辅助
- **体验差异**：
  - Thronefall："我布置的防线挡住了敌人"
  - 基地经营："我建造的基地支撑我守住了阵地"

**独特性**：
- 基地建造 + 操控战斗（不是自动战斗）
- 基地设施在战斗中提供支援（不是主要防御）
- 战斗直接影响基地状态（设施可能受损）
- **关键问题**：虽然和Thronefall有区别，但本质上仍是"建造类"玩法，区分度不如养成/工坊

## 潜在问题与解决方案

**问题1：基地建造和Thronefall经营的区别？**
- **核心区别**：
  - Thronefall建造的是**防御布局**（塔、墙、兵营），建筑本身就是防御力量
  - 基地经营建造的是**功能设施**（生产、训练、研究），设施产出资源/能力，不是直接防御
- **战斗中的作用**：
  - Thronefall：建筑是主要防御，玩家操控角色是辅助
  - 基地经营：玩家操控角色是主要防御，设施提供支援是辅助
- **体验差异**：
  - Thronefall："我布置的防线挡住了敌人"
  - 基地经营："我建造的基地支撑我守住了阵地"
- **但区分度仍然有限**：虽然有区别，但本质上仍是"建造类"玩法，不如养成/工坊的区分度高

**问题2：建造会不会太复杂？**
- 解决：设施类型不要太多（10-15种）
- 解决：建造规则简单清晰
- 解决：有推荐布局模板

**问题3：基地和战斗的关联够强吗？**
- 解决：方案A基地产出直接用于战斗
- 解决：方案B基地武器在战斗中自动开火
- 解决：方案C基地和角色协同作战

**问题4：建造的操作频次会不会太高？**
- 解决：建造是低频操作，不需要频繁调整
- 解决：设施建好后自动运行
- 解决：战斗是主要时间消耗

## 为什么是第二梯队

1. **认知负荷偏高**：建造需要空间规划和资源分配，不符合"极低认知负荷"要素
2. **操作频次偏高**：建造需要一定操作频次，虽然低于商店经营，但高于养成/工坊
3. **与Thronefall区分度有限**：虽然建造的是功能设施而非防御布局，但本质上仍是"建造类"玩法，区分度不如养成/工坊
4. **但仍然可行**：
   - 有成功案例（Fallout Shelter、RimWorld）证明基地建造本身是独立玩法
   - 四要素基本满足，双约束验证通过
   - 独特性较强：移动堡垒、太空站等包装有吸引力
   - 体验闭环完整：建造→产出→战斗→升级
5. **优于第三梯队的原因**：
   - 操作频次低于商店经营（建造是低频操作，商店需要频繁定价/进货）
   - 本能目标强于卡牌构筑（"我建造的基地"比"我的卡组"更有成就感）
   - 无操作冲突（建造在局外，战斗在局内，时间完全分离）
