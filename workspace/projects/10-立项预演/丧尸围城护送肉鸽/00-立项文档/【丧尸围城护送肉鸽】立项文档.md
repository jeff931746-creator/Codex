# 丧尸围城护送肉鸽

整理时间：2026-04-14

## 一句话定义

玩家护送一个必须存活的移动目标穿过丧尸围城区域，在不断升级的尸潮压迫下通过局内肉鸽成长完成突围。

## 立项结论

这不是纯围城守点，也不是纯护送跑图，而是一个把“移动目标的生存压力”和“局内 Build 成长”绑定到同一条战线上的中轻度爽游。

更准确的项目定位是：

- 前台：尸潮压境、护送目标移动、局部围困、突围成功反馈
- 中台：护送路线选择、局内三选一、临时救援决策
- 后台：护送单位 / 随行队伍 / 资源事件三条成长线

核心判断：

- 比单纯守点更有推进感
- 比单纯护送更有压力感
- 比单纯肉鸽割草更有明确目标感

## 核心体验目标

- 我知道自己在护送什么
- 我能感受到围城压力正在逼近
- 我会在路线上做取舍，而不是只往前走
- 我能通过肉鸽选项把这一局做出差异
- 我能在终局突围时感受到 Build 成型

## 核心玩法模型

### 主循环：护送推进

1. 护送对象持续向前移动
2. 丧尸从四周或前方压上来
3. 玩家清理威胁、处理拦截点
4. 护送对象通过关键节点
5. 到达下一安全区或中继点

### 中循环：围城压迫

1. 当前区域被丧尸包围
2. 玩家需要先打开一条可通过的路
3. 选择是强冲还是稳推
4. 压力越高，护送对象越容易受损

### 高潮循环：肉鸽成型与终局突围

1. 关键强化出现
2. 随行单位或护送对象能力变化
3. 尸潮升级为围城收束波
4. Boss 或破围单位登场
5. 玩家带着成型 Build 完成终局突围

## 角色与系统分工

### 护送目标

定位：

- 玩家要保护的核心对象
- 移动压力源
- 失败判定核心

职责：

- 自动推进
- 吸引部分战线压力
- 触发关卡节奏

### 玩家小队

定位：

- 主战力
- 局内 Build 承载层
- 护送过程中的机动清障单位

职责：

- 清怪
- 开路
- 临时救援
- 处理重点威胁

### 丧尸围城

定位：

- 空间压力
- 时间压力
- 收束压力

职责：

- 围堵路线
- 截断推进
- 迫使玩家做路线和资源决策

### 肉鸽成长

定位：

- 单局 Build 的变化来源
- 复玩动力来源
- 护送差异化来源

职责：

- 改变角色输出方式
- 改变护送对象能力
- 改变清怪与救援节奏

## 护送的重要性判断

结论：如果没有“移动目标必须存活并推进”，这条方向就会退化成普通的围城割草或波次塔防。

因此本项目保留的不是单纯清怪，而是：

- 目标必须一直被保护
- 路线必须持续推进
- 压力必须在移动中累积

这三者要绑成一个主循环。

## 丧尸围城系统

### 设计原则

- 压力必须明确
- 围堵必须可读
- 不能只做波次刷怪
- 不推进就会被围死

### 表现形式

- 普通尸潮
- 侧翼包抄群
- 拦截门卫型丧尸
- 区域 Boss

### 目标作用

- 提供围城压迫感
- 提供路线选择压力
- 让护送不只是“陪跑”

## 肉鸽系统

### 普通三选一

内容方向：

- 随行单位强化
- 护送目标护甲提升
- 清怪范围扩大
- 技能冷却缩短
- 事件奖励加成

### 高质量三选一

内容方向：

- 护送目标改为移动炮塔
- 随行单位变为自动开路队
- 终局突围时触发一次强力救援
- 经过围城点后生成临时安全区

## 围城和护送的关系

不是两个系统并列，而是同一条推进链里的两个压力面：

- 护送负责“前进目标”
- 围城负责“推进阻力”

如果只剩护送，局面会偏空；
如果只剩围城，局面会偏闷。

把两者合在一起，才有这条方向最重要的张力。
