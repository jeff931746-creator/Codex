# 立项初审稿 v1

项目名称：丧尸围城护送肉鸽  
当前阶段：方向选择 -> 玩法母型 -> 参与方式 -> 核心机制 -> 玩法循环（初审）

## 一、方向卡

### 事实

- 项目代号：`Dead Escort Rogue`
- 一句话方向定义：`一款围绕“护送移动目标穿越丧尸围城区”展开、并通过局内肉鸽成长实现突围的中轻度PVE爽游`
- 目标平台：`微信小游戏优先，后续可扩移动App`
- 目标用户：`喜欢丧尸压力、护送推进、Build 成型和短局成长的中轻度用户`
- 前台吸量壳子：`一边护送，一边从围城尸潮里杀出一条路`
- 后台长期承接：`护送目标 / 随行队伍 / 事件奖励三条成长线 + 局内三选一强化 + 关卡推进`
- 最像的竞品：
  - 《向僵尸开炮》式尸潮压力表达
  - 轻量护送关卡 / 生存撤离表达
  - 肉鸽割草 / Build 成型表达

### 判断

- 市场机会一句话：`丧尸压力和护送目标都是成熟语言，但把“围城推进”和“肉鸽突围”合成到同一主循环里，仍有清晰的前台差异空间。`
- 为什么是现在：
  - 1. 纯围城容易同质化成守点
  - 2. 纯护送容易变成跑图任务
  - 3. 肉鸽成长能把两者缝成一局里可复玩的目标推进体验
- 为什么是我们做：
  - 1. 已经有守线、打野、Build 的拆解经验
  - 2. 这条方向比纯塔防更有目标推进感
  - 3. 适合先做纸面原型验证主循环是否成立
- 当前最大机会：
  - 1. “护送”天然有目标感
  - 2. “围城”天然有压迫感
  - 3. “肉鸽”天然有重复游玩动力
- 当前最大风险：
  - 1. 护送对象如果太弱，会失去核心存在感
  - 2. 围城如果太重，会变成单纯守点
  - 3. 肉鸽如果太散，会把推进节奏拉乱

### 评分

- 市场机会：7
- 平台适配：8
- 团队匹配：7

### 结论

- 方向结论：`通过`

## 二、母型卡

### 事实

- 主母型：`护送`
- 辅助母型：`围城压迫`
- 组合一句话定义：`玩家必须把一个移动目标安全送出丧尸围城区，同时在局内不断处理被尸潮截断的推进压力。`

### 判断

- 主母型的核心乐趣：`推进、保人、开路、过点`
- 辅助母型的加入目的：`提供空间围堵、路线限制和持续时间压力`
- 为什么这两个组合成立：
  - 1. 护送给出明确目标
  - 2. 围城给出明确阻力
  - 3. 两者合起来自然形成“走不动就会死”的推进压力
- 两者冲突点：
  - 1. 如果围城过强，护送会被打断成清图
  - 2. 如果护送过慢，玩家会觉得像在等车
- 删掉辅助母型后产品是否仍成立：`是，但会退化成普通护送关卡`
- 不采用的母型：
  - 纯守点
  - 纯撤离跑图
  - 纯自动割草
- 放弃原因：
  - 1. 守点会削弱推进感
  - 2. 撤离跑图会削弱围城压力
  - 3. 纯割草会丢掉明确目标

### 评分

- 识别清晰度：8
- 组合合理性：8
- 学习成本：8
- 差异空间：7

### 结论

- 母型结论：`通过`

## 三、参与方式卡

### 事实

- 玩家身份一句话：`玩家作为护送队的战斗核心，在尸潮围城中开路并保护移动目标抵达安全区`
- 玩家控制对象：`主角 / 随行队伍 / 局内强化选择`
- 玩家高频动作：
  - 1. 观察围城缺口和尸潮方向
  - 2. 清理阻挡护送路线的威胁
  - 3. 在三选一里做强化取舍
- 玩家关键决策：
  - 1. 是先救护送目标还是先开路
  - 2. 当前 Build 更偏清怪、保人还是突围
- 玩家参与深度：`中操作`

### 判断

- 玩家代入感来源：
  - 1. 我在把一个重要对象送出去
  - 2. 围城压力会不断逼我改路线
  - 3. 我的强化会直接改变这一局能不能突围
- 如果玩家只玩 30 秒，他会觉得自己在做什么：`一边护送目标，一边清理围城丧尸并寻找出口`
- 高频动作是否过多：`否，当前集中在观察、开路、选择强化`
- 关键决策是否真的影响结果：`是，路线和 Build 选择都会立刻影响护送生存率`

### 评分

- 身份清晰度：8
- 动作负担：7
- 决策价值：8
- 代入感：8

### 结论

- 参与方式结论：`通过`

## 四、核心机制卡

### 事实

- 核心机制：`移动护送目标`
- 辅助机制：`局部围城压迫`
- 核心机制一句话定义：`玩家不是原地守点，而是围绕一个必须存活并持续推进的移动目标处理尸潮围堵与路线阻断`

### 判断

- 如果删掉核心机制，这个项目会变成什么：
  - `一个普通的丧尸围城守点肉鸽`
- 为什么它不可替代：
  - 1. 它是这条方向最直接的识别点
  - 2. 它把“围城”从静态防守变成动态推进
  - 3. 它决定了玩家不是清场者，而是护送者
- 它在单局中第一次出现的时间：`开局即出现`
- 它在单局中的使用频率：`全程高频`
- 它如何进入主循环：`每个路段都围绕护送目标是否能继续前进展开`
- 玩家为什么会主动使用它：
  - 1. 目标明确
  - 2. 压力强烈
  - 3. 结果反馈直接

### 评分

- 可感知性：9
- 主动使用意愿：8
- 不可替代性：8
- 与主循环贴合度：9
- 内容扩展性：8

### 结论

- 核心机制结论：`通过`

## 五、玩法循环卡

### 事实

- 20字以内核心循环：`护送推进 -> 尸潮围城 -> 开路突围 -> 选强化 -> 下一段`
- 循环节点：
  - 1. 护送目标出发
  - 2. 第一次围堵出现
  - 3. 玩家开路通过关键点
  - 4. 局内强化出现
  - 5. Boss 或终局围城收束

- 玩家目标：`把护送目标安全送到终点`
- 高频动作：`观察、清障、救援、选强化`
- 关键决策：`当前资源优先保目标还是冲出口`
- 奖励/成长：`普通三选一 + 关键事件奖励 + 终局成型`
- 下一轮压力：`更密尸潮、更窄路线、更强拦截点`

### 判断

- 单局时间轴：
  - 0:00-1:00
    - 玩家看到：护送目标出发，尸潮在周边形成第一层围堵
    - 玩家做：清开前进路线，第一次体会护送压力
    - 系统给：基础资源与首次强化
  - 1:00-3:00
    - 玩家看到：围城区变窄，拦截点出现
    - 玩家做：开始权衡保人、开路和冲刺
    - 系统给：第一次三选一强化
  - 3:00-5:00
    - 玩家看到：尸潮强度上升，护送目标进入高压段
    - 玩家做：围绕 Build 方向继续推进
    - 系统给：第二次关键强化与事件奖励
  - 5:00-8:00
    - 玩家看到：Boss 或终局围城波出现
    - 玩家做：带着成型 Build 突围
    - 系统给：结算与下一局动机

- 第一次目标感出现时间：`0-10秒`
- 第一次爽点出现时间：`约 30-60 秒`
- 差异机制出现时间：`开局即有，1 分钟内形成理解`
- 最终收束方式：`Boss 围堵或终局突围`
- 下一轮动机：`尝试不同护送目标成长和不同开路 Build`

### 评分

- 目标清晰度：8
- 节奏流畅度：8
- 爽点密度：7
- 差异机制融入度：8
- 收束完整性：8
- 复玩驱动：7

### 结论

- 玩法循环结论：`通过`

## 七、当前结论

### 当前最强点

- 护送目标天然给出明确任务
- 围城压力天然给出明确阻力
- 肉鸽成长天然给出局内变化和复玩动力

### 当前最弱点

- 护送对象如果不够“重要”，会削弱代入感
- 围城如果不够清晰，容易退化成普通打怪
- 肉鸽如果不够克制，容易把路线推进拆散

### 下一步建议

- 进入 `纸面原型阶段`
