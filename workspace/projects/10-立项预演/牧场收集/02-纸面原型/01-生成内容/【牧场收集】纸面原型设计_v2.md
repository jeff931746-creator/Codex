# 【牧场收集】纸面原型设计_v2（最小验证版）

## 验证目标

**核心问题：** 玩家是否会因为"差一点就能孵化出可展示、可使用的新生物"，主动再打一局？

**验证范围：** 1个区域，3只生物，1个战斗模板，10分钟内完成3-4轮循环

**成功标准：** 玩家能明确说出"我下一局想刷哪只生物"，并且生物进入牧场后产生可见价值

## 一、牧场核心价值设计

### 牧场是什么？

牧场是玩家的**生物展示空间**，每只生物孵化后会进入牧场，玩家可以：
1. **看到生物在牧场中活动**（展示价值）
2. **选择3只生物组成出战队伍**（功能价值）
3. **不同生物组合产生加成**（组合价值）

## 二、战斗系统（可计算版）

### 战斗模板（30秒/场）

**玩家数值：**
- 初始HP：100
- 移动速度：5 单位/秒
- 普通攻击：伤害10，攻击范围2单位，攻击间隔0.5秒
- 技能Q（范围AOE）：伤害30，范围5单位，冷却5秒
- 技能E（召唤生物）：召唤当前队伍生物助战5秒，冷却10秒

**敌人数值（3波）：**

| 波次 | 敌人类型 | 数量 | 血量 | 伤害 | 移动速度 | 出生点距离 | 到达时间 |
|------|----------|------|------|------|----------|-----------|----------|
| 第1波 | 绿史莱姆 | 2 | 30 | 5 | 2单位/秒 | 20单位 | 10秒 |
| 第2波 | 绿史莱姆 | 2 | 30 | 5 | 2单位/秒 | 20单位 | 10秒 |
| 第2波 | 蓝史莱姆 | 1 | 50 | 8 | 3单位/秒 | 20单位 | 6.7秒 |
| 第3波 | 蓝史莱姆 | 2 | 50 | 8 | 3单位/秒 | 20单位 | 6.7秒 |
| 第3波 | 红史莱姆 | 1 | 80 | 12 | 4单位/秒 | 20单位 | 5秒 |

**敌人攻击频率：** 到达玩家身边后，每2秒攻击一次

**防守点：** 玩家身后5单位，敌人到达防守点即失败

### 战斗时长计算

**第1波（0-10秒）：**
- 2只绿史莱姆，10秒后到达
- 玩家向前移动，5秒时开始攻击
- 每只史莱姆需要3次普攻（30HP / 10伤害）= 1.5秒
- 击败2只 = 3秒
- 第1波结束时间：8秒

**第2波（10-20秒）：**
- 2只绿史莱姆 + 1只蓝史莱姆
- 蓝史莱姆6.7秒后到达，绿史莱姆10秒后到达
- 玩家优先攻击蓝史莱姆（5次普攻 = 2.5秒）
- 使用技能Q清理2只绿史莱姆（30伤害，各剩0HP）
- 第2波结束时间：19秒

**第3波（20-30秒）：**
- 2只蓝史莱姆 + 1只红史莱姆
- 红史莱姆5秒后到达，蓝史莱姆6.7秒后到达
- 玩家使用技能E召唤生物（兔子加速 + 小鹿冲撞）
- 小鹿冲撞击杀红史莱姆（30伤害，剩50HP，再2次普攻）
- 兔子加速帮助玩家快速清理蓝史莱姆
- 第3波结束时间：30秒

**结论：** 战斗时长约30秒，玩家HP剩余70-80（受到2-3次攻击）

### 生物助战的具体影响

| 生物 | 助战效果 | 数值影响 |
|------|----------|----------|
| 兔子 | 玩家移速+50% | 缩短追击时间2-3秒 |
| 小鹿 | 冲撞伤害30 | 秒杀绿史莱姆，重伤蓝/红史莱姆 |
| 羊 | 治疗20HP | 容错率提升，允许多受2次攻击 |

**设计意图：** 生物助战能明显改变战斗结果，让玩家感受到"收集生物"的价值

## 三、掉落系统（反比算法 + 模拟验证）

### 反比算法（简化版）

```python
def calculate_drop_weight(fragment_type, inventory):
    """
    计算碎片掉落权重
    """
    owned = inventory.get(fragment_type, 0)
    required = 5  # 统一孵化需求
    gap = required - owned
    
    if gap <= 0:
        # 已孵化，权重降低
        return 10
    elif gap <= 2:
        # 差1-2个，权重大幅提升
        return 100
    else:
        # 差3-5个，正常权重
        return 50

def drop_fragments(inventory):
    """
    每场战斗掉落2个碎片
    """
    weights = {
        "兔子": calculate_drop_weight("兔子", inventory),
        "小鹿": calculate_drop_weight("小鹿", inventory),
        "羊": calculate_drop_weight("羊", inventory)
    }
    
    # 按权重随机抽取2个
    return weighted_random_choice(weights, 2)
```

## 八、失败判断标准

如果出现以下情况，说明核心假设不成立：

1. **玩家不关心碎片进度** - 战斗结算时直接跳过，不看进度条
2. **玩家不主动"再来一局"** - 差1个碎片时，选择返回而不是继续战斗
3. **玩家不查看牧场** - 孵化后直接开始下一场战斗，不看生物状态
4. **玩家不尝试组合** - 始终使用单只生物，不编辑队伍
5. **玩家10分钟后失去兴趣** - 明确表示"没意思""不想玩了"

如果出现上述情况，需要分析原因：
- 是反比算法不够明显？
- 是生物价值不够吸引人？
- 是战斗过于重复？
- 还是"牧场收集"方向本身不成立？
