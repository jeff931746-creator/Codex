1. 玩家能操作一个主英雄在防线附近移动和救场

## 4. 核心玩法定义

### 4.1 一句话玩法

玩家操控一名主英雄，在核心水晶前方的小型战场中布置功能不同的分身阵位，守住敌潮并通过升级选择逐步形成自己的守线 build。

### 4.2 核心循环

1. 敌人从右侧生成，向左侧核心水晶推进
2. 主英雄自动攻击最近敌人，玩家可移动主英雄并释放主动技能
3. 玩家消耗能量或 charges 在固定阵位部署分身
4. 分身持续输出、控场或增益
5. 击杀敌人获得经验
6. 升级后弹出 3 选 1 build 选择
7. 敌潮增强，出现精英和 Boss
8. 水晶血量归零则失败；击败最终 Boss 或撑过最终波次则胜利

## 5. 视角与地图

### 5.1 视角

- 使用 `俯视偏斜/平面俯视` 的简化战场
- 不做复杂地形
- 强调清晰和可读

### 5.2 地图结构

地图固定为一张单屏战场：

- 左侧：核心水晶 / 防守目标
- 中部偏左：主英雄活动区
- 中部：分身阵位区
- 右侧：敌人刷新与推进路径

### 5.3 路径设计

- 只有 `1条主路径`
- 路径可以轻微弯折，但不能复杂
- 目的是让玩家更容易看懂：
  - 怪从哪里来
  - 分身布在哪
  - 主英雄去哪里救场

## 6. 玩家操作

### 6.1 控制方式

- `WASD` 或方向键：移动主英雄
- `鼠标左键`：点击空的阵位部署当前选中的分身
- `1 / 2 / 3 / 4`：切换分身类型
- `Space`：释放主英雄主动技能
- `Esc`：暂停 / 恢复

### 6.2 操作原则

- 主英雄自动普攻
- 玩家主要做三件事：
  - 移动站位
  - 布置分身
  - 关键时刻开技能
- 不做复杂瞄准
- 不做高频闪避

---

## 7. 主英雄设计

### 7.1 职业

本 Demo 只做 `1个职业`：

- `风暴术士`

### 7.2 主英雄能力

- 自动攻击：
  - 发射短程雷击弹
  - 优先攻击最近的敌人
- 主动技能：
  - `雷暴脉冲`
  - 效果：以主英雄为中心释放圆形冲击波，对范围敌人造成伤害并短暂减速
  - 冷却：约 `8秒`

### 7.3 主英雄职责

- 处理漏怪
- 在精英/Boss 逼近时救场
- 补足分身火力空隙
- 作为战场中心给玩家代入感

### 7.4 主英雄数值方向

- 不要做成超模主 C
- 伤害约占整局输出的 `35%~45%`
- 分身应占主要阵地输出

## 8. 分身系统设计

### 8.1 分身理念

分身不是宠物，也不是普通召唤物，而是承担原“塔位功能”的角色化阵地单位。

### 8.2 阵位规则

- 场上固定提供 `6个阵位`
- 玩家开局可用其中 `2个`
- 后续可通过升级解锁更多可用阵位或强化已有阵位

### 8.3 分身类型

本 Demo 只做 `4类分身`：

1. `盾卫分身`
   - 作用：堵口、吸引敌人、减速推进
   - 表现：近战，血厚，攻击低

2. `寒霜分身`
   - 作用：持续减速、概率冻结
   - 表现：远程冰弹，输出低，控制强

3. `雷灵分身`
   - 作用：主力清杂输出
   - 表现：远程弹射雷球，中等伤害

4. `战旗分身`
   - 作用：为附近主英雄和其他分身提供攻速 / 伤害增益
   - 表现：不主动攻击，提供 aura

### 8.4 部署规则

- 部署分身需要消耗 `能量点`
- 能量点通过时间和击杀缓慢恢复
- 相同阵位上只能存在一个分身
- 已部署分身可被覆盖替换：
  - 替换后旧分身消失，新分身进场

### 8.5 分身行为原则

- 分身不移动
- 分身自动攻击或提供 aura
- 分身表现要清楚，让玩家一眼看出职责

## 9. 敌人与波次

### 9.1 敌人类型

本 Demo 只做 `4类敌人`：

1. `小型杂兵`
   - 血少、数量多、移动快

2. `持盾怪`
   - 血更厚，移动中等

3. `精英怪`
   - 血厚、伤害高，优先考验当前 build

4. `Boss`
   - 大体型、高血量、逼近水晶后造成高额伤害

### 9.2 波次节奏

- 总时长：`6-8分钟`
- 结构建议：
  1. `0:00 - 1:30` 新手理解期
  2. `1:30 - 3:30` build 成型期
  3. `3:30 - 5:30` 压力提升期
  4. `5:30 - 7:00` 精英 + Boss 收束期

### 9.3 敌人目标

- 所有敌人默认向左推进并攻击水晶
- 若路上遇到盾卫分身，可优先攻击盾卫

## 10. 三选一 Build 系统

### 10.1 升级方式

- 击杀敌人获得经验
- 满经验后暂停战斗，弹出 `3选1` 面板

### 10.2 选择池结构

本 Demo 只做约 `18-24个` 升级项，分三类：

1. `主英雄强化`
   - 雷暴脉冲范围增加
   - 主英雄普攻弹射 +1
   - 主英雄攻击速度提升

2. `分身强化`
   - 雷灵分身弹射次数 +1
   - 寒霜分身减速增强
   - 盾卫分身生命值提升
   - 战旗分身 aura 范围扩大

3. `混合强化`
   - 主英雄靠近分身时伤害提升
   - 冰冻敌人被雷击时额外爆炸
   - 场上分身数量达到 3 时，全体攻速提升
   - 解锁额外可用阵位

### 10.3 Build 目标

玩家在局中应能形成至少 `3种可感知风格`：

1. `冰雷控场流`
2. `多雷灵清杂流`
3. `盾卫拖线 + 术士救场流`

## 11. UI 设计

### 11.1 必要 UI

必须有以下最简 UI：

- 水晶血量
- 主英雄血量
- 主英雄技能冷却
- 当前能量点
- 当前等级 / 经验条
- 当前波次 / 倒计时

### 11.2 升级面板

- 居中弹窗
- 显示 3 个选项
- 每个选项有：
  - 标题
  - 1 句清晰说明
- 点击后立即生效，恢复战斗

### 11.3 调试与可读性

允许在界面角落显示简单 debug 信息，例如：

- 当前 FPS
- 场上敌人数
- 场上分身数

但 UI 必须保持简洁，不要像编辑器面板。
