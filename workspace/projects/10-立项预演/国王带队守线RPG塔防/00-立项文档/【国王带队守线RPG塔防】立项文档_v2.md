## 1. 项目摘要

### 1.1 一句话定义

- 项目一句话定义：
  一款以“主英雄带队守住最后防线”为主题、以“角色成长驱动战场构筑”为核心体验的中度 RPG 塔防产品。

### 1.2 立项结论

- 立项建议：`可试做`
- 核心理由：
  1. 该方向能把原始版本中“策略守线”“角色存在感”“中期成长承接”三件互相打架的事，重组为同一种产品语言。
  2. 相较原版偏 TF-like 的长局守城，新的 RPG 塔防定义更容易形成题材表达、素材表达和商业化表达的一致性。
  3. 方向具备明确的中度产品潜力，但必须通过 10 分钟战斗原型验证核心体感，不能直接按大制作推进。
- 最大机会点：
  把“守线爽感 + 职业 fantasy + 局内 build 成型”压缩进单局 8-10 分钟内，建立区别于纯塔防和纯割草的中间带产品。
- 最大风险点：
  产品极易做成四不像，即“塔不够重要、角色不够鲜明、数值盖过构筑、操作负担过重”。

### 1.3 项目概览

- 目标赛道：中度 RPG 塔防 / 守线 Build
- 目标平台：手游优先，兼容小游戏方法论
- 目标用户：
  1. 喜欢守线、Build、塔防和轻 RPG 成长的中度用户
  2. 对 War3 守家、Roguelite、僵尸like 有经验的男性向用户
  3. 对“最后防线”“带队守城”“职业成长”天然有兴趣的泛策略用户
- 核心玩法：
  带 1 名主英雄与 2 名副英雄进入战场，在少量关键塔位上构筑防线，通过局内升级、塔转职、遗物选择形成单局流派并守住终局 Boss。
- 核心题材：中世纪奇幻 / 王国最后防线 / 国王统御
- 核心破局点：
  1. 角色成长驱动，而非纯摆塔驱动
  2. 低负担移动 + 高价值站位
  3. 塔是流派放大器，而非独立养成宇宙

## 2. 找准切入点

### 2.1 市场时机判断

- 目标品类当前阶段：`红海中的结构性机会期`
- 判断结论：
  塔防和 RPG 都不是市场空白，但“中度守线 + 角色成长 + 局内构筑”的表达仍有切口，尤其适合承接“看得懂守城、玩进去有 build”的用户。
- 事实：
  1. 塔防在小游戏与移动端并未退出主流视野，且守线表达对素材依然友好。
  2. RPG 在中长期留存和商业承接上仍是成熟语法。
  3. 原版《无尽防线》说明“守城 + 英雄参与”具备阶段性市场接受度，但没有做到爆款级放大。
- 推断：
  1. 市场真正奖励的不是抽象的“更深”，而是“更容易讲清楚的爽点 + 更可持续的承接”。
  2. 原版的失败不意味着守城无机会，而意味着“长局、重理解、重运营”的那套表达不适合大盘放量。
- 建议：
  先把项目目标从“头部买量爆款”降到“可放量的中度产品”，再通过原型和素材测试决定是否加大投入。

### 2.2 用户判断

- 目标用户是谁：
  1. 塔防用户：想要守线压力和防守策略
  2. Build 用户：想要局内成型和明显跃迁
  3. RPG 用户：想养角色、组流派、刷进度
- 年龄段：25-40 为核心，18-24 为新增量
- 性别倾向：男性为主，可通过卡通奇幻和角色包装向中性扩展
- 游戏偏好：守线、Build、短局成长、Boss 压线、职业差异、装备/遗物搭配
- 消费能力：中度
- 主要需求：
  1. 杀时间：单局不拖沓
  2. 成就感：危局翻盘，build 成型
  3. 代入感：我是在带队守城，不是在旁观塔自动打
  4. 长期追求：职业、装备、塔科技、遗物图鉴
- 用户迁移动机：
  1. 纯塔防模板化，长期差异不足
  2. 纯割草守线队伍感弱、战场感弱
  3. 原版偏 hard 的守城理解门槛不利于泛用户进入

### 2.3 获客判断

- 预期获客成本区间：中等
- 是否适合走买量：适合，但不适合靠“复杂策略解释”买量
- 题材是否具备低成本吸量潜力：
  中世纪奇幻 + 最后防线 + 英雄带队，具备基础理解优势，但不是纯题材红利
- 画风是否适合当前投放环境：
  建议轻卡通中世纪奇幻，强调可读性、职业识别和战场密度，不建议偏灰暗写实
- 推断：
  该项目的买量优势来自“素材可讲职业成长与守线爽点”，而不是来自深策略本身

### 2.4 阶段结论

- 当前切入是否成立：`成立`
- 成立前提：
  1. 前 5 秒素材必须能看懂“守线 + 职业 + 反杀”
  2. 前 3 分钟实玩必须出现一次明确 build 成型感
  3. 角色与塔的职责必须清晰，不能互相抢戏
- 若不成立，主要问题是什么：
  一旦角色、塔、遗物三套系统不能形成统一体验，项目将退化成“系统更多但更难懂”的旧问题升级版

## 3. 竞品与缺口分析

### 3.1 竞品池范围

- 竞品池定义：
  1. 直接竞品：RPG 塔防、守线 Build、角色参与式塔防
  2. 邻近竞品：僵尸like 守线、War3 守家、动作塔防
- 核心对标竞品数量：3
- 样本筛选规则：
  以“用户认知相近”和“产品逻辑相近”为原则，不要求表层玩法完全一致

### 3.2 TOP3 竞品拆解

#### 竞品A

- 产品名称：向僵尸开炮
- 核心循环：守线 -> 升级三选一 -> 成型 -> 更高压波次
- 长期留存依赖：局外成长、词条、活动、数值承接
- 商业化模式：中度买量手游常见成长付费
- 核心卖点：爽点直给、素材表达强、理解成本极低
- 明显短板：队伍感和战场策略层较弱，长期体验容易回到数值承接
- 可借鉴点：前期爽点前置、素材语言简洁、build 成型反馈强

#### 竞品B

- 产品名称：无尽防线
- 核心循环：建造/守城 -> 英雄参与 -> 波次抗压 -> 局外成长
- 长期留存依赖：关卡推进、英雄养成、资源积累
- 商业化模式：中度成长、月卡、通行证、养成资源
- 核心卖点：守城感、地形与塔位协同、策略破局
- 明显短板：前期门槛高、角色与塔职责模糊、单局偏长、资源压力偏重
- 可借鉴点：守线题材、英雄参与感、阵地战压迫感

#### 竞品C

- 产品名称：Orcs Must Die! Deathtrap
- 核心循环：布置陷阱 -> 手操战斗 -> 获取成长 -> 抗更强敌潮
- 长期留存依赖：动作反馈、关卡变化、Roguelite 元进程
- 商业化模式：买断制，不可直接照搬
- 核心卖点：动作塔防融合、角色存在感强、战场互动密度高
- 明显短板：动作负担和表现成本高，不适合直接迁移到移动端中度产品
- 可借鉴点：角色亲自参战、塔与技能联动、Boss 压迫感
- **为何无法移动端迁移（关键推断）：**
  1. 操控层：OMD 核心爽感依赖鼠标/手柄的精准瞄准与快速方向切换，移至触屏后高频输入负担指数级上升，直接破坏战斗流畅度。
  2. 性能层：同屏大量物理陷阱、特效与三维环绕式怪群，超过主流中低端机承受范围；移动端用户基本盘与其所需硬件门槛倒挂。
  3. 商业层：买断制无法在免费流量买量场景复制，无法支撑手游投放逻辑。
- **本项目的回避策略：**
  以"低负担站位切换 + 技能无需精准瞄准"替代动作层；以"怪群沿路径移动"替代三维空间环绕；在操控与性能两侧主动降阶，保留角色存在感的同时不进入动作游戏赛道。

### 3.3 竞品共性与缺口

- 同类产品共性优势：
  1. 守线目标清晰
  2. build 与随机成长天然适合复玩
  3. 塔防与怪潮在素材表达上具备可视优势
- 同类产品共性问题：
  1. 纯塔防容易模板化
  2. 纯角色守线容易失去阵地感
  3. 角色与塔混搭时最容易职责混乱
  4. 一旦局外数值太重，局内构筑会失效
- 是否存在明显短板：`存在`
- 短板类型：`节奏慢 / 养成断层 / 玩法表达弱 / 产品定位不清`
- 这个缺口是否足以让用户迁移：
  有机会，但必须靠“职业差异”和“守线反杀”做出强识别

### 3.4 阶段结论

- 我们的切口是什么：
  从原版“策略守城 + 英雄参与”转向“英雄主导的守线构筑”
- 为什么这个切口值得做：
  它保留了原版最有价值的守线和阵地资产，又把表达方式调整到更适合买量与中期承接的形态

## 4. 破局点分析

### 4.1 胜率来源判断

- 主要靠什么赢：`靠产品 + 靠发行表达`
- 核心破局点不超过 3 个：
  1. 主英雄职业化，建立一眼可懂的战斗 fantasy
  2. 塔作为流派放大器，形成”角色 x 塔 x 遗物”的构筑快感
  3. 低负担移动与高价值站位，保留操作存在感但不演变为纯动作游戏
- **体验比例锚定（原型阶段设计约束基准）：**
  - 角色操作与战斗代入：目标占玩家注意力 **55%–65%**
  - 塔位构筑与战场读图：目标占玩家注意力 **35%–45%**
  - 说明：该比例为原型期的设计约束参照，而非成品期固定值。若测试者描述持续显示两侧偏移（如”感觉角色是主角但塔完全不重要”或”感觉在摆塔但角色像摆设”），优先以测试者反馈为准调整，而非坚持比例本身。

### 4.2 如果靠发行

- 是否存在明显地缘套利空间：否，暂不作为主前提
- 是否存在平台套利空间：有，手游和小游戏都可复用守线表达与短局思路
- 素材套利空间是否明显：中等偏高
- 研发速度是否足够快：只有在 MVP 明确聚焦时才足够快
- 产品综合体验是否过关：待 10 分钟战斗原型验证

### 4.3 如果靠产品

- 差异化类型：`组合创新`
- 超长板是什么：
  守线题材、职业代入、局内 build、塔与站位联动被压进同一个统一体验
- 用户是否能一眼感知到新意：
  若职业和战场同屏表达清楚，则能；若系统信息过多，则不能
- 竞品是否难以短期复制：
  中等。概念容易被抄，但节奏比例和职责划分不容易抄准

### 4.4 破局点逐条论证

#### 破局点1

- 破局点描述：主英雄职业化
- 为什么能吸引用户：
  不同职业天然具备可传播的 fantasy，例如“骑士堵门”“法师清屏”“游侠贯穿”“召唤师铺场”
- 为什么能促成迁移：
  比“普通塔防换皮”更有角色记忆点
- 为什么能支撑留存：
  职业本身就能承担多周目和中期目标
- 风险：
  若职业只是数值皮肤，反而会增加系统负担

#### 破局点2

- 破局点描述：塔是流派放大器
- 为什么能吸引用户：
  玩家能明确感受到“同样的塔，在不同职业和遗物下表现完全不同”
- 为什么能促成迁移：
  让塔防不再只是传统摆塔，而是 build 的一部分
- 为什么能支撑留存：
  塔转职和流派联动为中长期内容留出空间
- 风险：
  若塔和角色各自独立成长，将重新回到三核并立问题

#### 破局点3

- 破局点描述：低负担移动 + 高价值站位
- 为什么能吸引用户：
  有操作存在感，危机关头的“去哪里、何时开大”具备瞬时爽感
- 为什么能促成迁移：
  相比纯摆塔更像“我在指挥战场”，相比纯动作又不要求高频操作
- 为什么能支撑留存：
  站位与职业、塔位、波次关系会持续产生策略空间
- 风险：
  若操作频率过高，会压塌塔防节奏

## 5. 吸引力验证

### 5.1 题材向素材验证

- 验证方式：概念图 + 动态镜头板 + 3 组短视频脚本
- 素材方向：
  1. 最后防线被尸潮/魔潮压到城门，骑士堵门反杀
  2. 法师在寒霜塔与雷塔之间连锁清屏
  3. 召唤师铺场，战场友军越打越多
- 预期结论：
  角色主导的守线画面，比纯塔位经营更容易讲清楚

### 5.2 玩法向素材验证

- 验证方式：白模录屏 + 合成动效包装
- 素材方向：
  1. 同一关卡不同职业对比
  2. 选错遗物暴毙，选对遗物翻盘
  3. Boss 压门前，玩家切位开大并联动塔转职完成反杀
- 预期结论：
  “build 选择导致结果明显不同”是本项目可持续的素材来源

### 5.3 题材+玩法综合验证

- 验证方式：MVP 原型录屏 + A/B 包装文案
- 素材方向：
  1. 守城危机导向
  2. 职业成长导向
  3. build 反转导向
- 预期结论：
  真正有效的表达顺序应为“守线爽点 -> 职业差异 -> build 深度”

### 5.4 综合判断

- 哪类素材表现最好：
  预判为“危机反杀 + 职业清晰 + 一次明显升级变化”
- 素材表现是否稳定：待验证
- 素材拓展空间是否足够大：足够
- 是否值得继续投放验证：值得，但必须等核心原型可录屏后再做

## 6. 可玩性验证

### 6.1 当前验证方式

- 验证方式：`纸面原型 + 设计推演`
- 验证目标：
  验证“角色主导、塔放大、build 成型、移动不乱”四件事能否同时成立
- 参与测试人数：0
- 测试对象：待内部首测与小样本外部观察

### 6.2 核心体验判断

- 实操体验是否好玩：`待白模验证`
- 最强爽点：
  危机关头切位开技能，配合塔和遗物完成一波翻盘
- 最明显问题：
  三套系统一旦比例不对，会同时伤害理解成本和操作体感
- 用户是否愿意继续玩第二局：
  取决于第一局结束前能否形成一句清晰的自我归纳，例如“这把我是冰雷控制流”

### 6.3 玩法拓展空间

- 核心玩法拓展空间：高
- 是否支持中期成长：支持
- 是否支持长线内容：支持，但内容生产压力较高
- 是否存在快速内容枯竭风险：有，若职业、塔转职、遗物三者联动不够深，会很快被耗尽

### 6.4 结论

- 当前玩法是否值得继续做：`值得，但仅限进入 MVP 验证`
- 下一轮需要验证什么：
  1. 单局 10 分钟是否能出现一次明确 build 成型
  2. 操作负担是否低于动作守线产品
  3. 角色是否比塔更像主角，但塔仍然有存在感

## 7. 落地能力评估

### 7.1 产品设计难度

- 核心玩法设计难度：`中高`
- 关键系统设计难度：`中高`
- 核心风险点：
  1. 角色、塔、遗物三方比例不稳
  2. 首局爽点和中期深度很容易二选一
  3. 局外养成容易侵蚀局内构筑价值
- 团队是否有对应经验：
  需要同时具备塔防、RPG 和 Roguelite 节奏能力，单一类型经验不足以覆盖

### 7.2 技术落地能力

- 主要技术难点：
  1. 多单位同屏性能
  2. 自动普攻 + 技能施放 + 站位切换的操控手感
  3. 塔位、怪群、技能特效同时存在时的移动端稳定性
- 现有技术栈是否覆盖：理论上可覆盖
- 是否需要专项预研：需要
- 技术风险等级：`中`

### 7.3 美术与内容难度

- 美术风格难度：中
- 资源生产压力：中高
- 内容生产压力：高
- 是否依赖高成本表现：否，但需要稳定的职业识别和战场可读性

### 7.4 资源投入评估

- 预计人数：6-10 人可做 MVP，10-15 人可进入首版内容生产
- 预计周期：
  1. 玩法 MVP：6-8 周
  2. 可投放演示版：10-14 周
- 核心岗位：主策、战斗策划、客户端、TA/特效、角色/场景美术、制作
- 预估成本：中等
- 是否在公司资源带宽内：取决于是否能接受先做小样本验证

### 7.5 综合落地结论

- 能否落地：`能，但不适合一开始做大`
- 最大落地风险：
  原型期如果没有及时做减法，很容易陷入“每个系统都想保留”的泥潭
- 建议资源方案：
  先做极小范围 MVP，只保留 2 个职业、4 座塔、20 件遗物、1 个 Boss

## 8. 风险与止损条件

### 8.1 市场风险

- 风险描述：
  买量端用户可能只看到“普通守线游戏”，看不到差异
- 风险等级：中
- 应对方式：
  先做能录屏的高识别原型，再测“职业守线”素材而不是系统界面素材

### 8.2 产品风险

- 风险描述：
  角色过强会变守线 ARPG，塔过强会退化为传统塔防，系统过重会掉回原版问题
- 风险等级：高
- 应对方式：
  用原型强制验证四个体验标准，而不是先补系统

### 8.3 研发风险

- 风险描述：
  同屏单位、技能、塔和怪潮带来性能与调参复杂度
- 风险等级：中
- 应对方式：
  白模阶段优先压测同屏数量和低端机表现

### 8.4 投放风险

- 风险描述：
  素材如果先卖 RPG 深度而不是守线爽点，转化会受损
- 风险等级：中
- 应对方式：
  素材表达顺序固定为“守线危机 -> 职业反杀 -> build 成型”

### 8.5 止损条件

- 当 `10分钟MVP测试中，70%以上测试者无法说出自己这局是什么流派` 时，停止扩大制作并回炉玩法定义
- 当 `角色操作负担导致测试者持续忽略塔位、波次和遗物抉择` 时，调整方向，压低移动与主动技能复杂度
- 当 `首轮素材测试里，职业守线与反杀表达无法显著优于原版守城表达` 时，终止该方向或降级为中小体量项目

### 8.6 推进条件（满足全部后方可进入下一阶段）

> 来源：评审_v2 阶段性通过条件，与止损条件配套使用。

1. 内部测试中，**至少 70% 体验者能主动说出本局流派**（如"这把我是冰雷控制流"）
2. **至少 60% 体验者在结局后愿意立即开第二局**，且主动原因是"想换个打法"而非"刚才操作失误"
3. 体验反馈中，**"太乱""顾不过来"等多线程负担描述不高于 30%**
4. **录屏能剪出 3 条以上职业/流派差异足够大的素材脚本**，且发行侧判断可投放
5. **低端机或目标性能机型上，同屏压力波次仍可稳定运行**，无明显掉帧

  4. 美术：职业识别与战场读图
  5. 发行：素材脚本与表达验证

### 9.5 MVP 阶段设计约束项

> 来源：评审_v2 建议调整项，作为 MVP 期间的强制设计边界。

1. **主英雄定位：战场指挥官，不是超模输出单位。** 角色输出不应成为守住防线的唯一依赖，塔必须在无角色干预时仍能维持基础防线。
2. **塔的复杂度上限：只负责战场功能和流派放大，不建立独立大养成。** MVP 期间塔不做独立升级树，防止与角色养成形成竞争。
3. **前 5 分钟信息量控制：只让玩家学会"守一路、选一件遗物、开一次关键技能、看见一次 build 成型"。** 任何额外信息点（第二职业、多路分配、换塔）均推迟到第 5 分钟后。
4. **商业化后置：MVP 评审阶段禁止将商业化需求反推入原型设计。** 先验证乐趣，乐趣验证通过后再讨论商业化接口。
5. **素材验证同步推进：原型一旦可录屏，立即同步让发行测试素材方向。** 不等所有系统齐备再评估买量表达，发行和研发验证并行。

- 决策人：制作人 / 主策 / 发行负责人

## 10. 最终结论

### 10.1 立项判断

- 是否建议立项：`建议进入 MVP 立项`
- 建议立项级别：`中等级别、验证优先`

### 10.2 必须回答的三个问题

- 为什么现在做：
  因为塔防和守线需求仍在，而原版路线已经证明“重理解长局守城”不是最优表达，当前正适合用更统一的 RPG 塔防定义重做。
- 为什么我们能赢：
  因为我们不是简单复刻塔防或割草，而是抓住“职业守线 + 塔位放大 + build 成型”这个仍有缺口的组合。
- 为什么我们做得出来：
  因为该项目可先用小规模 MVP 验证，且核心乐趣可以用极少内容证明，不需要先搭完整内容宇宙。

### 10.3 最终建议

- 建议结论：`可试做，但必须先做极简 MVP，不建议直接进入大规模内容开发`
- 建议下一步：
  先做 2 职业、4 塔、20 遗物、1 Boss 的 10 分钟原型，用体感与素材双验证决定是否进入纸面原型深挖与资源升级

## 附录C：v1 → v2 设计继承与放弃决策

> 本附录说明 v2 相对 v1（立项初审稿 / 玩法设计继承摘要）的核心方向变化，避免后续设计中出现无意识的版本混用。

### C.1 v1 核心定义（已调整）

- 项目代号：守家打宝
- 一句话定义：守住核心防线 + 短程前压争夺高价值掉落的中轻度 RPG 塔防
- 最核心机制：**前场高价值目标系统**（精英营地、宝箱怪、神器祭坛、限时资源点）
- 核心循环：`守线 → 升级三选一 → 前场目标刷新 → 短程出击 → 获得神器 → Build 质变 → 回防 → Boss`
- 平台定位：微信小游戏优先

### C.2 v2 核心重定义

- 项目代号：Kingline TD
- 一句话定义：角色成长驱动的 RPG 塔防，以"主英雄带队守住最后防线"为主题
- 核心机制：**职业 × 塔 × 遗物的局内构筑**，角色是驱动力，塔是流派放大器
- 核心循环：`守线 → 职业技能 → 波次升级 → 遗物选择 → Build 成型 → 塔联动 → Boss 压线反杀`
- 平台定位：手游优先，兼容小游戏方法论

### C.3 继承项（v1 有效资产，v2 保留）

| 资产 | 在 v2 中的形态 |
|------|--------------|
| 守线题材与阵地战压迫感 | 保留，作为核心情绪底盘 |
| 英雄参与感与存在感 | 升级为"主英雄职业化"，是 v2 最强卖点 |
| Boss 压线节奏 | 保留，作为单局收束机制 |
| 局内成长三选一 | 保留，扩展为"遗物三选一" |
| 短局结构（8-10 分钟） | 保留，是中度产品节奏核心 |

### C.4 放弃项（v1 设计，v2 主动舍弃）

| v1 设计 | 放弃原因 |
|---------|---------|
| **前场高价值目标系统**（核心机制） | 前压出击与守线主循环存在节奏冲突；移动端地图空间难以支撑"出去→回来"的空间感；对首局理解成本和操控精度要求较高，不利于中轻度用户进入 |
| 微信小游戏优先 | v2 目标是中度产品，小游戏体量与职业差异、塔转职、遗物深度存在天花板矛盾；改为手游优先，保留小游戏方法论参考 |
| 卡牌养成外壳（英雄卡/佣兵卡/神器卡） | v2 收敛为职业体系，将卡牌外壳推后至长线内容阶段，MVP 期间不做 |
| "打野感"作为差异点 | v2 的差异点改为"职业成长 + 塔位放大 + build 成型"，打野感让位于更统一的 RPG 语言 |

### C.5 说明

前场高价值目标系统（v1 最核心机制）的放弃是 v2 最重要的方向决策，但在 v2 主文档中缺少显式说明，导致与《玩法设计继承摘要》之间存在版本混用风险。

**明确确认：《玩法设计继承摘要》描述的是 v1 阶段设计，已被 v2 方向调整部分覆盖。**

若后续希望重新评估"前场打野"机制，建议作为独立的玩法变体进行 A/B 原型对比，而不是在当前 v2 MVP 中混入。
