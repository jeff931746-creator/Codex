## 1. 项目摘要

### 1.1 一句话定义

- 一款以"桥头/城门前固定防守面"为主战场、以卡牌部署守军与法术为前台交互、以轻 Roguelite 强化为单局成长的 PVE 守线塔防产品。

### 1.2 立项结论

- 立项建议：`可进入初审`
- 核心结论：
  1. 这条方向比"纯类 CR"更轻，比"纯向僵尸开炮like"更有差异，前台识别清楚，适合买量素材表达。
  2. 它天然具备 `固定防守面 + 卡槽部署 + 费用条 + 波次Boss` 这套中轻度用户容易理解的语言。
  3. 如果严格控制复杂度，先做 `PVE短局守线卡组`，它比重 PvP 卡牌塔防和重 War3 守家都更适合作为新项目立项入口。

### 1.3 核心判断

- 这不是做一个"皇室战争 clone"。
- 这也不是做一个"向僵尸开炮多了卡槽"。
- 这条线真正成立的前提是：
  - 卡牌只承担 `部署方式`
  - 守线仍然是主体验
  - 单局成长要足够明显
  - 战场上要看到 `守军上场 -> 扛线 -> 技能支援 -> 波次反压`

### 1.4 最大机会与最大风险

- 最大机会：
  - 用"卡牌部署守军"解决前台新鲜感
  - 用"基地 + 两翼炮位 + 波次怪潮"保住守线直觉
  - 用"局内强化/神器"承接单局成长和复玩
- 最大风险：
  - 如果费用、卡组、单位种类太多，会迅速退化成轻量 CR clone
  - 如果单位参与感太弱，会退化成塔防 + 技能按钮
  - 如果局内强化不够强，这局就只是在机械出牌，爽点不足

## 2. 方向判断

### 2.1 方向定义

- 前台吸量壳子：`桥头最后防线 + 卡牌部署守军 + 怪潮压境`
- 后台长期承接：`单位卡/法术卡成长 + 局内轻 Roguelite 强化 + 章节波次`

### 2.2 目标平台

- 第一目标：微信小游戏 / 轻中度移动端
- 第二目标：后续可扩 App 端验证

### 2.3 目标用户

- 核心用户：
  - 喜欢《Clash Royale》这类短局自动战斗部署感的用户
  - 喜欢守线、怪潮、基地防守、Boss 波次的用户
  - 喜欢《向僵尸开炮》式单局成长和清场反馈的用户
- 泛用户：
  - 喜欢"最后一道防线"
  - 喜欢低理解成本策略战斗
  - 喜欢军团、兵种、技能协同的中轻度用户

### 2.4 为什么是这个方向

#### 事实

- `Clash Royale` 的短局自动部署与冲突表达仍然具备市场吸引力，且其后续模式如 `Merge Tactics` 证明"部署 + 自动战斗 + 短局成长"依旧有用户基础。
- 堡垒/基地防守 + 怪潮压境在移动端仍然是低理解成本、高传播效率的表达方式。
- 纯卡牌塔防在移动端已有样本，但多数会变重、偏 deck-builder 或 PvP，理解和立项成本都更高。

#### 判断

- 对当前团队而言，最合理的切法不是去做重 PvP 卡牌塔防，而是做：
  - `PVE`
  - `固定防守面`
  - `少量卡组`
  - `波次成长`
- 这条方向能同时吃到：
  - 低理解门槛
  - 足够的新鲜前台
  - 比传统塔防更适合买量
  - 比纯卡牌更容易短局验证

### 2.5 阶段结论

- 方向结论：`通过，可进入立项初审`

## 3. 市场与竞品调研

### 3.1 竞品锚点

本方向不找单一直接竞品，而是由三组锚点组成：

1. `短局部署对战锚点`
- Clash Royale 及其后续自动战斗衍生模式

2. `守线成长锚点`
- 向僵尸开炮一类"守线 + 局内强化 + Boss收束"产品

3. `PVE堡垒防守锚点`
- 固定防守面、怪潮压境、基地方向明确的产品

### 3.2 与现有路径的区别

#### 相比纯类 CR

- 我们不做 PvP
- 我们不做重平衡竞技
- 我们不做复杂卡组费用博弈
- 我们更强调：
  - 守线
  - 波次
  - 局内成长
  - Boss 检验

#### 相比向僵尸开炮like

- 我们不是单英雄站门口清怪
- 我们不是纯技能成长
- 我们把"守军部署"和"法术支援"抬到前台

#### 相比传统卡牌塔防

- 我们不做厚卡组构筑
- 我们先做小卡组、短局、明确防守面
- 卡牌是操作语言，不是主认知

### 3.3 竞品空档

当前可争取的空档主要是：

- `更直白的守线表达`
  - 比 CR 更容易懂"我在守什么"
- `更轻的卡牌操作`
  - 比纯卡牌塔防更少学习成本
- `更强的单局成长`
  - 比传统短局部署产品更有"这把起飞了"的爽点

### 3.4 市场判断

- 这条方向不是赛道红利型项目。
- 它成立的逻辑是：
  - 用成熟语言降低理解成本
  - 用组合创新拉开前台差异
  - 用短局波次和轻 Roguelite 承接留存

## 4. 方向拆解

### 4.1 主卖点

- 固定桥头/城门前防守面
- 底部卡槽部署单位与法术
- 自动守军与炮位共同挡潮
- 波次推进 + 精英 + Boss
- 局内轻 Roguelite 强化

### 4.2 核心体验

- 我不是在摆塔，而是在 `部署守军守线`
- 我不是在清怪，而是在 `守住桥头/城门`
- 我不是在打对战，而是在 `扛波次压力`
- 我不是在堆数值，而是在 `养成阵容`

### 4.3 核心玩法一句话

- 玩家在固定防守面前，用费用驱动的卡牌部署守军和法术，守住波次怪潮，并在单局中通过轻 Roguelite 强化逐渐把阵容养成。

### 4.4 核心爽点

- 守住了
- 阵容成型了
- 这波扛住了
- Boss 被打爆了
- 这把强化路线很顺

## 6. 原型建议

### 6.1 纸面原型优先验证点

最先验证 3 件事：

1. `玩家会不会把它看成轻 CR`
2. `玩家会不会觉得单位部署比技能按钮更有趣`
3. `波次强化后是否真的有"这把阵容成型"的感觉`

### 6.2 轻灰盒验证点

如果纸面原型通过，轻灰盒阶段优先验证：

1. 单位部署的手感和节奏
2. 费用条的压力感
3. 波次推进的节奏
4. Boss 战的收束感

## 7. 风险与机会

### 7.1 核心假设

1. 卡槽部署能成为主交互，而不是装饰
2. 单位存在感足够强，不会被炮位和法术盖过
3. 局内强化能显著改变战斗结果，而不是平推数值
4. Boss 波必须能收束，不然单局会发散

### 7.2 最大风险

#### 风险 1：退化成 CR clone

- 触发原因：
  - 太强调费用和卡组对局
  - 单位和技能都走对战逻辑
- 规避方式：
  - 锁 PVE
  - 简化费用体系
  - 把波次和守线做成主节奏

#### 风险 2：退化成塔防 + 技能按钮

- 触发原因：
  - 单位上场不重要
  - 玩家主要还是靠炮位和法术清怪
- 规避方式：
  - 保证单位承担核心扛线职责
  - 炮位偏辅助、不是唯一输出核心

#### 风险 3：局内成长过弱

- 触发原因：
  - 三选一都是小数值
  - 玩家体验不到"这把有变化"
- 规避方式：
  - 至少设计一批能改变战斗表现的强化件
