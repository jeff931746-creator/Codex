## 一、重构结论

本次不做旧稿迭代，直接重构。

重构后的主模型只有一句话：

`玩家作为国王，在中央防线带队守线；波次中出现近场神器事件时，玩家短程前压争夺高质量奖励，再回防处理精英和Boss。`

这次只保留 4 个不可删的核心点：

1. `国王带队`
2. `中央防线`
3. `近场争夺事件`
4. `神器驱动Build跃迁`

只要删掉其中任一项，产品都会退化成别的东西。

## 二、原型定义卡

### 1. 一句话定义

玩家守住中央城门防线，在怪潮压力中带队作战，并抓住两次短程前压机会抢神器，让本局 Build 成型后顶住 Boss。

### 2. 20字核心循环

`守线 -> 抢神器 -> Build成型 -> 回防 -> 打Boss`

### 3. 核心体验

- 我不是单英雄守门，而是在带一支队伍守线
- 我不是纯站桩防守，而是要抓时机短程前压抢收益
- 我拿到神器后，这一把的打法会明显变化
- Boss 来时，我会检验前面所有选择值不值

### 4. 本次只验证的假设

- 假设 1：`国王 + 固定小队` 比单英雄守门更有身份感
- 假设 2：`前场高价值目标` 能进入主循环，而不是打断主循环
- 假设 3：`高质量神器` 能在 8 分钟内制造一次明确 Build 跃迁

### 5. 本次不验证的内容

- 局外养成
- 多职业系统
- 自由建塔
- 多地图章节
- 商业化和外围系统

### 6. Kill 条件

- 玩家大多数时间仍感知为“一个人守门”
- 前压行为普遍被理解为离岗犯错
- 神器奖励只是数值变大，没有打法变化
- Boss 战只是血厚数值检定，没有额外压力

## 三、最小系统模型

### 1. 战场

只做 `1张横屏单战区地图`，结构固定为：

`前场事件区 -> 交战区 -> 中央防线 -> 城门`

约束：

- 任何前场事件都必须在离开中央防线 `3-5 秒` 内可完成
- 不做大地图
- 不做多线防守
- 不做长距离跑图

### 2. 玩家侧单位

固定 6 个战场要素：

1. `国王`
2. `盾卫`
3. `弓手`
4. `祭司`
5. `箭塔`
6. `路障`

职责固定：

- 国王：前压、补漏、处理精英、打断 Boss
- 盾卫：卡住第一接敌点
- 弓手：稳定清线
- 祭司：维持续航
- 箭塔：补基础伤害
- 路障：拖时间

### 3. 资源

只保留 2 种资源：

- `城门耐久`
- `升级经验`

不做金币，不做法力条，不做经营资源。

### 4. 成长

只保留 2 层局内成长：

1. `普通三选一`
- 小强化
- 每局约 3 次

2. `高质量神器`
- 机制变化
- 每局 1-2 件

Boss 遗物只做结算占位，不进本轮核心验证。

## 四、玩家动作集

### 高频动作

- 移动国王
- 释放国王技能
- 选择强化或拾取神器

### 中频动作

- 判断当前是否前压争夺
- 决定本局 Build 偏向

### 低频动作

- 处理精英
- 打断 Boss 关键动作

### 判断

- 高频动作控制在 3 个以内，符合纸面原型约束
- 关键决策只有 2 个，足以支撑单局差异
- 玩家应在前 30 秒内明确理解自己在“守中央城门”

## 五、单局时间脚本（7-8分钟）

### 0:00 - 1:00 建立守线目标

#### 玩家看到什么

- 城门和中央防线清晰可见
- 第一波近战怪从前场推进
- 盾卫在前、弓手和箭塔在后，初步形成防线

#### 玩家做什么

- 控制国王补伤害
- 观察漏怪风险
- 进行第一次普通三选一

#### 系统给什么

- 明确的城门受损反馈
- 第一次小强化

#### 这一段的目的

- 建立“守住中央城门”这一唯一目标
- 让玩家理解自己不是在纯割草

#### 风险点

- 如果城门压力太低，会被误解成普通幸存者式清怪

### 1:00 - 2:30 第一次压力抬升

#### 玩家看到什么

- 快跑怪开始出现
- 第一波明显漏怪风险到来

#### 玩家做什么

- 用国王技能补漏
- 开始判断小队哪个环节更弱

#### 系统给什么

- 第二次普通三选一前的经验积累
- 更明确的抗压反馈

#### 这一段的目的

- 让国王的“救场作用”成立
- 证明这是“带队守线”，不是自动守线

#### 风险点

- 如果国王不出手也能轻松过，这个身份会变弱

### 2:30 - 4:30 第一次前压争夺与第一次 Build 跃迁

#### 玩家看到什么

- 前场刷新 `神器祭坛`
- 主线怪潮没有暂停

#### 玩家做什么

- 判断当前防线能否独立撑住 3-5 秒
- 前压激活祭坛并拿到第一件神器

#### 系统给什么

- 3 选 1 高质量神器
- 第一只精英怪在拿完神器后不久出现

#### 这一段的目的

- 验证前压是否能进入主循环
- 验证神器是否能立刻改变处理方式

#### 风险点

- 如果事件点太远，前压会像跑图
- 如果神器不够强，玩家会觉得冒险不值

### 4:30 - 6:00 第二次压力抬升与第二次争夺

#### 玩家看到什么

- 精英怪推进
- 第二次前场事件 `宝箱怪` 出现并尝试逃跑

#### 玩家做什么

- 基于第一件神器后的 Build 做第二次取舍
- 决定保守守线还是再赌一次收益

#### 系统给什么

- 第二次高质量奖励，或强三选一
- Boss 前的最后调整窗口

#### 这一段的目的

- 验证第二次争夺是否仍有吸引力
- 验证 Build 是否已经形成方向差异

#### 风险点

- 如果第二次争夺吸引力明显下降，主循环会塌

### 6:00 - 8:00 Boss 收束

#### 玩家看到什么

- Boss `破城巨兽` 压近中央防线
- Boss 会撞击路障、召唤分流怪、读条重击城门

#### 玩家做什么

- 回防
- 用国王打断 Boss 关键动作
- 靠小队稳态输出和神器联动完成收束

#### 系统给什么

- Boss 战高潮
- 局终结算占位

#### 这一段的目的

- 检验前面所有成长和争夺是否有意义
- 让“国王必须亲自参与”成立

#### 风险点

- 如果 Boss 只会站桩挨打，收束会失去说服力

## 六、最小系统表

| 系统 | 是否必须 | 当前作用 | 不做会怎样 |
|---|---|---|---|
| 中央防线与城门 | 是 | 提供唯一守线目标 | 产品失去“守家”识别 |
| 国王主控 | 是 | 提供身份感、救场和前压能力 | 退化成自动守线 |
| 固定三随从小队 | 是 | 提供“带队感”和职责分工 | 退化成单英雄守门 |
| 箭塔与路障 | 是 | 提供守线下限和时间缓冲 | 前压容错不足 |
| 普通三选一 | 是 | 提供节奏型成长 | 单局反馈过平 |
| 高质量神器 | 是 | 提供 Build 跃迁 | 差异点不成立 |
| 前场事件系统 | 是 | 提供风险换成长的关键动作 | 退化成普通守线成长 |
| 精英怪 | 是 | 检验神器后处理方式变化 | 中段压力不足 |
| Boss | 是 | 负责单局收束 | 整局没有终点和检验 |
| 局外养成 | 否 | 不在本轮验证范围 | 不影响纸面推演 |
| 多职业系统 | 否 | 不在本轮验证范围 | 不影响纸面推演 |
| 自由建塔 | 否 | 不在本轮验证范围 | 反而更利于收敛 |

## 七、三轮纸面推演

### 推演 1：正常流

#### 玩家行为假设

- 开局正常守线
- 第一次事件选择前压
- 第二次事件视防线状态再决定争夺

#### 预期结果

- 第一件神器带来明确 Build 变化
- 小队职责逐渐清晰
- Boss 前形成一次完整成长弧线

#### 当前风险

- 第一件神器如果只提升数值，整局会仍然偏平

### 推演 2：保守流

#### 玩家行为假设

- 第一次事件不前压
- 第二次事件也偏保守
- 主要依赖普通三选一和基础防线过关

#### 预期结果

- 理论上应该能打到 Boss
- 但整局体验更稳、更平、缺少成型感

#### 要验证的问题

- 不前压是否仍能通过
- 但前压与不前压的体验差异是否足够大

### 推演 3：激进流

#### 玩家行为假设

- 两次事件都强行争夺
- 成长偏向联动或技能爆发
- 容忍城门承压换神器

#### 预期结果

- 如果成功，Build 成型速度最快
- Boss 战前最容易形成高潮

#### 要验证的问题

- 激进流是否只是“高风险更强”，还是会直接变成最优解
- 如果激进流始终最优，保守流会失去存在价值

## 八、首批神器方向

只做 6 件，避免扩表。

### 国王技能向

- `王令连斩`
- `王怒追击`

### 统御联动向

- `御前号角`
- `王旗前压`

### 守线稳态向

- `盾墙誓约`
- `圣坛余辉`

原则：

- 必须改变战场处理方式
- 不接受单纯攻击 +20% 作为核心神器

## 九、纸面原型问题清单

### 当前最可能成立的点

- `国王 + 固定小队 + 中央防线` 的身份模型清楚
- `前场近场争夺` 比大地图打野更适合小游戏节奏
- `神器驱动Build跃迁` 有机会形成明确中段爽点

### 当前最可能崩的点

- 小队存在感不足，退化成“国王带特效守门”
- 前压收益不够香，导致事件形同虚设
- Boss 机制不够强，导致整局没有真正收束

### 当前最需要在灰盒前确认的点

- 前场事件的最佳距离到底是 3 秒、4 秒还是 5 秒
- 小队/防线/国王的战力占比怎样才最稳
- 第一件神器的强度阈值应该多高，才会让玩家觉得“这把成型了”

---

## 十、继续与回滚条件

### 可以继续当前版本的条件

- 三轮纸面推演里，`正常流` 和 `激进流` 都能成立
- 保守流能打到 Boss，但明显不如争夺流有爽点
- 玩家身份稳定表现为“国王带队守线”

### 需要局部回调的条件

- 前压能成立，但事件距离或奖励强度不对
- 小队职责存在，但反馈不够强
- Boss 有收束，但机制压迫感不足

### 需要退回评审重看的条件

- 删除前场事件后玩法变化不大
- 删除小队后玩法变化不大
- 产品第一眼和实际体验都更像普通守线成长游戏

一句话总结：

`重构后的纸面原型不再追求“像War3多少”，而是只验证一个最小闭环：国王带队守线，冒险抢神器，再回防打Boss。`
