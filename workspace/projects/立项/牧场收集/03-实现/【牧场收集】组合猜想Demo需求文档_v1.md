# 【牧场收集】组合猜想Demo需求文档 v1

## 一、Demo验证目标与范围

### 1.1 核心问题

本Demo旨在验证"组合猜想→收集→验证→补强"的心理链路是否成立，即：
- 玩家能否在开局根据传闻和定位，说出组合猜想
- 玩家是否为了凑齐猜想的组合而继续收集
- 玩家验证成功后，能否说出组合为什么强
- 玩家发现缺口后，能否主动说出需要补充什么
- 玩家补强组合后，是否表现出"发现完美组合"的惊喜

### 1.2 验证的心理链路

```
传闻卡+生物定位 → 组合猜想 → 收集动机 → 验证成功 → 发现缺口 → 补强动机 → 完美组合
```

### 1.3 Demo范围界定

**做什么：**
- 6只生物（兔子、小鹿、羊、熊、鹰、狼）
- 3张传闻卡（高速突袭、铁壁防线、天地夹击）
- 2条剧本分支（突袭流、铁壁流）
- 3场战斗（单体强敌、群体小怪、混合战场）
- 组合效果展示（战斗反馈说明组合为什么强）

**不做什么：**
- 不做随机掉落（固定剧本发放碎片）
- 不做能量系统（Demo中无限战斗）
- 不做多个区域（只有一个测试区域）
- 不做长期养成（只验证组合猜想链路）
- 不做美术资源（占位图形+文字反馈）
- 不做音效音乐（可选，不影响验证）

## 二、玩家体验流程（验证路径）

### 2.1 开局：组合猜想阶段

**玩家看到什么：**
- 3张传闻卡，每张卡描述一种组合策略
  - 传闻卡1：高速突袭（"据说兔子和小鹿配合，能在敌人反应前结束战斗"）
  - 传闻卡2：铁壁防线（"据说羊和熊配合，能让敌人无法突破防线"）
  - 传闻卡3：天地夹击（"据说地面和空中同时进攻，能让敌人顾此失彼"）
- 6只生物的定位标签
  - 兔子：高速、脆皮、单体输出
  - 小鹿：高速、辅助、群体加速
  - 羊：低速、坦克、群体护盾
  - 熊：低速、坦克、反击
  - 鹰：高速、空中、单体输出
  - 狼：中速、地面、群体输出

**玩家做什么：**
- 选择一张传闻卡，决定想验证的组合方向

**验证什么：**
- 玩家能否在30秒内说出组合猜想（例如："我想试兔子+小鹿，应该能快速击杀"）
- 玩家是否表现出对组合的期待

**成功信号：**
- 玩家主动说出"我想试XX组合"
- 玩家能说出选择理由（例如："兔子快，小鹿加速，应该能抢先手"）

**失败信号：**
- 玩家说"不知道选哪个"
- 玩家随机选择，说不出理由

---

### 2.2 第一轮：收集+验证阶段

**玩家看到什么：**
- 战斗界面，敌人是单体强敌（高血量、中速度）
- 战斗胜利后，获得碎片（固定掉落，按剧本分支）
  - 突袭流：兔子碎片 x3
  - 铁壁流：羊碎片 x3
- 孵化界面，显示碎片进度（3/3），可以孵化生物

**玩家做什么：**
- 进行3场战斗，收集碎片
- 孵化第一只生物（兔子或羊）
- 继续战斗，收集第二只生物的碎片
- 孵化第二只生物（小鹿或熊）
- 组成2只生物的组合，进行验证战斗

**验证什么：**
- 玩家是否表现出"终于凑齐了"的兴奋
- 玩家验证成功后，能否说出组合为什么强

**战斗反馈示例（突袭流）：**
```
战斗开始
→ 小鹿使用"群体加速"，兔子速度+50%
→ 兔子抢先攻击，造成高额伤害
→ 敌人尚未行动，已被击败
战斗结束：胜利
组合效果：高速突袭生效，敌人无法反击
```

**成功信号：**
- 玩家说"终于凑齐了"或类似表达
- 玩家能说出"兔子快，小鹿加速，所以能抢先手"

**失败信号：**
- 玩家验证成功后无反应
- 玩家说不出组合为什么强

---

### 2.3 第二轮：发现缺口阶段

**玩家看到什么：**
- 新的战斗场景：群体小怪（低血量、高数量）
- 战斗过程中，组合效果不理想
  - 突袭流：兔子单体输出，无法快速清理群体
  - 铁壁流：羊和熊防御强，但输出不足，战斗拖延

**战斗反馈示例（突袭流）：**
```
战斗开始
→ 小鹿使用"群体加速"，兔子速度+50%
→ 兔子攻击敌人A，击败
→ 敌人B、C、D、E同时攻击，兔子血量下降
→ 兔子攻击敌人B，击败
→ 敌人C、D、E继续攻击，兔子阵亡
战斗结束：失败
组合缺口：缺少群体输出，无法应对多目标
```

**玩家做什么：**
- 尝试用现有组合挑战群体小怪
- 失败后，思考组合缺口

**验证什么：**
- 玩家能否主动说出组合缺口（例如："缺少群体输出"）
- 玩家是否表现出想要补强组合的动机

**引导提示（可选）：**
- 如果玩家说不出缺口，系统提示："你的组合在面对多个敌人时，似乎力不从心"

**成功信号：**
- 玩家主动说出"缺少群体输出"或类似表达
- 玩家说"我需要一个能打群体的生物"

**失败信号：**
- 玩家说"不知道为什么失败"
- 玩家放弃，不想继续

---

### 2.4 第三轮：补强组合阶段

**玩家看到什么：**
- 新的生物碎片掉落（固定剧本）
  - 突袭流：鹰碎片 x3（空中、单体输出，但攻击范围大）
  - 铁壁流：狼碎片 x3（地面、群体输出）
- 孵化第三只生物后，组合完整

**玩家做什么：**
- 收集第三只生物的碎片
- 孵化第三只生物
- 组成3只生物的完美组合，再次挑战群体小怪

**战斗反馈示例（突袭流）：**
```
战斗开始
→ 小鹿使用"群体加速"，兔子和鹰速度+50%
→ 兔子攻击敌人A，击败
→ 鹰使用"俯冲"，攻击敌人B、C，击败
→ 敌人D、E尚未行动，已被击败
战斗结束：胜利
组合效果：高速突袭+空中打击，完美应对群体
```

**验证什么：**
- 玩家是否表现出"发现完美组合"的惊喜
- 玩家能否说出补强后的组合为什么强

**成功信号：**
- 玩家说"这就对了"或"完美"
- 玩家能说出"鹰能打群体，补上了缺口"

**失败信号：**
- 玩家验证成功后无反应
- 玩家说不出补强后的组合为什么强

## 三、系统需求拆解

### 3.1 战斗系统

**功能：**
- 展示战斗过程（回合制或实时，Demo中可简化为回合制）
- 计算战斗结果（固定结果，按剧本配置）
- 展示组合效果（战斗中实时提示+战斗结束后总结）

**输入：**
- 玩家队伍（当前已孵化的生物）
- 敌人配置（单体强敌、群体小怪、混合战场）
- 剧本配置（固定战斗结果）

**输出：**
- 战斗过程文字反馈
- 战斗结果（胜利/失败）
- 组合效果说明（为什么强/为什么弱）
- 掉落碎片（固定掉落）

**与验证目标的关系：**
- 战斗反馈说明组合为什么强 → 验证玩家能否理解组合效果
- 战斗失败暴露组合缺口 → 验证玩家能否发现缺口

**关键设计：**
- 战斗结果固定，确保验证路径可控
- 战斗反馈必须说明"为什么"，不只是"胜利/失败"
- 组合效果在战斗中实时提示（例如："小鹿加速生效，兔子速度+50%"）

---

### 3.2 收集系统

**功能：**
- 战斗胜利后，掉落碎片
- 记录碎片进度（例如：兔子碎片 2/3）

**输入：**
- 战斗结果（胜利）
- 剧本配置（固定掉落）

**输出：**
- 碎片数量更新
- 碎片进度提示（例如："获得兔子碎片 x1，当前进度 2/3"）

**与验证目标的关系：**
- 固定掉落确保玩家能按剧本收集 → 验证收集动机
- 碎片进度提示让玩家知道"还差多少" → 强化收集动机

**关键设计：**
- Demo中不做随机掉落，固定剧本发放碎片
- 碎片进度可视化，让玩家知道"快凑齐了"

---

### 3.3 孵化系统

**功能：**
- 消耗碎片，孵化生物
- 将生物加入玩家队伍

**输入：**
- 碎片数量（例如：兔子碎片 3/3）

**输出：**
- 生物孵化成功
- 生物加入队伍
- 孵化动画/反馈（可选）

**与验证目标的关系：**
- 孵化成功是"凑齐组合"的关键时刻 → 验证玩家是否兴奋

**关键设计：**
- 孵化界面展示生物定位和技能
- 孵化成功后，提示"可以组成XX组合"

---

### 3.4 组合系统

**功能：**
- 检测当前队伍是否满足组合条件
- 在战斗中触发组合效果
- 展示组合效果说明

**输入：**
- 玩家队伍（当前已孵化的生物）
- 组合配置表（定义组合条件和效果）

**输出：**
- 组合检测结果（是否满足组合）
- 组合效果触发（战斗中生效）
- 组合效果说明（战斗反馈）

**与验证目标的关系：**
- 组合效果说明让玩家理解"为什么强" → 验证玩家能否说出原因
- 组合缺口暴露让玩家发现"缺什么" → 验证玩家能否主动补强

**关键设计：**
- 组合配置表定义：
  - 组合名称（例如：高速突袭）
  - 组合条件（例如：兔子+小鹿）
  - 组合效果（例如：速度+50%，抢先攻击）
  - 组合说明（例如："兔子快，小鹿加速，能在敌人反应前结束战斗"）
- 组合检测在战斗开始前触发
- 组合效果在战斗中实时展示

---

### 3.5 UI系统

**功能：**
- 开局界面：展示传闻卡和生物定位，玩家选择分支
- 战斗界面：展示战斗过程和组合效果
- 收集界面：展示碎片进度
- 孵化界面：展示生物信息和孵化按钮
- 队伍界面：展示当前队伍和组合状态

**与验证目标的关系：**
- 开局界面引导玩家做出组合猜想
- 战斗界面展示组合效果，帮助玩家理解"为什么强"
- 收集界面强化收集动机
- 孵化界面是"凑齐组合"的关键时刻

**关键设计：**
- 开局界面：传闻卡+生物定位并排展示，方便玩家对比
- 战斗界面：战斗反馈文字清晰，组合效果高亮显示
- 收集界面：碎片进度条可视化
- 孵化界面：生物定位和技能说明清晰

## 五、技术实现指南

### 5.1 场景结构

```
Demo场景结构：
- MainScene（主场景）
  - OpeningScene（开局场景）
    - RumorCardDisplay（传闻卡展示）

    - BranchSelector（分支选择器）
  - BattleScene（战斗场景）
    - BattleField（战场）
    - BattleFeedback（战斗反馈）

    - HatchButton（孵化按钮）
  - TeamScene（队伍场景）
    - TeamDisplay（队伍展示）
    - CombinationStatus（组合状态）
```

### 5.2 数据结构

**生物数据：**
```gdscript
class Creature:
  var id: String
  var name: String
  var tags: Array  # ["高速", "脆皮", "单体输出"]
  var skill: Dictionary  # {name: "快速攻击", effect: "单体高伤害"}
```

**组合数据：**
```gdscript
class Combination:
  var id: String
  var name: String
  var conditions: Array  # ["兔子", "小鹿"]
  var effects: Dictionary  # {speed_bonus: 0.5, first_strike: true}
  var description: String
```

**剧本数据：**
```gdscript
class ScriptBranch:
  var branch_id: String
  var rumor_card: String
  var battles: Array  # [BattleConfig, BattleConfig, ...]
  var drops: Array  # [{"creature": "兔子", "fragments": 3}, ...]
```

---

### 5.3 关键算法

**组合检测：**
```gdscript
func check_combination(team: Array, combinations: Array) -> Dictionary:
  for combo in combinations:
    var match = true
    for condition in combo.conditions:
      if not team.has(condition):
        match = false
        break
    if match:
      return combo
  return null
```

**战斗结果固定：**
```gdscript
func execute_battle(team: Array, enemy: Dictionary, script: Dictionary) -> Dictionary:
  var result = script.get_battle_result(team, enemy)
  var feedback = generate_feedback(team, enemy, result)
  return {result: result, feedback: feedback}
```

**组合效果展示：**
```gdscript
func generate_feedback(team: Array, enemy: Dictionary, result: Dictionary) -> String:
  var combo = check_combination(team, combinations)
  if combo:
    return "组合效果：" + combo.description + "\n战斗结果：" + result.outcome
  else:
    return "战斗结果：" + result.outcome
```

---

### 5.4 Godot实现建议

1. **使用信号系统连接各场景**
   - `signal branch_selected(branch_id)`
   - `signal battle_completed(result)`
   - `signal creature_hatched(creature_id)`

2. **使用Autoload单例管理游戏状态**
   - `GameState.gd`：存储当前分支、碎片进度、已孵化生物

3. **使用JSON配置文件存储数据**
   - `creatures.json`：生物数据
   - `combinations.json`：组合数据
   - `scripts.json`：剧本数据

4. **战斗系统简化**
   - 回合制战斗，固定结果
   - 战斗反馈用文字展示，不做复杂动画

## 六、验证与测试

### 6.1 每个阶段的观察点

| 阶段 | 观察点 | 成功信号 | 失败信号 |
|------|--------|----------|----------|
| 开局 | 玩家能否在30秒内说出组合猜想 | 玩家主动说出"我想试XX组合" | 玩家说"不知道选哪个" |
| 第一轮 | 玩家是否表现出"终于凑齐了"的兴奋 | 玩家说"终于凑齐了" | 玩家验证成功后无反应 |
| 第二轮 | 玩家能否主动说出组合缺口 | 玩家说"缺少群体输出" | 玩家说"不知道为什么失败" |
| 第三轮 | 玩家是否表现出"发现完美组合"的惊喜 | 玩家说"这就对了"或"完美" | 玩家验证成功后无反应 |

---

### 6.2 成功/失败信号

**整体成功信号：**
- 玩家能在每个阶段主动说出组合猜想、缺口、补强方案
- 玩家表现出兴奋、惊喜等情绪
- 玩家能说出组合为什么强

**整体失败信号：**
- 玩家在多个阶段说"不知道"
- 玩家验证成功后无反应
- 玩家说不出组合为什么强

### 6.3 测试流程

1. **招募测试玩家**
   - 3-5个测试玩家
   - 最好是对收集类游戏有一定了解的玩家

2. **测试流程**
   - 每人玩一遍Demo（10-15分钟）
   - 测试员在旁观察，记录玩家反应
   - 测试员不主动提示，只在玩家卡住时询问"你在想什么"

3. **记录内容**
   - 每个阶段的玩家反应（说了什么、做了什么）
   - 玩家是否主动说出组合猜想、缺口、补强方案
   - 玩家的情绪表现（兴奋、惊喜、困惑、放弃）

4. **统计成功率**
   - 成功率 ≥ 60%：心理链路成立，可以进入正式开发
   - 成功率 < 60%：心理链路不成立，需要调整设计或放弃方向
