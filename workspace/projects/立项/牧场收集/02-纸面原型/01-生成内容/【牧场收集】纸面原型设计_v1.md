# 【牧场收集】纸面原型设计_v1

## 一、数值框架

### 核心资源数值

| 资源 | 初始值 | 上限 | 恢复机制 | 用途 |
|------|--------|------|----------|------|
| 能量 | 10 | 20 | 5分钟/1点 | 触发战斗（每场消耗3点） |
| 生物碎片 | 0 | 999 | 战斗掉落 | 孵化/升级生物 |

### 生物孵化/升级数值

| 等级 | 所需碎片 | 战力提升 | 能量返还 |
|------|----------|----------|----------|
| 孵化（0→1） | 5 | +10 | 2 |
| 升级（1→2） | 8 | +15 | 3 |
| 升级（2→3） | 12 | +20 | 4 |

**设计意图：**
- 孵化阈值低（5个），让玩家快速获得第一只生物
- 升级成本递增，制造长期目标
- 能量返还机制鼓励持续孵化/升级

### 区域解锁条件

| 区域 | 解锁条件 | 能量奖励 | 战斗难度 | 碎片掉落数 |
|------|----------|----------|----------|-----------|
| 区域1（草原） | 初始解锁 | - | 低 | 2-3 |
| 区域2（森林） | 拥有1只生物 | 10 | 中 | 3-4 |
| 区域3（山地） | 拥有2只生物 | 15 | 高 | 4-5 |

**设计意图：**
- 解锁条件简单，确保 10 分钟内能体验 3 个区域
- 能量奖励补偿解锁成本，维持循环流畅
- 难度和收益成正比

## 二、战斗玩法设计

### 战斗流程（30-60秒/场）

```
开始战斗
→ 玩家操控角色站在防守点
→ 敌人分3波从右侧进攻（每波间隔10秒）
→ 玩家移动+攻击+释放技能
→ 击败所有敌人 → 胜利 → 掉落碎片
→ 敌人突破防守点 → 失败 → 不掉落碎片但返还能量
```

### 玩家角色操作

| 操作 | 按键 | 效果 | 冷却 |
|------|------|------|------|
| 移动 | WASD/方向键 | 上下左右移动 | - |
| 普通攻击 | 鼠标左键/空格 | 前方扇形攻击，伤害10 | 0.5秒 |
| 技能1 | Q | 范围AOE，伤害30 | 5秒 |
| 技能2 | E | 召唤当前队伍中的生物助战（持续5秒） | 10秒 |

**设计意图：**
- 操作简单，新手易上手
- 技能2让生物参与战斗，强化"收集养成"的意义

## 四、反比算法设计

### 碎片掉落权重计算

```python
def calculate_drop_weight(fragment_type, player_inventory):
    """
    计算某种碎片的掉落权重
    """
    base_weight = 100  # 基础权重
    
    # 获取玩家背包中该碎片的数量
    owned_count = player_inventory.get(fragment_type, 0)
    
    # 获取孵化该生物所需的碎片数
    required_count = get_hatch_requirement(fragment_type)
    
    # 计算距离孵化的差距
    gap = required_count - owned_count
    
    if gap <= 0:
        # 已经可以孵化，降低权重
        weight = base_weight * 0.3
    elif gap <= 2:
        # 差1-2个就能孵化，大幅提升权重
        weight = base_weight * 2.0
    elif gap <= 5:
        # 差3-5个，适度提升权重
        weight = base_weight * 1.5
    else:
        # 差距较大，正常权重
        weight = base_weight * 1.0
    
    # 稀有度修正
    rarity_multiplier = {
        "普通": 1.0,
        "稀有": 0.5,
        "史诗": 0.15,
        "传说": 0.02
    }
    weight *= rarity_multiplier.get(get_rarity(fragment_type), 1.0)
    
    return weight

def drop_fragments(region, player_inventory, drop_count):
    """
    根据区域和玩家背包状态掉落碎片
    """
    # 获取该区域的碎片池
    fragment_pool = get_region_fragments(region)
    
    # 计算每种碎片的权重
    weights = {}
    for fragment in fragment_pool:
        weights[fragment] = calculate_drop_weight(fragment, player_inventory)
    
    # 按权重随机抽取
    dropped = weighted_random_choice(weights, drop_count)
    
    return dropped
```

**设计意图：**
- "差1-2个"时权重x2，制造"再刷一次就能孵化"的紧迫感
- 已拥有的碎片权重降低到0.3，避免重复掉落过多
- 稀有度影响基础掉率，但反比机制仍然生效

### 关键代码示例（反比算法）

```gdscript
# drop_system.gd
extends Node

func calculate_drop_weight(fragment_type: String, inventory: Dictionary) -> float:
    var base_weight = 100.0
    var owned = inventory.get(fragment_type, 0)
    var required = CreatureData.get_hatch_requirement(fragment_type)
    var gap = required - owned
    
    var weight = base_weight
    if gap <= 0:
        weight *= 0.3
    elif gap <= 2:
        weight *= 2.0
    elif gap <= 5:
        weight *= 1.5
    
    var rarity = CreatureData.get_rarity(fragment_type)
    match rarity:
        "普通": weight *= 1.0
        "稀有": weight *= 0.5
        "史诗": weight *= 0.15
        "传说": weight *= 0.02
    
    return weight

func drop_fragments(region: int, inventory: Dictionary, count: int) -> Array:
    var pool = RegionData.get_fragment_pool(region)
    var weights = {}
    
    for fragment in pool:
        weights[fragment] = calculate_drop_weight(fragment, inventory)
    
    return weighted_random_choice(weights, count)
```

## 十、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
