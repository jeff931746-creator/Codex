# 【牧场收集】纸面原型设计_v3（纸面测试版）

## 验证目标

**核心问题：** 玩家是否会因为"差一点就能孵化出可展示、可使用的新生物"，主动再打一局？

**验证范围：** 1个区域，3只生物，固定战斗结算，10分钟内完成2-3次孵化

**成功标准：** 玩家能明确说出"我下一局想刷哪只生物"，并且生物进入牧场后产生可见价值

## 一、牧场核心价值设计

### 牧场是什么？

牧场是玩家的**生物展示空间**，每只生物孵化后会进入牧场，玩家可以：
1. **看到生物在牧场中活动**（展示价值）
2. **选择生物组成出战队伍**（功能价值）
3. **不同生物组合产生加成**（组合价值）

### 3只生物设计（草原区域）

| 生物 | 碎片颜色 | 孵化需求 | 战斗效果 | 牧场状态 | 组合效果 |
|------|----------|----------|----------|----------|----------|
| 🐰兔子 | 白色 | 5个 | 战斗用时-3秒 | 闲逛跳跃 | 兔子+小鹿=战斗用时-5秒 |
| 🦌小鹿 | 黄色 | 5个 | 战斗用时-3秒 | 低头吃草 | （本轮不验证其他组合） |
| 🐑羊 | 灰色 | 5个 | 战后HP+20 | 躺下休息 | （本轮不验证其他组合） |

**设计意图：**
- 每只生物在牧场中有独特动作，产生展示价值
- 每只生物有独特战斗效果，产生功能价值
- 本轮只验证一个组合：兔子+小鹿
- 孵化阈值统一为5个，简化验证

## 二、战斗系统（固定结算版）

### 战斗结算规则

**基础战斗：**
- 无生物：胜利，用时30秒，战后HP剩余70
- 消耗：无（纸面测试阶段不引入能量系统）

**生物加成：**
- 携带兔子：胜利，用时27秒，战后HP剩余70
- 携带小鹿：胜利，用时27秒，战后HP剩余70
- 携带羊：胜利，用时30秒，战后HP剩余90
- 携带兔子+小鹿：胜利，用时25秒，战后HP剩余70
- 携带兔子+羊：胜利，用时27秒，战后HP剩余90
- 携带小鹿+羊：胜利，用时27秒，战后HP剩余90
- 携带全部3只：胜利，用时25秒，战后HP剩余90

**设计意图：**
- 纸面测试阶段不需要证明实时战斗成立
- 用固定结算让玩家感受到生物的价值
- 组合效果明显（25秒 vs 30秒）

## 三、掉落系统（修正版算法 + 多路径验证）

### 反比算法（修正版）

```python
def calculate_drop_weight(fragment_type, inventory, hatched_creatures):
    """
    计算碎片掉落权重
    """
    # 如果已经孵化，权重降低
    if fragment_type in hatched_creatures:
        return 10
    
    owned = inventory.get(fragment_type, 0)
    required = 5  # 统一孵化需求
    gap = required - owned
    
    if gap <= 2:
        # 差1-2个，权重大幅提升
        return 100
    else:
        # 差3-5个，正常权重
        return 50

def drop_fragments(inventory, hatched_creatures):
    """
    每场战斗掉落2个碎片
    """
    weights = {
        "兔子": calculate_drop_weight("兔子", inventory, hatched_creatures),
        "小鹿": calculate_drop_weight("小鹿", inventory, hatched_creatures),
        "羊": calculate_drop_weight("羊", inventory, hatched_creatures)
    }
    
    # 按权重随机抽取2个（可重复）
    return weighted_random_choice(weights, 2)
```

**关键修正：**
- 增加`hatched_creatures`参数，孵化后权重固定为10
- 避免孵化后碎片归零导致权重重置

## 四、10分钟体验流程（纸面测试版）

### 纸面测试材料

**需要准备：**
1. 碎片进度卡（3种颜色，每种20个）
2. 生物卡（兔子/小鹿/羊各1张，正面是蛋，背面是生物）
3. 牧场板（3x2格子，放置生物卡）
4. 战斗结算卡（显示用时和HP）
5. 掉落抽取袋（不透明袋子，用于随机抽取）
6. 抽取袋配置卡（见下表）

### 抽取袋配置表

测试员根据当前状态快速配置抽取袋：

| 状态 | 兔子卡数量 | 小鹿卡数量 | 羊卡数量 | 说明 |
|------|-----------|-----------|---------|------|
| 初始状态 | 5 | 5 | 5 | 所有生物权重50 |
| 兔子差1-2个 | 10 | 5 | 5 | 兔子权重100 |
| 兔子已孵化 | 1 | 5 | 5 | 兔子权重10 |
| 小鹿差1-2个 | 5或1 | 10 | 5 | 小鹿权重100 |
| 小鹿已孵化 | 5或1 | 1 | 5 | 小鹿权重10 |
| 羊差1-2个 | 5或1 | 5或1 | 10 | 羊权重100 |
| 羊已孵化 | 5或1 | 5或1 | 1 | 羊权重10 |

**使用方法：**
- 每场战斗前，测试员根据当前碎片数和已孵化生物，查表配置袋子
- 从袋子中随机抽取2张卡（可重复）
- 抽取后立即放回，下一局重新配置

### 测试流程分组

**A组：固定路径测试（验证理解）**
- 目的：验证玩家是否理解进度条、孵化流程、牧场展示
- 方法：测试员按路径1固定掉落，不使用抽取袋
- 样本：1-2人
- 用途：检查流程时长、界面可读性、说明是否清晰

**B组：动态抽取测试（验证主动性）**
- 目的：验证玩家是否会因"差1个"主动再来一局
- 方法：测试员使用抽取袋随机抽取，按配置表动态调整
- 样本：3-5人
- 用途：判断核心假设是否成立，决策是否进入开发

**重要：正式决策只看B组结果。A组仅用于流程调试。**

### B组测试流程（动态抽取版）

**0:00-1:00 开场引导**
- 测试员："你是牧场守护者，通过战斗收集生物碎片，孵化生物保卫家园"
- 展示牧场板（空的）和3种生物卡（蛋状态）
- 说明："每种生物需要5个碎片孵化"
- 配置抽取袋：5兔/5鹿/5羊

**1:00-2:00 第一场战斗**
- 玩家："开始战斗"
- 测试员翻开战斗结算卡："胜利，用时30秒，HP剩余70"
- 测试员从袋子随机抽取2张卡
- 玩家放置碎片卡到进度区
- 测试员只报数字："兔子X/5，小鹿Y/5，羊Z/5"（不说"差几个"）
- 重新配置抽取袋（根据配置表）

**2:00-3:00 第二场战斗**
- 等待玩家主动说"继续战斗"或询问下一步
- 测试员："胜利，用时30秒"
- 测试员随机抽取2张卡
- 测试员只报数字，不做评价
- 重新配置抽取袋

**关键观察点1：第一次"差1-2个"**
- 当某种碎片达到3/5或4/5时
- 测试员只展示进度条，**不说"差1个"**
- **等待5秒**，观察玩家是否主动说出：
  - "我想凑兔子"
  - "再来一局"
  - "差一个了"
  - 或其他表示目标的话
- 记录玩家原话和行为
- 如果5秒内无反应，测试员提示："兔子差1个就能孵化了"
- 记录为"提示后理解"

**关键观察点2：第一次孵化**
- 当玩家凑齐5个碎片时
- 测试员："可以孵化了"
- 玩家孵化后，测试员翻转生物卡，放到牧场板上
- 测试员："兔子进入牧场，正在跳跃"
- **不主动说明战斗效果**
- 等待玩家主动询问或查看牧场
- 如果玩家询问，测试员回答："战斗时可以召唤兔子，用时-3秒"
- 记录玩家是否主动查看牧场、是否询问效果

**关键观察点3：战斗效果感知**
- 玩家下一场战斗时
- 测试员询问："要带兔子出战吗？"
- 如果玩家选择带兔子，测试员："胜利，用时27秒"
- 战斗结束后询问：
  - "你觉得兔子有帮助吗？"
  - "用时从30秒变27秒，这个效果值得追求吗？"
- 记录玩家原话

**继续测试（5:00-10:00）**
- 重复上述流程，直到玩家孵化2-3只生物
- 每次"差1-2个"时都观察主动性
- 每次孵化后都观察是否查看牧场
- 当玩家拥有兔子+小鹿时，询问："要不要试试两只一起出战？"
- 如果玩家同意，测试员："胜利，用时25秒"
- 询问："从30秒到25秒，你觉得组合有价值吗？"

**10:00 测试结束**
- 询问玩家：
  1. "你在哪个时刻最想'再来一局'？为什么？"
  2. "你孵化生物后，第一反应是什么？"
  3. "你觉得生物对战斗的帮助明显吗？"
  4. "如果继续玩，你下一步想做什么？"
  5. "用时-3秒和HP+20，哪个更吸引你？"

### 主动性判断标准

**主动：**
- 玩家自己说出"差一个""再来一局""我要凑XX"
- 玩家孵化后立即查看牧场或询问效果
- 玩家主动尝试不同队伍组合

**提示后：**
- 测试员提示"差1个"后，玩家表示理解并继续
- 测试员询问"要带XX出战吗"后，玩家才尝试
- 测试员说明效果后，玩家才关注

**否：**
- 提示后仍不关心进度
- 孵化后不查看牧场，直接继续战斗
- 对战斗效果无感，不调整队伍

- 至少3人在"差1-2个"时**主动**表现出"再来一局"

**B组动态抽取测试中，5人样本出现以下情况：**
- 少于3人在"差1-2个"时**主动**表现出"再来一局"
- 少于3人孵化后**主动**查看牧场
- 超过3人在5分钟后失去兴趣

**决策：退回机制重构，重新思考"牧场收集"方向**

**需要分析的问题：**
1. **反比算法问题** - "差1个"的感知不够明显？
2. **牧场价值问题** - 生物进入牧场后没有吸引力？
3. **战斗效果问题** - 用时-3秒的价值不够明显？
4. **方向问题** - "牧场收集"本身不成立？

### 决策流程

```
B组测试（5人）
    ↓
强通过？ → 是 → 进入Godot Demo开发
    ↓ 否
弱通过？ → 是 → 修改纸面原型 → 再测3人 → 重新判断
    ↓ 否
不通过 → 分析失败原因 → 退回机制重构
```

## 八、下一步

### 立即执行

1. **准备纸面测试材料**（1-2小时）
   - 制作碎片进度卡（可用彩色便签纸）
   - 制作生物卡（可用卡片正反面画蛋和生物）
   - 制作牧场板（可用A4纸画3x2格子）
   - 准备抽取袋（不透明袋子）
   - 准备抽取袋配置卡（打印配置表）
   - 准备测试记录表（打印或手写）

2. **招募测试玩家**（内部或外部）
   - A组：1-2人（固定路径，验证理解）
   - B组：3-5人（动态抽取，验证主动性）
   - 建议选择不同游戏经验的玩家

3. **执行A组测试**（调试流程）
   - 按固定路径执行，检查时长和可读性
   - 调整说明文字和引导方式
   - 确认测试员操作流程

4. **执行B组测试**（正式验证）
   - 使用抽取袋随机掉落
   - 降低提示强度，观察主动性
   - 详细记录玩家原话和行为

5. **收集反馈并分析**（1小时）
   - 整理测试记录表
   - 统计主动性比例
   - 分析失败案例的原因

6. **决策**
   - 强通过 → 进入Godot Demo开发
   - 弱通过 → 修改纸面原型，再测3人
   - 不通过 → 分析失败原因，退回重构

### 重要提醒

**不要跳过纸面测试，直接进入开发。**

纸面测试的成本是2-3小时，但能避免：
- 开发1-2周后发现核心假设不成立
- 浪费开发资源在错误方向上
- 无法定位失败原因（是机制、数值还是实现）

**只有B组测试达到强通过标准，才进入Godot开发。**
