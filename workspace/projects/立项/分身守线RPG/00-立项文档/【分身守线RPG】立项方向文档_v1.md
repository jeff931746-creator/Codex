## 1. 一句话定义

- 一句话定义：
  一款以“主英雄布置分身与随从守住最后防线”为核心体验、用局内 build 和局外 RPG 成长承接中长期留存的中度守线RPG。

---

## 2. 方向结论

### 2.1 当前判断

- 方向建议：`建议进入立项评审前验证`
- 核心结论：
  这不是传统塔防，也不是普通 RPG，而是一条更接近 `Hero Defense / Squad Defense` 的新方向。

### 2.2 为什么值得单独立项

- 事实：
  1. 原“角色 + 塔”的方向天然存在职责冲突，角色、塔、宠物、坐骑很容易互相抢戏。
  2. 如果继续强调 RPG 承接，塔往往会被做成一个既重要又不够主导的中间层，增加复杂度。
  3. 分身、宠物、坐骑本质都属于“我的战斗单位或战场延伸”，语言天然更统一。
- 推断：
  1. 把“塔”改写成“阵位分身”，可能比保留塔更适合做角色主导的守线产品。
  2. 新方向会更容易形成“我在布置自己的战场化身”这种独特心智。
  3. 如果方向成立，它会比传统 RPG塔防更统一，也更容易解释为什么要有宠物、坐骑和 build。
- 建议：
  不要把该项目继续定义成“RPG塔防变体”，应直接定义为“分身守线RPG”。

### 2.3 最大机会点与最大风险点

- 最大机会点：
  把传统塔防中的阵地功能拟人化，转译成“分身布置、随从协同、主角救场”的统一战斗体验。
- 最大风险点：
  一旦分身系统做成高操作、高常驻、高独立成长，就会让战场阅读和用户负担迅速失控。

## 3. 目标用户与市场切口

### 3.1 目标用户

- 主核心用户：
  1. 喜欢守线、守家、怪潮压力和防线反杀的用户
  2. 喜欢召唤流、分身流、军团流、编队流 build 的用户
  3. 喜欢中度 RPG 成长、养号、职业 fantasy 的用户
- 次核心用户：
  1. 对 War3 守家、Hero Defense、肉鸽成长有经验的玩家
  2. 喜欢“不是塔防但又有防守乐趣”的中度用户

### 3.2 主要需求

- 我想守住一条清晰的防线
- 我想操作一个有存在感的主角
- 我想把这局 build 养出来，而不是只看面板涨数值
- 我想有“我的军团/我的分身阵型”这种归属感
- 我想长期追求职业、宠物、坐骑和流派成长

### 3.3 市场切口

- 不是去拼最传统的塔防
- 也不是去拼最纯粹的 RPG
- 而是切入：
  `主角主导 + 分身布阵 + 守线抗压 + build 成型`

### 3.4 与传统 RPG塔防的区别

- 传统 RPG塔防：
  角色负责战斗，塔负责阵地，容易形成双核心甚至三核心
- 分身守线RPG：
  阵地功能由“角色化分身”承担，战斗单位体系统一，用户更容易理解成“我在布置我的战斗化身”

## 4. 产品定位

### 4.1 产品母型

- 主母型：`守线`
- 辅母型：`Roguelite成长`
- 承接语法：`RPG成长`

### 4.2 产品不是什么

- 不是传统摆塔塔防
- 不是高操作守线 ARPG
- 不是纯召唤刷怪游戏
- 不是重数值卡牌 RPG

### 4.3 产品是什么

- 是一款强调：
  1. 主英雄存在感
  2. 分身阵位部署
  3. 单局流派成型
  4. 中长期养成承接
  的中度守线RPG

### 4.4 成功定义

- 首局能否让玩家快速理解：
  “我不是在摆塔，而是在布置战场分身”
- 单局能否让玩家在 3 分钟内形成一句自我归纳：
  “这局我是堵门反伤流 / 冰雷分身流 / 军团铺场流”
- 中期能否让玩家因为职业、宠物、坐骑与分身组合而持续回流

## 5. 核心体验目标

- 我在守住最后一道防线
- 我带着自己的分身和随从守场，而不是只看建筑输出
- 我每一局都能围绕主角和分身做不同 build
- 怪潮压门时，我能亲自去关键位置救场
- 我养的宠物、坐骑和职业体系，真的会改变我的战斗风格

---

## 6. 核心玩法模型

### 6.1 主循环

1. 怪潮压线
2. 主英雄移动、站位、释放主动技能
3. 玩家在关键阵位布置分身
4. 击杀获得经验、能量或局内资源
5. 升级触发三选一
6. 分身、宠物、主角之间的协同逐渐成型
7. 顶住精英与 Boss 收束波次

### 6.2 中循环

1. 关键波次前刷新高价值选择点
2. 玩家决定是补分身、强化主角、偏向宠物协同，还是扩大战线
3. build 逐步清晰
4. 更高压波次检验当前阵型

### 6.3 高潮循环

1. Boss 或高压精英逼近防线
2. 主角切位救场
3. 分身体系联动爆发
4. 宠物/坐骑修正战场节奏
5. 通过成型 build 反杀收尾

## 7. 核心系统结构

### 7.1 主英雄

- 职责：
  1. 战场主控
  2. 关键救场
  3. 开启职业 fantasy
  4. 与分身和宠物形成联动核心
- 设计原则：
  1. 自动普攻
  2. 手动释放 2-3 个高价值技能
  3. 可移动，但强调站位价值而不是高频闪避

### 7.2 分身

- 定义：
  角色化的阵地单位，用来承担传统塔的功能，但表现和联动都归属于主角体系
- 职责：
  1. 堵口
  2. 控场
  3. 输出
  4. 增益
- 设计原则：
  1. 布置在固定阵位或有限阵位
  2. 不做高频移动操作
  3. 不做独立大养成线
  4. 由职业、遗物、宠物共同放大

### 7.3 宠物

- 定义：
  提供协同效果的长期成长单位
- 职责：
  1. 强化分身
  2. 补足主角短板
  3. 提供流派修正
- 设计原则：
  1. 首发轻做
  2. 重协同，轻数值坑
  3. 作为长期追求，不覆盖局内 build 价值

### 7.4 坐骑

- 定义：
  主英雄的战场风格修正器
- 职责：
  1. 改变转线能力
  2. 改变主角的救场节奏
  3. 强化职业差异
- 设计原则：
  1. 轻量接入
  2. 不做重度独立养成
  3. 优先服务战场运作方式

### 7.5 遗物/三选一

- 定义：
  单局构筑的决定性系统
- 职责：
  1. 定义这局主流派
  2. 改写主角与分身联动方式
  3. 形成明确 build 记忆点
- 设计原则：
  1. 不是纯数值
  2. 必须明显改变战斗方式
  3. 保持职业遗物、分身遗物和混合遗物三层结构

## 8. 与“角色+塔”路线的比较

### 8.1 新路线的优势

- 语言统一：
  主角、分身、宠物、坐骑都属于同一战斗体系
- 题材更顺：
  用户更容易理解“我在带着自己的化身作战”
- RPG承接更自然：
  宠物、坐骑和职业成长不再与塔冲突
- 买量表达更直给：
  “一人守不住，布下分身防线反杀”比纯塔位更具角色感

### 8.2 新路线的劣势

- 会削弱“塔防”这个传统认知标签
- 分身如果表现过多，战场可读性更差
- 平衡会从“建筑平衡”变成“单位联动平衡”，调试成本更高

### 8.3 方向判断

- 如果目标是更纯粹的塔防用户心智，保塔更稳
- 如果目标是更统一的 RPG 守线产品心智，去塔改分身更有机会做出新意

## 9. 可参照原型

> 这里写的是“母型参考”，不是要求直接复刻。

### 9.1 Thronefall

- 借什么：
  主角亲自守线、短局压迫感、终局危机感
- 不借什么：
  经济和建造为主的节奏组织

### 9.2 Boneraiser Minions

- 借什么：
  主角 + 军团 / 召唤物 build 的统一感
- 不借什么：
  完全偏幸存者式的自动刷怪母型

### 9.3 DEFENDUN : Hero Defense

- 借什么：
  用单位承担传统塔的防守职责
- 不借什么：
  过于合成化或强数值化的外层壳子

### 9.4 Kingdom Two Crowns

- 借什么：
  坐骑对战场节奏的修正价值
- 不借什么：
  偏资源经营和长线王国循环

### 9.5 结论

- 该项目最接近的成熟母型不是“传统塔防”，而是：
  `Hero Defense + Squad Build + 守线RPG`

## 10. 吸引力判断

### 10.1 素材吸引点

- 危局压门，主角与分身一起反杀
- 同职业不同分身组合，结果完全不同
- 选错 build 扛不住，选对 build 一波成型
- 宠物和坐骑改变主角守线方式

### 10.2 为什么可能好买

- 比传统摆塔更有角色感
- 比纯 RPG 更有清晰的守线危机画面
- 比原“角色 + 塔”路线更容易统一表达

### 10.3 为什么也可能不好买

- 如果素材只看到一群单位混战，用户会看不懂重点
- 如果分身表现像普通召唤物，独特性会不足
- 如果只卖成长不卖守线危机，会掉进普通 RPG 素材池

### 10.4 阶段结论

- 素材顺序建议固定为：
  `守线危机 -> 主角切位 -> 布置分身 -> build 反杀`

---

## 11. 风险判断

### 11.1 产品风险

- 最大产品风险：
  分身、宠物、坐骑三者一起上时，系统边界容易失控
- 风险表现：
  1. 玩家看不懂战场重点
  2. 主角存在感下降
  3. 分身变成纯数值单位
  4. 局外成长压住局内构筑

### 11.2 研发风险

- 多单位同屏性能压力
- 阵位分身 AI 与路径表现
- 分身与宠物联动的调试复杂度

### 11.3 发行风险

- 用户可能不把它认成塔防，也不把它认成典型 RPG
- 需要重新建立“分身守线”这个认知点

### 11.4 方向风险结论

- 这是一条统一度更高的方向
- 也是一条更需要用原型快速证明的新方向

## 12. 首发方向建议

### 12.1 建议保留

- 主英雄职业系统
- 分身阵位系统
- 遗物与三选一
- 轻量宠物
- 轻量坐骑
- 1 个 Boss 收束波次

### 12.2 建议砍掉

- 传统塔系统
- 复杂载具系统
- 分身独立大养成
- 开放式抓宠流程
- 坐骑装备和突破线
- 重度装备词条系统

### 12.3 推荐最小验证规模

- 2 个职业
- 4 类分身
- 20 件遗物
- 4 只宠物
- 2 种坐骑
- 1 张地图
- 1 个 Boss

## 13. 立项前必须验证的问题

1. 玩家能不能在 30 秒内理解“分身不是宠物，也不是塔，而是战场化身”
2. 玩家在 3 分钟内能不能说出自己这局的流派
3. 主角移动和分身布阵会不会让操作负担过高
4. 分身体系能不能提供足够稳定的守线结构，而不需要塔来兜底
5. 录屏素材能不能稳定剪出 3 条以上差异明显的表达
