# 项目区

`projects/` 管理所有游戏项目，按生命周期阶段组织。

## 目录结构

```
方向探索/     话题级发散探索，方向未收敛前放这里
  {话题名}/
    候选方向/   多个方向文档并列
    收敛结论.md 选定方向后填写，触发升格立项

立项/          已收敛到单一方向，走 6 阶段评审（00-05）
  {项目名}/
    00-立项文档/
    01-评审/
    02-纸面原型/
    03-实现/
    04-测试/
    05-发布/

开发中/        立项评审通过后的正式开发阶段（部门制）
  {项目名}/
    立项/       归档立项阶段文档
    策划/       输入/ + 输出/
    开发/       输入/ + 输出/
    美术/       输入/ + 输出/

上线/          已上线项目
```

## 流转规则

- 有想法 → `方向探索/{话题名}/候选方向/`
- 收敛选定方向 → 写 `收敛结论.md`，新建 `立项/{项目名}/`
- 立项 01-评审通过 → `git mv 立项/{项目名} 开发中/{项目名}`，建部门制目录
- 上线 → `git mv 开发中/{项目名} 上线/{项目名}`
- 中止（任意阶段）→ 移入 `archive/经验/{项目名}/`（需审批）
