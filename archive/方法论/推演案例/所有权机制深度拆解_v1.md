# 所有权机制深度拆解 v1

> 仅拆解分析，不推导立项方向。等待审批后再进入下一步。

---

## 机制 1 — 亲手获取

**心理学基础：禀赋效应（Endowment Effect）**
人对"自己获得过"的物品产生高于市场价值的主观估值。关键变量：获取过程的主动性和阻力越大，禀赋效应越强。捡到的和猎来的，所有权感完全不同。

**所有权对象：个体单位**（"我抓来的那只"，不是"我的守线系统"）

**成立前提：**
- 获取过程有阻力（需要技巧、判断、时机选择）
- 获取是玩家主动发起的，不是系统自动给予
- 获取结果有不确定性（存在失败可能）
- 被获取的对象有可识别的个体性（外观差异、特征、命名）

**失效条件：**
- 获取过程无难度（随手一抓）→ 所有权感等同于免费礼物，极弱
- 获取物完全同质（抓来的全一样）→ "我的那只"和别人的无区别，个体归属无法建立
- 获取频率过高（每波都能抓多个）→ 禀赋效应被稀释，单个单位不再珍贵

**强度曲线：瞬时触发，随时间衰减**
捕获那一刻所有权感最强。若后续没有配合其他机制（共同经历、因我而变），情感会逐渐淡化。单独存在时留存价值有限。

**典型游戏案例：**
- 宝可梦：捕获有技巧（削血 + 状态球）、有不确定性（捕获率）、每只有个体性（性格/个体值）→ 所有权感极强，经典公式
- Palworld：捕获机制类宝可梦，但Pal个体性弱 → 所有权感显著偏弱，用户容易流失

**RPG+防守中的边界：**
- 必须设计视觉差异明显的可捕获单位（种类 ≥20 才有足够的分辨度）
- 每局有效捕获次数需要严格限制（过多则稀释），建议每局上限3-5次有意义的捕获
- 捕获行为必须和高压波次分开（否则注意力争夺重现）

## 机制 2 — 共同经历

**心理学基础：关系形成加速效应**
共同经历危险会加速情感联结形成。游戏中的关键变量不是"一起经历了什么"，而是**玩家是否有干预行为**——"我去救了它"和"它自己活下来"对所有权感的贡献量级不同。

**所有权对象：个体单位，且是跨局积累的**
局内无法完整建立，需要多局堆叠才有感情深度。

**成立前提：**
- 单位有可识别的个体性（命名 + 外观区分，数量 ≤ 7个同时存在）
- 玩家必须有主动干预行为（去救场、去保护），而不是单位自动存活
- 历史记录对玩家可见（活了几局、经历了哪些波次）
- 永久死亡制：死亡要有真实代价，否则历史失去重量

**失效条件：**
- 单位太多无法区分 → 经历无法和特定个体绑定，情感弥散
- 单位局内完全重置 → 历史断开，跨局关系无法建立
- 玩家没有救场行为（单位自动防守活下来）→ "我们一起"变成"它自己活的"
- 没有永久死亡制 → 死亡可以撤销，历史失去重量，情感不值得投入

**强度曲线：累积建立，随时间增强，因死亡归零**
所有机制中留存价值最高，但建立周期最长（需要跨局积累），且破坏代价最大（单位死亡=情感归零）。

**典型游戏案例：**
- 火焰纹章：命名角色 + 永久死亡制，失去老角色是真实的哀伤，玩家会专门重开档
- FTL：船员有名字，某次近死被救后会记很久
- Darkest Dungeon：英雄有战斗历史标注，老角色的死亡代价真实可感

**RPG+防守中的边界：**
- 手游永久死亡制的用户接受度是核心未知量，必须先测试
- 单位数量硬控制在 ≤ 6 个命名守线单位，超出就无法区分个体
- 弱化替代方案：用"伤疤/创伤系统"代替永久死亡制（死过一次留下永久性状态改变），保留情感代价但降低损失门槛

## 机制 3 — 身份延伸

**心理学基础：自我一致性理论（Self-Consistency Theory）**
人对与自我形象一致的事物产生归属感，并倾向于保护和强化这种一致性。当防守系统"反映了我是谁"，玩家把对系统的攻击视为对自身的攻击，把系统的成功视为自身的成功。

**所有权对象：整个系统**（"我的守线军团"这个整体，不是某个单位）

**成立前提：**
- 玩家身份选择（职业/阵营）有明显的视觉和机制差异，选A和选B的结果看起来完全不同
- 防守系统的外观和行为明确对应玩家的身份选择（1:1映射，不模糊）
- 身份选择是玩家主动决定的，不是随机分配
- 身份之间没有数值强弱之分（否则变成"被迫选强的"，不是"选我的"）

**失效条件：**
- 职业差异只在数值层面，视觉和行为模式相同 → 身份认同无法建立
- 防守单位外观与玩家职业无关联 → "我是火法师但守线军团是剑士" → 割裂感
- 存在明显强势职业 → 玩家被迫选强势，身份选择变成功能性决策，认同感消失
- 职业过多但差异不足 → "其实选哪个都差不多" → 认同感被稀释

**强度曲线：快速建立，高度稳定，长期不衰减**
身份认同一旦建立，只要差异感维持，所有权感不会随时间衰减。这是7个机制中最稳定的。建立速度取决于职业差异的视觉冲击力——差异越大越明显，建立越快。

**典型游戏案例：**
- 魔兽世界职业系统：视觉外观 + 技能风格 + 社区标签三层共同构建"我是法师"的强烈认同
- Clash Royale：卡组是身份的延伸，"我的卡组"是强烈归属感来源（即使卡是系统给的）
- Vampire Survivors：角色选择改变游戏风格，但视觉差异不足 → 身份认同偏弱

**RPG+防守中的边界：**
- 职业=守线种族的映射必须是1:1且视觉差异极大（不能模糊过渡）
- 首版建议只做3个职业，每个职业守线种族在视觉上完全不同
- 强势职业会摧毁整个身份认同体系，数值平衡是该方向的研发红线

## 机制 4 — 投入成本

**心理学基础：沉没成本效应 + 宜家效应（IKEA Effect）**
人对自己付出代价的事物高估价值（沉没成本），且亲手构建的比直接购买的更有价值感（宜家效应）。两者叠加：我花了时间/资源强化它，它现在的强度是我造就的。

**所有权对象：个体单位**（我重点投入的那个特定单位）

**成立前提：**
- 投入的代价必须是真实有限的（无限资源的强化没有心理代价）
- 投入结果必须对玩家可见（花了资源，能感知到它变强了）
- 投入行为是玩家主动对特定对象的选择（不是"全体单位+10%"这种批量操作）
- 投入对象是可区分的个体

**失效条件：**
- 资源无限充裕 → 沉没成本感消失
- 强化效果不明显或不可见 → 宜家效应无法激活
- 批量强化（"所有单位升级"）→ 无法产生对特定单位的归属
- 代价过重导致挫败感 → 宜家效应变成负向情绪，玩家放弃

**强度曲线：随投入量累积，可叠加，遇到天花板后趋于平稳**
投入越多，所有权感越强，直到玩家认为"已经足够了"（满级/满配）后趋于稳定。

**典型游戏案例：**
- 宝可梦EV训练：愿意花几十小时训练一只，沉没成本极深
- Path of Exile装备改造：投入极深，复制成本极高，所有权感极强
- 手游养成系统（突破/星级/技能升级）：都是用投入成本机制建立所有权感和付费动机

**RPG+防守中的边界：**
- 和长期留存、付费变现天然兼容（投入越深越不愿意离开）
- 投入粒度要精确：对"特定单位"的定向投入 > 对"全体单位"的批量升级
- 在守线场景中，波次间对特定单位的资源分配可以作为自然的投入行为

## 机制 5 — 选择胜出

**心理学基础：决策后认知失调消解（Post-Decision Dissonance Reduction）**
人在多个选项中做出选择后，会倾向于高估被选中选项、贬低未选项，以消解决策后的焦虑。"选了它"这个行为本身就产生偏爱，即使选择前两者差不多。

**所有权对象：个体单位**（我在多个选项中挑中的那个）

**成立前提：**
- 选项之间有真实的、可感知的差异（不能都差不多）
- 玩家感受到"我在做一个有重量的选择"（决策时间和信息支持充足）
- 被放弃的选项不能立即再获得（选择必须有排他性）

**失效条件：**
- 选项差异不明显 → 选择无痛苦 → 认知失调不发生 → 偏爱不产生
- 选择可随时撤销/重选 → 排他性消失 → 所有权感为零
- 选项质量差异悬殊（一个明显最好）→ 玩家被迫选最好的，变成功能性选择，不产生情感性所有权

**强度曲线：瞬时触发，快速稳定，独立时强度偏弱**
选择那一刻触发，之后趋于稳定。但单独存在时所有权强度在7个机制中偏弱，更适合作为其他机制的触发前置（先选择→再投入→再共同经历，形成递进归属感）。

**典型游戏案例：**
- Slay the Spire三选一：选牌的瞬间开始偏爱这张牌
- 火焰纹章角色招募：从候选中选择，未选的消失 → 排他性极强

**RPG+防守中的边界：**
- 波次间的三选一招募可以激活该机制
- 关键设计约束：被放弃的选项必须真正消失（不能等会儿再选）
- 独立使用时所有权感有限，建议作为投入成本或因我而变的前置触发

## 机制 6 — 因我而变

**心理学基础：宜家效应（IKEA Effect）强化版 + 不可逆决策的价值加成**
宜家效应基础是"我参与构建"，而"因我而变"增加了"结果的唯一性"——这个单位走上了一条只属于它的、由我的决定造就的路。不可逆的路线选择 = 我对它当前状态的独特创造权。

**所有权对象：个体单位**，且是对"当前状态"的所有权（不是历史，是它现在的样子）

**成立前提：**
- 路线选择不可逆（可回退的选择不产生这种效应）
- 选择结果明显可见（外形改变、能力方向改变）
- 不同路线的结果截然不同（走A路线的和走B路线的看起来是两种东西）
- 玩家能感知到"是我的选择导致了现在的它"（因果链清晰）

**失效条件：**
- 路线可随时切换 → 不可逆性消失 → 效应消失
- 所有路线结果差不多 → 玩家感知不到"我的选择造就了它"
- 进化树过于复杂 → 玩家记不住自己做了什么选择 → 归属感无法建立

**强度曲线：每个不可逆节点触发一次，随节点数量累积**
每走一个进化节点，所有权感增加一次。进化层数不宜过多（≤3层），否则玩家疲劳且感知不到差异。

**典型游戏案例：**
- 宝可梦进化：不可逆，外观改变巨大（皮卡丘→雷丘，再也回不去）→ 极强"因我而变"
- Darkest Dungeon技能槽：4格技能槽有限，选哪4个是不可逆的定制，形成强归属

**RPG+防守中的边界：**
- 单局内完成1次不可逆进化已足够（幼年→成年）
- 进化路线之间外观差异必须足够大（不同路线=不同生物，不是同一生物+数值）
- 进化路线选择时机可以设计为波次间的三选一，和"选择胜出"机制自然组合

## 机制 7 — 实时联结

**心理学基础：代理感（Agency）+ 效能感（Efficacy）**
玩家行动直接、即时影响游戏状态时，产生"这是我力量的延伸"的感觉。这是**操作反馈感**的一种，核心是因果链的即时可见性。守线单位响应我的行为，感觉像身体的延伸，而不是独立的他者。

**所有权对象：整个系统**（被我驱动的那套防线，不是某个单位）

**成立前提：**
- 主角行为与守线单位状态变化之间延迟极短（< 0.5秒）
- 因果链对玩家清晰可见（我做了X，它发生了Y，之间有视觉连接）
- 不同操作产生不同结果（不是什么操作都触发相同效果）

**失效条件：**
- 反馈延迟过长 → 因果感消失 → 守线单位变成自己在动
- 因果关系不清晰 → 玩家不知道守线单位为什么在变化 → 代理感消失
- 主角行动和守线单位行为没有可感知关联 → 两套系统感觉独立运行

**强度曲线：完全依赖当前游戏状态，不跨局积累，停止游戏即消失**
即时性最强，但持久性最低——不玩时这个所有权感不存在。适合做操作爽感和即时反馈，不适合做长期留存锚点。

**典型游戏案例：**
- 守望先锋 Lucio：移动驱动治疗范围，团队跟着你的位置 → 强代理感
- 暗黑破坏神召唤术士：召唤物响应指令 → 代理感有，但长期所有权感一般

**RPG+防守中的边界：**
- 单独作为所有权锚点强度不足，留存价值低
- 最适合作为其他机制的体验增强层（职业映射守线 + 实时联结 = 视觉认同 + 操作爽感叠加）
- 核心作用：让防守系统在玩家操作时"感觉活着"，而不是在背景自动运转

## 横向分析 1：机制叠加关系

**可以叠加强化的组合：**

| 组合 | 效果 | 游戏案例 |
|---|---|---|
| 亲手获取 + 因我而变 | 我抓来的，而且走了我选的进化路线 → 双层所有权 | 宝可梦的核心公式 |
| 选择胜出 + 投入成本 | 选了它（归属触发）→ 花资源强化（归属加深） | 几乎所有手游养成 |
| 共同经历 + 投入成本 | 救了它（情感联结）→ 花资源治愈伤（代价加深情感） | 火焰纹章的情感深度来源 |
| 身份延伸 + 实时联结 | 防线种族反映我的职业（认同）+ 主角行为驱动它们（即时爽感） | 理想的RPG+防守体验层 |

**叠加后会互相稀释的组合：**

| 组合 | 问题 |
|---|---|
| 亲手获取 + 共同经历（同一产品高频使用） | 每局捕获新单位，老单位被替换，共同经历积累的情感随替换浪费 |
| 选择胜出 + 因我而变（同一步骤合并） | 两个机制合并为一个节点，不是叠加而是合并，没有额外收益 |
| 实时联结 + 共同经历（同一对象） | 实时联结是系统感，共同经历是个体情感；两者的所有权对象类型冲突（系统 vs 个体） |

## 横向分析 2：手游环境约束

| 机制 | 能在单局内建立？ | 需要跨局积累？ | 留存价值 | 变现兼容性 | 手游综合可行性 |
|---|---|---|---|---|---|
| 亲手获取 | 是 | 否 | 中（需叠加其他） | 高（捕获道具） | 高 |
| 共同经历 | 否 | 是（多局） | 最高 | 高（复活卷） | 中（需跨局系统支持） |
| 身份延伸 | 是（选职业即建立） | 否 | 高且稳定 | 高（新职业） | 高 |
| 投入成本 | 局内可开始 | 跨局加深 | 高 | 极高（加速/强化） | 高 |
| 选择胜出 | 是 | 否 | 低（单独使用） | 低 | 中（作为前置） |
| 因我而变 | 是（局内进化） | 可跨局加深 | 高 | 高（解锁路线） | 高 |
| 实时联结 | 是 | 否（不跨局） | 低（不产生留存） | 低 | 中（体验层使用） |

**结论：**
- 手游环境下有效的主力机制：**身份延伸、亲手获取、因我而变、投入成本**
- 需要专项设计才能在手游落地的：**共同经历**（跨局历史记录系统）
- 单独使用留存价值低、适合作辅助：**选择胜出、实时联结**
