# 操纵角色+防守 — 立项可能性推衍 v2

> **v2 变更说明：**
> - 合体并入变身（两者本质相同：主角形态改变）
> - 删除指挥官守线（买量弱，不建议独立立项）
> - 增加推衍约束框架，所有方向从框架中派生，不再随意列举
> - 新增：陷阱守线、感染守线、地形守线、共鸣守线

---

## 推衍约束框架

**核心约束：守线力量必须来自一个明确的、唯一的生成方式。**

每个立项方向对应一个不同的"核心动词"（玩家主要在做什么），动词不能重复。如果两个方向的核心动词相同，它们不是独立方向，而是同一方向的变体。

| 守线力量的生成方式 | 核心动词 | 立项方向 |
|---|---|---|
| 角色自身形态改变 | 变 | **C 变身守线**（含原合体概念） |
| 角色骑乘外部生命体 | 骑 | **B 骑战守线** |
| 角色主动捕获敌方单位 | 抓 | **A 抓宠守线** |
| 角色感染敌方单位并自动传播 | 染 | **F 感染守线** |
| 角色预判路径并埋设机关 | 埋 | **E 陷阱守线** |
| 角色培育已有守线单位进化 | 养 | **D 培育进化守线** |
| 角色改造战场地形本身 | 改 | **G 地形守线** |
| 角色的状态实时驱动守线单位 | 共 | **H 共鸣守线** |

**验证规则：**
- 任意两个方向的核心动词不得相同
- 如果一个方向的守线力量来源可以用另一个方向的动词描述，合并或删掉一个

## 现有项目基线（参照系）

| 项目 | 守线力量来源 | 核心动词 |
|---|---|---|
| 桥头卡组守线RPG | 牌库预设的自动技能 + 资源运营卡 | 运 |
| 国王带队守线RPG塔防 | 建造的塔 + 英雄主战 | 建 |
| 分身守线RPG | 主角化身的分身阵位 | 分 |

以下8个方向均与上表三个动词不同。

## 方向 A — 抓宠守线

**守线力量来源：主动捕获 / 核心动词：抓**

**一句话定义：** 玩家在守线中实时捕获敌方单位，被捕获的生物翻面成为己方守线力量；这局的防线取决于你抓到了什么。

**主角核心操作：** 识别高价值目标 → 削血到可捕获阈值 → 判断捕获时机 → 安置位置

**守线力量的边界（不是什么）：**
- 不是预设的固定卡池（vs 卡组守线）
- 不是主角的分身/化身（vs 分身守线）
- 不是感染自动传播，必须主角逐个主动捕获（vs F 感染守线）

**最大机会点：** "用敌人打敌人"天然有反转叙事；每局守线配置完全不同，复玩驱动强；捕获时机判断保留操作深度。

**最大风险点：** 捕获行为与守线压力同时存在，注意力分裂；如果可捕获单位种类不够多，几局后复玩价值崩塌。

**买量逻辑：** "Boss小弟血量低了，我一抓，它守门" → 捕获瞬间翻面 → 防线成型 → 反杀 Boss。

**立项建议：** `优先验证`——核心风险可以通过捕获时间减速/时停窗口规避。

---

## 方向 B — 骑战守线

**守线力量来源：骑乘外部生命体 / 核心动词：骑**

**一句话定义：** 主角骑乘不同坐骑在防线上作战，坐骑决定主角的攻击覆盖、站位逻辑和战场控制方式；切换坐骑即切换战术形态。

**主角核心操作：** 判断当前波次特性 → 选择合适坐骑 → 在防线上机动输出 → 关键时刻切换坐骑

**守线力量的边界（不是什么）：**
- 坐骑是独立的外部生命体，可以被击杀，可以被换下（vs C 变身：主角自身形态改变，无独立实体）
- 坐骑不只是移速加成，切换坐骑必须改变攻击模式（否则退化为换皮数值）
- 没有固定守线单位，主角+坐骑是唯一的守线主力（可以有少量辅助单位，但不是核心）

**最大机会点：** 坐骑形态视觉差异极大，买量素材天然多样；切换决策点清晰（这波密集 → 换范围，这波精英 → 换突进）。

**最大风险点：** 如果坐骑只是数值差异没有操作差异，退化为换皮；坐骑之外的系统（其他单位、阵位）边界需清晰，否则互相抢戏。

**买量逻辑：** "Boss来了，切换坐骑形态，反手一击" → 形态切换瞬间是天然高光。

**立项建议：** `优先验证`——关键验证问题：坐骑切换是否产生真实战术差异。

---

## 方向 C — 变身守线

**守线力量来源：角色自身形态改变 / 核心动词：变**

> 注：原"合体守线"并入此方向。合体与变身在结构上相同——主角形态改变，无论是自我变形还是与某物融合，结果都是主角变成不同的战斗形态。区分标准：合体后是否还有独立的外部实体存在？有 → 骑战；没有 → 变身。

**一句话定义：** 主角可以切换 3-4 种变身形态，每种形态对应不同的攻击类型、覆盖范围和守线站位；普通状态是基础形态，关键波次变身应对特殊压力。

**主角核心操作：** 判断当前威胁类型 → 切换匹配的变身形态 → 在该形态下发挥最大效果 → 变身冷却期回到基础形态

**守线力量的边界（不是什么）：**
- 主角自身形态改变，没有独立的外部单位（vs B 骑战：坐骑是外部生命体）
- 变身是主角的核心能力，守线依赖主角的形态选择（不是固定阵位单位）
- 变身之间有结构差异，不是同一技能的数值放大

**最大机会点：** 形态切换时刻视觉冲击强，素材高光明确；多形态构筑（局外强化哪个形态）给了复玩深度。

**最大风险点：** 如果形态切换冷却太长，大部分时间在普通形态战斗，变身存在感弱；如果变身太强玩家永远在变身，基础形态设计失去意义。

**买量逻辑：** "怪物太多了，变身，清屏" → 变身动画 + 范围爆发，是完整的视觉弧线。

**立项建议：** `值得验证`——变身切换节奏是核心设计挑战，建议先定义好"变身前/中/后"各自的体验价值。

---

## 方向 D — 培育进化守线

**守线力量来源：培育现有单位成长 / 核心动词：养**

**一句话定义：** 守线单位从弱小形态开始，在战斗中积累经验并进化；主角决定进化资源的分配和进化路线；防线越战越强，进化方向决定守线特性。

**主角核心操作：** 分配进化资源（给谁）→ 选择进化路线（走哪个方向）→ 关键时刻触发进化

**守线力量的边界（不是什么）：**
- 守线单位是己方已有单位，不是从敌方获取（vs A 抓宠、F 感染）
- 进化是局内动态发生的，不是局前构筑（vs 卡组守线的牌库构筑）
- 主角的核心价值在于进化资源分配，不是直接战斗输出

**最大机会点：** "越打越强"的正向反馈天然友好；进化瞬间是惊喜节点；进化路线给重度用户提供构筑深度。

**最大风险点：** 单局时长管理困难（防线越来越强，怪物压力如何同步提升）；前5分钟看不到进化，新手体验差。

**买量逻辑：** "这只小蜥蜴进化成巨龙守门" → 进化前后对比，视觉冲击强。

**立项建议：** `优先验证，控制进化层数`——首版建议只做2层进化（幼年→成年），进化路线用三选一复用已有机制。

## 方向 E — 陷阱守线

**守线力量来源：预判路径+埋设机关 / 核心动词：埋**

**一句话定义：** 主角在战场上布置陷阱（物理/元素/机关），怪物进入触发区时陷阱激活；守线依靠预判和陷阱组合，而不是主动输出或单位守门。

**主角核心操作：** 观察当前波次组成 → 预判怪物移动路径 → 在关键节点埋设陷阱 → 看陷阱触发/补置/组合连锁

**守线力量的边界（不是什么）：**
- 陷阱是无生命的，不主动移动追击（vs 抓宠、培育：守线单位是活体）
- 陷阱必须被触发才生效，主角不直接输出（vs 变身、骑战：主角是主战单元）
- 主角战斗能力弱，主要价值是布置和组合（不是主战，是机关师）

**最大机会点：** 预判成功的爽感极强（"我算到你要从这走"）；陷阱连锁触发是天然的连锁消消乐式爽感；和所有现有方向高度差异化。

**最大风险点：** 怪物路径随机性如果太强，预判失效，整个玩法崩溃（需要路径有规律可循）；陷阱布置界面如果复杂，认知负担压垮中轻度用户。

**买量逻辑：** 怪物大军踩进走廊 → 连锁陷阱触发 → 全部清空。节奏明确，视觉冲击强。

**立项建议：** `值得验证`——路径可预判性是设计基石，需要先定义路径规则再做陷阱设计。

---

## 方向 F — 感染守线

**守线力量来源：感染标记+自动传播 / 核心动词：染**

**一句话定义：** 主角对少数敌方单位施加感染标记，被感染的单位倒戈成为己方守线力量；感染还能在敌群中继续传播，守线力量可以自我扩散，无需主角逐个操作。

**主角核心操作：** 选择感染目标（谁传播范围最广）→ 施加感染 → 观察传播路径 → 关键时刻用主角补充感染或阻断传播

**守线力量的边界（不是什么）：**
- 守线力量通过被动传播扩散，不需要主角每次操作（vs A 抓宠：主角必须逐个主动捕获）
- 感染来自敌方单位，是临时状态，感染者死亡后不留任何守线力量（vs 培育：有固定的己方单位成长）
- 主角的核心操作是"选择谁来传播"，而不是直接战斗

**最大机会点：** 感染传播的视觉有独特性（敌军阵营中出现变色/变形的叛徒）；传播速度带来数学爽感（选对第一个感染目标，效果呈指数放大）。

**最大风险点：** 感染平衡极难：传播太快局面失控，传播太慢守线崩溃；感染状态的视觉识别要非常清晰，否则玩家分不清谁是己方谁是敌方。

**买量逻辑：** 感染在敌军中蔓延，一半敌人倒戈守门，最后一波自相残杀清屏。视觉反差极大。

**立项建议：** `原型验证后决策`——机制有强原创性，但平衡复杂度是最高的；建议先做最小原型确认传播节奏是否可控。

## 方向 G — 地形守线

**守线力量来源：改造战场地形 / 核心动词：改**

**一句话定义：** 主角没有守线单位，而是通过实时改造战场地形（挖沟、竖墙、设障碍、燃烧区域、冻结地面）让地形本身成为守线力量；怪物被阻挡、减速或引入死亡走廊。

**主角核心操作：** 观察怪物路径 → 挖/竖/点改变地形 → 将怪物引导进预设走廊 → 用主角输出击杀集中的怪物

**守线力量的边界（不是什么）：**
- 没有任何活体单位在守线，守线全靠地形（最大差异点）
- 主角本身有战斗能力，但主要价值是改造地形，而非主战
- 地形改造是实时的（vs 陷阱守线：陷阱布置后等待触发；地形守线是主动改变战场结构）

**最大机会点：** 与所有现有方向彻底不同（无单位守线）；地形组合形成"围杀通道"有强烈的智力成就感；和城市建设/沙盒玩家有交叉。

**最大风险点：** 没有守线单位，画面可能显得空旷和孤独；地形改造的流畅度要求极高（手感差则整个体验崩塌）；研发成本最高。

**买量逻辑：** 主角把路挖弯，怪物全挤进死胡同，集中爆炸清屏。战术感和反差感都有。

**立项建议：** `排在 A/B/D 之后验证`——差异点最强，但研发风险最高；适合作为技术验证型小项目。

---

## 方向 H — 共鸣守线

**守线力量来源：主角状态实时驱动守线单位 / 核心动词：共**

**一句话定义：** 主角与守线单位之间存在实时共鸣联结；主角进入特定状态（点燃、冰封、加速、超载），与主角共鸣的守线单位自动获得相同状态并发生效果改变；主角的行动即守线单位的行动。

**主角核心操作：** 主动进入/切换状态（跑进火焰区、施放元素技能、装备状态道具）→ 观察共鸣单位的效果变化 → 维持/切换状态以匹配当前压力

**守线力量的边界（不是什么）：**
- 守线单位是被动的，没有自主决策，完全靠主角状态驱动（vs 抓宠/感染：守线单位有独立行为）
- 主角不发命令，而是以自身状态作为实时 buff 来源（vs 指挥官守线：指挥官主动下达命令）
- 状态共鸣是实时同步的，主角离开状态则效果即时消失（不是一次性触发）

**最大机会点：** 主角的每个操作都直接体现在守线单位上，操作反馈感极强；状态切换的组合可以形成深度；视觉上"全线点燃/结冰"有冲击力。

**最大风险点：** 状态切换如果频率要求太高，中轻度用户跟不上；共鸣效果的视觉识别要非常清晰（必须让玩家在0.5秒内看懂"我做了什么，单位发生了什么变化"）。

**买量逻辑：** 主角冲进火焰区 → 全线守线单位燃烧爆发 → 一波怪物被清空。直觉性强，视觉冲击清晰。

**立项建议：** `值得验证`——操作反馈闭环天然强，适合做成中轻度手游；关键设计挑战是状态切换节奏。

## 推衍结论

**4个优先验证方向：A（抓宠）、B（骑战）、C（变身）、D（培育进化）**

共同条件：
- 核心动词可以在30秒素材里体现
- 与现有三个项目的差异一句话能说清
- 首版原型规模可控（不需要大量单位种类或地形系统）

**F（感染守线）**潜力最高，买量逻辑最强，但平衡难度也最高——建议等 A/B/D 中有一个跑通后再投入。

**G（地形守线）**差异最大，但研发成本最高——适合作为独立小项目技术验证，不适合作为主力立项。

**H（共鸣守线）**操作反馈天然强，建议考虑是否可以作为某个主力方向的子系统，而非独立立项。
