# RPG+防守 — 问题驱动推导

> 思路：先分析这个品类的结构性问题，再从问题中找解决方案，方案对应立项方向。
> 不从结构框架出发，从玩家真实体验的断裂点出发。

---

## 一、RPG+防守的优势（为什么值得做）

**RPG 带来的：**
- 可操控主角 → 代入感、存在感，买量有脸
- 职业/技能差异 → 构筑深度，重度用户有追求
- 成长线 → 长期留存锚点

**防守带来的：**
- 怪潮压力 → 持续的危机感，单局节奏清晰
- 守住某个目标 → 失败条件明确，玩家知道自己在干什么
- 空间策略 → 什么放在哪里，有布局感

**两者结合的机会：**
代入感（我在守护）+ 危机感（快撑不住了）+ 成就感（build成型反杀）。
这三件事叠在一起，是很强的情绪弧线。

---

## 二、核心体验问题

### 问题 1：主角和防线争夺注意力，两个都做不深

**现象：** 玩家同时需要操控主角战斗 + 管理守线单位/建造。
两件事互相打断，任何一件做深，另一件就被忽略。

**结果链：**
- 主角战斗设计做深 → 玩家专注操作主角，守线单位变摆设
- 守线设计做深 → 玩家专注布局，主角退到后方挂机，"RPG感"消失
- 两个都做 → 玩家不知道该关注哪个，体验稀薄

**核心矛盾：** RPG 要求"专注角色本身"，Defense 要求"专注全局战场"。同时要求两件事，两件事都做不好。

---

### 问题 2：主角的机动性使防守策略失效

**现象：** 主角能自由移动到任何位置。

**结果链：**
- 守线策略的核心趣味 = 在正确的位置放正确的东西
- 主角能随时移动去补任何位置的空缺
- 因此任何阵位部署的"失误"都能被主角移动弥补
- 阵位策略失效，守线退化为"主角在哪里哪里就赢"

**副作用：** 防线再薄也不会有紧张感，因为主角能去补。
玩家失去"边缘胜利"的爽感——永远是主角一出手解决，而不是"刚好守住"。

---

### 问题 3：成长节奏和守线压力节奏相互干扰

**现象：** 守线要求"专注当下压力"，RPG成长要求"感受自己在变强"。

**结果链：**
- 守线压力强时，玩家没空感受成长
- 刷成长期间，守线压力弱，怪潮变成背景噪音
- 两者都不强烈，体验就一直处于中间状态，不紧张也不爽

**更深层的问题：** 局内 Roguelite 成长 vs 局外 RPG 成长，互相稀释。
局外强到一定程度，局内选择失去意义；局内随机差异太大，局外成长被否定。

---

### 问题 4：守线单位/塔的存在感问题（派生问题）

这是问题1和问题2的共同结果：

- 主角抢注意力 → 守线单位是背景
- 主角能移动补位 → 守线单位不在也没关系
- 最终：守线单位要么是摆设（被主角覆盖），要么是保险（主角顾不到时才有用）

**没有一个好的答案让主角和守线单位各自有独立的、不可替代的价值。**

---

## 三、从问题推导解决方案

### 从问题1推导：解决注意力争夺

**思路：让两件事合并为一件事，或者分开到不同时间段。**

**方案 1-A：主角的行为直接产生守线效果（合并）**
主角不是独立战斗，也不是独立部署；主角的行为本身就是守线行为。
→ 例如：主角走到某个位置，附近守线单位被激活/强化；主角离开则效果消失。
→ 玩家的操作焦点只有一个：主角去哪里。守线效果是这个决策的自动结果。
→ **对应方向：共鸣守线**

**方案 1-B：彻底去掉守线单位，主角就是守线本身（取消一个核心）**
没有塔，没有分身，没有单位。主角就是防线，主角切换形态/坐骑来应对不同威胁。
→ 注意力只有一个对象：主角。
→ **对应方向：变身守线、骑战守线**

**方案 1-C：时间分离（部署期 vs 守线期）**
波次间：主角在战场上布置好所有守线手段。
波次中：主角退到安全区，守线手段自动运行；主角只做干预，不做主战。
→ 两件事不同时发生，不争夺注意力。
→ **对应方向：陷阱守线**（提前埋好，守线期自动触发；主角在守线期只补置陷阱）

---

### 从问题2推导：解决机动性破坏防守策略

**思路：让主角的位置本身有战略代价，或者限制主角的覆盖范围。**

**方案 2-A：主角的位置决定守线效果的触发（位置即策略）**
主角站在某处，才能激活该区域的守线能力。移动到另一处，前一处效果消失。
→ 主角的"在哪里"变成核心决策，而不是"去哪里补漏"。
→ 玩家主动选择"我现在应该守哪个位置"，而不是被动响应漏洞。
→ **对应方向：共鸣守线**

**方案 2-B：主角移动能力有结构限制（坐骑决定覆盖区域）**
不同坐骑决定主角的机动范围和攻击覆盖：
慢坐骑=高火力固定炮台，快坐骑=低火力高机动游击。
→ 玩家在机动性和火力之间做取舍，不能既快又强。
→ **对应方向：骑战守线**

**方案 2-C：把"在正确位置布置正确东西"做成核心操作**
主角的主要工作就是判断怪物路径、在合适节点埋设陷阱。
守线期不需要主角到处乱跑，因为陷阱已经布好了。
→ 位置策略前置到部署期，守线期主角位置影响有限。
→ **对应方向：陷阱守线**

---

### 从问题3推导：解决节奏冲突

**思路：让成长感和守线压力同时发生，或者把成长弧压缩进单局。**

**方案 3-A：成长行为本身就是守线行为（合并节奏）**
每次捕获敌人 = 我的守线力量增加了 = 成长感。
捕获行为发生在守线压力中，两件事不分离。
→ 玩家在守线压力中感受成长，两者不再是独立节奏。
→ **对应方向：抓宠守线**

**方案 3-B：在单局内完成完整成长弧（压缩到局内）**
守线单位从最弱形态开始，在同一局内进化到最强形态。
玩家在单局内就能体验从"刚刚够用"到"反杀碾压"的完整成长感。
→ 局外成长影响"起点和可选项"，局内成长决定"这局的爽感弧线"。
→ 两个成长系统不互相干扰，因为局内成长在每局都重置。
→ **对应方向：培育进化守线**

**方案 3-C：取消局外成长，成长完全在局内（彻底分离）**
局外只影响"可以带哪个坐骑/变身形态"，不影响数值强度。
局内的选择决定一切，局外成长不能替代局内决策。
→ 守线压力感和成长感都在局内完整呈现，不互相稀释。
→ **对应方向：骑战守线、变身守线**（局外解锁形态，局内切换形态，形态之间没有数值强弱）

---

### 从问题4推导：解决守线单位的存在感问题

问题4是问题1和2的结果，解决问题1和2后，问题4自然改善。
但有一个独立解法：

**方案 4-A：主角完全无法承担守线单位的职责**
主角只能输出或机动，不能堵路、不能吸伤害、不能长期占据一个阵位。
守线单位负责所有"阵位固定"的功能。
→ 主角和守线单位的职责不重叠，不互相取代。
→ **实现方式：主角是玻璃炮（高输出但不能堵路），守线单位是肉盾（不能输出但能挡路）**

**方案 4-B：彻底去掉守线单位，不存在这个问题**
→ **对应方向：变身守线、骑战守线、陷阱守线（地形替代单位）**

## 四、解决方案→立项方向对照

| 立项方向 | 解决了哪些核心问题 | 核心机制 |
|---|---|---|
| **骑战守线** | 问题1（去掉守线单位）、问题2（坐骑限制覆盖）、问题3（局内形态切换=成长感） | 坐骑切换=战术形态切换，主角+坐骑是唯一守线主力 |
| **变身守线** | 问题1（去掉守线单位）、问题3（局内形态解锁=成长弧） | 主角自身形态改变，形态决定守线方式 |
| **共鸣守线** | 问题1（主角行为合并为守线行为）、问题2（位置即策略） | 主角的状态/位置实时驱动守线单位，主角去哪里哪里被激活 |
| **抓宠守线** | 问题3（捕获行为=守线中的成长感） | 战场中捕获敌方单位变为守线力量，守线力量由局中决定 |
| **培育进化守线** | 问题3（单局内完成成长弧） | 守线单位局内进化，从弱到强，一局就能体验完整成长 |
| **陷阱守线** | 问题1（时间分离）、问题2（位置策略前置） | 波次前布置陷阱，守线期自动触发，主角不做主战 |

## 五、优先级重判

用问题解决能力重新评估：

**解决了最多核心问题的方向：**

**1. 骑战守线** — 同时解决问题1、2、3
- 去掉守线单位，注意力只有主角（解决问题1）
- 坐骑限制覆盖范围，位置有代价（解决问题2）
- 形态切换给局内成长感，不依赖局外数值（解决问题3）
- **最推荐：这是结构上最干净的方向**

**2. 共鸣守线** — 解决问题1+2，同时保留守线单位体系
- 主角行为合并为守线行为，不再争夺注意力（解决问题1）
- 主角位置即策略，移动有代价（解决问题2）
- 守线单位存在感强（因为完全靠主角驱动，主角行为有直接体现）
- **适合想保留守线单位体系的方向**

**3. 陷阱守线** — 时间分离解法，结构最特殊
- 两个核心完全分开在不同时间段（解决问题1最彻底）
- 位置策略是核心操作（解决问题2）
- **需要验证：纯陷阱守线是否有足够的操作爽感**

**4. 抓宠守线** — 解决问题3（成长感），但问题1和2仍然存在
- 捕获行为和守线压力仍然争夺注意力（问题1未解决）
- 需要叠加其他机制来解决问题1和2
- **建议：作为某个主方向的子系统，而不是独立方向**

**5. 培育进化守线** — 解决问题3，但问题1和2仍然存在
- 同抓宠守线，成长感解决了，但注意力分裂和阵位策略仍然需要解法
- **建议：同上，作为子系统**

**6. 变身守线** — 去掉守线单位，解决问题1，问题2和3需要额外设计
- 形态切换自然产生成长感，但需要设计保证形态之间有真实差异

## 六、结论

**这次推导和 v2 结构推导的核心差别：**

v2 是"角色与防守的关系有几种"——从结构出发，答案是8种，排列组合。
本文是"RPG+防守有哪些问题"——从问题出发，答案是解决方案的集合，数量少但每个有明确的存在理由。

**v2 的8个方向中，抓宠和感染守线的定位需要调整：**
- 抓宠守线：解决了复玩驱动，但没有解决注意力分裂和阵位策略问题——更适合作为"骑战守线"或"共鸣守线"的子系统，而不是独立立项方向
- 感染守线：机制独特，但同样没有解决核心的注意力分裂问题，且平衡极难——建议后置

**优先验证顺序（基于问题解决能力）：**
1. **骑战守线** — 结构最干净，解决问题最多
2. **共鸣守线** — 保留守线单位体系的最优解
3. **陷阱守线** — 时间分离方案，结构特殊值得验证
4. **变身守线** — 需要额外设计保证形态差异
5. 抓宠 / 培育 / 感染 — 作为子系统考量
