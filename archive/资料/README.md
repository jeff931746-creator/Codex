# 资料目录说明

该目录用于统一承载个人策划资料，当前采用 `用途层 -> 专题层` 的方式归档。

## 全局规则

- 所有知识沉淀、研究记录、可复用结论、专题拆解，统一放在 `research/资料/`
- 如果内容会长期复用，就不要放到项目目录里当临时稿
- 项目目录只放项目工作文件，知识本体回流到 `资料`
- 项目拆解必须先套用 `research/资料/项目拆解模板.md`，不能临时自由发挥结构

## 当前结构

- `竞品库`：记录外部产品、市场观察、商店页信息、竞品分析
- `复盘库`：记录项目复盘、版本复盘、活动复盘、方案复盘
- `机制库`：按单个游戏做专题性结构拆解，沉淀系统、模块、流派、经济、关卡与检定结论
- `立项方法论`：记录可复用的立项判断框架、市场验证方法、视频/文章学习笔记与决策检查表

## 归档原则

后续新增资料时，优先先判断它属于哪个“用途层”，再判断是否需要进入某个游戏专题。

归档路径格式：

`资料/<库类型>/<游戏或专题>/`

例如：

- `资料/竞品库/塔防/`
- `资料/复盘库/RPG/`
- `资料/机制库/小丑牌/`
- `资料/机制库/国王带队守线RPG塔防/`

## 使用规则

- 一份资料只放在一个主归档位置，避免重复存放。
- 如果资料横跨多个专题，优先放在最主要的目标专题目录。
- 暂时无法判断主题时，先放入 `其他待分类`，后续再整理。
- 文件名尽量体现 `产品/主题 + 内容类型 + 日期`。
- 机制库内部不再用游戏类型做主分类，改为“单个游戏主目录 + 多专题拆解”来检索。
