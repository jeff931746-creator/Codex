# 题材评估标准：逻辑链与证据链

更新时间：2026-05-11

> **关联文档**：平台参竞规则变化（2026 Q2 上线）参见 `平台策略变化_应用包资产x素材_2026Q2.md`。
> 该变化不替换本评估标准的双轴结构，但在"有效获量"轴下新增"参竞单元覆盖度"子维度。

## 一句话结论

题材评估不是判断“这个题材看起来好不好”，而是判断：

```text
这个题材在当前买量市场里，能否以可接受成本拿到目标用户，并用产品内 LTV 覆盖获量成本。
```

因此题材总表的决策必须基于：

```text
有效获量能力 × ROI 承接能力 × 证据等级
```

没有证据链的评分只能算“假设分”，不能作为立项结论。

## 决策逻辑链

每个题材必须按以下链路判断，不允许跳步。

```text
1. 题材定义
   这个题材到底是什么？
   是否能用“环境/时代 + 文化范式/战斗体系 + 叙事”说清？

2. 题材天然能力
   它本身是否有视觉符号、情绪刺激、行为想象、成长空间和付费容器？

3. 当前市场校正
   当前市场是否已经红海？
   同题材素材是否疲劳？
   CPI 是否可能过高？
   点击人群和付费人群是否匹配？

4. 我方进入切口
   我们靠什么进入？
   是新表达、新玩法、新人群、成本优势，还是投放素材角度？

5. 有效获量评分
   当前是否能以可接受成本拿到目标用户？

6. ROI 承接评分
   进入游戏后是否能形成留存、付费和可回本的 LTV？

7. 证据等级校正
   分数来自真实投放、竞品数据、广告素材观察、内容趋势，还是主观判断？

8. 推荐结论
   通过 / 补证 / 改写后复评 / 淘汰。
```

硬规则：

```text
题材天然能力成立，不等于市场成立。
市场信息不足，不允许标“通过”。
无进入切口，不允许标“通过”。
证据等级低于 B，不允许标“通过”。
有效获量或 ROI 承接任一低于 3，不允许标“通过”。
```

## 市场信息来源

市场信息必须按用途拆分。不同来源只能证明不同问题，不能互相替代。

| 来源类型 | 能证明什么 | 不能证明什么 | 示例 |
|---|---|---|---|
| 投放素材库 | 当前是否有人在买量、素材是否同质化、创意是否疲劳 | 不能直接证明 ROI | TikTok Creative Center、Meta Ads Library、Google Ads Transparency Center、巨量创意/巨量算数、DataEye、广大大、AppGrowing 等 |
| 商店榜单与下载收入估算 | 题材或玩法是否已有产品验证、收入体量、生命周期 | 不能证明具体素材能否跑通 | App Store、Google Play、Steam、Sensor Tower、data.ai、AppMagic、点点数据、七麦数据等 |
| 内容平台热度 | 题材是否有外部内容需求、情绪是否正在被消费 | 不能证明游戏内付费成立 | 抖音、TikTok、YouTube、B站、小红书、快手、起点、番茄、短剧平台、豆瓣、IMDb 等 |
| 竞品拆解 | 题材如何承接玩法、付费、长线内容 | 不能证明当前买量成本 | 同题材或相近题材游戏、短剧、小说、影视、动漫 |
| 内部素材测试 | 最直接证明素材点击、CPI、CVR、人群质量 | 样本小的时候不能证明长期 ROI | 小额投放、A/B 素材测试、落地页测试、可玩广告测试 |
| 内部产品数据 | 最直接证明留存、付费、LTV、回本周期 | 不能证明新题材外壳一定能获量 | Demo 测试、留存测试、付费测试、历史项目数据 |

使用原则：

- 判断“有效获量”优先看投放素材库、素材测试、CPI/CVR、内容平台热度。
- 判断“ROI 承接”优先看竞品收入、玩法商业化案例、内部留存付费、内容生产成本。
- 判断“蓝海”必须同时看供给少和需求存在。只看到供给少，不能叫蓝海，只能叫“未验证空白”。
- 判断“红海”必须区分题材红海、玩法红海、素材表达红海。不能把其中一个直接等同于全部红海。

## 证据等级

每条评分必须标证据等级。证据等级决定结论上限。

| 等级 | 定义 | 可支持的最高结论 |
|---|---|---|
| S | 自有投放数据或内部产品数据证明：有 CPI/CVR/留存/付费/LTV | 通过 |
| A | 至少 2 类外部强证据互相印证：广告素材库 + 商店收入/下载，或竞品拆解 + 榜单数据 | 通过 / 补证 |
| B | 有明确竞品或内容热度，并能解释进入切口，但缺少投放或收入数据 | 补证 |
| C | 只有题材热度、素材脑暴或单一来源案例 | 改写后复评 |
| D | 主观判断、灵感、未验证假设 | 改写后复评 / 淘汰 |

结论约束：

```text
S/A 级证据 + 双轴 >= 4：可标“通过”
B 级证据 + 双轴 >= 3.5：最多标“补证”
C/D 级证据：不得标“通过”
任一硬伤维度 < 3：不得标“通过”
```

## 有效获量评分标准

有效获量评分回答：

```text
这个题材在当前买量市场里，能不能以可接受成本拿到目标用户？
```

总分由 6 个维度组成，每项 1-5 分，取平均分。

| 维度 | 1 分 | 3 分 | 5 分 | 推荐证据 |
|---|---|---|---|---|
| 需求规模 | 没有可验证受众 | 有小众或间接受众 | 多平台存在稳定消费人群 | 内容平台热度、榜单、搜索趋势、竞品数量 |
| 第一眼刺激 | 画面无冲突，理解慢 | 有题材识别，但刺激一般 | 3 秒内能看懂冲突和情绪 | 素材样例、短视频脚本、封面测试 |
| 素材新鲜度 | 与现有素材高度同质化 | 有局部新角度 | 一眼不同，且仍能被目标用户理解 | 广告素材库、竞品素材对比 |
| 竞争压力 | 同题材同表达大量投放 | 有竞争但可找到切口 | 需求存在但同表达供给少 | 素材库密度、竞品数量、近 30-90 天投放观察 |
| CPI 风险 | 预估买量成本明显不可承接 | 成本不确定，需要小测 | 有低成本人群或素材切口 | 历史项目、行业投放经验、小额测试 |
| 参竞单元覆盖度 | 只能跑单一载体 + 单一转化目标，素材钩子单一只能换皮 | 可跑 2 个载体或 IAP/IAA 其中一组，素材有 ≥ 60% 差异度产能 | 可跑 ≥ 2 个载体 × IAP/IAA 双组合，素材具备 ≥ 90% 差异度（首发级）持续产能 | 应用包/资产 × 素材 × 载体 × 转化目标的实际跑通案例、竞品在多载体/多转化目标的覆盖情况、AIGC/试玩/星广联投等素材产能工具的适配度 |

注：参竞单元覆盖度的背景与原始策略见 `平台策略变化_应用包资产x素材_2026Q2.md`。

评分解释：

```text
4.0-5.0：高有效获量，当前市场有机会进人
3.0-3.9：中等有效获量，需要明确切口和小测
2.0-2.9：低有效获量，先改题材表达
1.0-1.9：不建议进入买量验证
```

## ROI 承接评分标准

ROI 承接评分回答：

```text
买进来的用户，能不能被产品内循环留住、付费，并覆盖买量成本？
```

总分由 5 个维度组成，每项 1-5 分，取平均分。

| 维度 | 1 分 | 3 分 | 5 分 | 推荐证据 |
|---|---|---|---|---|
| LTV 上限 | 题材只适合一次性体验 | 有成长或付费，但深度一般 | 天然支持长线成长、收集、外观、社交或策略 | 同类产品收入、品类 LTV、竞品系统拆解 |
| 留存深度 | 玩一次就结束 | 有阶段目标 | 有持续目标和重复行为 | Demo 留存、竞品循环、内容章节结构 |
| 付费自然度 | 付费破坏题材承诺 | 可设计部分付费点 | 付费自然服务成长、效率、收集、外观或身份 | 竞品 IAP、系统拆解、历史付费模型 |
| 内容效率 | 内容生产成本极高 | 可生产但成本需控制 | 可模块化、批量化、低成本迭代 | 美术/剧情/关卡成本评估、团队能力 |
| 回本压力 | 回本周期明显超团队能力 | 回本周期不确定 | 回本周期与成本、团队、投放节奏匹配 | 预算、制作周期、投放回收模型 |

评分解释：

```text
4.0-5.0：高 ROI 承接，有机会支撑买量回本
3.0-3.9：中等 ROI 承接，需要补长线或控成本
2.0-2.9：低 ROI 承接，先改产品循环
1.0-1.9：不建议作为商业立项
```

## 推荐结论规则

| 推荐结论 | 使用条件 | 下一步 |
|---|---|---|
| 通过 | 有效获量 >= 4，ROI 承接 >= 4，证据等级 A/S，进入切口明确，无硬伤 | 进入小额素材测试或立项预演 |
| 补证 | 双轴 >= 3，但证据不足、切口不完整或存在轻微硬伤 | 补市场证据、竞品拆解或素材小测 |
| 改写后复评 | 题材方向有潜力，但定义、切口、素材钩子或玩法承接不清 | 重写题材定义和素材钩子后复评 |
| 淘汰 | 双轴任一 < 3，且无法通过切口修复；或证据证明市场/ROI 不成立 | 移出候选池或仅保留为素材参考 |

## 表格字段要求

`题材总表.md` 至少应能追溯以下信息：

| 字段 | 要求 |
|---|---|
| 题材 | 必须是可被玩家理解的题材表达，不只是玩法或素材桥段 |
| 四象限 | 必须由有效获量和 ROI 承接自动推导，不手填主观结论 |
| 有效获量 | 必须能拆回 5 个子维度 |
| ROI 承接 | 必须能拆回 5 个子维度 |
| 推荐结论 | 必须受证据等级约束 |
| 玩法承接 | 必须说明题材如何进入游戏循环 |
| 进入切口 | 必须说明新表达、新玩法、新人群或成本优势 |
| 硬伤维度 | 必须写具体维度，不能只写“风险高” |
| 素材钩子 | 必须能直接变成首图或 3-5 秒视频 |
| 评审说明 | 必须解释“为什么这样打分”，不能只复述分数 |
| 来源 | 必须区分灵感来源和证据来源 |
| 证据等级 | 必须标 S/A/B/C/D |
| 下一步动作 | 必须说明补证、改写、小测或淘汰 |

## 质量门禁

题材进入总表时，可以是草稿；题材进入推荐池时，必须过门禁。

```text
草稿池：允许 C/D 级证据，允许未知分
候选池：至少 B 级证据，双轴不能为 0
推荐池：至少 A 级证据，必须有进入切口和硬伤处理
立项池：必须有 S 级或强 A 级证据，并完成素材小测或竞品拆解
```

## 对当前题材总表的直接影响

当前 `题材总表.md` 中：

- `通过` 结论必须回查证据等级；没有 A/S 证据的，应降为 `补证`。
- `蓝海`、`无竞争`、`高 ROI`、`CPI 风险` 等判断必须补证据来源。
- 出现 `dim6/dim7/dim10` 的说明，必须改为可读字段名。
- `进入切口` 为空的题材，不能标 `通过`。
- `未知/0.0` 项应留在草稿池，不参与排序。
