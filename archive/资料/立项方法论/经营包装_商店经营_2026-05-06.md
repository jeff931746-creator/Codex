# 商店经营包装

## 核心定位

**底层经营**：进货 → 定价 → 销售 → 获得金币 → 购买装备

**表层包装**：武器店、药水店、冒险者商店、黑市

**认知负荷**：中（定价、进货需要策略）

## 四要素验证

| 要素 | 验证 |
|---|---|
| 时间分离 | ✓ 局外经营，局内战斗 |
| 低操作频次 | ✗ 经营需要频繁操作（定价、进货、摆放） |
| 本能目标 | ✗ "我的商店"情感驱动弱，不如"我养大的宠物"或"我打造的装备" |
| 成长/策略 | ✓ 定价、进货有策略 |

**双约束验证**：
- **约束一（结构合法性）**：✓ 商店经营本身是独立玩法（Recettear、Moonlighter可以不冒险只经营）
- **约束二（体验闭环）**：⚠️ 经营→金币→购买装备→战斗，转化链条较长，中间环节（金币）削弱了直接成就感

## 核心循环

```
局外经营阶段
├─ 进货商品
├─ 摆放商品
└─ 定价销售
    ↓
获得金币
├─ 顾客购买商品
├─ 获得金币
└─ 商店升级
    ↓
购买装备
├─ 用金币购买装备
├─ 用金币升级装备
└─ 用金币解锁新商品
    ↓
进入战斗
    ↓
局内战斗阶段
├─ 操控角色防守
├─ 使用购买的装备
└─ 装备效果清晰可见
    ↓
战斗结束
├─ 获得商品
├─ 解锁新商品
└─ 获得金币
    ↓
回到局外经营
```

## 具体设计方案

### 方案A：武器店模式

**核心机制**：
- 经营武器店
- 进货武器
- 销售给冒险者
- 用金币购买自己的装备

**商品类型**：
- **武器**：剑、弓、法杖
- **防具**：盔甲、盾牌
- **消耗品**：药水、卷轴
- **材料**：矿石、皮革

**经营系统**：
- **进货决策**：选择进货商品类型和数量，不同商品有不同成本和利润率
- **定价策略**：定价过高影响销量，定价过低利润低，需要根据顾客需求调整
- **摆放优化**：商品摆放位置影响销量，热门商品放显眼位置
- **顾客系统**：不同顾客有不同需求和购买力，满足需求提升好评
- **商店升级**：扩大店面、增加货架、解锁新商品类型

**决策场景示例**：
- 场景1：稀有材料到货，是高价卖出赚快钱，还是留着自己用于战斗？
- 场景2：顾客需求集中在某类商品，是大量进货满足需求，还是保持多样性？
- 场景3：战斗前资金紧张，是降价快速回笼资金，还是保持利润等待合适买家？

**战斗关联**：
- 用金币购买自己的装备
- 装备在战斗中使用
- 战斗获得商品
- 商品可以销售或自用

**长线驱动**：
- 解锁所有商品
- 升级商店规模
- 成为最大的武器商

**参考对标**：
- Recettear（道具店经营）
- Moonlighter（商店+冒险）
- Shoppe Keep（商店经营）

### 方案B：冒险者商店模式

**核心机制**：
- 经营冒险者商店
- 销售装备给冒险者
- 冒险者带回战利品
- 战利品可以销售或自用

**商品类型**：
- **装备**：武器、防具
- **消耗品**：药水、食物
- **任务道具**：地图、钥匙
- **战利品**：宝石、材料

**经营系统**：
- **冒险者需求系统**：不同冒险者有不同职业和需求（战士需要武器防具，法师需要法杖药水）
- **声誉系统**：满足需求获得好评，好评吸引更高级冒险者，高级冒险者带回更好战利品
- **委托系统**：冒险者可以接受委托任务，完成后带回指定战利品
- **战利品鉴定**：冒险者带回的战利品需要鉴定，鉴定后可以销售或自用
- **商店升级**：扩大店面、增加商品种类、解锁稀有商品

**决策场景示例**：
- 场景1：传说冒险者需要昂贵装备，是满足他的需求换取稀有战利品，还是把装备留给自己？
- 场景2：战利品中有稀有材料，是高价卖出赚取金币，还是留着制作更强装备？
- 场景3：多个冒险者同时需要同一商品，是优先满足高级冒险者，还是平衡所有顾客满意度？

**战斗关联**：
- 用金币购买自己的装备
- 战斗获得战利品
- 战利品可以销售
- 销售获得金币

**长线驱动**：
- 解锁所有商品
- 吸引传说冒险者
- 成为最大的冒险者商店

**参考对标**：
- Recettear（道具店经营）
- Moonlighter（商店+冒险）
- Potion Craft（药水店经营）

### 方案C：黑市模式

**核心机制**：
- 经营黑市
- 销售违禁品
- 高风险高回报
- 用金币购买强力装备

**商品类型**：
- **违禁品**：禁药、禁术
- **赃物**：偷来的宝物
- **情报**：秘密情报
- **雇佣兵**：雇佣战士

**经营系统**：
- **违禁品交易**：销售违禁品利润高但有风险，需要判断顾客身份（是真买家还是卧底）
- **风险管理**：被查封会损失库存和金币，需要贿赂官员或雇佣保镖降低风险
- **情报系统**：购买情报了解哪些商品需求高、哪些官员可以贿赂、哪些时段安全
- **黑市网络**：建立黑市网络，解锁更稀有的违禁品和更高级的买家
- **商店伪装**：表面经营合法生意，暗地里交易违禁品

**决策场景示例**：
- 场景1：高价买家要求违禁品，但最近查封风险高，是冒险交易还是拒绝？
- 场景2：官员要求贿赂，是支付保护费降低风险，还是省钱自己承担风险？
- 场景3：情报显示某违禁品需求暴涨，是大量进货赚取暴利，还是保守经营避免风险？

**战斗关联**：
- 用金币购买强力装备
- 战斗获得赃物
- 赃物可以销售
- 销售获得金币

**长线驱动**：
- 解锁所有违禁品
- 成为黑市大亨
- 控制地下经济

**参考对标**：
- Moonlighter（商店+冒险）
- Recettear（道具店经营）
- Drug Dealer Simulator（黑市经营）

## 与操控防守的结合

**方案A（武器店）**：
- 我经营的武器店赚取金币
- 我用金币购买装备
- 装备在战斗中使用

**方案B（冒险者商店）**：
- 我经营的商店服务冒险者
- 冒险者带回战利品
- 战利品可以销售或自用

**方案C（黑市）**：
- 我经营的黑市赚取金币
- 我用金币购买强力装备
- 装备在战斗中使用

**资源循环**：
- 经营商店 → 获得金币 → 购买装备 → 战斗使用 → 获得商品 → 销售商品
- 循环清晰，但经营环节较重

**成长可见**：
- 商店规模越来越大
- 商品越来越多
- 金币越来越多

## 参考对标

**成功案例**：
- **Recettear（道具店经营）**：经营+地牢冒险，经营本身有深度（定价、进货、摆放），但转化链条长（经营→金币→装备→冒险）
- **Moonlighter（商店+冒险）**：白天经营商店，晚上进入地牢，战利品可以销售或自用，经营和冒险紧密结合
- **Potion Craft（药水店经营）**：炼制药水并销售，炼制过程本身是独立玩法，但销售环节较简单
- **Shoppe Keep（商店经营）**：纯商店经营模拟，无战斗部分，证明商店经营本身可以成为独立玩法

**独特性**：
- 商店经营 + 操控战斗（不是自动战斗）
- 经营获得金币，金币购买装备
- 战斗获得商品，商品可以销售
- **关键问题**：金币作为中间环节削弱了成就感，不如"我打造的装备"或"我养成的宠物"直接

## 潜在问题与解决方案

**问题1：商店经营会不会太复杂？**
- 解决：经营规则简单清晰
- 解决：有自动定价功能
- 解决：有推荐进货功能

**问题2：经营和战斗的关联够强吗？**
- 解决：金币直接用于购买装备
- 解决：战斗获得商品可以销售
- 解决：经营和战斗互相喂养

**问题3：经营的操作频次会不会太高？**
- 解决：经营可以自动化
- 解决：只需要定期进货和定价
- 解决：战斗是主要时间消耗

**问题4：商店经营和Thronefall经营的区别？**
- 解决：商店是销售商品，不是建造防御
- 解决：商店获得金币，不是直接防御
- 解决：金币用于购买装备，不是建造塔

## 为什么是第三梯队

1. **认知负荷偏高**：定价、进货、摆放需要频繁决策和策略思考
2. **操作频次高**：经营需要频繁操作，不符合"低操作频次"要素
3. **本能目标弱**："我的商店"情感驱动远弱于"我养大的宠物"或"我打造的装备"
4. **转化环节长**：经营 → 金币 → 装备 → 战斗，链条较长，金币作为中间环节削弱了直接成就感
5. **体验闭环弱**：战斗成果（商品）回到经营（销售）的循环不如直接使用来得有成就感
6. **但仍然可行**：有成功案例（Moonlighter、Recettear），商店经营本身是独立玩法，循环清晰
