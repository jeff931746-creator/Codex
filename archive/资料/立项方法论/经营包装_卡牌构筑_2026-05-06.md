# 卡牌构筑包装

## 核心定位

**底层经营**：收集卡牌 → 构筑卡组 → 战斗使用 → 获得新卡

**表层包装**：魔法卡牌、战术指令、技能组合、装备套牌

**认知负荷**：中到高（构筑需要策略思考）

## 四要素验证

| 要素 | 验证 |
|---|---|
| 时间分离 | ✓ 局外构筑，局内战斗 |
| 低操作频次 | ✗ 构筑需要频繁调整，战斗中使用卡牌也需要频繁操作 |
| 本能目标 | ✗ "我的卡组"情感驱动弱，不如"我养大的宠物"或"我打造的装备" |
| 成长/策略 | ✓✓ 卡组构筑有深度策略 |

**双约束验证**：
- **约束一（结构合法性）**：✓ 卡牌构筑本身是独立玩法（Slay the Spire可以不战斗只构筑）
- **约束二（体验闭环）**：⚠️ 构筑→战斗使用→获得新卡→优化构筑，循环成立，但战斗中使用卡牌可能与操控角色产生操作冲突

## 核心循环

```
局外构筑阶段
├─ 查看卡牌库
├─ 选择卡牌组成卡组
└─ 调整卡组策略
    ↓
进入战斗
    ↓
局内战斗阶段
├─ 操控角色防守
├─ 使用卡牌技能
└─ 卡牌提供增益/攻击
    ↓
战斗结束
├─ 获得新卡牌
├─ 解锁稀有卡
└─ 卡牌升级
    ↓
回到局外构筑
```

## 具体设计方案

### 方案A：魔法卡牌模式

**核心机制**：
- 收集魔法卡牌
- 构筑卡组（30-40张）
- 战斗中抽卡使用
- 卡牌提供技能/增益

**卡牌类型**：
- **攻击卡**：造成伤害
- **防御卡**：提供护盾
- **增益卡**：提升属性
- **召唤卡**：召唤生物

**构筑系统**：
- 卡组有张数限制
- 卡牌有费用限制
- 卡牌有稀有度
- 卡牌可以升级

**战斗应用**：
- 战斗中抽卡
- 消耗费用使用卡牌
- 卡牌效果立即生效
- 操控角色+卡牌技能协同

**长线驱动**：
- 收集所有卡牌
- 构筑多套卡组
- 发现强力组合

**参考对标**：
- **Slay the Spire（卡牌构筑+Roguelike）**：回合制战斗，构筑是核心玩法，但战斗节奏慢，不是操控角色
- **Monster Train（卡牌构筑+塔防）**：自动战斗+卡牌构筑，玩家不直接操控角色，只部署单位和使用卡牌
- **Griftlands（卡牌构筑+RPG）**：回合制战斗，卡牌构筑深度高，但不是实时操控

**双约束验证**：
- **约束一（结构合法性）**：✓ 卡牌构筑本身是独立玩法，Slay the Spire证明构筑本身有深度
- **约束二（体验闭环）**：⚠️ 构筑→战斗使用→获得新卡→优化构筑，循环成立，但**关键问题**：成功案例都是回合制或自动战斗，没有"卡牌构筑+实时操控角色"的成功案例

### 方案B：战术指令模式

**核心机制**：
- 收集战术指令卡
- 构筑指令组（20-30张）
- 战斗中使用指令
- 指令提供战术支援

**指令类型**：
- **火力支援**：空袭、炮击
- **战术支援**：烟雾弹、闪光弹
- **增援支援**：召唤援军
- **补给支援**：弹药、医疗

**构筑系统**：
- 指令组有张数限制
- 指令有冷却时间
- 指令有稀有度
- 指令可以升级

**战斗应用**：
- 战斗中选择指令使用
- 指令有冷却时间
- 指令效果范围攻击
- 操控角色+指令协同

**长线驱动**：
- 收集所有指令
- 构筑多套指令组
- 发现强力组合

**参考对标**：
- XCOM（战术指令）
- Into the Breach（战术策略）
- Advance Wars（战术指令）

### 方案C：技能组合模式

**核心机制**：
- 收集技能卡
- 构筑技能组（10-15个）
- 战斗中使用技能
- 技能有冷却和连携

**技能类型**：
- **主动技能**：需要手动释放
- **被动技能**：自动触发
- **连携技能**：多个技能组合
- **终极技能**：强力技能

**构筑系统**：
- 技能组有数量限制
- 技能有冷却时间
- 技能有连携关系
- 技能可以升级

**战斗应用**：
- 战斗中释放技能
- 技能有冷却时间
- 技能可以连携
- 操控角色+技能协同

**长线驱动**：
- 收集所有技能
- 发现连携组合
- 构筑多套技能组

**参考对标**：
- Hades（技能组合+Roguelike）
- Dead Cells（技能组合+动作）
- Risk of Rain（技能组合+射击）

## 潜在问题与解决方案

**问题1：卡牌构筑会不会太复杂？**
- 解决：卡牌数量不要太多（50-100张）
- 解决：卡组构筑有推荐模板
- 解决：卡牌效果简单清晰

**问题2：战斗中使用卡牌会不会打断操控？**
- 解决：方案A卡牌自动抽取，选择使用
- 解决：方案B指令是范围效果，不需要精确操作
- 解决：方案C技能是快捷键释放，不打断操控

**问题3：卡牌和操控的协同够强吗？**
- 解决：卡牌效果增强操控能力
- 解决：操控创造卡牌使用时机
- 解决：卡牌+操控产生连携

**问题4：构筑的认知负荷会不会太高？**
- 解决：有推荐卡组模板
- 解决：卡组可以保存多套
- 解决：构筑不是必须频繁调整

## 为什么是第三梯队

1. **认知负荷高**：构筑需要策略思考，不符合"低认知负荷"要素
2. **操作频次高**：构筑需要频繁调整，战斗中使用卡牌也需要频繁操作，不符合"低操作频次"要素
3. **本能目标弱**："我的卡组"情感驱动远弱于"我养大的宠物"或"我打造的装备"
4. **操作冲突风险**：战斗中使用卡牌可能与操控角色产生操作冲突，需要精心设计才能避免
5. **缺少成功案例**：所有卡牌构筑成功案例都是回合制或自动战斗，没有"卡牌构筑+实时操控角色"的验证
6. **但仍然可行**：卡牌构筑本身是独立玩法，有深度策略，如果能解决操作冲突问题，仍有可行性
