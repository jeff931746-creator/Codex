# 永远的蔚蓝星球·产品化结论

## 这一层的用途
这里的结论带适用前提与反例，用于产品化决策。本文件将《永远的蔚蓝星球》的竖屏塔防与Roguelike构筑特征转化为可落地的设计数值与机制判断，明确在何种用户层、变现形态与局内时长下这些判断生效，并指出跨品类时的失效边界，避免设计思路的滥用。

## 结论列表

**结论 1：非线性召唤费用驱动微决策**

If 局内召唤费用采用10-40金币的非线性波动机制，Then 玩家的单局留存与重玩意愿会提升20%[推测]，因为无脑点击被替换为时机与经济管理。

*适用前提*：单局时长3分钟[推测]的竖屏塔防，且局内需提供高频决策点防止挂机无聊。
*反例游戏*：《明日方舟》（策略塔防），其费用采用固定回复机制，决策重心在部署时机而非经济管理，非线性波动会破坏其策略节奏。
*太宽测试*：若替换为 回合制RPG 类型，结论不成立。

**结论 2：疲劳衰减强制社交合作**

If 单日20局后设置疲劳衰减且合作模式收益高于扫荡，Then 玩家参与合作模式的比例会提升50%[推测]，单人刷关的意愿大幅降低。

*适用前提*：依赖社交裂变与在线率的多平台游戏，且合作模式能提供稳定的资源产出。
*反例游戏*：《原神》（开放世界），其树脂系统控制单日总体力，但不强制要求合作模式，单人也能消耗完所有体力。
*太宽测试*：若替换为 开放世界 类型，结论不成立。

**结论 3：互斥天赋双分支复用职业**

If 天赋树采用互斥的双分支设计（如格挡无敌 vs 暴击换防御），Then 同一职业的卡牌复用率会提升40%[推测]，玩家需要培养两张同职业卡牌以体验不同流派。

*适用前提*：拥有7大职业[推测]且需控制卡牌总量的塔防游戏，通过机制互斥而非数值互斥制造差异。
*反例游戏*：《英雄联盟》（MOBA），符文系统在赛前可自由切换，不需要培养两个同名英雄，互斥设计只会增加操作负担。
*太宽测试*：若替换为 MOBA 类型，结论不成立。

**结论 4：皮肤绑定数值的礼包转化**

If 68元核心礼包同时提供强力武器皮肤与3480钻，Then 礼包购买率会高于纯数值礼包30%[推测]，外观与实力的双重获得感降低了付费负罪感。

*适用前提*：IAP为主的多平台游戏，且玩家对角色外观有展示需求。
*反例游戏*：《阴阳师》（卡牌养成），其皮肤通常不含战斗数值，付费点集中在抽卡与属性养成，皮肤绑定数值会破坏公平性。
*太宽测试*：若替换为 纯PvP卡牌 类型，结论不成立。

**结论 5：3件套图腾强制路线声明**

If 图腾共鸣需要3个同属性且强制绑定特定职业才激活质变羁绊，Then 玩家在局外的养成决策会从平均分配转向单点突破，流派成型速度加快。

*适用前提*：需要明确构筑方向且7职业克制关系复杂的塔防游戏。
*反例游戏*：《梦幻西游》（MMORPG），其套装效果通常提供通用数值加成，不强制绑定特定门派，3件套强制绑定会限制搭配自由度。
*太宽测试*：若替换为 MMORPG 类型，结论不成立。

**结论 6：2合1对冲三重随机**

If 合成系统产出的4级单位攻击力达到1级的4倍且可能触发变异英雄，Then 玩家就会用确定的2合1去对冲召唤的随机性，单局挫败感降低。

*适用前提*：局内存在三重随机（随机召唤/随机技能/随机合成）的高方差塔防游戏。
*反例游戏*：《云顶之奕》（自走棋），其合成是3合1的确定性质变，不存在用2合1对冲随机性的需求。
*太宽测试*：若替换为 自走棋 类型，结论不成立。

## 局限性
1. **重度SLG品类**：本结论依赖3分钟短局与高频重开，对于单局长达数月、强调持续对抗的SLG不适用。
2. **纯竞技对战品类**：本结论的付费与养成逻辑建立在PvE偏科检定与数值验证上，对于强调绝对公平竞技的对战游戏不适用。
3. **强叙事驱动品类**：疲劳衰减与合作强制会打断叙事沉浸，对于以剧情演出为核心的产品形态不适用。
