# 《永远的蔚蓝星球》总档

## 基本信息

- 游戏名：永远的蔚蓝星球
- 类型标签：多平台竖屏塔防 / Roguelike合成 / 职业克制构筑
- 平台：Android、iOS、PC（Windows）
- 日期：2024-05-20
- 资料来源：网络公开资料、玩家攻略、游戏介绍与评价分析

## 一句话定义

一款用“三重随机召唤-合成-强化”把塔防决策压进3分钟短局的竖屏塔防：前线用7职业英雄执行输出，局内靠2合1进阶与三选一强化声明路线，局外靠图腾共鸣、天赋树和装备洗练把单局随机性沉淀为流派底盘。

## 核心结构

- 核心循环：进入关卡 → 花费波动金币召唤英雄 → 2合1进阶与三选一强化 → 通过波次与合作Boss检定 → 结算资源与疲劳衰减 → 局外英雄/图腾/装备养成
- 主要资源：金币（局内召唤）、钻石（局外抽卡/体力）、招募券、锻造书、炼器石、英雄碎片
- 主要系统：英雄执行层、三重随机与合成层、英雄成长底盘、图腾与天赋构筑、资源循环、关卡节奏、商店与任务奖励、Boss与合作检定
- 主要节奏节点：开局召唤布阵、波次间隙合成与强化、3分钟单局结算、20局疲劳衰减线、合作Boss挑战窗口
- 主要检定节点：职业克制高压波次、合作模式Boss、高波次持续生存检定

## 系统总览

每个系统必须同时写：负责什么 / 不负责什么 / 和谁联动最紧密。

| 系统 | 层级 | 职责边界（是什么 + 不是什么） | 联动最紧密 |
| --- | --- | --- | --- |
| 英雄执行层 | 执行层 | 决定单局里7职业谁在打、怎么打；不负责路线声明，也不负责局外数值成长 | 三重随机与合成层、图腾与天赋构筑 |
| 三重随机与合成层 | 路线层 | 决定这局通过什么合成路径和随机Buff构筑打法；不负责底盘数值，只负责把局内随机件组合成更强结构 | 英雄执行层、图腾与天赋构筑 |
| 英雄成长底盘 | 底盘成长层 | 决定英雄等级、装备、披风属性沉淀；不负责当局随机路线选择，只负责让某条路线有足够数值兑现 | 资源循环、商店与任务奖励 |
| 图腾与天赋构筑 | 路线层 / 构筑层 | 决定图腾共鸣方向与天赋双分支选择；不负责具体单次攻击数值，只负责提供流派锚点和机制质变 | 英雄执行层、Boss与合作检定 |
| 资源循环 | 经济层 | 决定钻石、招募券、炼器石等资源怎么流转；不负责具体战斗结果，只负责把时间和付费转成养成进度 | 商店与任务奖励、英雄成长底盘 |
| 关卡节奏 | 节奏框架层 | 决定3分钟单局与疲劳衰减怎么推进；不负责给奖励种类，只负责规定压力和收益在什么时候出现 | Boss与合作检定、商店与任务奖励 |
| 商店与任务奖励 | 编辑层 / 奖励层 | 决定玩家在哪些节点买到、领到关键材料；不负责战斗执行，只负责修正路线和补短板 | 资源循环、英雄成长底盘 |
| Boss与合作检定 | 检定层 | 决定哪些偏科阵容会被卡住；不负责推进主线，只负责验证当前构筑是否过关 | 图腾与天赋构筑、关卡节奏 |

## 本游戏最值得拆的专题

- “三重随机”为什么能在单局内形成足够明确的路线，而不是让玩家觉得完全看脸？
- 2合1合成与非线性召唤费用，是怎么把“无脑抽卡”变成“经济管理”的？
- 图腾共鸣的3件套与天赋双分支，为什么是构筑的锚点而不是单纯的数值叠加？
- 合作Boss与疲劳衰减，是怎么把单机推图变成资源规划检定的？

## 总判断

- 这游戏靠什么成立：用“3分钟短局 + 局内2合1合成 + 局外图腾天赋”把塔防的复杂度拆散到多个低门槛决策点，玩家每次合成和强化都能立刻看到阵容质变
- 最大机会点：英雄执行、图腾共鸣和天赋树共享同一套7职业克制语言，所有资源最终都在为“哪条职业路线更完整”服务
- 最大风险点：如果三重随机的方差过大，导致玩家无法通过局外构筑稳定对冲局内霉运，游戏就会退化成纯看脸的放置塔防，策略感会大幅削弱
