# 《主宰世界》总档

## 基本信息

- 游戏名：主宰世界
- 类型标签：多平台修仙 RPG / 神通构筑养成
- 平台：Android、iOS、Windows
- 日期：2024-05-20
- 资料来源：网络公开攻略、玩家评价与测试数据、职业详解与新手十问

## 一句话定义

一款用“职业底座 + 神通构筑 + 修仙底盘”把修仙 RPG 的战力拆成多个维度的游戏：前线用职业和神通执行输出与构筑，局内靠神通替换声明路线，局外靠元婴、真灵和异兽把失败转成下一阶段更强的底盘。

## 核心结构

- 核心循环：选择职业 → 搭配主动与被动神通 → 挑战副本与任务 → 获取资源与神通升级 → 提升元婴、真灵与异兽 → 通过秒伤检定与高难门槛 → 结算奖励 → 回到局外继续构筑
- 主要资源：修为/经验类基础资源、元婴突破丹、真灵培养材料、神通升级残卷、异兽培养道具、钻石/虚拟货币
- 主要系统：职业执行层、神通构筑层、修仙底盘层、资源循环、检定与挑战、商店与奖励
- 主要节奏节点：职业选择期、神通搭配与测试期、元婴突破期、高难副本攻坚期
- 主要检定节点：秒伤数值门槛（如88亿/秒）、元婴前后的职业强度反转（万剑/御天）、高难副本机制

## 系统总览

每个系统必须同时写：负责什么 / 不负责什么 / 和谁联动最紧密。

| 系统 | 层级 | 职责边界（是什么 + 不是什么） | 联动最紧密 |
| --- | --- | --- | --- |
| 职业执行层 | 执行层 | 决定单局里“谁在打、基础动作是什么”；不负责构筑方向，也不负责长期数值兑现 | 神通构筑层、修仙底盘层 |
| 神通构筑层 | 路线层 | 决定这局要把职业往哪个方向强化（如爆发或持续）；不负责职业基础动作，只负责把执行层组合成更强结构 | 职业执行层、检定与挑战 |
| 修仙底盘层 | 底盘成长层 | 决定元婴、真灵、异兽的数值沉淀与质变；不负责当局神通怎么放，只负责让某条路线有足够数值兑现 | 资源循环、商店与奖励 |
| 资源循环 | 经济层 | 决定修为、突破丹、神通残卷怎么流转；不负责具体战斗结果，只负责把时间和付费转成养成进度 | 商店与奖励、修仙底盘层 |
| 检定与挑战 | 检定层 | 决定哪些偏科构筑或低数值会被卡住；不负责推进主线，只负责验证当前构筑是否过关 | 神通构筑层、修仙底盘层 |
| 商店与奖励 | 编辑层 / 奖励层 | 决定玩家在哪些节点买到、领到关键材料；不负责战斗执行，只负责修正路线和补短板 | 资源循环、修仙底盘层 |

## 本游戏最值得拆的专题

- 神通替换为什么能直接决定秒伤是从上限跌到下限，而不是只产生微小波动？
- 元婴系统为什么会造成职业强度的反转（如万剑元婴前弱后强），而不是单纯的数值放大？
- 秒伤测试为什么是这游戏最核心的检定器，而不是通关时间？
- 主动神通和被动神通的绑定关系，是怎么把“自由搭配”变成“路线锁定”的？

## 总判断

- 这游戏靠什么成立：用“职业底座 + 神通构筑 + 修仙底盘”把修仙 RPG 的复杂度拆散，神通负责路线声明，元婴/真灵负责数值兑现，玩家每次换一个神通都能立刻看到秒伤的涨跌
- 最大机会点：神通替换带来的秒伤剧烈波动，让构筑调优成为极具反馈的玩法核心
- 最大风险点：如果元婴等底盘系统的数值压制过强，神通构筑的路线感就会被纯数值堆叠掩盖，导致前期体验失衡
