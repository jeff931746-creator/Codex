# 百炼英雄·产品化结论

## 这一层的用途
这里的结论带适用前提与反例，用于产品化决策。本文件将《百炼英雄》的放置RPG与英雄收集特征转化为可落地的设计数值与机制判断，明确在何种用户层、变现形态与局内时长下这些判断生效，并指出跨品类时的失效边界，避免设计思路的滥用。

## 结论列表

**结论 1：机制型技能作为构筑拐点**

If 放置RPG中存在“100%减伤/无敌”等机制型技能（如圣骑士/天使），且玩家无法靠操作规避伤害，Then 玩家的养成目标会从“堆防御数值”转向“凑技能机制”，构筑深度与付费意愿提升。

*适用前提*：自动战斗的放置RPG，关卡存在高压生存检定。
*反例游戏*：《黑神话：悟空》（动作游戏），玩家靠操作翻滚闪避，无敌帧属于操作反馈，不需要靠养成凑机制型生存。
*太宽测试*：若替换为 动作游戏 类型，结论不成立。

**结论 2：等级发力点驱动阵容迭代**

If 关卡难度曲线与英雄的等级发力点（如45级）绑定，Then 玩家会在该节点产生强烈的阵容迭代需求，前期英雄自然退环境，长线留存增加。

*适用前提*：具有长线养成周期和多英雄池的RPG，且前期易获取英雄数值成长低于后期稀有英雄。
*反例游戏*：《云顶之弈》（自走棋），单局内英雄不存在跨局等级发力点，每局从零开始。
*太宽测试*：若替换为 自走棋 类型，结论不成立。

**结论 3：分模式检定驱动多维养成**

If 后期节奏通过多模式分化（试炼塔检定爆发、矿坑检定续航）来加压，Then 单一阵容挂机的策略失效，玩家必须进行多维养成。

*适用前提*：需要维持多英雄养成生态，且玩家每日有足够时长应对多模式挑战的RPG。
*反例游戏*：《旅行青蛙》（放置休闲），无多模式检定，也不需要多维养成，结论不适用。
*太宽测试*：若替换为 休闲放置 类型，结论不成立。

**结论 4：觉醒转化冗余抽卡负反馈**

If 觉醒系统消耗重复掉落的英雄碎片并提供质变属性（如王昭君觉醒加30%攻击），Then 玩家对抽卡歪掉的抵触会降低，冗余产出转化为长线养料。

*适用前提*：IAP驱动的高稀有度抽卡游戏，玩家频繁面临抽到重复低价值卡的情况。
*反例游戏*：《阴阳师》（回合RPG早期），未推出返魂/转换机制前，重复SSR负反馈极强，无法转化为长线养料。
*太宽测试*：若替换为 无抽卡系统 的类型，结论不成立。

**结论 5：乘区绑定提升辅助价值**

If 辅助位的增益效果（如王昭君攻速+攻击）与核心位的输出系数是乘法关系，Then 辅助位的战略价值会远超同战力的输出位，构筑深度增加。

*适用前提*：拥有明确核心输出+辅助分工，且伤害计算公式支持乘区叠加的RPG。
*反例游戏*：《暗黑破坏神2》（ARPG），辅助光环效果多为基础面板加成，最终伤害更依赖装备底材和技能等级的独立乘区，辅助绑定效应较弱。
*太宽测试*：若替换为 纯单人ARPG 类型，结论不成立。

## 局限性
1. **强操作竞技品类**：本结论依赖自动战斗与机制型技能的数值兑现，对于靠玩家操作规避伤害的MOBA或动作游戏不适用。
2. **单局重置品类**：本结论的长线养成与阵容迭代逻辑建立在跨局资源累积上，对于每局从零开始的自走棋或BR游戏不适用。
3. **纯休闲放置品类**：分模式检定与多维养成会增加理解压力，对于以低负担收菜为核心的休闲放置不适用。
