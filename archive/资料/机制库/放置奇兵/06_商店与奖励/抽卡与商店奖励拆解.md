# 放置奇兵·抽卡与商店奖励拆解

## 这个专题要回答什么

《放置奇兵》的抽卡与商店为什么不是附属系统，而是直接参与阵营构筑成型的修正器？

## 总体判断

这套奖励系统的作用不是“发福利”，而是把玩家的阵营缺口转成可被修补的节点。抽卡解决核心英雄从哪来，先知祭坛兑换解决定向补缺，付费则解决时间不够的问题。

## 怎么判断某个东西属于奖励层

- 它如果在“缺什么就补什么”这个逻辑里工作，就属于奖励层
- 它如果主要决定英雄获取速度，而不是直接决定关卡胜负，就属于奖励层
- 它如果可以被玩家按需兑换或抽取，就更像奖励层而不是战斗层

## 奖励节点是怎么工作的

### 1. 抽卡负责核心件的来源

- 触发条件：玩家消耗钻石或抽卡券进行召唤
- 具体结果：随机获得各阵营英雄，产出核心输出或关键辅助
- 为什么有效：抽卡是放置游戏制造惊喜和付费冲动的核心装置

### 2. 定向兑换负责修正构筑缺口

- 触发条件：玩家在先知祭坛使用先知球兑换指定阵营英雄
- 具体结果：定向获取当前构筑所需的关键拼图
- 为什么有效：纯随机会让构筑无法成型，定向兑换是给路线兜底

### 3. 付费礼包负责加速时间流转

- 触发条件：玩家购买成长礼包、月卡或特惠道具
- 具体结果：获得大量资源、抽卡次数或直接高星英雄
- 为什么有效：放置游戏的付费本质是买时间，礼包直接兑现这一预期

## 奖励层和经济层的分工

- 奖励层决定“现在给你什么英雄和材料”
- 经济层决定“这些材料怎么被花掉、时间怎么转化”
- 只给抽卡不给出路，会让阵容散乱；只给出路不给定向获取，会让构筑遥遥无期

## 这个模块不负责什么

- 不负责关卡卡点本身，那是关卡与放置节奏的职责
- 不负责英雄自动战斗机制，那是执行层的职责
- 不负责阵营羁绊加成本身，那是阵容构筑层的职责

## 可迁移结论

- If 抽卡系统在随机召唤外，提供先知球定向兑换指定阵营英雄，Then 玩家就能在随机惊喜与稳定构筑之间找到平衡，避免纯随机构筑流产
- If 付费礼包的核心卖点是“立即获得12小时挂机收益”，Then 商业化就是在卖时间而非卖数值特权，不会破坏免费玩家的长线数值上限
