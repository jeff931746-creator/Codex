# 放置奇兵·底层设计公式

## 这一层的用途
这里的结论脱离本游戏语境仍应成立。它们揭示了“放置+策略RPG”品类在底层结构上的因果关系——当游戏试图将操作负担转化为战前决策、并让挂机时长与构筑深度绑定时，必须遵守的机制传导逻辑。这些命题不依赖具体的英雄题材或手游平台，而是该设计范式成立的基石。

## 结论列表

**结论 1：时长货币律**

If 基础成长资源的产出唯一绑定挂机时长且设有离线收益上限，Then 付费行为就会自然转化为“购买时间”，且日活回流节点会被收益上限严格锚定。

*反面测试*：把 If 取反后（基础资源产出不绑定时长或无离线上限），Then 是否仍成立？不成立，若无上限则日活节点消失，若不绑定时长则付费买时间逻辑不成立。

**结论 2：冗余升星律**

If 局外成长底盘必须消耗同名卡牌作为升星核心材料，Then 抽卡系统的冗余产出就会从负反馈源转化为长线养成驱动力。

*反面测试*：把 If 取反后（升星不需要同名卡牌，只需通用材料），Then 是否仍成立？不成立，冗余卡牌失去消耗出口，重新变为负反馈。

**结论 3：阵营锁构筑律**

If 阵营羁绊加成随同阵营单位数量指数级上升，Then 构筑决策就会从“单卡强度比较”跃迁为“路线排他声明”。

*反面测试*：把 If 取反后（羁绊加成为线性或无阵营要求），Then 是否仍成立？不成立，玩家会继续选择单卡最强的混搭，不会产生路线排他性。

**结论 4：自动执行律**

If 战斗执行层完全自动且不可干预，Then 策略重心就会完全转移到战前编队与局外养成底盘上。

*反面测试*：把 If 取反后（战斗可手动操作或干预），Then 是否仍成立？不成立，策略重心会部分或全部转移到局内即时操作上。

**结论 5：定向兜底律**

If 获取系统在纯随机抽取外提供定向兑换指定角色的途径，Then 玩家的构筑路线就能避免因随机性而无法兑现。

*反面测试*：把 If 取反后（只有纯随机抽取，无定向途径），Then 是否仍成立？不成立，玩家路线极易因缺少关键件而崩溃。

## 边界
这套公式在缺乏局外长线养成或战斗过程强操作干预的品类中不适用；在重即时反馈轻策略构筑的动作类或射击类游戏中也不适用。当游戏的核心乐趣不再依赖于“战前构筑的深度与局外时长的沉淀互相反哺”时，上述因果链便会断裂。
