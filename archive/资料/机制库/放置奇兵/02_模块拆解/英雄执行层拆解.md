# 放置奇兵·英雄执行层拆解

## 这个专题要回答什么

《放置奇兵》的英雄为什么是执行层，而不是路线层？

## 总体判断

英雄执行层负责把阵容构筑的数值和协同效果兑现成自动战斗结果。它决定“这队能不能自动打赢”，但不决定“这局要养谁、走哪条阵营路线”，路线声明来自阵容构筑和成长底盘。

## 英雄执行层到底在做什么

### 1. 英雄上场后按预设逻辑自动战斗

- 触发条件：玩家编队进入PvE推图或PvP对战后
- 具体结果：英雄按速度属性决定出手顺序，按技能CD自动普攻和释放技能（资料未覆盖具体能量规则，需后续补充）
- 为什么有效：放置游戏的核心是降低操作负担，执行层必须全自动且结果可预期

### 2. 职业分工决定英雄在战斗中的行为模式

- 触发条件：英雄的职业标签（如刺客、牧师）确定
- 具体结果：刺客AI优先攻击后排血量最低单位，牧师AI优先治疗己方低血量单位
- 识别边界：只要英雄的核心行为是“对敌军造成威胁”，它就是输出/承伤；只要核心行为是“改变友军状态或敌军debuff”，它就是辅助

### 3. 技能触发改写战局形态

- 触发条件：英雄能量满100点或血量低于30%触发被动
- 具体结果：释放大招或被动技能，产生爆发伤害、群体控制或关键治疗
- 为什么有效：让自动战斗不是简单的互砍，而是有节奏的技能爆发期

### 4. 高阶英雄的关键不在名字，而在“能否承接更多乘区”

- 触发条件：英雄升星至9星/10星、装备强化后
- 具体结果：技能系数或触发频率提升，能更好地利用阵营羁绊加成
- 为什么有效：让强度来源从“抽到稀有卡”转为“围绕稀有卡继续加乘”

## 这个层级的边界在哪里

- 它不负责声明“这队要走光明阵营还是暗影阵营”，那是阵容构筑层的职责
- 它不负责长期解锁新上限，那是英雄成长底盘的职责
- 它不负责产出资源，那是放置资源循环的职责

## 典型识别标准

如果一个元素满足下面任意两条，就更接近执行层：
- 上场后立刻参与自动战斗结算
- 行为主要由职业AI和速度属性预设逻辑决定，玩家不可局内干预
- 其强度主要体现为“当前关卡能不能自动推过”

如果它主要满足下面这些条件，就不是执行层：
- 需要靠局外资源长期养成
- 本身不直接参与战斗结算（如羁绊光环）
- 更像路线选择或奖励修正

## 这个模块不负责什么

- 不负责阵容搭配方向，那是阵容构筑层的职责
- 不负责英雄升星与数值上限，那是英雄成长底盘的职责
- 不负责挂机收益产出，那是放置资源循环的职责

## 可迁移结论

- If 放置RPG的战斗执行层将英雄行为完全托管于速度属性与技能CD自动循环，Then 玩家的策略重心就会从局内微操转移到局外的速度调整与战前能量节点规划上
- If 放置RPG将英雄的职业标签（如刺客/牧师）与其AI寻敌逻辑（刺客切后/牧师保排）强制绑定，Then 玩家在战前编队时就能仅凭职业标签预判该英雄的承伤/输出站位，从而降低阵容功能识别成本
