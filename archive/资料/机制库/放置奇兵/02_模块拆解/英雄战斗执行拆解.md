# 放置奇兵·英雄战斗执行拆解

## 这个专题要回答什么

英雄在《放置奇兵》里为什么是纯粹的执行层，而不是构筑层？

## 总体判断

英雄战斗执行层负责把阵容搭配和养成数值兑现成战斗结算画面。它决定"这队能不能打赢"，但不决定"这局要养哪个阵营"，路线声明来自阵营羁绊和升星养成系统。战斗全自动且不可干预的设定，刻意剥夺了玩家的局内操作权，把策略重心全部压到了战前布阵与局外养成上。

## 怎么判断某个东西属于英雄战斗执行

- 只要它的主要作用是"当前一场战斗里直接造成伤害、承受伤害或提供即时增益/减益"，它就属于执行层
- 如果它的主要作用是"决定后续资源往哪投"或"长期抬高数值上限"，它就不属于执行层

## 英雄战斗执行由哪些部分组成

### 1. 自动战斗与技能循环

- 触发条件：玩家进入PVE推图或PVP竞技场，点击开战
- 具体结果：英雄按速度条行动，自动释放普攻与技能，玩家无法手动选择目标或释放时机
- 设计原因：将操作负担降至零，落实"护肝"标签，让策略胜负在战前决定而非战中

### 2. 职业分工定位

- 触发条件：编队时放入战士、法师、游侠、牧师、刺客等不同职业英雄
- 具体结果：战士承伤，游侠/刺客切后，法师群伤，牧师治疗，各司其职
- 设计原因：提供最基础的阵容搭配逻辑，如果没有职业分工，阵容构筑就退化成纯数值比拼

### 3. 站位与目标选择逻辑

- 触发条件：战前布阵，将英雄放置在前排或后排的1-6号位
- 具体结果：前排优先承受单体伤害，刺客/游侠优先攻击对位及相邻后排脆皮
- 设计原因：用简单的空间规则制造"肉盾保护输出"的战术面

## 这个层级不负责什么

- 不负责英雄的基础数值和升星质变，那是英雄养成底盘的职责
- 不负责阵容羁绊加成，那是阵营职业构筑的职责
- 不负责产出战斗外的资源，那是放置资源循环的职责

## 可迁移结论

- If 一款RPG将战斗过程设计为全自动且不可干预，且战前布阵决定目标选择优先级，Then 玩家的策略重心会完全前移至战前阵容搭配与局外养成，战斗本身退化为数值验证动画
- If 职业分工的目标选择逻辑与站位（前后排）脱钩，Then 前后排的空间布阵会退化为纯数值卡槽，阵容构筑的深度会大幅降低
