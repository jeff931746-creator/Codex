# 三国：冰河时代·联盟协作与构筑拆解

## 这个专题要回答什么

《三国：冰河时代》的联盟协作为什么是路线声明，而不是单纯的互助奖励？

## 总体判断

联盟系统不只是发福利的公会，而是决定玩家和服务器局势走向的路线层。联军科技的互斥选择、万人跑商的军费转化、雄据一方的战略，都在声明这个联盟要走“军事碾压”还是“经济控制”的路线。联盟路线一旦确定，个体玩家的养成和资源流向必须跟进。

## 怎么判断某个东西属于联盟协作与构筑

- 只要它会改变联盟后续的资源分配和战略方向（如选择军事科技会挤占资源科技进度），它就属于路线层
- 如果它只改变个体玩家的武将属性（如升星），它属于底盘层
- 如果它只是发邮件送道具，它属于奖励层

## 核心结构

### 1. 联军科技声明联盟的发展方向

- 触发条件：联盟成员捐献资源研发联军科技
- 具体结果：选择军事类科技（如“全军突击”提升全军攻击）会挤占资源类科技（如“富国强兵”提升采集速度）的研发进度
- 设计原因：科技树互斥让联盟从“人群聚集”变成“路线共同体”，选错科技会导致后期竞争落后

### 2. 万人跑商把经济行为变成战略博弈

- 触发条件：S6赛季开启“富甲天下”剧本，触发跑商玩法
- 具体结果：联盟需要规划商路、保护商队并抢夺对方资源，跑商资金直接转化为战争军费
- 设计原因：把枯燥的资源采集转化为宏观战略博弈，联盟不仅要会打仗，还要会赚钱

### 3. 协作任务强制个体跟进联盟节奏

- 触发条件：联盟发布协作任务（如集体攻打某城、集体捐献某资源）
- 具体结果：玩家必须调整个人日常优先级来完成任务，否则拿不到联盟奖励
- 设计原因：用集体目标纠正个体散漫，确保联盟行动步调一致

### 4. 雄据一方形成势力版图与防线

- 触发条件：联盟占领关键城池与关卡
- 具体结果：形成割据版图，获得区域加成，同时面临防守压力
- 设计原因：让领土不只是坐标，而是具有战略纵深和防御优势的实体

## 这个模块不负责什么

- 不负责武将的具体战斗伤害和升星觉醒，那是底盘层和战斗执行层的职责
- 不负责赛季何时重置及跨服匹配算法，那是节奏框架层的职责
- 不负责元宝的直接产出，那是经济循环的职责

## 可迁移结论

- If 联盟科技树的选择存在互斥研发进度（如军事科技与经济科技抢夺捐献资源），且会显著改变联盟整体的资源获取与战斗属性，Then 联盟就不再只是社交容器，而是路线声明器，加入不同联盟等于选择不同游戏体验
- If 跑商类玩法的资金产出直接挂钩战争军费，Then 经济行为就会升格为战略博弈，而非纯粹的休闲副玩法
