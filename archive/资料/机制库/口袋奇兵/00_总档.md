# 《口袋奇兵》总档

## 基本信息

- 游戏名：口袋奇兵
- 类型标签：移动端 / 合成策略 / 轻量化SLG
- 平台：iOS / Android
- 日期：2026-04-20
- 资料来源：百度百科、TapTap攻略站、应用商店页面公开信息

## 一句话定义

一款用"即时合成"把SLG的建造和造兵压缩成极短操作链的轻量化策略手游：前线用海陆空三军和英雄执行输出，局内靠合成声明兵种路线，局外靠科技树、英雄养成和联盟协作把单点优势转成可持续扩张的底盘。

## 核心结构

- 核心循环：生产单位/建筑 → 拖拽合成升级 → 消耗资源解锁科技/招募英雄 → 兵种与阵容成型 → 通过PVE关卡与PVP争霸检定 → 获取资源与领地扩张 → 回到局外继续养成
- 主要资源：金币、石油、电力（基础建设与造兵资源）、钻石（高级加速与招募）、英雄碎片/招募券、加速道具
- 主要系统：兵种执行层、合成升级层、英雄与阵容路线层、科技与底盘成长层、基地与资源循环、战役与争霸节奏、PVP与高难检定
- 主要节奏节点：基地建筑合成节点、PVE野怪/战锤刷新节点、英雄招募与升星节点、跨服争霸与王座激战开启节点
- 主要检定节点：荒原野怪高难关卡、战锤机器人、王座激战、跨服争霸

## 系统总览

每个系统必须同时写：负责什么 / 不负责什么 / 和谁联动最紧密。

| 系统 | 层级 | 职责边界（是什么 + 不是什么） | 联动最紧密 |
| --- | --- | --- | --- |
| 兵种执行层 | 执行层 | 决定战斗里"谁在打、打谁、克制谁"；不负责生产速度，也不负责英雄技能加成 | 合成升级层、英雄与阵容路线层 |
| 合成升级层 | 路线层 | 决定这局/这个阶段要把哪些兵种和建筑合到高等级；不负责英雄数值，只负责把低级单位快速转化为高战力结构 | 兵种执行层、基地与资源循环 |
| 英雄与阵容路线层 | 路线层 / 构筑层 | 决定阵容走陆军、海军还是空军路线，以及输出/辅助分工；不负责具体造兵速度，只负责给特定兵种提供乘区加成 | 兵种执行层、科技与底盘成长层 |
| 科技与底盘成长层 | 底盘成长层 | 决定兵种科技树的永久解锁与属性沉淀；不负责单次合成操作，只负责让同等级单位拥有更高基础数值 | 基地与资源循环、英雄与阵容路线层 |
| 基地与资源循环 | 经济层 | 决定金币、石油、电力和钻石怎么产出与消耗；不负责战斗胜负，只负责把时间和付费转成造兵与养成速度 | 合成升级层、科技与底盘成长层 |
| 战役与争霸节奏 | 节奏框架层 | 决定PVE关卡推进和PVP争霸的时间压力；不负责给具体兵种，只负责规定资源掠夺和对抗在什么时候发生 | PVP与高难检定、基地与资源循环 |
| PVP与高难检定 | 检定层 | 决定单一兵种极限养成会被克制方击穿；不负责日常资源产出，只负责验证阵容深度和三军发展均衡度 | 英雄与阵容路线层、战役与争霸节奏 |

## 本游戏最值得拆的专题

- "即时合成"为什么能把SLG的漫长等待变成路线声明，而不是只省掉了点击升级的动画？
- 英雄与兵种的强绑定为什么会让阵容从"全都要"变成"必须选一条路走到底"？
- 三军克制循环为什么在PVP里是偏科检定器，而在PVE里却只是数值墙？
- 基地建筑的合成升级为什么直接改写了资源产出公式，而不是只改变外观？

## 总判断

- 这游戏靠什么成立：用"拖拽合成"把传统SLG的重度建造和造兵等待压缩成即时反馈，让玩家在极短时间内看到战力跃迁，同时保留了科技树和三军克制的策略深度
- 最大机会点：合成、兵种、英雄和科技共享同一套海陆空分类语言，所有资源最终都在为"哪条兵种线更完整"服务
- 最大风险点：如果三军克制在PVP中无法有效击穿单兵种极限养成，游戏就会退化成只看数值的合成放置，路线感和策略感都会变弱
