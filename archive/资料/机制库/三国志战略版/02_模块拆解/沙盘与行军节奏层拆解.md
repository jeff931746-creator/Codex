# 三国志战略版·沙盘与行军节奏层拆解

## 这个专题要回答什么

《三国志战略版》的沙盘行军为什么能把"自动战斗"变成"时间与空间的战术协同"？

## 总体判断

沙盘与行军系统是这游戏把战斗从"比数值"变成"比战术"的核心。自由行军和地形阻隔制造了空间纵深，行军时间和压秒合击制造了时间纵深，攻城分工制造了组织纵深。这三层纵深让战前构筑的数值不再是唯一解。

## 怎么判断某个东西属于沙盘与行军节奏层

- 它如果主要控制部队在什么时间到达什么位置，就属于沙盘节奏层
- 它如果利用地形或距离制造战术优势，就属于沙盘节奏层
- 它如果只改变伤害数值而不改变空间位置，就不属于沙盘节奏层

## 核心结构

### 1. 自由行军与地形阻隔制造空间纵深

- 触发条件：玩家点击沙盘目标地发起行军
- 具体结果：部队沿路径行军，遇山脉河流必须绕行，关卡未攻破无法穿越
- 设计原因：让地图不只是背景，关口和河道成为天然防线，防守方可以依托地形以少挡多

### 2. 行军时间与压秒合击制造时间纵深

- 触发条件：多支部队向同一目标行军，距离不同
- 具体结果：玩家需精确计算到达时间，使主力、器械、补给在1-5秒内先后抵达
- 设计原因：让"协同"成为可量化的战术，压秒合击能集中火力突破防线，行军脱节会被逐个击破

### 3. 攻城分工强制同盟协作

- 触发条件：攻击玩家城池或NPC城池
- 具体结果：主力部队消灭守军，器械部队拆除耐久，主力无法高效拆耐久，器械无法打赢守军
- 设计原因：一个人无法单挑城池，必须有人出主力清人，有人出器械拆城，同盟组织度直接决定攻城效率

### 4. 驻守与藏兵制造防御博弈

- 触发条件：敌军即将抵达我方城池
- 具体结果：玩家可将部队驻守在地块上迎击，或将部队藏入城内待敌军接触时再突然拉出
- 设计原因：让攻防不只是数值对撞，防守方可以通过藏兵制造信息差，诱导敌军深入后反制

## 这个模块不负责什么

- 不负责武将战法怎么触发和伤害怎么算，那是战斗与兵种执行层的职责
- 不负责阵容怎么搭配，那是战法与阵容构筑层的职责
- 不负责行军消耗的粮食和征兵资源怎么来，那是资源与征兵循环的职责

## 可迁移结论

- If 攻城伤害结算强制拆分为"消灭守军"与"拆除耐久"两个互斥阶段，且器械兵种对战斗兵种极弱，Then 同盟内会出现专门的攻城职能分工，而非全员只出最强战斗队
- If 沙盘行军时间精确到秒级且山脉/关卡等地形强制阻隔路线，Then 战术重心就会从"怎么配将"延伸到"怎么卡位和压秒"，空间和时间成为可操作的战术变量
