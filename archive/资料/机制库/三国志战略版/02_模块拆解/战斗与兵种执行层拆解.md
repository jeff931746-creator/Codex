# 三国志战略版·战斗与兵种执行层拆解

## 这个专题要回答什么

《三国志战略版》的战斗为什么是"战前决策，战后看报"，而不是"战中操作"？

## 总体判断

战斗执行层把操作门槛降到零，把策略重心完全压在战前构筑和沙盘行军时机上。战力等于兵量，兵种克制和战法触发全自动运行，玩家看到的是结果，决策的是前提。

## 怎么判断某个东西属于战斗执行层

- 它如果在交战接触后自动发生，不需要玩家手动干预，就属于战斗执行层
- 它如果决定伤害倍率和战损比例，但不决定武将怎么配，就属于战斗执行层
- 它如果需要玩家在大地图上走位或卡时间点，就不属于战斗执行层，而是沙盘节奏层

## 核心结构

### 1. 兵量即战力，自动结算战损

- 触发条件：两支部队在沙盘上接触
- 具体结果：战斗自动进行，战力直接等于兵量，战报显示双方战损和剩余兵量
- 设计原因：剥离操作负担，把SLG的核心焦虑从"操作失误"转移到"兵力损耗与征兵资源"上

### 2. 兵种克制循环决定基础倍率

- 触发条件：骑兵碰盾兵、盾兵碰弓兵、弓兵碰枪兵、枪兵碰骑兵等具体克制组合交战
- 具体结果：克制方获得伤害加成，被克制方受到伤害减免削弱
- 设计原因：给战前阵容搭配提供最基础的决策锚点，避免纯数值碾压

### 3. 非战斗状态可自由切换兵种

- 触发条件：部队处于非交战状态
- 具体结果：玩家可随时将部队切换为骑兵、盾兵、弓兵、枪兵或器械
- 设计原因：增加战术弹性，让一支部队能应对不同的克制场景，同时与兵种适性系统联动限制切换收益

### 4. 器械兵种专职攻城，被所有战斗兵种克制

- 触发条件：部队切换为器械兵种
- 具体结果：对城池耐久度造成极高伤害，但对战斗兵种造成极低伤害且承受极高战损
- 设计原因：强制同盟内部分工，攻城不能靠主力硬砸，必须有人保护器械

## 这个模块不负责什么

- 不负责武将怎么搭配和战法怎么选，那是战法与阵容构筑层的职责
- 不负责部队怎么行军和卡位，那是沙盘与行军节奏层的职责
- 不负责兵量怎么补充，那是资源与征兵循环的职责

## 可迁移结论

- If 在自动战斗的SLG中，战斗力完全等价于可消耗的兵量且战斗过程不可操作，Then 玩家的策略重心会完全前置于战前配将与战法联动，战中焦虑感极低
- If 兵种克制形成无死角的封闭循环（如骑盾弓枪骑），Then 战前侦查和针对性出兵就会成为核心战术，而不是纯堆一队全能兵
