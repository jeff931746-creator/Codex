# 《菇勇者传说》总档

## 基本信息

- 游戏名：菇勇者传说（Legend of Mushroom / 冒险大作战）
- 类型标签：放置挂机 / 装备收集 / 轻量化MMO
- 平台：iOS + Android
- 日期：2024-05-20
- 资料来源：TapTap介绍、鸟哥笔记深度拆解、百景网深度分析、出海帆深度报道、巴哈姆特攻略板

## 一句话定义

一款用"点灯抽装备-出售养神灯"的老虎机循环把挂机推关变成高频装备筛选，再用11条互不干扰的垂直养成管线和转职系统把随机装备收束为明确构筑的放置游戏。

## 核心结构

- 核心循环：自动推关 → 击杀怪物获取神灯点数 → 点灯抽取随机装备 → 筛选穿戴或出售 → 出售获得金币与经验 → 升级神灯与英雄 → 推关战力提升
- 主要资源：神灯点数、金币、经验、装备、各系统碎片/扭蛋币、宝石
- 主要系统：推关战斗执行层、神灯与装备构筑层、垂直养成底盘、转职与流派形成、神灯经济循环、日常与留存节奏、付费与Pass矩阵、排行榜与公会战检定
- 主要节奏节点：24小时神灯刷新、家园种植与偷菜、偷灯贼日常、新系统解锁节点（前7天每日）
- 主要检定节点：推关Boss节点、排行榜竞争、公会副本与GVG战斗

## 系统总览

每个系统必须同时写：负责什么 / 不负责什么 / 和谁联动最紧密。

| 系统 | 层级 | 职责边界（是什么 + 不是什么） | 联动最紧密 |
| --- | --- | --- | --- |
| 推关战斗执行层 | 执行层 | 决定战力是否足够过关及神灯点数产出速度；不负责装备怎么来，也不负责养成数值怎么堆 | 神灯经济循环、神灯与装备构筑层 |
| 神灯与装备构筑层 | 路线层 | 决定这局要留什么属性装备、走什么流派方向；不负责基础数值的长期沉淀，也不负责点灯概率 | 转职与流派形成、推关战斗执行层 |
| 垂直养成底盘 | 底盘成长层 | 决定11大系统的碎片沉淀与数值上限；不负责单局装备筛选，也不负责发材料 | 付费与Pass矩阵、神灯经济循环 |
| 转职与流派形成 | 路线层 / 构筑层 | 决定技能侧重与装备属性偏好；不负责具体数值源头，数值来自底盘与装备 | 神灯与装备构筑层、推关战斗执行层 |
| 神灯经济循环 | 经济层 | 决定点数、金币、经验、装备怎么流转；不负责具体战斗结果，也不负责社交压力 | 推关战斗执行层、垂直养成底盘 |
| 日常与留存节奏 | 节奏框架层 | 决定24小时刷新、偷菜、任务解锁的时间节拍；不负责发极品装备，只负责规定什么时候给机会 | 神灯经济循环、付费与Pass矩阵 |
| 付费与Pass矩阵 | 编辑层 / 奖励层 | 决定首日促销、各系统Pass和扭蛋怎么发材料；不负责战斗执行，只负责修正养成速度 | 垂直养成底盘、日常与留存节奏 |
| 排行榜与公会战检定 | 检定层 | 决定纯数值排名与公会GVG的胜负校验；不负责推进主线，只负责验证战力绝对值 | 垂直养成底盘、付费与Pass矩阵 |

## 本游戏最值得拆的专题

- "点灯-出售-升级神灯"为什么能把垃圾装备的负反馈变成核心长线驱动力，而不是让玩家觉得在刷垃圾？
- 11个互不干扰的垂直养成管线为什么能撑起极高的ARPU，而不是让玩家觉得系统太碎？
- 转职系统怎么把神灯的纯随机装备筛选变成有方向的构筑决策？
- 为什么排行榜和公会战只做纯数值检定，游戏依然能维持极高的收入？

## 总判断

- 这游戏靠什么成立：用"老虎机式点灯"提供高频随机刺激，用"出售垃圾养神灯"形成闭环，再用转职给随机装备一个筛选漏斗，让挂机推关不再是纯等时间，而是高频的装备评估决策
- 最大机会点：11条垂直养成管线和Pass矩阵把付费点拆散到每个系统，大R可以全管线拉满获得维度碾压，0氪也能在单管线慢慢积累
- 最大风险点：检定层只做纯数值排名而无追赶机制，晚入场玩家追赶成本指数级增长，长线收入必然呈现鲨鱼鳍形衰减
