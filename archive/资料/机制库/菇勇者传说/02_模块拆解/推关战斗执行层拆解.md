# 菇勇者传说·推关战斗执行层拆解

## 这个专题要回答什么

《菇勇者传说》的推关战斗为什么是纯自动的执行层，而不是策略决策点？

## 总体判断

推关战斗在这款游戏里只负责两件事：验证当前战力是否达标，以及按击杀数产出神灯点数。玩家不需要在战斗中做任何操作，所有决策都压在战前的装备筛选和养成投入上。这种设计把执行层完全交给了数值，把玩家的精力全部释放给了构筑层和经济层。

## 怎么判断某个东西属于推关战斗执行层

- 它如果主要决定"这关能不能打过"且过程中无需玩家操作，就属于执行层
- 它如果主要改变神灯点数的获取速度，就属于执行层
- 它如果需要玩家在战斗中做决策，就不属于这款游戏的执行层（本游戏不存在此类设计）

## 推关战斗的关键结构

### 1. 4小关+1Boss的固定结构

- 触发条件：玩家战力达到当前关卡推荐值，点击推关
- 具体结果：小蘑菇自动战斗，依次通过4个小关和1个Boss关，击杀怪物获得神灯点数
- 设计原因：固定结构提供稳定的节奏预期，4+1的切割让Boss战成为每个小阶段的验证点

### 2. 战力达标即自动推进

- 触发条件：当前战力超过关卡推荐值
- 具体结果：战斗快速结算，跳过战斗过程直接通关
- 设计原因：减少无效时间，让玩家尽快回到装备筛选和养成界面

### 3. 战力不足则卡关停留

- 触发条件：当前战力低于关卡推荐值
- 具体结果：角色在当前关卡挂机，持续获得神灯点数但无法推进
- 设计原因：卡关是这套系统的核心压力源，逼迫玩家回到神灯、装备和养成系统寻找增量

## 这个层级的边界在哪里

- 它不负责装备怎么来，那是神灯经济循环的职责
- 它不负责养成数值怎么堆，那是垂直养成底盘的职责
- 它不负责给玩家提供操作乐趣，这款游戏的执行层刻意排除了操作

## 这个模块不负责什么

- 不负责装备产出机制，那是神灯系统的职责
- 不负责战力提升方法，那是养成底盘和构筑层的职责
- 不负责社交和竞争压力，那是排行榜与公会战的职责

## 可迁移结论

- If 挂机游戏的战斗层完全自动且只验证战力绝对值，Then 玩家的所有策略精力都会被挤压到战前的装备筛选和养成资源分配上
- If 推关卡点只看战力数值而不检定阵容偏科，Then 玩家在卡关时只会追求最简单的数值提升，而不会尝试调整构筑结构
