# 菇勇者传说·神灯与装备构筑层拆解

## 这个专题要回答什么

神灯系统为什么不是简单的"抽卡"，而是这套游戏里最核心的构筑声明器？

## 总体判断

神灯系统表面上是老虎机式的随机抽装备，实际上它通过"10个装备槽位+随机二级属性+穿戴/出售决策"把每一次点灯都变成了构筑筛选。玩家不是在单纯追求高品质，而是在海量随机装备里寻找"符合当前转职属性偏好"的那件。神灯是这套游戏把随机性转化为构筑感的发动机。

## 怎么判断某个东西属于神灯与装备构筑层

- 它如果主要改变"这局要走什么属性方向"，就属于构筑层
- 它如果需要玩家在多选项中做"留还是卖"的决策，就属于构筑层
- 它如果只改变装备掉落概率而不改变属性方向，就不属于构筑层，属于经济层

## 神灯构筑的关键结构

### 1. 点灯产出10件随机装备

- 触发条件：消耗1点神灯点数进行1次点灯
- 具体结果：产出10件覆盖6个部位、6种品质、随机生成二级属性（暴击/攻速/闪避等）的装备
- 设计原因：10件装备同时产出制造了"批量评估"的心理刺激，类似老虎机的多线开奖

### 2. 二级属性随机决定构筑方向

- 触发条件：玩家查看装备的二级属性
- 具体结果：属性符合当前转职需求的装备被穿戴，不符合的被出售
- 设计原因：如果装备只有固定属性，点灯就是纯数值比大小；二级属性随机让每件装备都有"是不是我要的路线"的判断价值

### 3. 穿戴或出售的二元决策

- 触发条件：获得装备后进行评估
- 具体结果：高品质且二级属性匹配转职偏好的穿戴，低品质或属性不匹配的出售换取金币和经验
- 设计原因：二元决策把复杂的构筑选择简化为"留还是走"，降低认知负荷的同时保留了筛选乐趣

### 4. 神灯等级控制高品质概率

- 触发条件：投入金币和时间升级神灯
- 具体结果：高品质装备出现概率按神灯等级阶梯提升
- 设计原因：用长期投入把"运气"逐渐转化为"确定性"，给玩家长线追求

## 这个层级不负责什么

- 不负责神灯点数怎么来，那是推关战斗执行层的职责
- 不负责金币和经验的长期流转，那是神灯经济循环的职责
- 不负责转职具体有什么属性偏好，那是转职与流派形成的职责

## 可迁移结论

- If 装备系统提供高度随机的二级属性且转职系统提供明确的属性偏好，Then 玩家会在海量随机产出中主动寻找"本职业能用"的漏斗，抽卡的重复感会被构筑目标冲淡
- If 抽卡产出同时存在"出售换取升级资源"和"穿戴改变构筑"两个出口，Then 玩家对抽到垃圾的负反馈会被转移为升级动力
