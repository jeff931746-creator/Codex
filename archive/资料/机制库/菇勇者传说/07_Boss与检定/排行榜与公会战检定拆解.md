# 菇勇者传说·排行榜与公会战检定拆解

## 这个专题要回答什么

为什么排行榜和公会战只做纯数值检定，游戏依然能维持极高的收入？

## 总体判断

这套检定系统不检定偏科，只检定战力绝对值。排行榜竞争和公会GVG战斗的本质是"谁砸的钱多谁赢"。这种设计在前期极为有效，因为数值差距可以直接转化为付费动力；但长线会因为无追赶机制导致晚入场玩家追赶成本指数级增长，收入呈现鲨鱼鳍形衰减。

## 怎么判断某个节点属于检定层

- 它如果会强迫玩家暴露战力差距，就属于检定层
- 它如果会改变玩家对"要不要继续投钱"的判断，就属于检定层
- 它如果在检定后提供社交炫耀功能，就属于检定层

## 检定是怎么发生的

### 1. 推关Boss检定当前战力是否达标

- 触发条件：章节推进到Boss关
- 具体结果：战力达到关卡推荐值则通过，未达到则卡关停留
- 设计原因：Boss关是卡关压力的核心来源，逼迫玩家回到养成和付费

### 2. 排行榜竞争检定战力绝对值

- 触发条件：排行榜刷新
- 具体结果：玩家看到自己与前面的战力差距
- 设计原因：排行榜是最直接的战力对比工具，"只差一点就能超过去"是极强的付费动力

### 3. 公会副本与GVG检定集体战力

- 触发条件：公会活动开启
- 具体结果：公会成员的战力总和决定GVG胜负和公会副本进度
- 设计原因：集体检定把个人付费压力放大为社交压力，"不能拖公会后腿"比"自己不够强"更有驱动力

### 4. 公会城镇+装备检查面板提供炫耀出口

- 触发条件：玩家访问公会城镇
- 具体结果：可以查看其他玩家的装备和养成进度
- 设计原因：检定后的炫耀是付费的回报，没有炫耀出口的检定只会制造挫败感

## 检定层和节奏层的分界

- 节奏层回答"什么时候给资源"
- 检定层回答"你现在够不够强"
- 推关卡关是节奏层和检定层的交叉：节奏层控制卡关的时间点，检定层控制卡关的战力门槛

## 这个模块不负责什么

- 不负责装备和养成材料怎么来，那是神灯经济循环和付费矩阵的职责
- 不负责推关过程中的平滑节奏，那是节奏层的职责
- 不负责具体的战斗机制，那是执行层的职责

## 可迁移结论

- If 检定系统只看绝对数值排名而无追赶机制，Then 晚入场玩家会因为追赶成本指数级增长而直接放弃竞争，游戏收入会呈现鲨鱼鳍形（前高后低）
- If 检定系统同时提供装备检查面板等炫耀出口，Then 大R的付费会获得社交正反馈，进一步强化付费动力
