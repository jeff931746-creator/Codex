# 菇勇者传说·转职与流派形成拆解

## 这个专题要回答什么

转职系统怎么把神灯的纯随机装备筛选变成有方向的构筑决策？

## 总体判断

30级解锁的转职系统是这套游戏从"堆数值"转向"走构筑"的关键开关。在转职之前，玩家只看装备战力高低；转职之后，不同职业的技能侧重让装备的二级属性有了明确的偏好。转职给了神灯的随机性一个收口，让"留还是卖"的决策不再只看品质，而是看"是不是我要的属性"。

## 怎么判断某个东西属于转职与流派形成

- 它如果会改变玩家对装备二级属性的偏好，就属于流派形成
- 它如果会让玩家开始排斥另一类属性，就属于流派形成
- 它如果只是线性提升数值而不改变选择方向，就不属于流派形成

## 流派形成的关键结构

### 1. 30级转职提供第一次路线声明

- 触发条件：英雄等级达到30级
- 具体结果：玩家可以从战士菇、弓弩菇、愈灵师菇中选择一个转职
- 设计原因：30级之前玩家已经熟悉了神灯和装备的基础逻辑，此时给路线选择才能让玩家理解选择的意义

### 2. 职业分化制造属性偏好

- 触发条件：选择不同职业
- 具体结果：战士侧重生存和近战属性，弓弩侧重暴击和攻速属性，愈灵师侧重技能和回复属性
- 设计原因：没有属性偏好，神灯的二级属性随机就是纯噪音；有了属性偏好，随机性就变成了构筑筛选

### 3. 中后期细分转职深化路线

- 触发条件：达到70级/觉醒阶段并完成前置转职
- 具体结果：每个基础职业进一步细分，技能更加专精
- 设计原因：细分转职让路线层有足够深度，支撑长线的装备追求

### 4. 装备筛选标准随转职改变

- 触发条件：转职后查看神灯产出的装备
- 具体结果：原本只看品质高低的玩家开始看二级属性是否匹配职业需求
- 设计原因：这是转职系统对神灯系统的反向塑造，也是构筑感的核心来源

## 不同流派之间的边界

- 只堆高品质装备不看属性匹配，不叫流派，只叫战力堆叠
- 只有属性偏好而没有对应的技能侧重，不叫流派，只叫属性偏科
- 同时拥有职业技能侧重和装备属性匹配，才更像真正的流派

## 这个模块不负责什么

- 不负责装备的具体产出和随机机制，那是神灯系统的职责
- 不负责基础数值的长期沉淀，那是垂直养成底盘的职责
- 不负责检定流派是否过关，那是排行榜与公会战的职责

## 可迁移结论

- If 转职系统在30级才解锁且不同职业只对特定二级属性有乘区放大，Then 玩家在30级前的装备决策会是纯战力比大小，30级后才会因为属性排斥而形成构筑取舍
- If 流派形成只改变属性偏好而不改变技能形态，Then 构筑深度会很快见底，玩家在收集齐偏好属性装备后就会失去目标
