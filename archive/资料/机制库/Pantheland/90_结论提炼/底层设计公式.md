# Pantheland·底层设计公式

## 这一层的用途

这里的结论脱离《Pantheland》具体题材后仍应成立。它们关注的是“武器驱动职业化 + 编辑层构筑 + 长线成长沉淀”这一轻 MMO / 动作 RPG 结构本身的因果关系。

## 结论列表

**结论 1：武器前置职业律**

If 一个成长型动作 RPG 让武器切换直接改变攻击方式与技能表现，Then 武器就会承担职业入口的职责，而不再只是装备槽。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

**结论 2：编辑层成流派律**

If 模块系统能直接改写攻击效果、形态和属性，而不是只提升数值，Then 模块层就会从附魔层跃迁为流派编辑层。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

**结论 3：专属成长沉淀律**

If 每类武器拥有独立成长树且需要持续投入，Then 玩家对武器的选择就会从短线尝试转化为长期路线承诺。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

**结论 4：三层解耦清晰律**

If 一个系统把即时战斗身份、战斗表现编辑和长期成长沉淀拆成三层，Then 它更容易同时兼容低上手门槛与中长期构筑深度。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

**结论 5：公开资料降级律**

If 开放测试产品的公开证据主要来自商店页与搬运型实机视频，Then 对经济层与社交层的结论必须降级为框架判断，不能伪造成熟运营细节。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

## 边界

这套公式不适用于纯挂机、纯自动战斗或武器只做面板加成的 RPG；也不适用于没有长期成长沉淀、只做局内短线构筑的 Roguelite。当武器既不承担即时身份声明、模块也不改写可见战斗表现时，这组因果链会失效。
