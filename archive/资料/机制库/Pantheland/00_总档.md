## 一句话定义

一款试图用“武器即职业”替代固定职业框架的轻 MMO 动作 RPG: 玩家通过切换武器、插入模块、分配武器天赋来声明战斗路线，再用多人协作与副本推进承接长期成长。

## 核心结构

- 核心循环：选定武器与预设构筑 → 进入战斗地图/副本 → 普攻/位移/技能清怪 → 用模块与武器天赋强化当前打法 → 完成阶段目标后回到局外继续养成
- 主要资源：武器体系、模块槽/模块效果、武器专属天赋、局外养成材料、内购货币、广告补偿机会
- 主要系统：武器驱动执行层、模块编辑层、武器天赋成长层、多人协作/副本层、局外成长与商业化层
- 主要节奏节点：进入副本、首次拿到可见技能/模块强化、武器天赋分支形成、阶段性副本目标完成
- 主要检定节点：多人协作与谜题节点（商店页确认存在，公开视频未完整验证）、高压战斗段（公开视频能确认存在持续战斗场景）

## 系统总览

| 系统 | 层级 | 职责边界（是什么 + 不是什么） |
| --- | --- | --- |
| 武器驱动执行层 | 执行层 | 决定角色当前攻击形态、技能表现与手感；不负责长期成长分支，那是武器天赋层的职责 |
| 模块编辑层 | 编辑层 / 路线修正层 | 负责改写攻击效果、形态和属性，使同一武器出现变体；不负责基础动作模板，那是武器执行层的职责 |
| 武器天赋成长层 | 底盘成长层 / 路线层 | 为每类武器提供独立长期成长树，决定后续投资方向；不负责即时战斗表现的底层输入，那是执行层的职责 |
| 多人协作/副本层 | 节奏框架层 / 检定层 | 负责把“合作与解谜”转化为关卡推进目标；不直接提供成长深度，只提供压力场景与协作需求 |
| 局外成长与商业化层 | 经济层 / 留存框架 | 负责把开放测试期的推进、付费与广告补偿挂到长线体验上；不定义具体战斗手感，只改变获取节奏 |

## 本游戏最值得拆的专题

- 为什么这款游戏把“职业选择”前移成“武器选择”，而不是先给固定职业后再换技能？
- 模块系统为什么不是单纯词条装配，而是把同一武器改写成不同打法的编辑层？
- 当一款 MMO 轻量化产品想保留构筑感时，武器天赋树应该承担多少路线声明职责？

## 总判断

- 这游戏靠什么成立：靠“武器即职业 + 模块即攻击编辑器 + 武器专属天赋树”三层叠加，把传统 MMO 的固定职业感改写成可切换、可持续成长的构筑体验
- 最大机会点：如果武器差异和模块改写足够直观，它能同时吃到动作 RPG 的手感爽点与轻 MMO 的长线成长动力
- 最大风险点：如果多人协作、谜题与副本目标只停留在宣发文案，没有形成明确的高压检定与社交收益，产品会退化成常规数值养成 MMO，核心卖点会被快速稀释
