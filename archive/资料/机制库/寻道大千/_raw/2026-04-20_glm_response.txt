# 寻道大千 - 总档

## 基本信息
- 游戏名：寻道大千
- 平台：微信小游戏
- 类型：修仙挂机放置 RPG（IAP 内购驱动）
- 开发/发行：中清龙图
- 上线时间：2023年6月，至今仍为微信小游戏畅销榜头部

## 一句话定义
以“砍树爆装”为极简视觉表现，以“灵兽流派构筑”为长线验证的修仙放置卡牌 IAP 小游戏。

## 核心结构
游戏的核心循环由三个层级嵌套而成：
1. **执行层（砍树挂机）**：玩家消耗桃子（体力）砍树，每0.5秒掉落1件装备，装备提供基础攻防血数值。这是最外层的视觉包装与基础资源产出源。
2. **构筑层（灵兽与流派）**：玩家通过消耗御兽石洗练灵兽技能，搭配精怪与神通，形成如“连爆流”、“闪避流”等特定乘区组合。这是中层的心智占据与深度追求。
3. **验证层（推图与 PVP）**：通过战力门槛验证推图进度，推图层数决定挂机收益倍率。卡图时弹出限时礼包诱导付费。这是最内层的商业化闭环。

## 系统总览表格
| 系统 | 分类 | 主要产出 | 主要消耗 | 核心功能 |
|---|---|---|---|---|
| 砍树挂机 | 执行层 | 装备、仙桃、仙玉 | 桃子（体力） | 基础数值投放与视觉反馈 |
| 灵兽系统 | 构筑层 | 灵兽技能、属性加成 | 御兽石、仙玉 | 流派乘区定义与深坑洗练 |
| 精怪/神通 | 构筑层 | 被动效果、战斗机制 | 挑战券、灵尘 | 流派补强与战斗策略丰富度 |
| 推图/异兽 | 验证层 | 挂机倍率、稀有道具 | 无 | 付费卡点与进度验证 |
| 商城/礼包 | 付费层 | 桃子、御兽石、仙玉 | 人民币 | IAP 核心变现 |

## 本游戏最值得拆的专题
**灵兽与流派构筑**：传统放置游戏多靠单一数值堆叠，而《寻道大千》通过灵兽技能与装备词缀的乘区叠加，成功制造了极强的流派差异化和洗练深坑，这是其 LTV 远超同类小游戏的核心原因。

## 总判断
《寻道大千》的本质是“开箱游戏”的修仙变体，其成功不在于砍树动作本身，而在于将极低门槛的视觉反馈（砍树）与极深数值坑的流派构筑（灵兽洗练）强行绑定，利用前端的低门槛拉新，用后端的深坑变现。

# 砍树挂机与装备驱动

## 专题名称
砍树挂机与装备驱动

## 这个专题要回答什么
高频重复的“砍树”动作与随机爆装机制，如何避免枯燥并转化为玩家的持续在线与付费动力？（命题：砍树的视觉反馈和随机爆装机制是维持玩家前7日留存的核心原因，而非单纯的数值成长。）

## 总体判断
砍树本质是披着动作外衣的“高频抽卡”。游戏通过极短的反馈回路（0.5秒一次掉落）和极低的单次消耗（1桃子），将传统开箱游戏的爽感最大化。体力（桃子）是控制抽卡次数的阀门，也是游戏最硬通的付费锚点。

## 拆解正文

### 1. 砍树基础机制
- **触发条件**：玩家点击屏幕或开启自动砍树，每次扣除1点桃子（体力）。
- **具体结果**：对树木造成1次伤害，树木血量归零时重置，掉落1-3件装备、少量仙铁（装备分解货币）或仙玉。自动砍树间隔为0.5秒/次。
- **设计原因**：0.5秒的间隔创造了极高频的视觉与听觉刺激，模拟老虎机的拉杆体验，让玩家在短时间内获得大量“爆装”多巴胺。

### 2. 暴击与连击反馈
- **触发条件**：玩家面板暴击率或连击率达标（如暴击率20%），在砍树时按概率触发。
- **具体结果**：触发暴击时，伤害数字放大并变红，掉落装备数量翻倍（从1件变2件）；触发连击时，单次点击扣除1点桃子但连续播放2次砍树动画及掉落。
- **设计原因**：将战斗属性映射到日常行为中，让玩家在非战斗阶段也能直观感受到属性成长的收益，强化“变强”的体感。

### 3. 装备词缀与阶级系统
- **触发条件**：击杀推图 Boss 或开启妖王宝箱。
- **具体结果**：掉落更高阶级的装备（如 T1 青铜、T2白银...T9混沌）。同阶级装备分绿/蓝/紫/金/红5个品质。红色品质装备必定附带2条随机战斗词缀（如连击率+12.5%、暴击率+11.3%）。
- **设计原因**：阶级控制基础数值（攻防血），品质控制词缀条数。红色极品词缀是游戏前中期的核心追求，制造了巨大的随机性鸿沟，驱动玩家反复消耗桃子刷取。

### 4. 挂机收益机制
- **触发条件**：玩家离线，时间累计。
- **具体结果**：每分钟自动获得推图层数对应的桃子、仙铁、仙玉。离线收益上限为12小时。上线后可观看1次 IAA 广告使本次离线收益翻倍。
- **设计原因**：保底收益机制，确保玩家不登录也有基础成长。12小时上限强制玩家每日至少登录2次以清空收益，维持 DAU。IAA 广告提供微量补充，不破坏 IAP 生态。

## 这个模块不负责什么
1. 不负责灵兽技能的数值计算与流派搭配（由《灵兽与流派构筑》模块负责）。
2. 不负责 PVP 匹配与排名结算（由活动与验证系统负责）。
3. 不负责仙玉与桃子的充值兑换比例设计（由《经济循环拆解》模块负责）。

## 关键判断
砍树模块的内核是“体力消耗器”与“词缀投放器”。其设计精髓在于将传统卡牌游戏的“抽卡”动作高频化、日常化，使得体力（桃子）成为玩家最渴望的资源，从而为直接贩卖体力的礼包铺平道路。

## 可迁移结论（If-Then）
If 一款放置游戏将核心资源获取包装为高频点击且每次点击都有独立掉落动画的视觉反馈，Then 必须设置极低的单次消耗（如1体力）和极快的动画演出（<0.5秒），否则高频操作会因等待动画而引发疲劳，导致玩家迅速流失。

## 相关专题
- 灵兽与流派构筑（说明装备词缀如何与灵兽技能形成乘区，驱动玩家追求特定词缀组合）
- 经济循环拆解（说明桃子作为核心消耗品如何与付费挂钩）

# 灵兽与流派构筑

## 专题名称
灵兽与流派构筑

## 这个专题要回答什么
灵兽系统如何通过技能组合与洗练机制，驱动玩家进行深度的长线付费与资源倾注？（命题：灵兽的被动技能组合是决定流派上限的唯一因素，而非灵兽的基础面板。）

## 总体判断
灵兽系统是《寻道大千》的数值深坑与长线锚点。游戏通过“主战+协战”的位置设计划分了灵兽的职能，通过“技能洗练”制造了极深的随机坑。流派构筑的核心逻辑是“乘区叠加”，即通过装备词缀、精怪被动、灵兽技能共同将某一属性（如连击率）推至100%阈值，从而触发质变。

## 拆解正文

### 1. 灵兽分类与识别边界
- **识别边界**：按出战位置与技能机制划分。**主战灵兽**每回合主动攻击1次，造成基于自身攻击力比例的伤害，并释放1个主动技能（如青龙造成28%攻击伤害并增加10%暴击率）；**协战灵兽**不主动攻击，每回合有概率（如30%）释放1个被动技能（如玄武提供主角最大生命11%的护盾）。判断标准：看灵兽是否占用攻击回合，占用即为主战，否则为协战。
- **设计原因**：主战灵兽负责拓展伤害乘区，协战灵兽负责拓展防御/回复乘区，两者互不干扰，使玩家需要同时培养两只不同维度的灵兽，翻倍了养成坑度。

### 2. 技能洗练机制
- **触发条件**：消耗御兽石对灵兽技能进行重置。
- **具体结果**：随机生成1-4个技能格子及技能内容。技能池包含“强化灵兽自身”（如灵兽攻击力+15%）、“强化主角属性”（如连击率+8.5%）、“特殊机制”（如主角被攻击时概率眩晕攻击者1回合）。洗练时锁定1个技能需消耗5个御兽石，锁定2个消耗20个，锁定3个消耗100个。
- **设计原因**：指数级增长的锁定消耗是典型的深坑设计，逼迫玩家在“保留次品继续用”和“重氪追极品”之间抉择。金色/红色特殊机制技能的极低出现率（约0.5%）是长线付费的抽水机。

### 3. 流派构筑逻辑
- **触发条件**：玩家将装备词缀、精怪属性、灵兽技能指向同一乘区，且总值超过阈值。
- **具体结果**：形成特定流派。例如“连爆流”：装备提供连击率+30%、暴击率+25%；精怪“大明王”提供暴击率+15%；灵兽青龙被动提供连击率+20%、暴击率+15%。合计连击率65%，暴击率65%。实战中高频触发连击，每次连击均判定暴击，伤害产生乘积级质变。
- **设计原因**：让玩家从散装数值过渡到体系化数值。一旦确立流派，玩家更换任何单一组件都会导致阈值断裂（如连击率跌破50%），从而产生极强的沉没成本，极大提高了换流派的付费门槛。

### 4. 塑魂与升星
- **触发条件**：消耗同名灵兽碎片或通用塑魂花。
- **具体结果**：灵兽星级提升（1星至5星），主动技能伤害比例提升（如从28%提升至45%），被动技能数值提升（如从8.5%提升至12.5%）。
- **设计原因**：提供稳定的线性成长，弥补洗练纯随机带来的挫败感，确保中小R玩家也能通过时间积累获得基础提升。

## 这个模块不负责什么
1. 不负责装备的掉落概率与基础数值投放（由《砍树挂机与装备驱动》模块负责）。
2. 不负责御兽石与仙玉的产出与兑换比例（由《经济循环拆解》模块负责）。
3. 不负责战斗的先后手判定与速度计算（由战斗结算系统负责）。

## 关键判断
灵兽系统的本质是“阈值驱动器”。流派的强弱不在于绝对数值的高低，而在于能否通过灵兽技能将某项属性推过质变阈值（如闪避流需将闪避率堆至100%以上以应对命中属性）。这种设计让付费目标极其明确：差5%闪避率，就必须去洗练或买礼包。

## 可迁移结论（If-Then）
If 游戏允许玩家自由搭配装备词缀、宠物技能和被动天赋，Then 必须设计至少3个互相排斥的乘区（如连击、闪避、击晕），并在高阶 PVP 验证中强制要求单一乘区达到80%以上阈值才能生效，否则玩家会趋向于全水桶加点，导致流派构筑系统失效，失去长线付费深度。

## 相关专题
- 砍树挂机与装备驱动（说明装备如何提供基础词缀支撑灵兽流派阈值）
- 经济循环拆解（说明御兽石与塑魂花如何通过礼包变现）

# 经济循环拆解

## 专题名称
经济循环拆解

## 这个专题要回答什么
多代币体系如何控制玩家进度，并精准诱导仙玉（付费货币）消耗以实现 IAP 转化？（命题：桃子（体力）的稀缺性是驱动玩家购买日常礼包的直接原因，而非灵兽洗练材料的稀缺性。）

## 总体判断
《寻道大千》的经济循环是典型的“双轨制”：明线上，桃子（体力）是限制玩家获取装备和基础数值的阀门，制造日常焦虑；暗线上，御兽石（洗练材料）是吞噬仙玉的无底洞，制造深度付费。仙玉作为万能润滑剂，将明暗两线强行绑定，使得无论哪个环节卡点，最终都会转化为对人民币的直接需求。

## 拆解正文

### 1. 核心代币与产出消耗
- **桃子（体力）**
  - 产出：挂机每分钟产出1个，12小时上限720个；日常任务产出约500个；推图首通产出。
  - 消耗：砍树1次消耗1个；挑战妖王1次消耗50个。
  - 设计原因：严格控制每日获取上限，确保玩家在无付费情况下，每日游戏时长被锁定在1-2小时内，制造“想玩但没体力”的焦虑。
- **御兽石（洗练货币）**
  - 产出：妖王挑战少量掉落；活动兑换；仙玉购买（50仙玉/个）。
  - 消耗：灵兽洗练1次消耗1个，锁定1技能消耗5个，锁定2技能消耗20个。
  - 设计原因：极低的日常产出与极高的消耗形成巨大剪刀差，是中后期仙玉消耗的最大黑洞。
- **仙玉（付费货币）**
  - 产出：充值（1元=10仙玉）；砍树极小概率掉落（约0.1%）；成就少量奖励。
  - 消耗：购买桃子（50仙玉买500个）；购买御兽石；购买挑战次数。

### 2. 经济循环路径
- **日常循环**：桃子 -> 砍树 -> 装备/仙铁 -> 战力提升 -> 推图层数增加 -> 挂机桃子产出微增。
- **付费循环**：人民币 -> 仙玉 -> 桃子/御兽石 -> 砍树/洗练 -> 装备/极品灵兽技能 -> 战力质变 -> 推图/ PVP 优势。

### 3. 付费卡点与礼包触发机制
- **触发条件1（体力耗尽）**：玩家桃子降至0且未开启自动回复。
  - 具体结果：主界面弹出“每日特惠”礼包，定价6元，包含500桃子+2个御兽石。性价比标注为日常的5倍。
  - 设计原因：利用玩家刚建立起“砍树爽感”却被强制中断的瞬间，用极低客单价破冰。
- **触发条件2（卡图超过2小时）**：玩家推图战力不足，连续失败2次。
  - 具体结果：弹出“战力飞跃”礼包，定价30元，包含大量仙铁（强化装备）+塑魂花。
  - 设计原因：精准定位玩家受挫时刻，提供即时战力提升方案，转化焦虑为付费。
- **触发条件3（洗练出极品技能）**：灵兽洗练出现红色技能。
  - 具体结果：锁定该技能需消耗20个御兽石，若不足则弹出“洗练助力”礼包，定价12元，包含25个御兽石。
  - 设计原因：利用“沉没成本”和“损失厌恶”，玩家如果不买礼包，下次洗练极品技能就会被覆盖，逼迫玩家必须买单。

### 4. 通货膨胀控制
- **机制**：装备迭代。推图层数每提升100层，开启下一阶装备（如 T3 升 T4）。高阶装备基础数值是低阶的1.5倍。
- **结果**：玩家之前积累的低阶红色极品词缀装备，在100层后必须被替换为高阶绿色无词缀装备。仙铁（强化材料）返还80%。
- **设计原因**：通过阶层硬重置，不断清空玩家既有数值积累，制造持续的“数值匮乏感”，防止老玩家资源溢出，确保桃子和仙玉的消耗需求永不过期。

## 这个模块不负责什么
1. 不负责装备词缀的随机分配算法（由《砍树挂机与装备驱动》模块负责）。
2. 不负责灵兽技能的强弱平衡（由《灵兽与流派构筑》模块负责）。
3. 不负责 PVP 赛季的奖励发放（由活动系统负责）。

## 关键判断
《寻道大千》的经济系统核心在于“制造匮乏与填补匮乏”。桃子匮乏是浅层匮乏，解决的是在线时长与首充破冰；御兽石匮乏是深层匮乏，解决的是大 R 的无限付费需求。阶层装备迭代是维持这两种匮乏永续存在的底层引擎。

## 可迁移结论（If-Then）
If 游戏中存在一种可由付费货币无限购买的基础消耗品（如体力），且该消耗品是获取随机数值装备的唯一门槛，Then 随着游戏进程，基础消耗品的边际收益会急剧下降（出红装概率不变但需求提升），此时必须引入新的高价值消耗品（如洗练石）来承接付费货币的消耗，否则大 R 玩家将面临有钱无处花的窘境，导致营收见顶。

# If-Then 结论

1. If 一款放置游戏将核心资源获取包装为高频点击且每次点击都有独立掉落动画的视觉反馈，Then 必须设置极低的单次消耗（如1体力）和极快的动画演出（<0.5秒），否则高频操作会因等待动画而引发疲劳，导致玩家迅速流失。

2. If 游戏允许玩家自由搭配装备词缀、宠物技能和被动天赋，Then 必须设计至少3个互相排斥的乘区（如连击、闪避、击晕），并在高阶 PVP 验证中强制要求单一乘区达到80%以上阈值才能生效，否则玩家会趋向于全水桶加点，导致流派构筑系统失效，失去长线付费深度。

3. If 游戏中存在一种可由付费货币无限购买的基础消耗品（如体力），且该消耗品是获取随机数值装备的唯一门槛，Then 随着游戏进程，基础消耗品的边际收益会急剧下降，此时必须引入新的高价值消耗品（如洗练石）来承接付费货币的消耗，否则大 R 玩家将面临有钱无处花的窘境，导致营收见顶。

4. If 游戏采用阶层装备迭代机制（高阶基础数值碾压低阶），Then 必须在迭代时提供80%以上的养成材料返还，否则玩家会因为沉没成本过高而拒绝更换装备，导致数值膨胀失控与阶层断层。
