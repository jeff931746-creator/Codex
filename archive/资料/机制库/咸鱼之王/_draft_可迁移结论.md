# 咸鱼之王·可迁移结论草稿

## 这份文件是什么

完整拆解还没做（等 Gemini worker 吃资料后出初稿）。这里先把已经盘清楚的 5 条可迁移结论按模块归位，将来并进对应拆解文件的"可迁移结论"小节。

每条都配了反面测试，用来验证因果链是不是真的。反面测试的写法：把 If 反过来，看 Then 还成不成立；不成立才算过。

## 归位到「植物成长底盘 / 角色成长底盘」模块

### 1. 不可逆升星阶梯 → 每日碎片错过成本非线性放大

**If** 核心战力靠"碎片 → 升星阶梯"沉淀且升星不可逆（咸鱼之王咸将升星 + 觉醒），
**Then** 玩家对每日碎片掉落的错过成本会随下一个星级阈值临近而非线性放大。

*反面测试*：If 升星可无损重置或碎片可退还，Then 错过成本仍非线性放大？{不成立，会回落到线性}

## 归位到「关卡节奏」模块

### 2. 战力建议值 + Boss 血墙 → 卡关日付费窗口收窄

**If** 主线推图设了战力建议值 + 章节 Boss 血墙（咸鱼之王章节关 + 关底咸将），
**Then** 卡关会把玩家的付费动机从"想变强"收窄成"想过这一关"，礼包命中率在卡关当日激增。

*反面测试*：If 推图难度连续平滑递增、没有明显墙点，Then 礼包命中率仍在某日激增？{不成立，退回日常低频}

## 归位到「日常循环 / 资源循环」模块

### 3. 三件套日活钩子 → 登录被锚定

**If** 日活钩子是"每日任务进度 + 挂机收益 12 小时封顶 + 七日签到断档不可补"三件套（咸鱼之王离线挂机 + 签到），
**Then** 登录从"想玩"变成"不登就亏"，日活频次被锚定到每日至少一次。

*反面测试*：If 挂机不封顶或签到可补签，Then 登录频次仍被锚定到每日一次？{不成立，回落到内容驱动}

## 归位到「商店与奖励」模块

### 4. 情绪触发礼包 → 内容必须贴痛点

**If** 商店收入主要来自"情绪触发礼包"（咸鱼之王卡关礼包、抽歪补偿、开服基金、首充），
**Then** 礼包内容必须和玩家当下的具体痛点一一对应（卡关给战力、抽歪给碎片），否则转化率会坍塌到常规商店水平。

*反面测试*：If 触发礼包发通用元宝/钻石，和当下痛点脱钩，Then 转化率仍能保持在触发礼包档位？{不成立，回落到常规商店}

## 归位到「流派形成 / PVP 检定」模块

### 5. PVP 克制环 → 主力阵容回报变平、被迫分流

**If** PVP 引入阵容职业/属性克制（咸鱼之王巅峰对决、跨服争霸的克制环），
**Then** 单一主力阵容的养成回报会在达到克制阈值后迅速变平，迫使玩家开二队、三队分流资源。

*反面测试*：If PVP 只看战力总和、无克制，Then 玩家仍会开二队、三队？{不成立，会继续堆主力}
