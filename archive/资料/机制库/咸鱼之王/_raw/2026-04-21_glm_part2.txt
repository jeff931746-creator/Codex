<<<FILE: 06_商店与奖励/商店与奖励拆解.md>>>
专题名称：商店与奖励拆解

这个专题要回答什么：
咸鱼之王的商店与奖励体系如何通过多层级定价与碎片化投放，驱动微信小游戏用户的持续留存与冲动付费？

总体判断：
咸鱼之王的商店本质是"伪商店，真礼包"，奖励体系是"高密度微反馈+长线碎片锁定"。它不依赖传统商店的"比价购物"逻辑，而是通过黑市折扣、限时礼包、推图挂机宝箱三类载体，将付费与免费奖励深度捆绑。日常商店的核心作用是建立价格锚点，制造"不买即亏"的损失厌恶；而真正的IAP收入爆发点在于基于情绪峰值的条件触发礼包。

拆解正文：

1. 黑市商店（日常付费锚点与金砖回收器）
- 触发条件：每日0点刷新，部分档位可用金币购买，核心档位仅限金砖购买。
- 具体结果：提供木箱子、青铜箱、黄金箱、铂金箱、火把、精铁等。例如初级礼包500金砖[推测]，进阶石宝箱1800金砖[推测]。每日限购1次。
- 设计原因：建立低价锚点。黑市商品标价常驻5折[推测]，让玩家产生"原价买亏了，今日不买亏了"的心理。同时，金砖作为通行货币，通过黑市日常消耗，控制免费金砖的通货膨胀，迫使玩家在后续卡关时因金砖不足而直接充值。

2. 挂机奖励与推图宝箱（留存基石与微反馈源）
- 触发条件：每分钟自动产出，累积3小时/12小时达到收益上限需手动领取；推图通过特定关卡触发宝箱掉落。
- 具体结果：产出基础经验、金币、各类进阶石与随机武将碎片。推图宝箱有概率开出完整红将（概率约0.5%[推测]）。
- 设计原因：限制离线时长强制日活。3小时上限要求玩家每天至少登录3次以上才能吃满挂机收益，拉起日活频次。推图宝箱的随机完整红将，是极强的高光微反馈，利用抽卡快感对冲推图的枯燥感。

3. 限时/条件触发礼包（IAP爆发点与情绪定价）
- 触发条件：卡关（如连续挑战同一关卡失败3次[推测]）、核心武将升星、开服特定天数。
- 具体结果：弹出6元、30元、68元等档位礼包，内容高度定制化（如卡关礼包必含大量进阶石与突破石；升星礼包必含该武将碎片）。
- 设计原因：情绪峰值精准收割。当玩家"差一点就能过"或"刚抽到核心武将急需升星"时，付费意愿达到顶峰。此时礼包性价比远超黑市（如6元礼包性价比常达日常3倍[推测]），将情绪冲动直接转化为IAP流水。

4. 月度通行证与月卡（LTV稳定器）
- 触发条件：花费30元[已验证(游戏内实测)]购买月卡，或68元[推测]购买通行证，之后每日登录领取。
- 具体结果：月卡每日发放100金砖[推测]；通行证通过推图层数解锁大量金砖与红将万能碎片。
- 设计原因：沉没成本拉扯。一旦购买，玩家为赚回本金必须保持30天连续登录，用极低的日均成本锁定长线留存。

这个模块不负责什么：
不负责具体武将数值养成曲线（属于02_模块拆解/养成），不负责战斗胜负判定与怒气循环（属于02_模块拆解/战斗），不负责金砖的宏观产出与消耗平衡（属于04_经济循环）。

关键判断：
咸鱼之王的商店系统不是为"自由交易"设计的，而是为"损失厌恶"和"冲动消费"设计的。黑市是价格锚点，限时礼包才是利润中心。

可迁移结论：
If 微信小游戏IAP依赖碎片化日常付费，Then 必须设立日常低价商店作为价格锚点制造损失厌恶，并在情绪峰值点（卡关/升星）投放限时高性价比礼包完成收割。

相关专题：

<<<FILE: 90_结论提炼/底层设计公式.md>>>
专题名称：底层设计公式

这个专题要回答什么：
放置卡牌IAP小游戏在长线留存与付费设计上，存在哪些不可违背的底层机制逻辑？

总体判断：
底层逻辑围绕"可逆压力与不可逆养成"展开，通过高频微反馈对冲长线枯燥，用情绪定价取代价值定价。放置游戏的商业化本质不是卖数值，而是卖"数值缩小的差距感"与"等待时间"。

拆解正文：
放置卡牌的底层公式建立在行为心理学的斯金纳箱变体之上。传统斯金纳箱强调随机奖励，而放置IAP游戏强调"可预期的压力与付费捷径"。玩家在游戏中面临的推图压力是周期性可逆的（卡关-变强-通过-再卡关），但投入的养成资源是不可逆的。这种不对称结构迫使玩家在压力期寻求解法，而付费是最直接的解法。同时，为了维持长线运营，游戏必须通过高频的碎片化奖励维持每日登录的微反馈，避免玩家在长草期流失。

这个模块不负责什么：
不负责具体品类的数值参数设定（如掉落率具体数值），不负责美术与叙事逻辑，不负责跨品类（如SLG、MOBA）的机制解释。

关键判断：
底层机制的核心变量是"压力的可逆性"与"养成的不可逆性"的乘积效应。

可迁移结论（If-Then）：

1. If 核心养成路径是碎片化且不可逆的，Then 玩家对日常资源获取的强迫感会随时间递增。
*反面测试*：If 核心养成可逆（如可随时无损重置），Then 强迫感仍递增？{不成立}

2. If 局内压力呈现周期性可逆瓶颈（卡关），Then 玩家在瓶颈期的付费意愿会随突破预期的升高而激增。
*反面测试*：If 压力无瓶颈且平滑递增，Then 付费意愿仍激增？{不成立}

3. If 日常奖励的领取存在时间上限且需要手动重置，Then 用户的日活登录频次会被强制锚定。
*反面测试*：If 日常奖励无时间上限且无需手动重置，Then 日活频次仍被强制锚定？{不成立}

4. If 商店定价体系以情绪峰值（卡关/抽卡）触发为主，Then 常规商店的锚点价格必须高于触发礼包以维持性价比错觉。
*反面测试*：If 商店定价不以情绪峰值触发为主，Then 常规商店锚点价格仍必须高于触发礼包？{不成立}

5. If 阵容构筑存在明确的职业/属性克制环，Then 玩家在单一养成线上的沉没成本会被克制关系稀释，从而增加多线养成需求。
*反面测试*：If 阵容构筑无克制环（纯数值比拼），Then 单线沉没成本仍被稀释并增加多线需求？{不成立}

<<<FILE: 90_结论提炼/产品化结论.md>>>
专题名称：产品化结论

这个专题要回答什么：
针对微信小游戏平台放置卡牌品类，如何将底层公式转化为带具体数值和落地条件的产品设计判断？

总体判断：
产品化落地需要紧扣"微信生态用户低耐心、高比价"特征，用极低首充破冰，用高密度卡关逼氪，用碎片化时间适配。具体数值设计必须服务于"缩小玩家与卡关战力的差距感"，而非绝对数值的堆砌。

拆解正文：
在底层公式的基础上，产品化结论需要解决"做多少"的问题。微信小游戏用户对沉没成本极度敏感，首充必须提供远超预期的回报以建立信任；卡关礼包的定价必须卡在用户的冲动消费阈值内；离线收益的设计必须在"让玩家觉得没亏"和"逼玩家上线"之间找到最优解。这些结论基于咸鱼之王的成功实践提炼，带有明确的数值推断与适用边界。

这个模块不负责什么：
不负责跨平台（如PC端游、主机）的数值移植，不负责纯IAA（广告变现）游戏的设计判断，不负责重度SLG的付费点设计。

关键判断：
具体数值的设定是情绪阈值的具象化，6元与30元是微信小游戏IAP的两个核心生死线。

可迁移结论（If-Then）：

**结论1：首充破冰锚点**
If 放置卡牌小游戏首发获客，Then 首充6元必须直接赠送当前版本T0级核心武将（获取率100%[推测]）。
*适用前提*：F2P+IAP变现、微信小游戏轻度用户、游戏核心乐趣依赖特定流派构筑。
*反例游戏*：原神（买断制倾向的RPG，首充给的是通用抽卡货币而非直接送T0角色，因为其商业模式靠角色池概率赚钱，送T0会破坏抽卡商业模式）。
*太宽测试*：若替换为卡牌对战类型，结论{不成立}。

**结论2：卡关礼包转化率**
If 玩家在推图/爬塔中遭遇卡关（存活超过30秒[推测]但伤害不足），Then 弹出的6元/30元专属资源礼包转化率最高。
*适用前提*：单局时长<3分钟、数值卡点明确、战斗可自动播放的放置游戏。
*反例游戏*：艾尔登法环（买断制ARPG，无内购，卡关只能靠技术或游戏内刷级，弹出礼包会破坏体验与买断契约）。
*太宽测试*：若替换为单机RPG类型，结论{不成立}。

**结论3：离线收益上限**
If 游戏核心循环依赖离线挂机，Then 离线收益累积上限必须设定在12小时[推测]且需手动点击领取。
*适用前提*：需要高频日活的移动端免费游戏。
*反例游戏*：旅行青蛙（纯放置，收益无上限且无需频繁手动，因为其商业模式是周边与轻度广告，无需逼迫日活）。
*太宽测试*：若替换为MOBA类型，结论{不成立}。

**结论4：日常商店折扣锚**
If 游戏存在多种货币（免费+付费），Then 日常黑市商店必须长期提供5折[推测]的养成材料包作为价格锚点。
*适用前提*：多货币体系、重度养成消耗的IAP游戏。
*反例游戏*：星露谷物语（单机模拟经营，无内购商店，无需价格锚点，资源获取全靠游戏内行为）。
*太宽测试*：若替换为单机RPG类型，结论{不成立}。

**结论5：献祭流构筑容错**
If 游戏主打献祭/自残流派，Then 流派核心输出武将的生存力必须低于队伍平均值的50%[推测]，且击杀回复必须达到最大生命值的30%[推测]。
*适用前提*：有怒气/能量循环的回合制自动战斗卡牌。
*反例游戏*：守望先锋（FPS，无血量自残机制，靠枪法与技能配合，生存力由操作决定而非数值设定）。
*太宽测试*：若替换为MOBA类型，结论{不成立}。
