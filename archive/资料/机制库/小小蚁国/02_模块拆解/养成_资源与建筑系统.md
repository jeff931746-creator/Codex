# 专题名称：资源与建筑系统

## 这个专题要回答什么
《小小蚁国》的七资源体系，为什么不会因为单一资源过剩或匮乏就破坏整体养成节奏？仓库保护上限对玩家行为的驱动机制是什么？

## 总体判断
游戏通过七资源的功能分工（直接采集类 vs 间接发酵类 vs 高消耗货币类）构建了资源的自然依赖链：植物必须经切叶蚁发酵才能产出菌类，菌类不足会降低其他资源效率；蜜露是最稀缺的高消耗品，只能由建筑产出。这个设计使得玩家即使在某个资源充足时，仍会被另一个资源的瓶颈制约，从而持续推动建筑升级。仓库保护上限则作为"溢出惩罚"，强制玩家在累积到上限时选择投入消耗（升级建筑/孵化兵蚁）或接受被掠夺损失，消除了放置类游戏中"囤积不动也安全"的收益方案。

## 拆解正文

### 1. 七资源功能分工

| 资源 | 产出方式 | 主要消耗去向 | 能否野外采集 |
|---|---|---|---|
| 肉 | 肉采集地（最多10个） | 工蚁数量维持、兵蚁孵化 | 是 |
| 植物 | 植物采集地 | 切叶蚁发酵→菌；建筑升级 | 是 |
| 水 | 蓄水池 | 工蚁数量维持 | 否（建筑产出） |
| 菌 | 切叶蚁栖息地将植物发酵产出 | 建筑升级、科技研究 | 否（建筑产出） |
| 湿土 | 湿地建筑 | 建筑升级的主要消耗品 | 是 |
| 沙子 | 沙子采集地 | 蚁道挖掘、特定建筑 | 是 |
| 蜜露 | 蜜露专属建筑 | 进化研究、特化蚁培养（高消耗） | 否（建筑产出） |

**关键依赖链**：肉 + 水 → 工蚁数量上限 → 同时升级建筑数量上限；植物 → 菌（via 切叶蚁）→ 建筑升级速度；蜜露 → 进化树 + 特化蚁（独立产出，无野外补充）。

菌的供应约束最为精妙：切叶蚁栖息地需要工蚁维持，而工蚁数量受肉和水制约，意味着提升菌产量需要先解决肉和水的供给，形成三阶梯制约。

### 2. 建筑数量约束机制

- **工蚁孵化地**：最多建造8个，工蚁总数量受肉+水资源存储总量上限决定
- **肉采集地**：最多10个
- **所有建筑数量上限**：受蚁后等级间接控制（蚁后升级解锁新建筑槽位）
- **同时升级数量**：由当前工蚁数量决定（工蚁多 = 可以同时多队施工）

这意味着玩家面临资源-建筑的双层瓶颈：没有足够肉和水，工蚁数量上不去；工蚁不够，同时升级数量受限；建筑等级不够，蚁后无法升级；蚁后不升级，新建筑槽位不开放。

### 3. 仓库保护上限机制

资源系统有三个关键数值：
- **已存储量**：当前实际持有量
- **每小时生产量**：建筑持续产出速率
- **仓库保护上限**：低于此量的资源不会被掠夺带走

**行为驱动逻辑**：
- 当玩家资源低于保护上限：可以挂机累积，无需登录操作
- 当玩家资源超过保护上限：超出部分完全暴露在被掠夺风险中
- **强制行为**：玩家必须选择 (a) 主动消耗（升级建筑/孵化兵蚁）以降低暴露量，或 (b) 在线保护，或 (c) 接受损失

这个机制与放置游戏常见的"无限累积挂机"截然相反：累积过多资源不是安全的，而是变成给别人的猎物，从而将纯放置玩家引导至周期性的主动消耗行为。

### 4. 资源获取双轨

- **建筑生产**：稳定但缓慢，受建筑等级上限约束
- **野外掠夺/打野**：高峰高谷，依赖兵力投射效率，受兵蚁数量和特化蚁等级制约

两轨互补：发展习性玩家主要依赖建筑生产，掠夺习性玩家需要建筑产出提供基础，再靠掠夺加速增长。这使得习性选择不只是buff方向，而是实际决定了主要资源获取行为。

## 这个模块不负责什么
本模块不负责野外大地图上的资源点分布与争夺规则（属于野外地图系统）；不负责仓库保护上限的具体数值如何随等级变化（需更多一手数据）；不负责不同习性对特定资源产出的具体加成数值（属于习性系统模块）。

## 关键判断
仓库保护上限是把"被动放置留存"转化为"主动消耗循环"的关键阀门。它不是安全感机制，而是资源溢出惩罚机制。

## 可迁移结论
If SLG类游戏设计了仓库保护上限，且超出部分完全暴露于PvP掠夺风险，Then 玩家的资源消耗行为会从"等到积够再升"转变为"持续小额投入以避免暴露"，付费点前移至日常防护道具和小额加速购买。

If 资源体系中存在"间接发酵类资源"（A→B的转化依赖建筑C），Then 建筑C的等级就同时承压两个资源维度的玩家焦虑，付费点密度高于单纯的直接采集资源。
