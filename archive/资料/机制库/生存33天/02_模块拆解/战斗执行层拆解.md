# 生存33天·战斗执行层拆解

## 这个专题要回答什么

《生存33天》的战斗为什么是“阵营克制与阵型搭配”的执行层，而不是单纯的数值对撞？

## 总体判断

战斗执行层负责把阵营属性、克制关系和阵型站位兑现成战斗结果。它决定“这波遭遇战能不能赢”，但不决定“去哪里打”，路线声明来自迷雾探索系统。

## 战斗执行层到底在做什么

### 1. 三阵营克制决定伤害放大与减免

- 触发条件：我方幸存者攻击被克制的敌方阵营（物理克神圣、神圣克魔法、魔法克物理）
- 具体结果：造成 **150%基础伤害**；反之，攻击克制我方的敌人，仅造成 **50%基础伤害**
- 设计原因：用明确的150%与50%数值差距，强制玩家维持三阵营均衡，替代纯数值比拼

### 2. 阵型搭配决定谁承受伤害与谁释放技能

- 触发条件：战前将幸存者拖入前排（1-2号位）或后排（3-5号位）
- 具体结果：前排单位承受 **70%的单体攻击**，后排单位承受 **30%**；1号位有20%概率触发“反击”类技能，5号位有20%概率触发“远程支援”类技能
- 为什么有效：让同一种阵营的幸存者因站位不同产生截然不同的战斗职能，增加局内操作空间

### 3. 战斗结算直接消耗队伍状态

- 触发条件：战斗结束（无论胜负）
- 具体结果：所有参战幸存者扣除本次战斗损失的生命值（需消耗食物/药品恢复），且全队“疲劳度”增加10点，疲劳度超过80时全属性下降20%
- 设计原因：把战斗从“无损清怪”变成“资源消耗事件”，与生存循环强绑定

## 典型识别标准

如果一个元素满足下面任意两条，就更接近执行层：

- 在遭遇战里直接参与伤害结算与状态扣除
- 核心价值是利用150%克制加成或阵型位效果改变当前战斗结果
- 战斗结果会直接削减队伍的生存资源（生命值、疲劳度）

如果它主要满足下面这些条件，就不是执行层：

- 决定去哪个区域寻找敌人
- 需要靠局外资源长期养成
- 本身不直接参与战斗结算

## 这个模块不负责什么

- 不负责决定探索路线和遭遇风险，那是迷雾探索与路线的职责
- 不负责幸存者的长期属性成长，那是角色成长底盘的职责
- 不负责战斗后的物资分配与生存消耗，那是资源与生存循环的职责

## 可迁移结论

- If 战斗系统采用三阵营循环克制且克制收益（150%）显著高于平级输出，Then 玩家会优先培养多阵营阵容而非单阵营极限战力
- If 战斗结算会直接扣除队伍生命与状态且不可自动回满，Then 战斗就从收益事件变成了风险决策，与生存资源循环绑定
