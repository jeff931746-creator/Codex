# 生存33天·产品化结论

## 这一层的用途
这里的结论带适用前提与反例，用于产品化决策。本文件将《生存33天》的末日生存与阵营克制特征转化为可落地的设计数值与机制判断，明确在何种用户层、变现形态与局内时长下这些判断生效，并指出跨品类时的失效边界，避免设计思路的滥用。

## 结论列表

**结论 1：探索风险权衡驱动路线声明**

If 迷雾探索需要权衡路线风险（敌人等级+5）、时间成本（消耗加倍）和小队状态，Then 探索就变成了路线声明而非单纯移动，玩家单局留存与重玩意愿会提升[推测]。

*适用前提*：移动端竖屏，单次探索决策时长 1-3分钟[推测]的轻量生存策略，且地图需提供至少 2条[推测]差异化风险路线（低危/高危）。
*反例游戏*：《原神》（开放世界），其探索重环境交互与沉浸驱动，时间成本不构成生存压力，此结论不适用。
*太宽测试*：若替换为 开放世界 类型，结论不成立。

**结论 2：生存消耗对冲探索收益**

If 时间推移会自动消耗基础物资（每4小时消耗一次）且探索是物资唯一主动来源，Then 探索就从可选行为变成了强制行为，玩家无法通过停滞来规避风险。

*适用前提*：IAP+IAA混合变现，依赖高频上线维持留存的生存游戏，且存在明确的倒计时或消耗机制。
*反例游戏*：《明日方舟》（塔防），无自动消耗生存物资的压力，玩家可通过停滞规避风险，探索(作战)非生存必须。
*太宽测试*：若替换为 无生存消耗的塔防 类型，结论不成立。

**结论 3：阵营克制检定驱动即时补缺**

If 遭遇战采用纯阵营敌人进行克制检定（如纯魔法队要求神圣输出），Then 商店中对应克制阵营幸存者的兑换率会提升[推测]，检定直接转化为付费动力。

*适用前提*：拥有多阵营构筑且商业化深度绑定阵营补齐的生存游戏，玩家已形成 1-2个[推测]主力阵营。
*反例游戏*：《阴阳师》（回合制RPG），PVE检定不强制要求纯阵营克制，驱动力更多来自数值而非阵营补缺。
*太宽测试*：若替换为 无阵营克制检定的RPG 类型，结论不成立。

**结论 4：倒计时压缩决策窗口**

If 生存游戏采用固定天数倒计时作为终局节点，Then 资源分配就从无限养成变成了限量时间内的最优化问题，中后期付费窗口会集中在加速与状态恢复。

*适用前提*：单局有明确终局（如33天），且后期生存消耗速率显著提升（从每8小时到每3小时）的生存游戏。
*反例游戏*：《辐射：避难所》（模拟经营），无固定终局倒计时，资源分配是无限长线运营，不存在最优化时间窗口。
*太宽测试*：若替换为 无终局的模拟经营 类型，结论不成立。

**结论 5：广告加速绑定战损恢复**

If 广告加速绑定在战损恢复（生命值<30%）和探索返回节点，Then IAA收入会与生存高压期的高频需求强绑定，广告观看率会高于常规功能加速。

*适用前提*：网络要求高且战斗存在状态损耗的在线生存手游，玩家对时间敏感。
*反例游戏*：《开心消消乐》（三消），无战损概念，加速仅用于节省体力等待时间，非紧急止损。
*太宽测试*：若替换为 无战损的三消 类型，结论不成立。

## 局限性
1. **重度SLG品类**：本结论依赖固定倒计时与局内生存压力，对于单局长达数月、强调持续对抗的SLG不适用。
2. **纯竞技对战品类**：本结论的付费与养成逻辑建立在PvE阵营检定与生存消耗上，对于强调绝对公平竞技的对战游戏不适用。
3. **强叙事驱动品类**：倒计时与高频战损恢复会打断叙事沉浸，对于以剧情演出和情感体验为核心的产品形态不适用。
