# 向僵尸开炮·底层设计公式

## 这一层的用途
这里的结论脱离本游戏语境仍应成立。它们是对「射击肉鸽」这一品类底层运作逻辑的抽象提取，揭示的是压力框架、构筑感生成、用户留存与系统分工之间的决定性因果关系，可直接作为同品类新项目的设计推演基石。

## 结论列表

**结论 1：可逆压力与挽回动力**

If 压力信号可逆（如防线距离），Then 压力高峰期玩家仍有挽回动力。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。当压力信号不可逆时，高峰期体验必然滑向绝望放弃，可逆性是动力的决定性来源。

**结论 2：单一防守目标与认知负荷**

If 波次守线游戏提供单一防守目标，Then 轻度用户的认知负担处于可接受范围。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。多防线目标必然导致轻度用户认知超载，单一目标是维持其注意力集中的决定性前提。

**结论 3：失败补偿与卡点留存**

If Roguelite 失败产出可累积的局外资源，Then 轻度用户在卡关节点仍会保持留存。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。纯归零的失败在卡关节点必然导致快速流失，局外累积是把高失败率机制做成大众产品的决定性改造。

**结论 4：路线件与构筑感**

If 强化选项包含路线件（改变后续最优选项方向），Then 玩家单局选择体验具有构筑感。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。仅有数值量变的选项必然使体验退化为比大小，路线件的存在是构筑感生成的决定性条件。

**结论 5：强度检定与轻度用户正反馈**

If Boss检定设计为强度检定（数值压制），Then 轻度用户能通过构筑成长获得通关正反馈。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。机制检定（特定克制）必然构成行为门槛筛走轻度用户，强度检定是轻度用户通过数值成长跨越障碍的决定性保障。

**结论 6：系统解耦与上手门槛**

If 执行层（基础属性）与路线层（流派方向）解耦，Then 轻度用户能在低理解成本下完成首局构筑。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。两层强耦合必然要求前置理解全局技能组合，解耦是控制轻度用户上手成本的决定性分工。

**结论 7：行为多样性与流派宽度**

If 敌人设计引入行为模式多样性，Then 不同流派能维持各自的发挥场景与宽度。

*反面测试*：把 If 取反后，Then 是否仍成立？不成立

含义：
- 「不成立」= If 是 Then 的决定性变量。仅靠数值提升难度必然使最优解收敛至单一最高输出路线，行为多样性是维持流派宽度的决定性变量。

## 边界
这套公式在核心向硬核 Roguelite、纯策略塔防或长线大型 RPG 中不适用。核心向产品恰恰需要不可逆压力与机制检定来建立用户壁垒；纯塔防无局外长线养成承载失败补偿；长线 RPG 的单局时长与复杂度远超轻中度射击框架，其认知负荷与构筑节奏的分配逻辑完全不同。
