# 《向僵尸开炮》总档

## 基本信息

- 游戏名：向僵尸开炮
- 类型标签：射击 Roguelite / 波次守线 / 局内构筑
- 平台：移动 App + 微信小游戏
- 资料来源：App Store 公开信息、立项调研文档（`projects/10-立项预演/小丑牌向僵尸开炮/`）、产品实机分析
- 市场信号：App Store 评分约 4.5 / 约 63 万条评分；发行商 2024 年全球收入近 1.8 亿美元

## 一句话定义

一款用明确的压线目标框架承接 Roguelite 局内成长的轻中度射击游戏——僵尸波次制造目标压力，局内技能选择制造流派差异，局外养成承接长期留存。

## 核心结构

- 核心循环：出击 → 射击清怪 → 过波奖励三选一 → 技能成型 → Boss 检定 → 局外结算成长
- 主要资源：体力（局次限制）、局内金币（即时消费）、软货币（局外成长）、硬货币（高级解锁）
- 主要系统：角色/机甲系统、武器系统、技能强化系统、波次压力系统、局外成长系统、商业化运营层
- 主要节奏节点：每波清怪结束、三选一技能节点、精英波/Boss 波
- 主要检定节点：Boss 波（检验当前 Build 能否压制爆发伤害需求）、压线失守（检验清怪效率下限）

## 系统总览

| 系统 | 层级 | 职责边界（是什么 + 不是什么） |
|---|---|---|
| 角色/机甲系统 | 执行层 | 决定基础攻击属性、站位与移动方式；不决定流派方向，那是技能系统的职责 |
| 武器系统 | 执行层 | 决定弹道形态、伤害类型与攻击节奏；不决定目标优先级，那是玩家操作层 |
| 技能强化系统 | 路线与构筑层 | 每波后三选一，声明并叠加流派方向；不提供数值底盘，那是局外成长层的职责 |
| 波次压力系统 | 检定层 / 节奏框架 | 通过僵尸数量、速度、类型组合制造阶梯性压力；不直接影响玩家成长，只作为检验目标 |
| 局外成长系统 | 底盘成长层 | 永久提升角色、武器、基地的基础属性；不影响局内流派方向，只影响能否过基础门槛 |
| 局内奖励节点 | 修正层 | 三选一、宝箱、特殊事件提供流派修正机会；不决定最终流派，只提供调整窗口 |
| Boss 与精英波 | 检定层 | 要求当前 Build 能在有限时间内完成爆发输出；不是单纯数值门槛，而是针对清怪效率的具体检验 |
| 商业化运营层 | 留存框架 | 用体力、活动、卡池驱动长期登录和消费；不影响核心战斗逻辑，只影响获取节奏 |
| 社交系统 | 局间动力层 | 用军团归属感、好友竞争、分享拉新解决局与局之间的回来动力；不干预局内体验 |

## 本游戏最值得拆的专题

- 波次压力系统为什么是"目标感"而不是"难度感"——它如何让轻度玩家也能清楚感知压力而不产生挫败？
- 技能三选一如何在每局 5–10 分钟的时间内让玩家感受到明确的流派差异？
- 局外成长如何解决 Roguelite 的"高失败率留存危机"——在局内死亡后玩家为什么还想再来？
- Boss 波在这套系统里实际在检定什么——它检验的是清怪效率上限，而不是机制理解？

## 总判断

- 这游戏靠什么成立：用"压线目标 + 波次节奏"给 Roguelite 成长提供了清晰的外部压力框架，让玩家每次技能选择都有直接可见的意义；局外成长把失败转化为进度，解决了纯 Roguelite 对轻度用户的留存难题
- 最大机会点：压力框架极度精简（守住防线这一个目标），可以承载各种主题和执行层变体而不改变核心体验逻辑
- 最大风险点：如果局内技能组合的差异感不够强，整体会退化成纯数值堆叠的割草游戏，失去 Roguelite 的构筑爽感
