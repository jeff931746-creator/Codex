# 向僵尸开炮·系统分工拆解

## 这个专题要回答什么

《向僵尸开炮》的各个系统是如何分工的，它们之间的边界在哪里，以及这套分工为什么能支撑一个 5–10 分钟的轻中度 Roguelite 产品？

## 总体判断

这个游戏的系统分工遵循一个清晰的三层结构：**压力框架（波次系统）提供目标，成长层（技能 + 局外）提供手段，执行层（角色 + 武器）负责兑现**。三层各司其职，不互相越界，这使得整体复杂度被控制在轻度用户可接受的范围内。

---

## 系统分层结构

```
第一层：压力框架
  └── 波次压力系统（提供"必须做什么"）
  └── Boss 与检定（提供"做得够不够好"的验证点）

第二层：成长层
  └── 技能强化系统（局内"如何变强"的选择权）
  └── 局外成长系统（跨局"如何累积进步"的保障）

第三层：执行层
  └── 角色/机甲系统（谁来打）
  └── 武器系统（用什么打）
```

---

## 系统边界的关键判断

### 边界 1：波次系统 vs 技能系统

- 波次系统说"现在威胁有多大"，不说"应该选什么技能"
- 技能系统说"这局能做什么路线"，不说"这局会遇到什么威胁"
- 两者的交叉点：波次类型（密集波 vs 精英波）会让不同技能路线的效率产生差异——但这是隐性的，游戏不会明说"这一波选 AOE 更好"

### 边界 2：技能系统 vs 局外成长

- 技能系统的效果在局结束后完全重置
- 局外成长的效果永久保留，跨局生效
- 两者的交叉点：局外的科技树可以扩展技能池（解锁新技能）——局外成长会影响局内技能出现的可能性，但不影响局内选择本身

### 边界 3：执行层 vs 技能系统

- 技能系统的效果最终落地在执行层的具体属性上（武器弹道变化、伤害提升）
- 执行层本身不主动影响技能选项——武器品质不会改变技能池，只影响技能在执行层上的兑现效率
- 两者的交叉点：部分技能专门改变武器属性（弹射、穿透），这些技能的价值高度依赖当前装备的武器类型

---

## 系统间的核心咬合链

```
玩家进入关卡
  → 波次系统产生压力（"必须清怪"）
  → 执行层基础属性决定能否维持基础清怪效率
  → 技能三选一提供路线选择机会（"变得更强"）
  → 选择的技能改变执行层的输出方式和效率
  → 累积的技能组合形成流派
  → Boss 波检验累积后的总输出上限
  → 局结束后，资源产出流入局外成长系统
  → 局外成长提升下局的执行层基础属性
```

---

## 这套分工为什么适合轻度用户

1. **压力框架极简**：只有一个目标（守住防线），不需要理解复杂的胜利条件
2. **成长层可感知**：每波结束有三选一节点，变强的感受是即时且可见的
3. **失败不归零**：局外成长保证失败仍有积累，低挫败感
4. **不需要记住组合**：5–10 分钟的局内时长，玩家不需要记住上百种技能组合，只需要认出当前局的 2–3 个协同点

## 可迁移结论

如果一个轻中度产品的系统分工不清晰（比如让角色系统同时承担"执行"和"路线声明"的职责），则新玩家上手时需要同时理解多个层级，认知负担快速超出轻度用户的接受范围。**执行层和路线层的分离是控制上手门槛的核心手段**，即使牺牲部分构筑深度。
