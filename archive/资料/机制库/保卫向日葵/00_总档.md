# 《保卫向日葵》总档

## 基本信息

- 游戏名：保卫向日葵
- 类型标签：微信小游戏 / 背包肉鸽塔防 / 合成养成
- 平台：微信小游戏
- 日期：2026-04-17
- 资料来源：微信小游戏公开试玩信息、App Store 页面、TapTap 游戏介绍与更新说明、B 站试玩视频、攻略站公开资料

## 一句话定义

一款用“种植-摆位-合成-升级”把塔防决策拆成多个短步骤的微信小游戏：前线用植物执行输出，局内靠杂交和升级声明路线，局外靠植物养成、庄园和资源投放把失败转成下一局更强的底盘。

## 核心结构

- 核心循环：进入关卡 → 放置/种植植物 → 战斗中获取资源与升级选项 → 合成杂交或强化单体植物 → 通过波次与 Boss 检定 → 结算奖励 → 回到局外继续养成
- 主要资源：金币/阳光类即时资源、钻石类高级资源、植物碎片/植物源液、基因萃取液、体力、庄园币、庄园经验、种子与培育品
- 主要系统：植物执行层、杂交合成层、植物成长底盘、流派形成、资源循环、关卡节奏、商店与庄园奖励、Boss 与挑战检定
- 主要节奏节点：开局布阵、每波出怪间隙、局内升级/抽取/合成节点、章节推进节点、庄园解锁与日常收菜节点
- 主要检定节点：Boss 波、精英挑战、虚空之境、极限挑战、中后期章节门槛

## 系统总览

每个系统必须同时写：负责什么 / 不负责什么 / 和谁联动最紧密。

| 系统 | 层级 | 职责边界（是什么 + 不是什么） | 联动最紧密 |
| --- | --- | --- | --- |
| 植物执行层 | 执行层 | 决定单局里“谁在打、怎么打、打多快”；不负责路线声明，也不负责长期养成效率 | 杂交合成层、植物成长底盘 |
| 杂交合成层 | 路线层 | 决定这局要把哪些植物合成到哪个方向；不负责底盘数值，只负责把局内件组合成更强结构 | 植物执行层、流派形成 |
| 植物成长底盘 | 底盘成长层 | 决定植物等级、星级、觉醒与碎片沉淀；不负责当局路线选择，只负责让某条路线有足够数值兑现 | 资源循环、商店与庄园奖励 |
| 流派形成 | 路线层 / 构筑层 | 决定输出流、辅助流、杂交流等构筑怎么成形；不负责具体数值源头，数值来自成长与奖励系统 | 杂交合成层、Boss 与挑战检定 |
| 资源循环 | 经济层 | 决定钻石、体力、碎片、萃取液等资源怎么流转；不负责具体战斗结果，只负责把时间和付费转成养成进度 | 商店与庄园奖励、植物成长底盘 |
| 关卡节奏 | 节奏框架层 | 决定章节、波次和阶段压力怎么推进；不负责给奖励种类，只负责规定奖励在什么时候出现 | Boss 与挑战检定、商店与庄园奖励 |
| 商店与庄园奖励 | 编辑层 / 奖励层 | 决定玩家在哪些节点买到、领到、换到关键材料；不负责战斗执行，只负责修正路线和补短板 | 资源循环、植物成长底盘 |
| Boss 与挑战检定 | 检定层 | 决定哪些偏科流派会被卡住；不负责推进主线，只负责验证当前构筑是否过关 | 流派形成、关卡节奏 |

## 本游戏最值得拆的专题

- “放置-合成-升级”为什么能在单局内形成足够明确的路线切换，而不是只剩下战力堆叠？
- 杂交合成为什么会直接改写后续资源投放，而不是只产出一张更稀有的卡？
- 庄园与日常奖励为什么能改变主线推进速度，而不是只提供额外收集乐趣？
- Boss 与挑战为什么主要在检定偏科，而不是单纯在检定面板战力？

## 总判断

- 这游戏靠什么成立：用“短循环塔防 + 局内合成 + 局外植物养成”把复杂度拆散到多个低门槛决策点，玩家每次操作都能立刻看到阵容变强或阵型成形
- 最大机会点：杂交、成长、庄园和活动奖励共享同一套养成语言，所有资源最终都在为“哪条植物线更完整”服务
- 最大风险点：如果杂交不能持续改变后续选择，游戏就会退化成只看数值的放置塔防，路线感和构筑感都会变弱
