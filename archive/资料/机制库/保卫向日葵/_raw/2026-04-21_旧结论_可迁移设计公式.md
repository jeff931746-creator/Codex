# 保卫向日葵·可迁移设计公式

## 可以直接迁移的结论

- 如果一个塔防游戏要适配微信小游戏用户，就应把“复杂决策”拆到多个短节点里，而不是集中在一个超长战斗回合里
- 如果合成系统会改变后续资源投放方向，它就已经是路线层，不再只是收集系统
- 如果局外养成材料能由重复掉落转化而来，玩家对冗余掉落的负反馈会明显下降
- 如果奖励系统同时负责即时补缺和长期供给，它就会从“福利”变成“流派修正器”
- 如果检定节点不针对偏科，只做纯数值门槛，玩家失败后的行为只会是继续堆战力

## 设计公式

- If 执行层单位能被独立理解，Then 新手可以只靠“摆出来就会打”的直觉上手
- If 路线层会在局内锁定方向，Then 构筑感会比纯收集感更强
- If 底盘成长能吃掉重复资源，Then 养成会更耐久
- If 节奏层能把压力切成短波次，Then 每次补强的意义都会更清楚
- If Boss 层专门检定偏科，Then 玩家会主动补短板而不是只刷数值

## 这套结构里最不能拿掉的部分

- 不能拿掉杂交合成层，否则局内路线就会退化成平推
- 不能拿掉局外成长底盘，否则重复游玩没有积累感
- 不能拿掉挑战检定层，否则中后期只能靠数值膨胀维持难度
