## 与机制库的关系

[机制库](../机制库/) 是具体游戏的机制拆解（如 `向僵尸开炮/`、`保卫向日葵/`），玩法承接库是抽象主题（如 `波次防守 + 局外塔型升级`）。

**对应关系**：

```text
玩法承接（抽象主题）       机制库（具体游戏案例）
─────────────         ─────────────────────────
PLY-TD-001            向僵尸开炮、保卫向日葵、保卫萝卜...
（波次防守+局外升级）   （多个游戏案例）
```

每个玩法承接条目的 `mechanism_refs` 字段引用机制库的路径。

## 添加新玩法承接

按 [填库工作流](../../../reference/部门标准/立项/三位一体框架/05-填库工作流.md) 操作：

1. 主 agent 先 plan，说明拟新增的玩法承接
2. 用户审批后，按 [玩法承接模板](../../../reference/部门标准/立项/三位一体框架/模板/玩法承接模板.md) 填写草稿
3. 通过 [独立玩法五问](../../../reference/部门标准/立项/三位一体框架/03-玩法承接标准.md#独立性判定必填字段-independence_check) 验证
4. 通过自检和交叉引用检查
5. 分配 ID 并归档
6. 更新 [玩法索引](玩法索引.md)
