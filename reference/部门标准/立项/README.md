## 立项定义阶段

在进入正式立项评审前，需要先完成立项定义。

### 产出物

- 项目代号和一句话定义
- 目标用户、目标市场
- 核心买量承诺
- 核心玩法和爽点
- 主要变现方式
- 3-5 个最像的竞品

### 进入条件

- 有明确的产品方向
- 有初步的市场判断

### 退出条件

- 项目基线清晰
- 竞品明确
- 核心承诺可描述

### 下一步

完成立项定义后，进入 [01-立项评审](立项流程机制/01-立项评审/)

## 当前分类

- `发行制作人/`：偏立项和发行判断的角色提示词、分析框架
- `立项流程机制/`：按立项步骤归档的评审规则、原型闸门、纸面原型补证方法和长线经营假设模板

当前口径：

- 立项阶段只回答两件事：
  这个方向是否值得进入下一阶段验证；
  这个方向的长线经营逻辑是否已经被写清。
- 原型或纸面原型阶段不直接验证 `长线 ROI` 和 `长线留存` 结果。
- 原型阶段只验证两类前提：
  `买量回收前提`
  `付费留存前提`
- `长线 ROI / 长线留存` 在立项里只能写成经营假设、内容引擎和后续验证路径，不能写成已经成立的结果。
- 小测和后续投放阶段才负责看真实数据；立项阶段负责的是“是否值得花下一阶段成本去验证”。

不应放在这里的内容：

- 系统策划、玩法策划、团队协作流程
- 已经进入制作阶段的项目规则
