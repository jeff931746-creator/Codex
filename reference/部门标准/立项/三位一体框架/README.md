# 三位一体立项框架

## 这是什么

三位一体框架（Triple-Cluster Framework）是立项阶段的结构化决策骨架，由三个独立但相互关联的库组成：

```text
人群簇库（AUD-*）   →   卖给谁
题材母题库（THM-*） →   讲什么
玩法承接库（PLY-*） →   怎么玩
```

任何立项决策都需要在这三个维度上同时成立，才能进入下一步评审。

---

## 为什么需要

题材方法论已经齐全（[题材选择方法论_2026-05-08.md](../../../../archive/方法论/立项/题材选择方法论_2026-05-08.md)），但落到操作层面存在三个缺口：

| 缺口 | 表现 | 后果 |
|---|---|---|
| 人群无结构化定义 | "目标受众" 是自由文本 | 无法判断两个题材是否在抢同一群人；无法系统估 CPI |
| 玩法承接无结构化定义 | "玩法承接" 字段填 "塔防/RPG" 这种品类标签 | 无法解释"为什么这个题材适合这个玩法"，组合判断靠经验 |
| 三者无交叉引用 | 题材评分中 "有效获量" 和 "ROI承接" 都依赖人群和玩法 | 评分依据弱，难以复用，每个项目都要重新分析 |

三位一体框架建立后：

- 题材评估可以引用具体人群簇 ID 和玩法承接 ID，评分有据可查
- 同象限题材可以按"是否抢同一人群"、"玩法承接是否可复用"做二次筛选
- 立项决策可沉淀：一旦某个 `AUD × THM × PLY` 组合验证成功，整组合可作为下一项目的参考

## 三库关系图

```text
                  ┌──────────────────────┐
                  │   题材母题库（THM）   │
                  │  （讲什么 - 抽象层） │
                  │   ─────────────────  │
                  │  例：末世求生母题    │
                  └──────────┬───────────┘
                             │
              引用 audience_ids  │  引用 playstyle_ids
                  ↓              │              ↓
   ┌──────────────────────┐      │      ┌──────────────────────┐
   │   人群簇库（AUD）     │      │      │   玩法承接库（PLY）   │
   │   （卖给谁）          │      │      │   （怎么玩）          │
   │   ───────────────    │      │      │   ─────────────────  │
   │  例：压力释放型       │←─────┴─────→│  例：塔防、Roguelike │
   └──────────────────────┘             └──────────────────────┘
```

**关联规则**：题材母题作为中心节点，引用人群簇和玩法承接的 ID。反向查询通过题材母题的 `audience_ids` 和 `playstyle_ids` 字段实现。

## 使用流程

### 立项前（题材筛选）

1. 在 [题材总表](../../../../archive/资料/买量组合库/题材总表.md) 中筛出候选题材
2. 查看每条候选题材的 `mother_theme_id`（关联的母题）
3. 通过母题在 [题材母题库](../../../../archive/资料/买量组合库/题材库/) 中查到关联的 `audience_ids` 和 `playstyle_ids`
4. 在 [人群簇库](../../../../archive/资料/人群簇库/) 和 [玩法承接库](../../../../archive/资料/玩法承接库/) 中查具体定义
5. 判断三者是否自洽（人群偏好玩法、玩法能承接题材、题材能吸引人群）

### 立项中（评审依据）

题材评审时，[评估模板](../../../../archive/资料/买量组合库/评估模板/) 中的 `audience_ids` 和 `mother_theme_id` 字段需填写，作为评分依据：

- "有效获量" 评分 → 引用人群簇的获量难度
- "ROI 承接" 评分 → 引用玩法承接的商业化方式

### 立项后（沉淀）

- 验证成功的 `AUD × THM × PLY` 组合，作为新条目加入 [组合案例库](../../../../archive/资料/买量组合库/组合案例库/)
- 验证失败的组合，分析失败原因并更新对应母题/人群簇的"风险"字段
