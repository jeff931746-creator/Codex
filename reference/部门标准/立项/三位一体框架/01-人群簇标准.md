# 人群簇标准

## 这是什么

人群簇是在游戏消费行为上具有稳定共性的玩家群体。本文档定义人群簇的字段、命名规则、分类原则和填库流程。

参考：[00-框架定义.md](00-框架定义.md) 中的"维度 1：人群簇"。

---

## 大类分类

人群簇按以下大类分类，大类代码用于构成 ID：

| 大类代码 | 大类名称 | 核心特征 | 典型代表 |
|---|---|---|---|
| `Y` | Young 年轻向 | 18-30岁，娱乐型消费，对新奇内容敏感 | 二次元抽卡用户、潮流休闲用户 |
| `M` | Middle 中年向 | 30-45岁，时间碎片化，付费力强 | SLG重氪用户、放置型用户 |
| `S` | Senior 成熟向 | 45+岁，习惯于熟悉品类，对学习成本敏感 | 麻将棋牌用户、宫斗剧情用户 |
| `F` | Female 女性向 | 强女性向偏好，情感叙事驱动 | 乙女、换装、宠物模拟用户 |
| `C` | Core 核心玩家 | 重度玩家，追求深度和挑战 | Steam硬核用户、动作游戏爱好者 |
| `H` | Hybrid 混合型 | 跨多个人群特征，无明显边界 | 全民休闲、合家欢用户 |

**注意**：大类不是互斥的，一个人群簇可能在两个大类之间（如"年轻女性向" 可以放 Y 或 F）。**归大类的依据是核心买量识别特征**：如果 "性别向" 比 "年龄段" 更能决定素材表达，就归 F；反之归 Y。

## 字段定义

每个人群簇条目必须包含以下字段：

### 必填字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | 字符串 | 格式 `AUD-{大类}-{编号}`，永不修改 |
| `name` | 字符串 | 人群簇名称，简洁有辨识度（≤12字） |
| `category` | 枚举 | 大类代码（Y/M/S/F/C/H） |
| `description` | 文本 | 一句话描述（≤30字） |
| `emotion_need` | 数组 | 核心情绪需求（如 `[压力释放, 身份代入]`） |
| `playstyle_preference` | 数组 | 偏好玩法（如 `[策略, 放置]`） |
| `payment_pattern` | 数组 | 付费习惯（如 `[数值养成, 月卡]`） |
| `acquisition_difficulty` | 整数 1-5 | 获量难度（1=易获，5=极难） |
| `ltv_potential` | 整数 1-5 | LTV潜力（1=低LTV，5=高LTV） |
| `evidence_level` | 枚举 | 证据等级 S/A/B/C/D（参考 [题材评估标准](../../../../archive/资料/买量组合库/题材评估标准.md)） |

### 选填字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `age_range` | 字符串 | 年龄段（仅供参考，不作为唯一判据） |
| `gender_lean` | 枚举 | 性别倾向 male/female/neutral |
| `geo` | 数组 | 地域分布（如 `[国内一二线, 东南亚]`） |
| `info_channels` | 数组 | 信息渠道（如 `[抖音短视频, B站, 应用商店]`） |
| `representative_games` | 数组 | 代表游戏（用于对照） |
| `boundary` | 文本 | 与相邻人群簇的差异点（必填，避免重复定义） |
| `risk_notes` | 文本 | 风险说明（如 "素材疲劳快"、"流失高"） |

## 命名原则

### 名称要求

- ≤ 12 字
- 包含核心识别特征（如 "压力释放型休闲玩家"、"SLG重氪用户"）
- 避免使用人口学标签（如 "25-35岁男性"），改用行为+情绪描述
- 避免使用产品名（如 "口袋奇兵用户"），用情绪/玩法描述代替

### 好的名称示例

✅ "压力释放型休闲玩家"
✅ "二次元养成抽卡用户"
✅ "SLG策略重氪用户"
✅ "宫斗剧情代入用户"

### 不好的名称示例

❌ "25-35岁男性玩家"（人口学描述，无法指导买量）
❌ "口袋奇兵核心用户"（绑死产品，不可复用）
❌ "玩SLG的人"（太宽泛）

## 边界（boundary）字段填写要点

**boundary 字段是人群簇标准的核心防呆机制**。

每个新增人群簇必须明确说明：

```text
本人群簇与 AUD-XX-XXX 的区别：______
本人群簇与 AUD-YY-YYY 的区别：______
```

如果说不清边界，说明这个人群簇没有独立存在的必要，应合并到现有人群簇。

例：

```text
AUD-Y-002 "二次元养成抽卡用户" 的 boundary：
- 与 AUD-Y-001 "年轻男性休闲玩家" 区别：付费习惯不同，前者抽卡，后者数值养成或广告变现
- 与 AUD-F-001 "二次元女性向用户" 区别：性别倾向更宽，对题材的偏好集中在战斗向而非情感向
```

## 评分维度说明

### 获量难度（acquisition_difficulty）

| 分数 | 含义 |
|---|---|
| 1 | 极易获，主流买量渠道任何素材都能命中 |
| 2 | 容易获，主流买量渠道，标准素材模板可覆盖 |
| 3 | 中等难度，需要针对性素材策略 |
| 4 | 较难，人群分散或CPI较高 |
| 5 | 极难，仅特定渠道或长期内容运营才能触达 |

### LTV潜力（ltv_potential）

| 分数 | 含义 |
|---|---|
| 1 | 极低，主要靠广告变现 |
| 2 | 较低，付费转化率低，ARPPU 低 |
| 3 | 中等，有月卡/小额付费 |
| 4 | 较高，有抽卡或大R付费 |
| 5 | 极高，重氪渠道明确，大R潜力强 |

## 填库工作流（简要版）

详细流程见 [05-填库工作流.md](05-填库工作流.md)，本节给出简要步骤：

1. **提案阶段**：在 [人群簇库](../../../../archive/资料/人群簇库/) 新建草稿文档，填写所有必填字段
2. **boundary 校验**：明确说明与现有人群簇的差异
3. **代表游戏验证**：至少举出 3 个代表游戏，说明这些游戏的用户重合度高
4. **评审**：用户审批后分配 ID，加入索引

## 关联到其他维度

人群簇本身**不主动引用**题材母题和玩法承接的 ID。关联关系由题材母题库通过 `audience_ids` 字段建立。

反向查询"这个人群簇适配哪些题材/玩法"的方式：

```text
grep "AUD-Y-001" archive/资料/买量组合库/题材库/*.md
grep "AUD-Y-001" archive/资料/玩法承接库/*.md
```
