# ID 与交叉引用规则

## 设计原则

```text
1. 所有条目必须有 ID
2. 所有跨库引用必须用 ID，不用名称
3. ID 一旦分配，永不复用，永不修改
4. 名称可以改，ID 不能改
```

为什么用 ID 而不是名称：

- 名称可能修改（措辞优化、改名），ID 不变才能保证引用稳定
- 名称可能重复或近似（"末世求生" vs "末日求生"），ID 强制唯一
- ID 便于工具自动化校验关联完整性

---

## ID 命名规则

### 通用格式

```text
{库前缀}-{大类代码}-{三位编号}
```

例：`AUD-Y-001`、`THM-SUR-001`、`PLY-TD-001`

### 三个库的前缀

| 库 | 前缀 | 含义 |
|---|---|---|
| 人群簇库 | `AUD` | Audience |
| 题材母题库 | `THM` | Theme Motif |
| 玩法承接库 | `PLY` | Playstyle |

### 大类代码

每个库的大类代码由各自的标准定义：

- 人群簇：见 [01-人群簇标准.md](01-人群簇标准.md)
- 题材母题：见 [02-题材母题标准.md](02-题材母题标准.md)
- 玩法承接：见 [03-玩法承接标准.md](03-玩法承接标准.md)

### 编号规则

- 三位数字，从 `001` 开始
- 在同一大类内顺序编号
- 删除条目时编号不复用（条目删除后编号永久作废）

### ID 示例

```text
AUD-Y-001     人群簇库 - 年轻向 - 第1个条目
AUD-Y-002     人群簇库 - 年轻向 - 第2个条目
THM-SUR-001   题材母题库 - 求生压力类 - 第1个条目
THM-SUR-002   题材母题库 - 求生压力类 - 第2个条目
PLY-TD-001    玩法承接库 - 塔防类 - 第1个条目
PLY-RPG-003   玩法承接库 - RPG类 - 第3个条目
```

## 交叉引用语法

### 在条目内引用其他库

每个条目使用 YAML frontmatter 声明引用关系：

```yaml
---
id: THM-SUR-001
name: 末世求生
audience_ids: [AUD-Y-001, AUD-M-002, AUD-S-005]   # 关联的人群簇
playstyle_ids: [PLY-TD-001, PLY-SLG-002, PLY-RG-003]  # 关联的玩法承接
---
```

### 三个库的引用方向

```text
人群簇（AUD）  ──┐
                 │
                 ↓
题材母题（THM）── 中心节点，主动声明引用
                 ↑
                 │
玩法承接（PLY）──┘
```

**关键约束**：

- **题材母题**主动引用人群簇和玩法承接的 ID
- 人群簇和玩法承接**不主动引用**题材母题（避免双向维护）
- 反向查询（"这个人群偏好哪些题材"）通过遍历题材母题库实现

### 在文档正文中引用

正文中引用 ID 时使用以下两种格式之一：

```markdown
适配人群：[AUD-Y-001](../../../archive/资料/人群簇库/AUD-Y-001.md)（年轻男性向休闲玩家）

或

适配人群：AUD-Y-001（年轻男性向休闲玩家）
```

第一种用于需要跳转的场合，第二种用于表格内或频繁引用时。

## 变更管理

### 允许的变更

| 字段 | 允许变更？ | 说明 |
|---|---|---|
| ID | ❌ 永不变更 | 一旦分配即固定 |
| 名称 | ✅ 可优化 | 措辞改进、表述更准确 |
| 描述/字段内容 | ✅ 可更新 | 随认知深化更新 |
| 关联 IDs | ✅ 可增删 | 但需校验引用完整性 |

### 不允许的变更

- 删除已被其他条目引用的 ID
- 修改 ID 值
- 复用已删除的 ID

### 删除条目流程

如果某个条目确认作废：

1. 检查是否有其他条目引用此 ID（grep 全库）
2. 若有引用，必须先在引用方移除此 ID
3. 在原条目顶部加 `status: deleted` 和作废原因
4. **不删除文件**，保留作废记录便于追溯

### 名称修改流程

1. 修改条目内的 `name` 字段
2. 更新 [索引文件](../../../archive/资料/) 中该 ID 对应的名称
3. 不需要修改其他条目的引用（因为引用用的是 ID）

---

## 引用完整性校验

每次新增或修改条目后，需校验：

1. 所引用的 ID 真实存在
2. 不存在自引用（A 引用 B，B 又引用 A 形成环）
3. 索引中的 ID 与实际文件名一致

校验方式：手动 grep（暂无脚本）。后续可在 `archive/tools/scripts/` 下增加自动化校验工具。

---

## ID 分配权限

| 操作 | 权限要求 |
|---|---|
| 新增条目（分配 ID） | 按 [05-填库工作流.md](05-填库工作流.md) 走审批 |
| 修改条目内容 | 同上 |
| 删除条目 | 必须用户明确授权 |

AI 不得自主分配 ID 或新增条目，必须先在 plan 中说明意图并获得授权。
