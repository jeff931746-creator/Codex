# 框架定义

## 一句话定义

```text
立项决策 = 选对人群簇 × 选对题材母题 × 选对玩法承接
```

任何缺一不可。三者要互相支撑：人群偏好玩法、玩法承接题材、题材吸引人群。

---

## 三个维度的定义

### 维度 1：人群簇（Audience Cluster）

**定义**：在游戏消费行为上具有稳定共性的玩家群体。

**关键判断**：两个玩家是否属于同一人群簇，看的不是年龄性别，而是：

- 偏好的核心情绪需求（压力释放、身份代入、社交炫耀、收集满足……）
- 偏好的玩法类型（策略、放置、动作、模拟……）
- 付费习惯（数值养成、抽卡、外观、月卡、广告变现……）

**例**：

- "压力释放型休闲玩家" ≠ "策略沉浸型核心玩家"，即便他们性别年龄相同
- "二次元抽卡用户" 和 "日漫塔防用户" 在题材偏好上有重合，但在付费习惯上是不同人群簇

**为什么是人群簇而不是用户画像**：

- "用户画像" 太粗，常用人口学维度（25-35岁男性）作为画像，无法指导买量
- "人群簇" 强调行为聚类，每个簇有明确的可识别行为（看什么素材会点击、玩什么类型留存、买什么道具）

### 维度 2：题材母题（Theme Motif）

**定义**：玩家在题材中能识别的稳定情绪+冲突结构。

**关键判断**：题材母题不是题材标签，而是题材背后稳定的情绪驱动力。

**对照**：

| 概念 | 例子 | 数量级 |
|---|---|---|
| 题材标签 | "末世铁路王国"、"维多利亚蒸汽朋克"、"修仙"、"赛博朋克"  | 数百个 |
| 题材母题 | "末世求生"、"权力博弈"、"成长复仇"、"探索未知" | 15-25 个 |

题材母题是题材标签的**上位抽象**。同一母题可以对应多个题材标签：

```text
末世求生母题 → 末世丧尸、末世铁路王国、核冬天废土、海底文明崩溃
权力博弈母题 → 三国、宫斗、黑帮、政治、修真宗门
```

**为什么用母题而不是直接用题材**：

- 题材标签太多太散，无法系统比较
- 母题数量可控，可以建立母题之间的关系（相似/对立/可融合）
- 母题与人群簇和玩法承接的对应关系更稳定（玩家选的是情绪体验，不是题材标签）

参考：[题材选择方法论](../../../../archive/方法论/立项/题材选择方法论_2026-05-08.md) 中"题材三要素"的定义。

### 维度 3：玩法承接（Playstyle Carrier）

**定义**：能独立成立的、能承接题材情绪和驱动人群留存的游戏玩法系统。

**关键判断**：参考 [独立玩法结合法](../../../../archive/方法论/立项/独立玩法结合法_2026-05-04.md) 中"独立玩法"的五问标准：

1. 能否单独做成一个**成功**的游戏？
2. 玩家会不会为了这个玩法本身而玩？
3. 这个玩法是否有自己的目标和反馈循环？
4. 这个玩法是否能撑起长线留存？
5. 这个玩法是否能容纳商业化方式？

**对照**：

| 概念 | 例子 |
|---|---|
| 品类标签 | "RPG"、"塔防"、"SLG"、"卡牌"、"放置" |
| 玩法承接 | "波次防守战 + 局外塔型升级"、"卡牌构筑 + Roguelike 跑图"、"资源链路自动化" |

玩法承接 = **品类标签 + 具体循环 + 商业化承接方式** 的组合。

**为什么是玩法承接而不是玩法品类**：

- 品类标签太粗，"塔防" 可以是 PVZ 也可以是 Bloons TD 也可以是向僵尸开炮，玩法承接能力完全不同
- 玩法承接强调"承接"——既要承接题材情绪（题材母题导出来的情绪要在玩法里得到释放），也要承接人群偏好（人群簇要在玩法里找到舒适区）

## 为什么是这三个维度

立项决策面临的核心问题是：

```text
1. 谁会下载（点击素材）→ 人群簇
2. 看到什么会下载（题材） → 题材母题
3. 下载后能留住和付费（玩法+商业化）→ 玩法承接
```

这三个问题一一对应到买量漏斗的三个关键节点：

```text
看到素材 → 点击/下载 → 进入游戏留存付费
   ↑           ↑              ↑
题材母题    人群簇        玩法承接
（情绪共鸣）（点击转化）  （ROI 承接）
```

任何一个维度判断错，整个立项就立不住：

- 题材对了人群错了 → 素材点击率高，但下载后留存崩盘（错配）
- 人群对了题材错了 → 素材打不中目标人群，CPI 失控
- 题材人群都对了玩法错了 → 用户进来后没事可做或玩不下去，付费转化崩盘
