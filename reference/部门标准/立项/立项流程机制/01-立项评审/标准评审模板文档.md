# 标准评审模板文档

## 目标

这份模板用于买量型产品的立项评审。

它解决的不是“谁讲得更好”，而是：

- 这个方向是否值得进入下一阶段验证
- 当前是否已经具备进入小测的资格
- 哪些问题必须进入 `02-纸面原型` 或其他补证环节
- 长线 `ROI / 留存` 的经营逻辑是否被写清
- 当前应该 `进入纸面原型补证 / 补其他证据再评 / 进入小测 / 终止`

---

## 使用规则

### 1. 评审顺序

固定按以下顺序进行：

1. 项目基线
2. 原型立项闸门评审
3. 长线经营假设评审
4. 总决策

### 2. 评审结构

每个模块都必须分成三部分：

- `事实`
- `判断`
- `结论`

### 3. 决策结果

整场评审只能给出以下四种结论：

- `进入纸面原型补证`
- `补其他证据再评`
- `进入小测`
- `终止`

### 4. 评审原则

- 不允许用“感觉不错”代替结论
- 不允许用“后面再补”解释当前问题
- 不允许把 `长线 ROI` 当成已被原型直接验证的结果
- 不允许同时保留多个主方向不做取舍
- 不允许跳过失败条件和止损条件

### 决策矩阵

参考 [原型立项闸门表.md](../原型立项闸门表.md#决策规则) 的决策矩阵：

| 通过数 | 补证数 | 不通过数 | 一票否决 | 决策 |
|--------|--------|----------|----------|------|
| 6 | 0 | 0 | 否 | **进入小测** |
| 5 | 1 | 0 | 否 | **进入纸面原型补证** |
| 4 | 2 | 0 | 否 | **进入纸面原型补证** |
| 4 | 1 | 1 | 否 | **补其他证据再评** 或 **退回立项定义** |
| 3 | 2 | 1 | 否 | **补其他证据再评** 或 **退回立项定义** |
| 3 | 3 | 0 | 否 | **补其他证据再评** 或 **退回立项定义** |
| ≤2 | - | ≥2 | 否 | **退回立项定义** 或 **终止** |
| - | - | - | 是 | **终止** |

### 决策判断

**进入纸面原型补证 vs 补其他证据再评**：
- 如果证据不足的闸门可以通过纸面原型或轻灰盒回答 → 进入纸面原型补证
- 如果证据不足的闸门需要补充题材/竞品/发行信息 → 补其他证据再评
- 如果不通过的闸门属于方向性问题（如核心承诺不成立）→ 退回立项定义

**补证 vs 退回立项定义**：
- 如果核心假设仍成立，只是证据不足 → 补证
- 如果核心假设被证伪，需要改动冻结基线 → 退回立项定义
- 冻结基线包括：目标用户、核心买量承诺、核心玩法一句话、首个爽点来源、首付理由、爽点成本结构

**连续补证失败的止损规则**：
- 第 1 轮补证失败：分析原因，针对性补证
- 第 2 轮补证失败：评估核心假设是否成立，决定继续补证或退回立项定义
- 连续 2 轮补证仍未通过：强制终止或退回立项定义重新设计

### 最终结论

- `进入纸面原型补证 / 补其他证据再评 / 进入小测 / 退回立项定义 / 终止`

## 说明

- 旧的 `方向卡 / 母型卡 / 参与方式卡 / 核心机制卡 / 玩法循环卡` 可以继续作为内部准备材料或附录。
- 但正式立项会上的决策，不再按五张卡逐张给结论，而是统一折叠进上面的 `原型立项闸门` 和 `长线经营假设`。
- 原型阶段不能把 `长线 ROI` 当成已验证结果；它最多只能证明“这个方向是否值得进入下一阶段验证”。
